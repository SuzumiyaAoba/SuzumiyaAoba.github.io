## スレッドを作る ― Thread と Runnable

それでは、実際にスレッドを作って、動かしてみましょう。
スレッドは、**`Thread`** クラスと、第22章で学んだ **`Runnable`**（処理を表す関数型インターフェース）を使って作ります。

---

## スレッドを作って動かす

スレッドにやらせたい処理を、ラムダ式（`Runnable`）で書き、`new Thread(...)` に渡します。
そして、**`start()`** を呼ぶと、その処理が**別のスレッドで**始まります。

```java
Thread thread = new Thread(() -> {
    IO.println("別スレッドの処理");
});

thread.start();   // 別スレッドで処理を始める
IO.println("メインの処理");
```

`new Thread(() -> { ... })` で、「この処理をするスレッド」を作ります。
渡しているラムダ式は、第22章で学んだ `Runnable`（引数なし・戻り値なしの処理）です。

そして **`start()`** で、その処理が、**メインスレッドとは別の流れ**で動き始めます。

> **注意: `run()` ではなく `start()`**
>
> スレッドを動かすのは、**`start()`** です。`run()` を直接呼んではいけません。
> `run()` を呼ぶと、**別スレッドにならず**、ふつうのメソッド呼び出しとして、その場で実行されてしまいます。
> 「新しいスレッドで動かすなら、必ず `start()`」と覚えておきましょう。

---

## 出力の順番は、決まらない

ここで、大切なことを確認します。
`thread.start()` と `IO.println("メインの処理")` は、**どちらが先に表示されるか、決まっていません**。

```java
Thread thread = new Thread(() -> IO.println("別スレッドの処理"));
thread.start();
IO.println("メインの処理");
```

```text
（実行の一例 ― 順番は毎回変わりうる）
メインの処理
別スレッドの処理
```

`start()` を呼ぶと、別スレッドの処理が「始まる」だけで、それが終わるのを**待ちません**。
メインスレッドは、すぐ次の行（`IO.println("メインの処理")`）へ進みます。
そのため、「メインの処理」が先に出ることも、「別スレッドの処理」が先に出ることも、あります。
**何度か実行すると、順番が変わる**かもしれません。これが、並行処理の「順番が決まらない」性質です。

---

## スレッドの終了を待つ ― join

「別スレッドの処理が**終わるのを待ってから**、次へ進みたい」ときは、**`join()`** を使います。

```java
Thread thread = new Thread(() -> IO.println("別スレッドの処理"));
thread.start();
thread.join();                    // このスレッドが終わるまで待つ
IO.println("メインの処理");
```

```text
別スレッドの処理
メインの処理
```

`thread.join()` は、「`thread` が終わるまで、ここで待つ」という意味です。
これで、「別スレッドの処理」が**必ず先**に終わり、そのあとに「メインの処理」が表示されます。

> **補足: join は例外を投げる可能性がある**
>
> `join()` は、待っている間に中断されると、`InterruptedException`（チェック例外、第20章）を投げる可能性があります。
> そのため、`join()` を使うメソッドは、`try-catch` で囲むか、`throws InterruptedException` を付ける必要があります（第20章のチェック例外のとおりです）。

## 複数のスレッドを動かす

複数のスレッドを作れば、複数の処理を同時に進められます。

```java
for (int i = 1; i <= 3; i++) {
    int id = i;
    Thread t = new Thread(() -> IO.println("スレッド " + id + " が動いています"));
    t.start();
}
```

```text
（実行の一例 ― 順番は毎回変わりうる）
スレッド 2 が動いています
スレッド 1 が動いています
スレッド 3 が動いています
```

3つのスレッドが、それぞれ動きました。
ですが、表示される順番は、`1 → 2 → 3` とは限りません。**どのスレッドが先に動くかは、決まっていない**からです。
（`int id = i;` としているのは、第22章で学んだ「ラムダ式が使う変数は実質的 final でなければならない」ためです。ループ変数 `i` をそのまま使えないので、`id` に写しています。）

---

## まとめ

- スレッドは、**`new Thread(() -> { 処理 })`** で作り、**`start()`** で動かす
- 処理はラムダ式（`Runnable`、第22章）で渡す
- **`run()` ではなく `start()`**（`run()` を直接呼ぶと別スレッドにならない）
- `start()` は処理の終了を**待たない**ので、出力の**順番は決まらない**
- **`join()`** で、スレッドの終了を待てる（`InterruptedException` に注意）
