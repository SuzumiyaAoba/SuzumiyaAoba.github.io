## はじめに ― この章で学ぶこと

これまでのプログラムは、すべて「**上から下へ、1つずつ順番に**」処理してきました。
1つの処理が終わってから、次の処理へ ―― このような流れを、**逐次処理**（ちくじしょり）と呼びます。

ですが、現実には、「**複数の処理を、同時に進めたい**」場面がたくさんあります。

- 重いファイルのダウンロードを待つ間に、別の作業を進めたい
- たくさんのユーザーからの依頼を、同時にさばきたい（Web サーバーなど）
- 複数の計算を、手分けして同時に行い、全体を速くしたい

このように、**複数の処理を同時に進める**ことを、**並行処理**（へいこうしょり、Concurrency）と呼びます。
この第29章では、その入口を学びます。

並行処理は、プログラミングの中でも、特に**難しいテーマ**の1つです。
この章では「完全にマスターする」ことは目指しません。
「**スレッドとは何か**」「**なぜ難しいのか**」「**Java 21 の画期的な新機能（仮想スレッド）とは何か**」――
この3点をつかむことを、目標にします。

---

## スレッド ― 処理の流れ

並行処理の主役は、**スレッド**（Thread）です。
スレッドとは、「**処理の流れ（実行の筋道）**」のことです。

これまでのプログラムは、1本のスレッド（メインスレッド）だけで動いていました。
複数のスレッドを使えば、「処理の流れ」を複数に分けて、**同時に進められる**ようになります。

```text
逐次処理（1本の流れ）:   A → B → C → D
並行処理（複数の流れ）:   A → B
                         C → D   （同時に進む）
```

この章では、スレッドの作り方から始めて、その難しさ（共有データの問題）、そして現代的な扱い方（`ExecutorService`・仮想スレッド）まで、順に見ていきます。

## この章で学ぶこと

第29章は、次の7つの節で構成されています。

| 節 | タイトル | 内容 |
|---|---|---|
| 1 | 並行処理とは | スレッドの考え方。逐次処理との違い |
| 2 | スレッドを作る | `Thread` と `Runnable` |
| 3 | 共有データの問題 | 競合状態と、なぜ難しいのか |
| 4 | ExecutorService | スレッドを管理する、現代的な方法 |
| 5 | 仮想スレッド | Java 21 の画期的な新機能 |
| 6 | よくあるつまずき | 順序・共有・例外 など |

前半（1〜3節）で、スレッドの基本と、その難しさを学びます。
後半（4〜5節）で、現代的なスレッドの扱い方と、仮想スレッドを学びます。

## この章を読み終えると

第29章を読み終えるころには、次のことができるようになっています。

- 並行処理とスレッドの考え方を、説明できる
- `Thread` でスレッドを作り、動かせる
- 「共有データの競合」がなぜ起きるか、その危険性を理解できる
- `ExecutorService` で、スレッドを管理しながらタスクを実行できる
- 仮想スレッド（Virtual Threads）が、なぜ画期的なのかがわかる

> **補足: 出力の順番は、毎回変わることがある**
>
> 並行処理では、複数のスレッドが同時に動くため、**出力される順番が、実行のたびに変わる**ことがあります。
> 本書のサンプルの出力例は「一例」であり、あなたが試すと、違う順番になるかもしれません。
> これは間違いではなく、並行処理の本質です。この章では、その「順番が決まらない」という性質も、しっかり学びます。

それでは、最初の節「並行処理とは」から始めましょう。
