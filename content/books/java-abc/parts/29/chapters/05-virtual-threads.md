## 仮想スレッド ― Java 21 の画期的な新機能

**仮想スレッド**（Virtual Threads）は、**Java 21** で正式導入された、並行処理の常識を変える画期的な機能です[^jep444-vt]。
「**とても軽い、大量に作れるスレッド**」だと考えてください。
この節では、仮想スレッドが、なぜすごいのかを学びます。

---

## これまでのスレッドの限界

これまで使ってきたスレッド（`new Thread(...)`）は、**プラットフォームスレッド**と呼ばれます。
これは、OS（オペレーティングシステム）が管理する、本物のスレッドに、1対1で結びついています。

プラットフォームスレッドには、限界がありました。

- **重い** … 1つ作るのに、それなりのメモリ（数MB）を使う
- **数に限りがある** … 数千個も作ると、コンピュータがパンクする

そのため、「待ち時間の多い処理を、何万個も同時にさばきたい」（たとえば、何万人ものユーザーに同時対応する Web サーバー）といった用途では、スレッドの数が足りず、工夫が必要でした。

---

## 仮想スレッド ― 大量に作れる軽いスレッド

**仮想スレッド**は、この限界を打ち破ります。
仮想スレッドは、OS の本物のスレッドに、つねには結びつきません。
Java（JVM）が、少数の本物のスレッドの上で、**たくさんの仮想スレッドを、巧みに切り替えて**動かします。

その結果、

- **とても軽い** … 1つ作るコストが、けた違いに小さい
- **大量に作れる** … **何百万個**でも作れる

「待っている」スレッドは、本物のスレッドを占有しないので、待ち時間の多い処理を、**いくらでも同時に**走らせられます。

---

## 仮想スレッドの作り方

仮想スレッドは、**`Thread.ofVirtual().start(...)`** で作れます。

```java
Thread vt = Thread.ofVirtual().start(() -> {
    IO.println("仮想スレッドの処理");
});
vt.join();
IO.println("完了");
```

```text
仮想スレッドの処理
完了
```

使い方は、ふつうのスレッドとほとんど同じです（`start`・`join` も同じ）。
違うのは、「とても軽くて、大量に作れる」という点だけです。

---

## ExecutorService と組み合わせる

実際には、前の節の `ExecutorService` と組み合わせて使うのが、おすすめの方法です。
**`Executors.newVirtualThreadPerTaskExecutor()`** を使うと、「**タスクごとに、仮想スレッドを1つ作る**」実行係になります。

```java
import java.util.concurrent.*;

try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {
    Future<Integer> future = executor.submit(() -> 3 + 5);
    IO.println("計算結果: " + future.get());
}
```

```text
計算結果: 8
```

仮想スレッドは大量に作れるので、「タスクごとに1スレッド」という、ぜいたくな使い方ができます。
前の節の `newFixedThreadPool(2)`（数を制限する）とは対照的に、「**作りすぎを心配しなくてよい**」のが、仮想スレッドの強みです。
たとえば、1万件のネットワーク通信を、1万個の仮想スレッドで、同時に待つ ―― そんな処理が、軽々と書けるのです。

---

## 仮想スレッドの、なにが画期的か

仮想スレッドの本当のすごさは、「**シンプルなコードで、高い性能が出せる**」ことにあります。

これまでは、「待ち時間の多い処理を、たくさんさばく」には、スレッドの数を節約するための、複雑な書き方（非同期プログラミングなど）が必要でした。
仮想スレッドなら、「**1つの依頼につき、1つのスレッドで、ふつうに（逐次的に）書く**」という、分かりやすいスタイルのまま、大量の処理をさばけます。

| | プラットフォームスレッド | 仮想スレッド |
|---|---|---|
| 重さ | 重い（数MB） | とても軽い |
| 作れる数 | 数千個まで | 何百万個でも |
| 向いている用途 | CPU を使う重い計算 | 待ち時間の多い処理（通信・I/O） |

「難しい工夫なしに、分かりやすいコードで、たくさんの処理を同時にさばける」――
これが、仮想スレッドが「画期的」と言われる理由です。

> **補足: 使い分け**
>
> 仮想スレッドは万能ではありません。「**待ち時間が多い処理**」（ネットワーク・ファイル・データベース）に向いています。
> 一方、「CPU をフル回転させる重い計算」には、従来のプラットフォームスレッド（CPU のコア数ぶん）のほうが適しています。
> とはいえ、初心者のうちは、「待ち時間の多い処理を大量にさばくなら、仮想スレッド」と知っておけば十分です。

---

## まとめ

- **仮想スレッド**（Java 21+）は、「**とても軽く、大量に作れる**」スレッド
- 従来の**プラットフォームスレッド**は重く、数千個が限界だった
- 仮想スレッドは、**何百万個**でも作れ、待ち時間の多い処理を大量にさばける
- 作り方は **`Thread.ofVirtual().start(...)`**、または **`Executors.newVirtualThreadPerTaskExecutor()`**
- 「分かりやすいコードのまま、高い性能」が、仮想スレッドの画期的な点
- 待ち時間の多い処理（通信・I/O）に向く（重い計算には従来のスレッド）

次の節では、並行処理でつまずきやすいポイントを、まとめて確認します。

[^jep444-vt]: JEP 444: Virtual Threads, <https://openjdk.org/jeps/444>。Java 21（2023年9月）で正式機能となった。JVM が管理する軽量スレッドで、ブロッキング I/O 中も OS スレッド（carrier thread）を占有しないため、何百万単位の同時実行が可能。プレビュー段階は JEP 425（JDK 19）、JEP 436（JDK 20）。
