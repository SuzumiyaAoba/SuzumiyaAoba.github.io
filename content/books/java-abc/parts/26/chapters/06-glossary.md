## 用語集 ― この章で学んだ言葉

第26章では、型や構造に応じた分岐「パターンマッチング」を学びました。
節ごとに、言葉を整理しておきます。

---

## instanceof と switch（第1〜2節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| パターンマッチング | Pattern Matching | 型や構造に応じて、確認・取り出し・分岐をまとめて行う |
| パターン変数 | Pattern Variable | `instanceof String s` の `s`。マッチしたら値が入る |
| switch のパターンマッチング | ― | `case 型 変数 ->` で、型による分岐（Java 21+） |
| `case null` | ― | switch で `null` を安全に扱うラベル |

---

## レコードパターン（第3節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| レコードパターン | Record Pattern | レコードの中身を分解して取り出す（`case Point(int x, int y)`、Java 21+） |

ネストしたレコードもまとめて分解できる。中の型は `var` で省略可。

---

## ガードと網羅性（第4節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| `when` ガード | Guard | 型に加えて条件を付ける（`case Integer i when i < 0`） |
| 網羅性 | Exhaustiveness | switch がすべての場合をカバーしているか |
| `default` | ― | 「それ以外」をカバーするラベル |

`case` は上から順。狭いパターンを上、広いパターンを下に書く。

---

## おわりに

この章では、型や構造に応じた分岐「パターンマッチング」を学びました。

- **`instanceof` パターン** … 確認と取り出しを1つに
- **`switch` のパターンマッチング** … 型による分岐をすっきりと
- **レコードパターン** … レコードの中身を分解
- **`when` ガード** … 条件つきのパターン

第15章で「使いすぎに注意」とされた `instanceof` の連打が、安全で読みやすい `switch` に生まれ変わりました。

ところで、この章では何度か、「**取りうる型が限られていれば、`default` なしで網羅できる**」と予告しました。
たとえば「図形は、円・長方形・三角形のどれか」と決まっていれば、その3つを挙げた時点で、`switch` は完成します。

次の第27章「シールドクラス」では、この「**取りうる型を、あらかじめ限定する**」しくみを学びます。
シールドクラスとパターンマッチングは、まさに「一心同体」の機能です。
2つが組み合わさることで、「決まった種類のデータを、もれなく安全に処理する」という、モダン Java の強力な設計が完成します。
