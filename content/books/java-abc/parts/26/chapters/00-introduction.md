## はじめに ― この章で学ぶこと

第15章で、`instanceof` を使った**パターンマッチング**を、少しだけ学びました。

```java
if (obj instanceof String s) {   // 型を確認しつつ、s に入れる
    IO.println(s.length());
}
```

「型を確認して、その型の変数に入れる」を、1つにまとめて書けるしくみでしたね。
この第26章では、このパターンマッチングを、本格的に学びます。

パターンマッチングは、近年の Java で、もっとも力を入れて拡張されてきた機能の1つです。
`instanceof` だけでなく、**`switch`** と組み合わせたり、第17章の**レコードを分解**したりと、「**型や構造に応じた分岐**」を、安全かつ簡潔に書けるようになります。

たとえば、「図形の種類に応じて、面積を計算する」処理が、こう書けます。

```java
double area = switch (shape) {
    case Circle(double r)         -> r * r * 3.14;   // 円なら、半径を取り出して計算
    case Rectangle(double w, double h) -> w * h;     // 長方形なら、幅と高さを取り出して
    case Triangle(double b, double h)  -> b * h / 2; // 三角形なら…
};
```

`switch` で型を見分けつつ、その場で中身（半径や幅）を取り出して、計算しています。
第15章で「`instanceof` の使いすぎは避けよう」と学びましたが、パターンマッチングは、そうした分岐を**ずっと安全で読みやすい形**にしてくれるのです。

## この章で学ぶこと

第26章は、次の6つの節で構成されています。

| 節 | タイトル | 内容 |
|---|---|---|
| 1 | instanceof パターン | 第15章のおさらいと、`&&` との組み合わせ |
| 2 | switch のパターンマッチング | 型に応じた分岐を switch で |
| 3 | レコードパターン | レコードを分解して取り出す |
| 4 | when ガードと網羅性 | 条件つきパターン・null・もれの検査 |
| 5 | よくあるつまずき | 順序・網羅性 など |

前半（1〜2節）で、`instanceof` と `switch` のパターンマッチングを学びます。
後半（3〜4節）で、レコードの分解と、条件つきパターンを学びます。

---

## この章を読み終えると

第26章を読み終えるころには、次のことができるようになっています。

- `instanceof` パターンを、`&&` と組み合わせて使える
- `switch` で、型に応じた分岐を、安全に書ける
- レコードパターンで、レコードの中身を分解して取り出せる
- `when` で、条件つきのパターンを書ける
- パターンマッチングが、`instanceof` の連打より優れている理由を説明できる

> **補足: パターンマッチングの導入時期**
>
> パターンマッチングは、段階的に導入されてきました。
> `instanceof` のパターンは **Java 16**、`switch` のパターンマッチングと**レコードパターン**は **Java 21** で正式機能になりました。
> 本書が対象とする **Java 25** では、これらすべてが、安心して使えます。

それでは、最初の節「instanceof パターン」から始めましょう。
