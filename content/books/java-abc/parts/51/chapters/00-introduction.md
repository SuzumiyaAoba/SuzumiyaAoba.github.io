## はじめに ― この章で学ぶこと

第6部の最初は、生成パターン（Creational Patterns）の代表格、**Singleton**（シングルトン）です。

「アプリケーション全体で**ただ 1 つだけ**しか存在してはいけないオブジェクト」を、Java で安全に作るためのパターン ―― それが Singleton です。
有名で名前を聞いたことがある人も多いはずですが、**「安易に使うとアンチパターンになる」**ことでも有名なパターンでもあります。

この章では、

- Singleton が**どんな問題を解こうとしているか**
- 単純な実装に潜む**落とし穴**（スレッド安全性、リフレクション、シリアライズ）
- 現代の Java での**もっとも素直な書き方**（`enum`）
- そして、**Singleton を使わずに済ます方法**

までを順に見ていきます。
読み終えるころには、「**いつ Singleton を使うべきで、いつ使うべきでないか**」が、自分の判断で言えるようになります。

## 第6部の歩き方 ― GoF 23 パターンとの付き合い方

第6部では、GoF（Gang of Four）の **23 パターンすべて**を 1 章ずつ取り上げます。
GoF のパターンは、次の 3 つに分類されています。

| 分類 | 関心ごと | 本書での章 |
|---|---|---|
| **生成パターン**（Creational） | **どう作るか** | 第51〜55章 |
| **構造パターン**（Structural） | **どう組み合わせるか** | 第56〜62章 |
| **振る舞いパターン**（Behavioral） | **どうやり取りするか** | 第63〜73章 |

各章では、

1. パターンが**解こうとしている問題**を見せる
2. **古典的な書き方**（GoF の本どおりの形）を示す
3. **現代の Java での書き方**（ラムダ・レコード・シールドクラスなど）を示す
4. **使いどころ**と**よくあるつまずき**を整理する

という流れで進みます。
第34章の「デザインパターン入門」を読み終えていれば、そのまま続けて読める内容です。

---

## この章を読み終えると

- Singleton パターンの**意図**を、自分の言葉で説明できる
- **スレッドセーフな Singleton** を 2 通り（DCL と `enum`）で書ける
- **リフレクションやシリアライズに耐える Singleton** を書ける
- 「**Singleton にしない**」という選択肢を取れる

それでは、Singleton の本体に入っていきましょう。
