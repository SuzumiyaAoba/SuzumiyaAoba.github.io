## はじめに ― この章で学ぶこと

第46章で、**実行時**にアノテーションを読み取る仕組みを学びました。
ですが、Java の世界には、それ以前の段階 ――**コンパイル時**にアノテーションを使ってコードを生成する仕組みがあります。
これが **アノテーション処理**（Annotation Processing）です。

`@Override`、`@SuppressWarnings`、`@Data`（Lombok）、`@Generated`（MapStruct） ―― これらすべてが、**コンパイル時に**処理されています。
第47章では、その仕組みをほどき、**自分でも書ける**ようになることを目指します。

---

## なぜ「コンパイル時」が大事か

リフレクションで実行時に処理するのと、コンパイル時に処理するのとの違いは、こうです。

| | リフレクション（実行時） | アノテーション処理（コンパイル時） |
|---|---|---|
| いつ走る | アプリ起動時、または毎リクエスト | **`javac` 実行時のみ** |
| コスト | 実行時に発生 | **コンパイルが少し遅くなるだけ**、実行時はゼロ |
| 型安全 | キャストが必要 | 生成コードは普通のコード ―― 完全に型安全 |
| デバッグ | スタックトレースから追える | 生成されたコードを見れば追える |
| 制約 | `setAccessible` などで侵入可能 | コンパイル時に決まることだけ |

アノテーション処理は、リフレクションのコストとリスクを**ゼロ**にしながら、似たような「**メタプログラミング**」を実現します。
そのおかげで、Lombok や MapStruct のような優れたライブラリが生まれました。

---

## 「**生成されたコード**」を見たことはありますか?

`@Override` や `@SuppressWarnings` は、コンパイル時に**チェック**するだけで、コードを生成しません。
一方、Lombok の `@Data` を付けると、

```java
@Data
public class User {
    private String name;
    private int age;
}
```

これに、

- `getName()` / `getAge()`
- `setName(String)` / `setAge(int)`
- `equals` / `hashCode` / `toString`
- 引数なしコンストラクタ

が**コンパイル時に**生成されます。
ソースには出てきませんが、`.class` ファイルにはちゃんと入っています。

このような「コード生成」を、自分でも書けるようにする ―― それがアノテーション処理の入り口です。

---

## レコードとの違い

「`record` を使えば、Lombok の `@Data` と同じことができるのでは?」と思うかもしれません。
おおむねその通りで、Java 14 以降の `record` は、Lombok の `@Data` の典型的な用途を**言語機能**でカバーしました。

ですが、

- Lombok の `@Builder` ―― ビルダーパターンの自動生成
- MapStruct の `@Mapper` ―― DTO 変換コードの自動生成
- AutoService の `@AutoService` ―― ServiceLoader 用ファイルの生成

など、`record` ではカバーできない**コード生成**は、依然として必要です。
アノテーション処理は、これらすべての裏側で動いています。

---

## この章で扱う範囲

アノテーション処理を**ガッツリ実装する**には、`javax.annotation.processing` の API を使い、**プロセッサを別 jar として配布**するなど、本格的なライブラリ作りが必要です。
本書では、

- アノテーション処理の**全体像**を理解する
- 既存のプロセッサ（Lombok、MapStruct）の動きを**想像できる**ようになる
- 簡単なプロセッサを**動かしてみる**

までを目指します。**自分でライブラリを作る**レベルまでは踏み込みません。

## この章を読み終えると

第47章を読み終えるころには、次のことができるようになっています。

- アノテーションの保持期間（`Retention`）の違いを使い分けられる
- `@Override`・`@SuppressWarnings`・`@Deprecated` の意図を読める
- 簡単なアノテーションプロセッサを書ける
- Lombok や MapStruct が**裏で何をしているか**を想像できる
- ビルドツール（Maven / Gradle）でプロセッサを動かす設定が分かる

それでは、最初の節「**アノテーションの種類とライフサイクル**」から始めましょう。
