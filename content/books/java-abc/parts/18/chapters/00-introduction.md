## はじめに ― この章で学ぶこと

前の第17章では、「データを束ねる」ためのレコードを学びました。
この第18章では、もう1つの特別な型 ―― **列挙型**（れっきょがた、Enum）を学びます。

列挙型は、「**決まったいくつかの値の、どれか1つ**」を表す型です。
世の中には、選択肢が決まっているデータが、たくさんあります。

- 信号の色 … 赤・黄・青の、どれか
- 曜日 … 月〜日の、どれか
- 注文の状態 … 受付・発送・完了の、どれか
- トランプのスート … スペード・ハート・ダイヤ・クラブの、どれか

こうした「**取りうる値が、あらかじめ決まっている**」データを、安全に表すのが列挙型です。

---

## なぜ列挙型が必要なのか

たとえば、信号の色を、文字列で表すとしましょう。

```java
String signal = "あか";
```

一見、これで困らなそうですが、問題があります。

- `"あか"` なのか `"赤"` なのか `"RED"` なのか、**表記がばらつく**
- `"あ"` や `"みどり"`、`"あお-"`（打ち間違い）のような、**ありえない値も入ってしまう**
- 打ち間違えても、**コンパイル時には気づけない**（実行して、初めておかしくなる）

数値で表す（`0`＝赤、`1`＝黄…）方法もありますが、これも「`5` のような、ありえない番号」を防げませんし、`0` が何を意味するのか、コードを見ても分かりません。

**列挙型**を使えば、「信号の色は、赤・黄・青の3つだけ」と Java に教えられます。
すると、それ以外の値は**コンパイルエラーではじかれ**、打ち間違いも防げます。
「取りうる値を、型のレベルで限定する」 ―― これが列挙型の最大の利点です。

## この章を読み終えると

第18章を読み終えるころには、次のことができるようになっています。

- `enum` で、取りうる値が決まった型を定義できる
- 列挙型が、文字列や数値より安全な理由を説明できる
- `switch` と列挙型を、相性よく組み合わせられる
- 各列挙値に、フィールドやメソッドを持たせられる
- `values()`・`valueOf()` などの便利なメソッドを使える

それでは、最初の節「列挙型とは」から始めましょう。
