## よくあるつまずき

列挙型で、初心者がつまずきやすいポイントを、まとめて確認します。

---

## 1. enum を new で作ろうとする

列挙型の定数は、定義に書いたものが、Java によって自動で用意されます。
そのため、`new` で新しく作ることは**できません**（コンストラクタは自動的に `private` でした）。

```java
Signal s = new Signal();   // ✕ enum は new できない
```

```text
エラー: 列挙クラスはインスタンス化できません
```

列挙型を使うときは、`new` ではなく、`Signal.RED` のように、定義済みの定数を使います。
「列挙定数は、あらかじめ用意された、決まったオブジェクト」だと考えましょう。

> **補足: switch の case は、`RED` でも `Signal.RED` でもよい**
>
> switch の `case` ラベルは、`case RED ->` のように**定数名だけ**で書くのが、簡潔で一般的です。
> なお、新しい Java（21 以降）では、`case Signal.RED ->` のように**列挙型名を付けても**書けるようになりました。
> 本書では、簡潔な `case RED ->` を使います。

---

## 2. 定数の後ろの「;」を忘れる

列挙型に、フィールドやメソッドを書くときは、**定数の並びの最後にセミコロン `;`** が必要です。

```java
enum Signal {
    RED, YELLOW, GREEN     // ✕ ; がない
    private final String label;
    ...
}
```

```text
エラー: ','、'}'または';'がありません
```

正しくは、`GREEN;` のように、最後の定数のあとに `;` を付けます。

```java
enum Signal {
    RED, YELLOW, GREEN;    // ◯ ; を付ける
    ...
}
```

「定数だけの enum なら `;` は不要、メソッドなどを足すなら `;` が必要」と覚えておきましょう。

---

## 3. valueOf に、存在しない名前を渡す

`valueOf("名前")` は便利ですが、**存在しない名前を渡すと、実行時にエラー（例外）**になります。

```java
Signal s = Signal.valueOf("PURPLE");   // 存在しない
```

```text
Exception in thread "main" java.lang.IllegalArgumentException: No enum constant Signal.PURPLE
```

外部（ファイルやユーザー入力）から受け取った文字列を `valueOf` に渡すときは、特に注意が必要です。
正しい名前かどうか分からない文字列を変換するときは、エラーへの備えが必要になります（くわしくは第20章「例外処理」）。

---

## 4. ordinal() の番号に頼る

`ordinal()`（並び順の番号）に、重要な処理を頼るのは危険です。

```java
if (signal.ordinal() == 0) { ... }   // 「0 番なら赤」のつもり
```

このコードは、`RED` を先頭に書いているうちは動きます。
ですが、あとで定数の順番を変えたり、先頭に新しい定数を足したりすると、`0` が指すものが変わり、**こっそりバグになります**。
番号に意味を持たせたいなら、`ordinal()` ではなく、第4節のように**フィールドとして明示的に持たせ**ましょう。
判定には、`signal == Signal.RED` のように、定数そのもので比べるのが安全です。

---

## 5. enum を == で比べてよいか迷う

第10章で「比較は `equals`」と強く言われたので、列挙型でも `equals` を使うべきか、迷うかもしれません。
列挙型は、**`==` で比べてかまいません**（むしろ推奨）。各定数は世界に1つだけなので、`==` で正しく判定でき、`null` でも安全だからです。

```java
if (signal == Signal.RED) { ... }   // ◯ enum は == でよい
```

「文字列は `equals`、列挙型は `==`」と、対にして覚えておきましょう。

---

## まとめ

- 列挙型は **`new` できない**。定義済みの定数（`Signal.RED`）を使う
- フィールドやメソッドを足すときは、定数の並びの最後に **`;`** を付ける
- `valueOf` に存在しない名前を渡すと**例外**になる（外部入力に注意）
- **`ordinal()` の番号には頼らない**。意味はフィールドで持つ
- 列挙型の比較は **`==`** でよい（文字列は `equals`）

次は、この章で学んだ言葉を、用語集としてまとめます。
