## JDK / JRE / JVM の違い

Java を学び始めると、よく似た3つの言葉に出会います。

- **JVM**
- **JRE**
- **JDK**

どれもアルファベット3文字で、「J」で始まり「VM」「RE」「DK」と続くため、最初はとても紛らわしく感じます。

しかし、この3つの関係はとてもシンプルです。
一度整理してしまえば、もう迷うことはありません。
この節で、すっきりと理解しておきましょう。

---

## 3つの言葉の関係は「入れ子」になっている

先に結論を言ってしまいましょう。

この3つは、**大きな箱の中に小さな箱が入っている**ような、入れ子の関係になっています。

![JDK・JRE・JVM の入れ子の関係。いちばん内側に JVM があり、それを JRE が、さらに JDK が包んでいる。](./images/jdk-jre-jvm.svg)

- いちばん内側に **JVM** がある
- JVM を含み、プログラムを動かすために必要なものをそろえたのが **JRE**
- JRE を含み、さらに開発に必要な道具を加えたのが **JDK**

それぞれが何なのか、内側から順に見ていきましょう。

---

## JVM ― バイトコードを実行する係

**JVM**（Java Virtual Machine、Java 仮想マシン）は、前の節で学んだとおり、**バイトコードを読み込んで実行する仕組み**です。

3つのなかでは、いちばん中心にある部品です。

JVM の主な役割は、次のとおりです。

- バイトコード（`.class`）を読み込む
- そのバイトコードが安全なものかを確認する
- 動いている環境の機械語に変換しながら実行する
- メモリの管理（ガベージコレクション）を行う

JVM は、いわば**バイトコードを実際に動かす実行係**です。
ただし、JVM だけが単独であっても、プログラムを動かすことはできません。
実行には、次に説明する標準ライブラリなども必要になるからです。

---

## JRE ― プログラムを「動かす」ための一式

**JRE**（Java Runtime Environment、Java 実行環境）は、**Java のプログラムを動かすために必要なものを一式まとめたもの**です。

JRE には、次のものが含まれます。

- **JVM**（実行係）
- **標準ライブラリ**（Java に最初から付属している、便利な機能のまとまり）
- そのほか、実行に必要な部品

前の節で、JVM はバイトコードを実行すると説明しました。
しかし、実際のプログラムは、文字や数値を扱う機能、データをまとめて管理する機能など、標準ライブラリの力を借りて動いています。
そのため、プログラムを動かすには、JVM だけでなく標準ライブラリもセットで必要になります。

この「動かすために必要な一式」をまとめたものが、JRE です。

```text
JRE ＝ JVM ＋ 標準ライブラリ ＋ 実行に必要な部品
```

JRE は「プログラムを動かす」ことに特化しています。
すでに完成した Java のプログラムを動かすだけなら、JRE があれば十分です。
ただし、JRE には、自分でプログラムを**作る**ための道具は含まれていません。

---

## JDK ― プログラムを「作る」ための一式

**JDK**（Java Development Kit、Java 開発キット）は、**Java のプログラムを開発するために必要なものを一式まとめたもの**です。

JDK には、次のものが含まれます。

- **JRE に含まれるものすべて**（つまり JVM や標準ライブラリ）
- **開発のための道具**

開発のための道具には、たとえば次のようなものがあります。

- **コンパイラ**（`javac`） ： ソースコード（`.java`）をバイトコード（`.class`）に変換する道具
- そのほか、プログラムをまとめたり、調べたり、文書化したりするための各種ツール

前の節で、ソースコードはコンパイルしてバイトコードにすると学びました。
このコンパイルを行うコンパイラ（`javac`）は、JDK に含まれています。
JRE には含まれていません。

つまり、

```text
JDK ＝ JRE ＋ 開発のための道具（コンパイラなど）
```

という関係になります。

自分でプログラムを書いて、コンパイルして、実行する。
この一連の作業をすべて行うには、JDK が必要です。

---

## 結局、何をインストールすればよいのか

ここまでの説明で、もっとも大切な結論はこれです。

> **これから Java を学ぶ人は、JDK をインストールすればよい**

理由は単純です。
私たちはこれから、プログラムを**自分で書いて、コンパイルして、実行する**ことを学びます。
そのためには、開発の道具まで含んだ JDK が必要だからです。

そして、JDK の中には JRE が、JRE の中には JVM が含まれています。
つまり、JDK を1つインストールすれば、実行に必要なものもすべて手に入る、というわけです。

> **補足: 単独の JRE について**
>
> 以前は、プログラムを動かすだけの人向けに、JRE が単独で配布されていました。
> しかし Java 11 以降、Oracle は単独の JRE を配布しなくなりました[^java11-no-jre]。
> 現在は、開発する人も動かす人も、JDK を入れるのが基本的な形になっています。
> この点からも、「まずは JDK を入れればよい」と覚えておけば大丈夫です。

実際のインストール手順は、次の第2章「環境構築」でくわしく説明します。

---

## OpenJDK と、いろいろな配布元

JDK について調べると、**OpenJDK**（オープンジェイディーケー）という言葉や、いくつもの提供元の名前を目にすることがあります。

前の節で、Java の中核部分はオープンソース化され、その実装が OpenJDK の土台になったと学びました。
現在配布されている多くの JDK は、この OpenJDK をもとに作られています。

OpenJDK をもとにした JDK は、さまざまな団体や企業から提供されています。
たとえば、次のようなものがあります。

- Oracle JDK[^oracle-jdk]
- Eclipse Temurin（テムリン）[^temurin]
- Amazon Corretto（コレット）[^corretto]
- Azul Zulu（ズール）[^zulu]

名前は違っても、これらはいずれも OpenJDK を土台にしているため、基本的な動作は共通しています。
そのため、初めて学ぶうちは「どれを選ぶか」で深く悩む必要はありません。

どの JDK を選ぶかについては、次の第2章であらためて案内します。

## まとめ

この節では、JVM・JRE・JDK の違いを整理しました。

- この3つは「入れ子」の関係になっている（JDK ⊃ JRE ⊃ JVM）
- **JVM** は、バイトコードを実行する中心的な係である
- **JRE** は、プログラムを「動かす」ための一式（JVM ＋ 標準ライブラリなど）である
- **JDK** は、プログラムを「作る」ための一式（JRE ＋ コンパイラなどの開発ツール）である
- これから Java を学ぶ人は、**JDK をインストールすればよい**（実行に必要なものもすべて含まれている）
- Java 11 以降、単独の JRE は配布されなくなり、JDK を入れるのが基本となった
- 現在の多くの JDK は、オープンソースの **OpenJDK** を土台にしており、さまざまな提供元がある

3つの言葉の関係がわかれば、Java を始める準備はもうすぐです。
