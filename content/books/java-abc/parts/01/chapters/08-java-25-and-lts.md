## Java 25 について ― LTS とリリースサイクル

本書が対象とするのは、**Java 25** というバージョンです。

なぜ Java 25 を選ぶのでしょうか。
そして、よく聞く「LTS」や「リリースサイクル」とは、いったい何を意味するのでしょうか。

この章の最後の節では、Java のバージョンにまつわる話を整理します。
これがわかると、なぜ本書が Java 25 を選んでいるのか、その理由がはっきりします。

---

## Java はバージョンを重ねてきた

「Java の誕生と歩み」の節でも触れたように、Java は1996年の最初の版から、何度もバージョンアップを重ねてきました。

初期のバージョンは「1.0」「1.1」…「1.4」と数えられていましたが、2004年の **Java 5** 以降は、「5」「6」「7」…というシンプルな番号で呼ばれるようになりました。

そして現在、最新の長期サポート版が **Java 25** です。
番号が大きいほど新しいバージョンであり、新しい機能や改善が取り入れられています。

---

## 半年ごとの新バージョン ― リリースサイクル

Java のバージョンアップには、現在、決まったリズムがあります。

**2017年に公開された Java 9 以降、Java は半年ごと（3月と9月）に新しいバージョンを公開する**ようになりました。

この「どのくらいの間隔で新しいバージョンを出すか」というリズムのことを、**リリースサイクル**（Release Cycle）と呼びます。

半年ごとに新バージョンが出るため、Java のバージョン番号は速いペースで増えていきます。

```text
… → Java 21（2023年9月）
   → Java 22（2024年3月）
   → Java 23（2024年9月）
   → Java 24（2025年3月）
   → Java 25（2025年9月）→ …
```

このように、Java は半年ごとに着実に進化を続けています。

---

## すべてのバージョンが長く使えるわけではない

ここで、ひとつ大切な点があります。

半年ごとに公開される新バージョンですが、**そのすべてが長期間サポートされるわけではありません**。

多くのバージョンは、次の新バージョンが出るまでの**約半年間**だけ、公式の更新（不具合の修正など）が提供されます。
半年後に次のバージョンが出ると、サポートはそちらへ引き継がれていきます。

このような、半年間でサポートが切り替わっていくバージョンを、**フィーチャーリリース**（Feature Release、機能リリース）と呼びます。

フィーチャーリリースは、最新の機能をいち早く試したい人にとっては魅力的です。
しかし、長く使い続けるシステムにとっては、半年ごとにバージョンを上げ続けなければならず、負担が大きくなってしまいます。

そこで登場するのが、**LTS** という考え方です。

---

## LTS ― 長く使える特別なバージョン

**LTS**（Long-Term Support、長期サポート）とは、**特に長い期間サポートされる、特別なバージョン**のことです。

半年ごとに公開されるバージョンのうち、いくつかが LTS として選ばれます。
LTS に選ばれたバージョンは、何年にもわたって不具合の修正などの更新が提供されます。

LTS には、次のような利点があります。

- **長期間、安心して使い続けられる** ： 何年もサポートが続くため、頻繁にバージョンを上げる必要がない
- **安定している** ： 多くの利用者が長く使うため、情報が豊富で、トラブルにも対処しやすい
- **学習の土台にしやすい** ： 長く使われるため、学んだ知識が長持ちする

企業の重要なシステムの多くは、安定性を重視して、この LTS を選びます。

---

## これまでの LTS と、その間隔

これまでに LTS として選ばれてきた主なバージョンは、次のとおりです。

| バージョン | 公開時期 | 備考 |
|---|---|---|
| Java 8 | 2014年 | 長く使われてきた古くからの LTS |
| Java 11 | 2018年 | 新しいリリース方式での最初の LTS |
| Java 17 | 2021年 | |
| Java 21 | 2023年 | |
| Java 25 | 2025年 | 本書が対象とする最新の LTS |

これを年表にすると、次のようになります。

![これまでの LTS の歩み。Java 8（2014年）・11（2018年）・17（2021年）・21（2023年）・25（2025年）。Java 17 以降は 2 年ごとに公開されている。](./images/lts-timeline.svg)

LTS が公開される間隔も、少しずつ見直されてきました。

当初、LTS はおよそ3年ごとに公開される予定でした（Java 11 から Java 17 までが約3年です）。
その後、3年は長すぎるという声を受けて、**Java 17 以降は2年ごと**に見直されました。
そのため、Java 17（2021年）→ Java 21（2023年）→ Java 25（2025年）と、2年おきに LTS が公開されています。

このペースが続けば、次の LTS は2027年ごろに公開される見込みです。

---

## 本書が Java 25 を選ぶ理由

ここまでの話をふまえると、本書が **Java 25** を選んでいる理由が見えてきます。

- **最新の LTS だから** ： 2025年9月に公開された、もっとも新しい長期サポート版である
- **長く使えるから** ： LTS なので、これから何年も使い続けられる。学んだ知識が長持ちする
- **新しい機能を学べるから** ： 最新版なので、これまでに積み重ねられた便利な機能をすべて学べる

実際、Java 25 は2025年9月16日に正式公開された LTS であり、長期にわたるサポートが予定されています。
これから Java を学び始めるなら、長く使えて、かつ最新の機能まで学べる Java 25 は、最適な選択肢のひとつです。

> **補足: 新機能のまとまり「JEP」**
>
> Java の新しいバージョンには、たくさんの改善が盛り込まれます。
> こうした改善は、**JEP**（Java Enhancement Proposal、Java 改善提案）という単位で提案・検討されます。
> Java 25 には、18件の JEP が取り入れられました。
> JEP という言葉は、これから Java の情報を調べるときに目にすることがあるので、頭の片隅に置いておくとよいでしょう。

---

## Java 25 は、初学者にもやさしくなった

Java 25 には、これからプログラミングを学ぶ人にとって、とくにうれしい改善が含まれています。

これまで、Java で「画面に文字を表示するだけ」の簡単なプログラムを書くにも、次のように書く必要がありました。

```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

初めて見ると、`public` や `class`、`static`、`String[]` など、意味のわからない言葉がたくさん並んでいます。
本当にやりたいのは「Hello, World! と表示する」ことだけなのに、そのまわりに多くの決まり文句が必要だったのです。

Java 25 では、同じ内容を次のように書けるようになりました。

```java
void main() {
    IO.println("Hello, World!");
}
```

ずいぶんすっきりしました。
`class` を書く必要がなくなり、`public` や `static`、`String[]` といった言葉も省けるようになっています。

このしくみは、**コンパクトソースファイル**と**インスタンスメインメソッド**と呼ばれる、Java 25 で正式に導入された新しい機能によるものです（先ほど触れた JEP のひとつ、JEP 512 にあたります）。
プログラミングを学び始めたばかりの人が、最初から多くの決まり文句に戸惑わずにすむように、という配慮から生まれた機能です。

> **補足: `IO.println` の `IO.` は省略できません**
>
> 新しい書き方では `IO.println(...)` のように書きます。
> この `IO.` の部分は、現時点では省略せずに付ける必要があります。
> 細かな理由は今は気にしなくて大丈夫です。「文字を表示するときは `IO.println(...)` と書く」と覚えておけば十分です。

本書では、これらの書き方を順を追って紹介していきます。
従来の書き方（`public class …`）も、Java を理解するうえで大切な基礎となるため、あわせて学んでいきます。

## 自分の Java のバージョンを確認するには

実際に Java をインストールしたあとで、自分のコンピュータに入っている Java のバージョンを確認するには、次の命令を使います。

```text
java --version
```

これを実行すると、インストールされている Java のバージョンが表示されます。
たとえば、Java 25 が入っていれば、`25` という数字を含む情報が表示されます。

実際にインストールして確認する手順は、次の第2章「環境構築」でくわしく説明します。
ここでは、「`java --version` でバージョンを確認できる」とだけ覚えておいてください。

## まとめ

この節では、Java 25 と、バージョンにまつわる考え方を学びました。

- Java は、**2017年の Java 9 以降、半年ごと（3月と9月）に新バージョンを公開**している。このリズムを**リリースサイクル**と呼ぶ
- 半年ごとのバージョン（**フィーチャーリリース**）は、次の版が出るまでの約半年間だけサポートされる
- そのなかで、特に長くサポートされる特別なバージョンが **LTS**（長期サポート版）である
- これまでの LTS は、Java 8・11・17・21・25 で、現在は2年ごとに公開されている
- **Java 25** は、2025年9月16日に公開された最新の LTS であり、長期サポートが予定されている
- 本書が Java 25 を選ぶのは、「最新の LTS で長く使え、最新の機能まで学べる」からである
- Java 25 では、初学者向けに、より簡潔にプログラムを書ける機能（コンパクトソースファイルなど）が導入された
- 自分の Java のバージョンは、`java --version` で確認できる

これで、第1章「プログラミングとは・Java とは」は終わりです。

ここまでで、プログラムとは何か、コンピュータの仕組み、Java の歴史・特徴・用途、そして Java が動く仕組みと必要な道具について学びました。
いよいよ次の第2章では、実際に Java を使う準備 ―― **環境構築**に取りかかります。
自分のコンピュータに JDK をインストールし、最初のプログラムを動かす準備を整えていきましょう。
