## 用語集 ― この章で学んだ言葉

この第1章では、たくさんの新しい言葉が登場しました。

ここでは、それらの用語を**節ごとに**まとめておきます。
「あの言葉は何だったかな」と思ったときに、辞書のように使ってください。

すべてを覚える必要はありません。
本書を読み進めるなかで、わからなくなったらここに戻ってくる ―― そんな使い方で十分です。

---

## プログラミングの基礎（第1節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| プログラム | Program | コンピュータに実行してほしい処理を、決められた形式で書いた命令の集まり |
| プログラミング | Programming | プログラムを書く作業 |
| プログラミング言語 | Programming Language | プログラムを書くために使う言葉。Java もその一つ |
| ソースコード | Source Code | 人間がプログラミング言語で書いたプログラムそのもの |
| 入力・処理・出力 | Input / Process / Output | 多くのプログラムに共通する基本の流れ |
| 条件分岐 | ― | 条件によって、実行する処理を変える考え方 |
| 繰り返し（ループ） | Loop | 同じ処理を何度も実行する考え方 |
| 無限ループ | ― | 終わる条件を誤り、いつまでも終わらなくなった繰り返し |
| データ | Data | プログラムが扱う情報（名前・点数・金額など） |
| アルゴリズム | Algorithm | 問題を解決するための手順や考え方 |
| バグ | Bug | プログラムが期待どおりに動かない原因となる誤り |
| デバッグ | Debug | バグを見つけて修正する作業 |
| 写経（しゃきょう） | ― | 見本のコードを、自分の手で書き写して学ぶ学習方法 |

## コンピュータの仕組み（第2節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| CPU | Central Processing Unit | 命令を1つずつ高速に実行する「頭脳」（料理人にあたる） |
| コア | Core | CPU の中で実際に処理を行う部分。複数あると同時に処理を進められる |
| クロック周波数 | Clock Frequency | CPU が動作する速さの目安。単位は Hz（ヘルツ） |
| メモリ | Memory（RAM） | 作業中のデータを一時的に置く高速な場所（調理台にあたる） |
| ストレージ | Storage | データを長期間しまっておく大容量の場所（冷蔵庫・倉庫にあたる） |
| 揮発性（きはつせい） | Volatile | 電源を切ると内容が消える性質。メモリの特徴 |
| 不揮発性（ふきはつせい） | Non-volatile | 電源を切っても内容が残る性質。ストレージの特徴 |
| ビット | bit | 0 または 1 で表される、情報の最小単位 |
| バイト | byte | ビット8個分のまとまり |
| 機械語（マシン語） | Machine Code | CPU が直接理解できる、0 と 1 で書かれた命令 |

## Java という言語（第3〜5節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| プラットフォーム | Platform | OS（Windows・Mac・Linux など）や機種といった、プログラムが動く環境 |
| プラットフォーム非依存 | Platform Independence | 環境を選ばずに、同じプログラムを動かせる性質 |
| Write Once, Run Anywhere | ― | 「一度書けば、どこでも動く」という Java を象徴するスローガン |
| オブジェクト指向 | Object-Oriented | プログラムを、データと操作をまとめた「部品」の集まりとして組み立てる考え方 |
| 型 | Type | データの種類（数値・文字など） |
| 静的型付け | Static Typing | プログラムを実行する前に、型の誤りをチェックする仕組み |
| 堅牢性（けんろうせい） | Robustness | 予期せぬ原因で止まりにくく、安定して動きやすい性質 |
| ガベージコレクション | Garbage Collection（GC） | 使い終わったメモリの後片付けを、自動で行う仕組み |
| ライブラリ | Library | あらかじめ用意された、便利な機能のまとまり |
| 標準ライブラリ | ― | Java に最初から付属しているライブラリ |
| 後方互換性（こうほうごかんせい） | Backward Compatibility | 古いプログラムが、新しい環境でもそのまま動きやすい性質 |
| サーバーサイド | Server-Side | 利用者からは見えない、サーバー側で動くプログラム |
| フレームワーク | Framework | 開発を効率化するための土台となる道具（Spring など） |

## Java が動く仕組み（第6〜7節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| コンパイル | Compile | 人間が書いたソースコードを、別の形（バイトコード）に変換すること |
| コンパイラ | Compiler | コンパイルを行う道具。Java では `javac` |
| バイトコード | Bytecode | コンパイルで作られる、環境に依存しない中間的な命令（`.class` ファイル） |
| JVM | Java Virtual Machine | バイトコードを読み込み、その環境の機械語に変換しながら実行する仕組み |
| JRE | Java Runtime Environment | プログラムを「動かす」ための一式（JVM ＋ 標準ライブラリなど） |
| JDK | Java Development Kit | プログラムを「作る」ための一式（JRE ＋ コンパイラなどの開発ツール） |
| OpenJDK | ― | オープンソース化された Java の実装。現在の多くの JDK の土台 |
| インタプリタ | Interpreter | 命令を1つずつ解釈しながら実行する方式 |
| JIT コンパイラ | Just-In-Time Compiler | よく使う部分を実行時に機械語へ変換し、高速に実行する仕組み |

## バージョンと開発（第8節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| リリースサイクル | Release Cycle | 新しいバージョンを公開する間隔。Java 9 以降は半年ごと（3月・9月） |
| フィーチャーリリース | Feature Release | 約半年でサポートが次の版に切り替わる、通常のバージョン |
| LTS | Long-Term Support | 長期間サポートされる特別なバージョン（Java 8・11・17・21・25） |
| JEP | Java Enhancement Proposal | Java の改善を提案・検討する単位（Java 改善提案） |
| コンパクトソースファイル | ― | `class` 宣言などを省き、簡潔にプログラムを書ける Java 25 の新機能 |
| インスタンスメインメソッド | ― | `public static` を付けずに `main` を書ける Java 25 の新機能 |
