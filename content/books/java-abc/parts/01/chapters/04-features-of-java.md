## Java の特徴

世の中には、たくさんのプログラミング言語があります。
そのなかで、Java はどのような特徴を持つ言語なのでしょうか。

この節では、Java を代表する特徴を見ていきます。

専門的な用語も出てきますが、いまの段階ですべてを深く理解する必要はありません。
「Java にはこういう良いところがあるのか」と、雰囲気をつかむつもりで読み進めてください。
それぞれの特徴は、本書を読み進めるなかで、少しずつ実感できるようになります。

---

## どこでも動く（プラットフォーム非依存）

Java のもっとも有名な特徴は、**一度書いたプログラムを、さまざまな環境で動かせる**ことです[^java-platform-independence]。

ここでいう「環境」とは、Windows・Mac・Linux といった OS（オペレーティングシステム）の種類や、コンピュータの機種のことです。
こうした動作環境のことを、**プラットフォーム**（Platform）と呼びます。

ふつう、プログラムは特定のプラットフォーム向けに作られるため、環境が変わると動かないことがよくあります。
ところが Java は、書いたプログラムを多くのプラットフォームでそのまま動かせます。
この性質を、**プラットフォーム非依存**（Platform Independence）と呼びます。

前の節でも登場した、有名なスローガンで表現すると、次のようになります。

```text line-numbers=false
Write Once, Run Anywhere
（一度書けば、どこでも動く）
```

なぜこのようなことが可能なのかは、後の節「Java の動く仕組み」でくわしく説明します。
ここでは、**Java で書いたプログラムは特定の環境に縛られにくい**、と覚えておいてください。

## オブジェクト指向の言語である

Java は、**オブジェクト指向**（Object-Oriented）という考え方をもとに作られた言語です。

オブジェクト指向とは、プログラムを「データ」と「そのデータに対する操作」をひとまとめにした**部品**の集まりとして組み立てていく考え方です。

たとえば、買い物アプリを作るとします。
このとき、「商品」「カート」「注文」といったものを、それぞれ部品として表現し、それらを組み合わせてプログラムを作っていきます。

この考え方には、次のような利点があります。

- 大きなプログラムを、意味のあるまとまりに分けて整理できる
- 同じような部品を、別の場所でも再利用しやすい
- 修正や機能追加をするときに、影響する範囲を把握しやすい

オブジェクト指向は、最初は少し難しく感じられる考え方です。
本書では、第2部でじっくりと時間をかけて学んでいきます。
いまは、「Java は、プログラムを部品の組み合わせとして作っていく言語なのだ」と理解しておけば十分です。

## まちがいに気づきやすい（静的型付け）

Java は、データの種類を**型**（Type）としてはっきり区別する言語です。

たとえば、`100`（数値）と `"100"`（文字）は、人間の目には似て見えても、Java では別の種類のデータとして扱われます。

そして Java では、プログラムを実行する前の段階で、「数値を入れるべき場所に、誤って文字を入れていないか」といったことが自動的にチェックされます。
このように、実行する前に型のまちがいを見つけられる仕組みを、**静的型付け**（Static Typing）と呼びます[^jls-types]。

この仕組みには、大きな利点があります。

- プログラムを動かす前に、多くのまちがいに気づける
- 「ここには数値が入る」とはっきりしているため、コードが読みやすくなる
- 開発を助ける道具（IDE）が、入力ミスや誤りを指摘してくれる

まちがいは、後になって見つかるほど、原因を探すのが大変になります。
Java は、まちがいをできるだけ早い段階で教えてくれる、慎重で堅実な言語だといえます。

> **補足: 型については後でくわしく学びます**
>
> 「型」は、プログラミングを学ぶうえでとても大切な考え方です。
> 第4章「変数とデータ型」で、あらためてていねいに説明します。

## 安定して動きやすい（堅牢性）

Java は、プログラムが予期せぬ原因で突然止まってしまうことを、できるだけ防ぐように設計されています。
このような「壊れにくさ」「安定して動きやすい性質」を、**堅牢性**（けんろうせい、Robustness）と呼びます。

堅牢性を支える仕組みには、たとえば次のようなものがあります。

- **自動的なメモリ管理** ： 使い終わったメモリの後片付けを、Java が自動で行ってくれる（後述）
- **例外処理のしくみ** ： エラーが起きたときに、プログラムをいきなり止めるのではなく、適切に対応するための仕組みが用意されている
- **きびしい型のチェック** ： 先ほど説明した静的型付けにより、あやしい処理を事前に防ぐ

これらの仕組みのおかげで、Java で書かれたプログラムは、長時間動かし続けても安定しやすいという特徴があります。
銀行や企業の重要なシステムなど、止まると困る場面で Java が選ばれてきたのは、この堅牢性が理由のひとつです。

---

## メモリの後片付けを任せられる（ガベージコレクション）

プログラムが動くとき、データはメモリ（前の節で学んだ「調理台」）の上に置かれます。

使い終わったデータをそのまま放置すると、調理台がどんどん散らかっていき、やがて新しい作業をする場所がなくなってしまいます。
そのため、本来は「もう使わなくなったデータが占めているメモリ」を、適切に片付ける必要があります。

言語によっては、この後片付けをプログラマー自身が手作業で行わなければなりません。
これはとても手間がかかるうえ、片付けを忘れたり、まちがえたりすると、不具合の原因になります。

Java では、この**メモリの後片付けを自動で行う仕組み**が備わっています。
これを **ガベージコレクション**（Garbage Collection、ごみ集め）と呼びます[^jvms-gc]。

ガベージコレクションのおかげで、Java を使うプログラマーは、メモリの後片付けをほとんど意識せずにすみます。
本来やりたい処理に集中できるのです。

## たくさんの便利な道具が最初からそろっている

プログラムを書くとき、私たちはすべてをゼロから作るわけではありません。
よく使う機能は、あらかじめ用意されたものを利用します。

このような「あらかじめ用意された便利な機能のまとまり」を、**ライブラリ**（Library）と呼びます。

Java には、**標準ライブラリ**と呼ばれる豊富な道具一式が、最初から付属しています。
たとえば、次のような機能が標準で使えます。

- 文字や数値を扱う機能
- たくさんのデータをまとめて管理する機能
- 日付や時刻を計算する機能
- ファイルを読み書きする機能
- ネットワークを通じて通信する機能

さらに、世界中の開発者が作った膨大な数のライブラリも利用できます。
このように、すぐに使える道具が豊富にそろっていることも、Java の大きな強みです。

「やりたいことを実現する道具が、たいてい見つかる」というのは、開発をするうえでとても心強いことです。

---

## 古いプログラムが動き続けやすい（後方互換性）

Java は、**過去に書かれたプログラムを、新しい環境でもできるだけそのまま動かせる**ことを、長年とても大切にしてきました。

この性質を、**後方互換性**（こうほうごかんせい、Backward Compatibility）と呼びます。

新しいバージョンが出るたびに古いプログラムが動かなくなってしまうと、利用者はバージョンアップをためらってしまいます。
書き直しに大きな手間がかかるからです。

Java は後方互換性を重視してきたため、何年も前に書かれたプログラムが、今の Java でも動くことが少なくありません。
この安心感は、長期間にわたって使われるシステムにとって、非常に重要な価値です。
企業が安心して Java を採用できる、大きな理由のひとつになっています。

---

## まとめ

この節では、Java という言語の代表的な特徴を学びました。

- **プラットフォーム非依存** ： 一度書けば、さまざまな環境で動かせる
- **オブジェクト指向** ： プログラムを部品の組み合わせとして組み立てていく
- **静的型付け** ： データの種類を型で区別し、実行前にまちがいに気づける
- **堅牢性** ： エラーや不具合に強く、安定して動きやすい
- **ガベージコレクション** ： 使い終わったメモリの後片付けを自動で行う
- **豊富なライブラリ** ： すぐに使える便利な道具が最初からそろっている
- **後方互換性** ： 古いプログラムが新しい環境でも動き続けやすい

これらの特徴により、Java は「学びやすく、安定していて、長く使える」言語として、世界中で選ばれ続けています。
