## はじめに ― この章で学ぶこと

**Proxy**（プロキシ、代理人）パターンは、**「本物の代わりに振る舞う代理オブジェクト」**を立てることで、

- アクセス制御
- 遅延ロード
- 通信の隠蔽
- ログや計測の差し込み

などを行うパターンです。

呼び出し側からは、本物と Proxy の**区別はつきません**。同じインターフェースを実装しているからです。
Spring の AOP（Aspect Oriented Programming）も、内部では Proxy を生成しています。

---

## この章を読み終えると

- Proxy の**4 つの主要な種類**を区別できる
- 仮想 Proxy（遅延ロード）を自分で書ける
- Java の**動的プロキシ**の使いどころを言える
