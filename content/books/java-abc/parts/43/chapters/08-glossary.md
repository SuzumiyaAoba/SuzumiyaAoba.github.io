## 用語集 ― この章で学んだ言葉

### Java メモリモデル（Java Memory Model、JMM）

スレッド間で、データがどう見えるかを定める **JLS §17** の規定。
Java 5（JSR-133）で抜本的に整理された。

### 可視性（visibility）

あるスレッドの書き込みが、他のスレッドからいつ・どう見えるかという問題。

### 順序 / 並べ替え（ordering / reordering）

書いた順番どおりに他のスレッドから見えるかという問題。
JIT・CPU は最適化のために命令を**並べ替える**ことが許される。

### アトミシティ（atomicity）

操作が「**途中で割り込まれずに 1 つの単位として完了する**」性質。
プリミティブの `++`・`+=` はアトミックではない。

### happens-before（→）

JMM が定める「**先に起きた**」関係。
A → B のとき、A の効果は B から**必ず見える**ことが保証される。
プログラム順序・モニタロック・volatile・start/join・final・推移律から成る。

### データレース（data race）

happens-before の道がつながっていない 2 つのアクセスで、少なくとも一方が書き込みである状態。
未定義の振る舞いを引き起こす。

### `volatile`

フィールド宣言に付けて、**可視性**と**並べ替え制限**を保証する修飾子。
**複合操作**（`++` など）はアトミックにならない。

### メモリバリア（memory barrier）

CPU の命令で、キャッシュとメインメモリを同期し、並べ替えを制限するもの。
`volatile` や `synchronized` の解放/取得で挿入される。

### `synchronized`

ブロックまたはメソッドに付ける、**排他制御**と **happens-before の橋**を提供する仕組み。
**再入可能**（同じスレッドがロックを再取得できる）。

### モニタロック / 内部ロック（intrinsic lock）

`synchronized` が使う、**任意のオブジェクトが持つロック**のこと。
`this`、`Class<?>`、または **private final Object** をロック先として使う。

### デッドロック（deadlock、膠着状態）

複数のロックを **逆順**で取り合うと起きる「**互いに相手の解放を待ち続ける**」状態。
`jstack` で検出できる。
予防は**ロック取得順序の一貫**。

### `ReentrantLock`

`java.util.concurrent.locks` の高機能ロック。
**`tryLock`**、**タイムアウト**、**割り込み可能**、**公平モード**などが使える。
**`try-finally` で `unlock()` を必ず呼ぶ**。

### CAS（Compare And Swap、比較交換）

「**期待値と一致したら新しい値に書き換える**」を 1 命令でアトミックに行う CPU 操作。
`AtomicXxx` の中で使われる。

### `AtomicInteger` / `AtomicLong` / `AtomicReference`

CAS を使った、ロックなしの整数・参照などの安全な更新を提供する型。
`incrementAndGet()`、`updateAndGet(...)`、`compareAndSet(...)` などを持つ。

### `LongAdder`

高頻度カウンタ用の `AtomicLong` の代替。
内部で**複数セル**に分散して書き、合計時に合算する。
**書き込みは速い**が、`sum()` は O(N)。

### オプティミスティック・コンカレンシー（optimistic concurrency）

「**競合は稀**」と楽観し、競合したら**リトライ**する並行制御の発想。
CAS の動作原理。

### ABA 問題

CAS の落とし穴で、値が `X → Y → X` と変わると「**書き換えがなかった**」ように見えてしまう問題。
`AtomicStampedReference` でバージョン番号を併用して回避する。

### 不変オブジェクト（immutable object）

生成後、状態が一切変わらないオブジェクト。
**すべてのフィールドが `final`**、**コンストラクタで防衛的にコピー**、**ゲッターも安全な値を返す**。
レコードは不変クラスの**標準形**。

### `record`

Java 14 以降の不変クラス専用構文。フィールド・コンストラクタ・ゲッター・`equals`・`hashCode`・`toString` を自動生成。

### 防衛的コピー（defensive copy）

可変オブジェクト（`Date`・`List` など）をフィールドに持つとき、**コンストラクタ**と**ゲッター**で**コピーを作る**こと。
外部からの改ざんを防ぐ。

### `with` メソッド

不変オブジェクトの「1 つだけ変えた新しいインスタンス」を返すメソッド。
`copy with modifications` のイディオム。

### 二重チェックロック（DCL、Double-Checked Locking）

「ロックを最小化するため、ロック内外で 2 回 `null` チェックする」遅延初期化パターン。
**`volatile` 付き**でないと壊れる。
新規コードでは `enum` シングルトンが推奨。

### `ConcurrentHashMap`

`java.util.concurrent` の、内部でセグメント化された CAS を使う**スレッドセーフな Map**。
読み多 / 書き少の用途に向く。

### `Collections.synchronizedMap` / `Collections.unmodifiableList`

既存コレクションを**ラップ**してスレッドセーフに / 不変ビューにするユーティリティ。
`unmodifiableList` は**ビュー**で、元のリストが変わると見え方も変わる。
**真の不変が欲しいなら `List.copyOf` を使う**。

### スレッド安全性の階層

1. **イミュータブル** ―― 不変なので、何もしなくて安全
2. **スレッドセーフ** ―― 内部で同期、使う側は何もしなくていい
3. **条件付きスレッドセーフ** ―― 単一操作は安全、複数操作の組み合わせは外部同期が要る
4. **スレッドホスタイル / 単スレッド** ―― 並行アクセス不可

### 並行処理設計の原則

> **共有するなら不変、変えるなら共有しない。**

スレッド間で共有するなら不変オブジェクト、変えたいならそのスレッド専用にする。
