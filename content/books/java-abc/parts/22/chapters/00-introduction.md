## はじめに ― この章で学ぶこと

前の第21章では、コレクションを学びました。
その最後に、こんな1行が出てきたのを覚えているでしょうか。

```java
names.removeIf(name -> name.equals("佐藤"));   // 条件に合うものを削除
```

この `name -> name.equals("佐藤")` という、見慣れない `->`（矢印）。
これが、この章で学ぶ **ラムダ式**（Lambda Expression）です。

ラムダ式は、ひとことで言えば、**「処理（ふるまい）そのものを、値のように受け渡す」**ためのしくみです。
これまで、メソッドに渡せるのは「データ（値）」だけでした。
ラムダ式を使うと、「**こういう処理をしてほしい**」という**処理の中身**を、メソッドに渡せるようになります。

この発想は、最初は少し不思議に感じるかもしれません。
ですが、これこそが、次の第23章「ストリーム API」をはじめ、**モダン Java の中心**となる考え方です。
この章で、その土台をしっかり作りましょう。

---

## なぜラムダ式が必要なのか

たとえば、「リストを並べ替える」場面を考えます。
「どういう順番で並べるか」という**ルール（処理）**を、並べ替えメソッドに教えたい ―― そんなとき、ラムダ式が活躍します。

```java
List<String> names = new ArrayList<>(List.of("Charlie", "Alice", "Bob"));
names.sort((a, b) -> a.compareTo(b));   // 「a と b をこう比べてね」という処理を渡す
IO.println(names);
```

```text
[Alice, Bob, Charlie]
```

`sort` に渡した `(a, b) -> a.compareTo(b)` が、「2つを比べるルール」です。
このように、「**何をするか**」をその場でさっと書いて、メソッドに渡せる ―― それがラムダ式の便利さです。

実は、ラムダ式が登場する前（Java 8 より前）は、同じことを、もっと長々と書く必要がありました。
ラムダ式は、その「決まり文句」を、すっきり短く書くために生まれたのです（くわしくは第1節）。

## この章で学ぶこと

第22章は、次の7つの節で構成されています。

| 節 | タイトル | 内容 |
|---|---|---|
| 1 | ラムダ式とは | 処理を値として渡す。匿名クラスとの比較 |
| 2 | 関数型インターフェース | ラムダ式が実装する「型」の正体 |

## この章を読み終えると

第22章を読み終えるころには、次のことができるようになっています。

- ラムダ式が「処理を値として渡すしくみ」だと理解できる
- 関数型インターフェースと、ラムダ式の関係がわかる
- ラムダ式を、引数や本体の省略ルールも含めて書ける
- `Function`・`Predicate`・`Consumer`・`Supplier` を使い分けられる
- メソッド参照（`String::length` など）を読み書きできる

> **補足: ラムダ式は Java 8 から**
>
> ラムダ式と、この章で学ぶ関数型インターフェースは、**Java 8**（2014年）で導入されました。
> それ以前の Java には、この書き方はありませんでした。
> いまでは、Java を書くうえで欠かせない、基本中の基本となっています。

それでは、最初の節「ラムダ式とは」から始めましょう。
