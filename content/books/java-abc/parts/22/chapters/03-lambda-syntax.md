## ラムダ式の書き方 ― 省略のルール

ラムダ式は、状況に応じて、いろいろな書き方ができます。
「引数の書き方」「本体の書き方」には、いくつかの省略ルールがあり、慣れると、とても短く書けます。
この節では、その書き方を整理します。

---

## 基本の形

ラムダ式の、いちばん基本的な形は、次のとおりです。

```text
(引数の型 引数名, ...) -> { 処理; }
```

たとえば、「2つの整数を足す」ラムダ式を、省略なしで書くと、こうなります。

```java
(int a, int b) -> { return a + b; }
```

- `(int a, int b)` … 引数（型つき）
- `-> { ... }` … 本体（処理）
- `return a + b;` … 戻り値を返す

これがフル装備の形ですが、実際には、多くの部分を**省略**できます。

---

## 省略その1 ― 引数の型を省く

引数の型は、**省略できます**。
ラムダ式を代入する関数型インターフェースから、Java が型を**推論**してくれるからです。

```java
// 型あり
Calculator add = (int a, int b) -> { return a + b; };

// 型を省略（推論される）
Calculator add = (a, b) -> { return a + b; };
```

`Calculator` のメソッドが `int calculate(int a, int b)` だと分かっているので、`a` と `b` が `int` だと、Java が自動で判断します。
実際の開発では、**型を省略するのが一般的**です。

---

## 省略その2 ― 本体が1つの式なら、`{ }` と return を省く

本体が「**1つの式（値を計算するだけ）**」のときは、`{ }` と `return` を省略できます。

```java
// { } と return あり
Calculator add = (a, b) -> { return a + b; };

// 省略（1つの式なら、その値がそのまま戻り値になる）
Calculator add = (a, b) -> a + b;
```

`-> a + b` のように書くと、「`a + b` を計算して、その結果を返す」という意味になります。
`return` を書かなくても、式の値が、自動的に戻り値になるのです。
これが、もっともよく見る、すっきりした形です。

ただし、本体が**複数の文**になるなら、`{ }` が必要で、`return` も自分で書きます。

```java
Calculator add = (a, b) -> {
    int sum = a + b;
    IO.println("計算します: " + sum);
    return sum;            // 複数の文のときは return が必要
};
```

---

## 省略その3 ― 引数が1つなら、`( )` を省く

引数が**ちょうど1つ**のときは、引数を囲む `( )` を省略できます。

```java
// ( ) あり
Predicate<Integer> isEven = (n) -> n % 2 == 0;

// ( ) を省略（引数1つのときだけ）
Predicate<Integer> isEven = n -> n % 2 == 0;
```

`n -> ...` のように、引数名だけを書けます。
（`Predicate` は、次の節で学ぶ「条件を判定する」関数型インターフェースです。`test(...)` で `true`/`false` を返します。）

ただし、**引数がない**ときと、**2つ以上**のときは、`( )` が必要です。

```java
() -> IO.println("やあ")     // 引数なし → ( ) が必要
n -> n * 2                   // 引数1つ → ( ) は省略できる
(a, b) -> a + b              // 引数2つ → ( ) が必要
```

---

## 書き方のまとめ

これまでの省略ルールを、表で整理します。

| 場面 | 書き方 |
|---|---|
| 引数なし | `() -> 処理` |
| 引数1つ | `x -> 処理`（`( )` 省略可） |
| 引数2つ以上 | `(x, y) -> 処理` |
| 本体が1つの式 | `-> 式`（`{ }`・`return` 省略） |
| 本体が複数の文 | `-> { 文; 文; return 値; }` |
| 引数の型 | ふつうは省略（推論される） |

迷ったら、「**まずフル装備で書き、省略できるところを削る**」と考えるとよいでしょう。
慣れてくると、`n -> n * 2` や `(a, b) -> a + b` のような、短い形が自然に書けるようになります。

---

## 実際に動かしてみる

いくつかのラムダ式を、実際に使ってみましょう。

```java
import java.util.function.*;

Predicate<Integer> isEven = n -> n % 2 == 0;        // 偶数か判定
IO.println(isEven.test(4));
IO.println(isEven.test(5));

Function<String, Integer> length = s -> s.length(); // 文字数を返す
IO.println(length.apply("hello"));
```

```text
true
false
5
```

`isEven.test(4)` は「4 は偶数か」で `true`、`length.apply("hello")` は「文字数」で `5` です。
ラムダ式で書いた処理が、`test` や `apply` で呼び出されています。
（`Predicate`・`Function` は、次の節でくわしく学びます。`java.util.function` パッケージにあるので、ファイルに書くときは `import` が必要です。）

---

## まとめ

- ラムダ式のフル装備は **`(型 引数, ...) -> { return 式; }`**
- **引数の型**は省略できる（関数型インターフェースから推論される）
- 本体が**1つの式**なら、**`{ }` と `return` を省略**できる（`-> a + b`）
- 引数が**1つ**なら、**`( )` を省略**できる（`n -> ...`）。引数なし・2つ以上は `( )` が必要
- 本体が**複数の文**なら、`{ }` で囲み、`return` も書く

次の節では、Java が標準で用意している、便利な関数型インターフェースを学びます。
