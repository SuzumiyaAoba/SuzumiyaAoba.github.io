## メソッド参照 ― ラムダ式をさらに短く

ラムダ式の中には、「**ただ、あるメソッドを呼ぶだけ**」のものが、たくさんあります。
そういうときは、**メソッド参照**（Method Reference）を使うと、さらに短く書けます。
この節では、その書き方を学びます。

---

## 「ただメソッドを呼ぶだけ」のラムダ式

次のラムダ式を見てください。

```java
Function<String, Integer> length = s -> s.length();
```

これは、「受け取った `s` の `length()` を呼ぶ**だけ**」です。
`s` を受け取って、それに対して `length()` を呼ぶ、という以上のことは、何もしていません。

こういう「ただメソッドを呼ぶだけ」のラムダ式は、**メソッド参照**で、もっと短く書けます。

```java
Function<String, Integer> length = String::length;
```

`String::length` ―― これがメソッド参照です。
「`String` の `length` メソッドを、そのまま使う」という意味で、`s -> s.length()` と**まったく同じ**働きをします。

書き方は、**`型名::メソッド名`**（コロンを2つ）です。
ラムダ式の「引数を受け取って、それに対してメソッドを呼ぶ」という部分が、まるごと省けるのです。

```java
Function<String, Integer> length = String::length;
IO.println(length.apply("hello"));
```

```text
5
```

---

## いちばんよく見る形 ― forEach での出力

メソッド参照が、もっともよく登場するのは、`forEach` での出力です。

```java
List<String> names = List.of("佐藤", "鈴木", "高橋");

// ラムダ式
names.forEach(name -> IO.println(name));

// メソッド参照（同じ意味）
names.forEach(IO::println);
```

```text
佐藤
鈴木
高橋
```

`name -> IO.println(name)` は、「受け取った `name` を、`IO.println` に渡すだけ」です。
これを `IO::println` と書けます。「各要素を、`IO.println` で出力する」と、とても読みやすくなります。

---

## メソッド参照の4つのパターン

メソッド参照には、何を参照するかによって、いくつかのパターンがあります。
細かく覚える必要はありませんが、「こういう形がある」と知っておきましょう。

| パターン | 書き方 | ラムダ式での同等 |
|---|---|---|
| 静的メソッド | `Integer::parseInt` | `s -> Integer.parseInt(s)` |
| インスタンスメソッド（型から） | `String::length` | `s -> s.length()` |
| 特定オブジェクトのメソッド | `IO::println` | `s -> IO.println(s)` |
| コンストラクタ | `ArrayList::new` | `() -> new ArrayList<>()` |

いずれも、共通点は「**`型やオブジェクト :: 名前` で、メソッド（やコンストラクタ）をそのまま指す**」ことです。
ラムダ式が「`x -> 何かのメソッド(x)`」のように、受け取った値をそのまま渡すだけなら、メソッド参照に書きかえられる、と考えてください。

---

## 静的メソッドの参照

第4章で使った `Integer.parseInt(...)` のような静的メソッドも、参照できます。

```java
Function<String, Integer> toInt = Integer::parseInt;   // s -> Integer.parseInt(s) と同じ
IO.println(toInt.apply("123"));
```

```text
123
```

「文字列を受け取って、`Integer.parseInt` で数値に変換する」処理が、`Integer::parseInt` だけで表せました。

---

## 無理に使わなくてよい

メソッド参照は便利ですが、**無理に使う必要はありません**。
「ただメソッドを呼ぶだけ」のときに、すっきり書ける、という道具です。

少しでも処理が加わるなら、ふつうのラムダ式のままが、かえって読みやすいことも多いです。

```java
names.forEach(name -> IO.println("名前: " + name));   // 文字を足している → ラムダ式のまま
```

この場合、「`"名前: "` を足してから出力する」という処理があるので、メソッド参照にはできません（ラムダ式で書きます）。

「**ただ呼ぶだけならメソッド参照、少しでも手を加えるならラムダ式**」と、使い分ければ十分です。
読んだときに分かりやすいほうを選びましょう。

---

## まとめ

- **メソッド参照**は、「**ただメソッドを呼ぶだけ**」のラムダ式を、さらに短く書く方法
- 書き方は **`型名::メソッド名`**（`String::length`、`IO::println` など）
- `s -> s.length()` は `String::length`、`name -> IO.println(name)` は `IO::println` と書ける
- 静的メソッド・インスタンスメソッド・コンストラクタ（`ArrayList::new`）などを参照できる
- 無理に使わず、「ただ呼ぶだけ」のときに使う。手を加えるならラムダ式のままでよい
