## よくあるつまずき

ラムダ式と関数型インターフェースで、初心者がつまずきやすいポイントを、まとめて確認します。

---

## 1. ラムダ式の中から、ローカル変数を書き換える

ラムダ式の中では、外側のローカル変数を**読む**ことはできますが、**書き換える**ことはできません。
メソッドの中で、次のように書いたとしましょう。

```java
void countUp() {
    int count = 0;
    Runnable r = () -> { count++; };   // ✕ 外のローカル変数を書き換えようとする
}
```

```text
エラー:
ラムダ式から参照されるローカル変数は、finalまたは事実上のfinalである必要があります
```

ラムダ式が使える外側のローカル変数は、**実質的に final**（一度入れたら変わらない）でなければならない、というルールがあります。
これは、ラムダ式が「あとから別の場所で実行されるかもしれない」ため、変数の値が変わると、つじつまが合わなくなるからです。

> **補足: jshell では再現しないことがある**
>
> このルールは「メソッドの中の**ローカル変数**」に対するものです。
> jshell のトップレベルに直接 `int count = 0;` と打った変数は、ローカル変数ではなく**フィールドに近い扱い**になるため、ラムダ式から書き換えてもエラーになりません。
> このエラーを jshell で試すときは、上の例のように**メソッドの中に**書いてみてください。

**読むだけ**なら問題ありません。

```java
int base = 10;
Function<Integer, Integer> addBase = n -> n + base;   // ◯ base を読むだけ
IO.println(addBase.apply(5));
```

```text
15
```

「合計を数えたい」のように、値を変えながら集計したいときは、ラムダ式の外でループを書くか、第23章のストリーム（`sum` など）を使います。

---

## 2. 関数型インターフェース以外に、ラムダ式を代入する

ラムダ式を代入できるのは、**関数型インターフェース**（抽象メソッドが1つ）だけです。
`String` のようなふつうの型や、メソッドが2つ以上あるインターフェースには、代入できません。

```java
String s = () -> "hello";   // ✕ String は関数型インターフェースではない
```

```text
エラー: 不適合な型: Stringは機能インタフェースではありません
```

「このラムダ式は、何の型（どの関数型インターフェース）になるのか」を、いつも意識しましょう。
`Runnable`・`Function`・`Predicate` など、**1つの抽象メソッドを持つ型**にだけ代入できます。

---

## 3. どのメソッドを呼ぶのか、迷う

標準の関数型インターフェースは、それぞれ呼び出すメソッド名が違います。
ラムダ式を「実行する」とき、どのメソッドを呼ぶのか、最初は迷いがちです。

| インターフェース | 呼ぶメソッド |
|---|---|
| `Runnable` | `run()` |
| `Supplier<T>` | `get()` |
| `Consumer<T>` | `accept(値)` |
| `Function<T,R>` | `apply(値)` |
| `Predicate<T>` | `test(値)` |

```java
Predicate<Integer> isEven = n -> n % 2 == 0;
isEven.apply(4);   // ✕ Predicate のメソッドは test
isEven.test(4);    // ◯
```

「変換は `apply`、判定は `test`、消費は `accept`、供給は `get`」と、役割とセットで覚えると、混乱しにくくなります。

---

## 4. ラムダ式を、長く書きすぎる

ラムダ式は「短く書ける」のが魅力ですが、本体に何行も詰め込むと、かえって読みにくくなります。

```java
list.forEach(item -> {
    // 何十行もの複雑な処理…
});
```

本体が長くなりそうなときは、その処理を**ふつうのメソッドに切り出し**、メソッド参照で渡すほうが、すっきりします。

```java
list.forEach(this::処理する);   // 長い処理はメソッドにして、参照で渡す
```

「ラムダ式は、短い処理のためのもの」と心得ておきましょう。長い処理は、名前のあるメソッドにするほうが、読みやすく、再利用もできます。

---

## まとめ

- ラムダ式の中から、外のローカル変数は**読めるが書き換えられない**（実質的 final）
- ラムダ式を代入できるのは、**関数型インターフェース**（抽象メソッド1つ）だけ
- 呼ぶメソッドは型ごとに違う（`apply`・`test`・`accept`・`get`・`run`）
- ラムダ式は**短い処理**のためのもの。長くなるならメソッドに切り出す
