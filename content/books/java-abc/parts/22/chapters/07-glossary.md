## 用語集 ― この章で学んだ言葉

第22章では、処理を値として渡す「ラムダ式」と「関数型インターフェース」を学びました。
節ごとに、言葉を整理しておきます。

---

## ラムダ式（第1〜3節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| ラムダ式 | Lambda Expression | 処理を値のように渡す簡潔な書き方（`(引数) -> 処理`） |
| 匿名クラス | Anonymous Class | 名前のないクラスをその場で作る、ラムダ式以前の書き方 |
| `->` | アロー | ラムダ式で「引数を受け取って、こうする」を表す |

ラムダ式の省略：引数の型は推論、本体が1式なら `{ }`・`return` 省略、引数1つなら `( )` 省略。

---

## 関数型インターフェース（第2節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| 関数型インターフェース | Functional Interface | 抽象メソッドがちょうど1つのインターフェース。ラムダ式を代入できる |
| `@FunctionalInterface` | ― | 「抽象メソッドが1つ」をコンパイラに確認させる注釈 |

---

## 標準の関数型インターフェース（第4節）

| 用語 | メソッド | 役割 |
|---|---|---|
| `Supplier<T>` | `get()` | 引数なしで値を供給する |
| `Consumer<T>` | `accept(t)` | 値を受け取り消費する（戻り値なし） |
| `Function<T, R>` | `apply(t)` | 値を受け取り変換して返す |
| `Predicate<T>` | `test(t)` | 値を受け取り真偽を判定する |
| `Runnable` | `run()` | 引数も戻り値もない処理 |

引数が2つなら `BiFunction`・`BiConsumer` など `Bi` 版を使う。`java.util.function` パッケージ。

---

## メソッド参照（第5節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| メソッド参照 | Method Reference | 「ただメソッドを呼ぶだけ」のラムダ式を短く書く（`型::メソッド`） |

`s -> s.length()` は `String::length`、`x -> IO.println(x)` は `IO::println`。

---

## おわりに

この章では、モダン Java の中心となる「ラムダ式」を学びました。

- **ラムダ式** … 処理を、値のように渡す（`(a, b) -> a + b`）
- **関数型インターフェース** … ラムダ式を受け止める「型」（抽象メソッド1つ）
- **標準の関数型インターフェース** … `Function`・`Predicate`・`Consumer`・`Supplier`
- **メソッド参照** … `String::length` のように、さらに短く

「処理を値として渡せる」 ―― このシンプルな発想が、Java の表現力を大きく広げました。

そして、ラムダ式が**真価を発揮する**のが、次の第23章「ストリーム API」です。
ストリームは、「コレクションのデータを、流れ作業のように処理する」しくみで、その各工程に、ラムダ式を渡します。
たとえば「リストから偶数だけを取り出し、2倍して、合計する」といった処理が、`filter`・`map`・`sum` を `->` でつなぐだけで、流れるように書けるようになります。

モダン Java の醍醐味を、味わっていきましょう。
