## ラムダ式とは ― 処理を値として渡す

**ラムダ式**（Lambda Expression）は、「**処理（ふるまい）を、値のように受け渡す**」ための、簡潔な書き方です[^jsr335-lambda]。
この節では、ラムダ式が「なぜ生まれたのか」を、昔の書き方と比べながら理解します。

---

## これまでは「データ」しか渡せなかった

第9章で、メソッドに**引数**を渡すことを学びました。
渡せるのは、`3` や `"hello"` のような**データ（値）**でした。

ですが、ときには「**処理そのもの**」を渡したいことがあります。
たとえば、「ボタンが押されたら、この処理をして」「リストを、このルールで並べ替えて」のように、**「何をするか」を相手に教えたい**場面です。

ラムダ式は、まさにこの「**処理を渡す**」を実現します。

---

## 昔の書き方 ― 匿名クラス

ラムダ式の便利さを実感するために、まず「ラムダ式がなかったころ」の書き方を見てみましょう。

第16章で学んだインターフェースを思い出してください。
「処理を渡す」には、そのインターフェースを実装したオブジェクトを作って渡す必要がありました。
たとえば、`Runnable`（「実行する」という1つのメソッド `run()` を持つ、標準のインターフェース）に処理を持たせるには、こう書きます。

```java
Runnable task = new Runnable() {
    @Override
    public void run() {
        IO.println("実行されました");
    }
};
task.run();
```

```text line-numbers=false
実行されました
```

この `new Runnable() { ... }` という書き方を、**匿名クラス**（Anonymous Class）と呼びます。
「名前のないクラスを、その場で作って渡す」という書き方です。

ですが、見てのとおり、とても**長くて、ごちゃごちゃ**しています。
本当にやりたいのは「`IO.println("実行されました")` を実行する」という**1行だけ**なのに、その周りに `new Runnable() { @Override public void run() { ... } }` という、たくさんの「決まり文句」が必要でした。

---

## ラムダ式なら、こう書ける

まったく同じことを、ラムダ式で書くと、こうなります。

```java
Runnable task = () -> IO.println("実行されました");
task.run();
```

```text line-numbers=false
実行されました
```

**たった1行**になりました。
`() -> IO.println("実行されました")` が、ラムダ式です。
匿名クラスの「決まり文句」（`new Runnable()`、`@Override`、`public void run()`、`{ }`）が、すべて消えています。

2つを並べてみると、その差は一目瞭然です。

```java
// 匿名クラス（長い）
Runnable task = new Runnable() {
    @Override
    public void run() {
        IO.println("実行されました");
    }
};

// ラムダ式（短い）
Runnable task = () -> IO.println("実行されました");
```

ラムダ式は、「**本当に必要な部分（処理の中身）だけ**を書く」ための書き方なのです。
余計な決まり文句を、Java が補ってくれる、と考えてください。

---

## ラムダ式の形 ― 矢印 `->`

ラムダ式の基本の形は、次のとおりです。

```text line-numbers=false
(引数) -> 処理
```

- **`(引数)`** … 受け取る引数（なければ `()`）
- **`->`** … 「を受け取って、こうする」を表す矢印（アロー）
- **処理** … 実際に行う処理

たとえば、

```java
() -> IO.println("やあ")        // 引数なし → やあ を表示する
n  -> n * 2                     // n を受け取って → 2倍する
(a, b) -> a + b                 // a と b を受け取って → 足す
```

「**引数を受け取って、矢印の先のことをする**」 ―― これがラムダ式の読み方です。
第6章の switch 式で出てきた `->` と、見た目は同じ矢印ですが、こちらは「処理を表す式」です。

---

## ラムダ式は「処理を持った値」

大切なのは、ラムダ式が**1つの「値」として扱える**ことです。

```java
Runnable task = () -> IO.println("実行されました");
```

この行では、ラムダ式 `() -> ...` を、`task` という**変数に代入**しています。
「処理」が、まるで `3` や `"hello"` のような値のように、変数に入っているのです。
だから、変数に入れて持ち運んだり、メソッドの引数として渡したりできます。

「処理を、値のように扱える」 ―― この発想が、次の節以降、そして第23章のストリームで、大きな力を発揮します。

---

## まとめ

- **ラムダ式**は、「**処理を、値のように受け渡す**」ための簡潔な書き方
- 昔は**匿名クラス**（`new Runnable() { ... }`）で書いていたが、長く、決まり文句が多かった
- ラムダ式なら、`() -> IO.println("...")` のように、**処理の中身だけ**を短く書ける
- 基本の形は **`(引数) -> 処理`**（`->` は「を受け取って、こうする」）
- ラムダ式は1つの**値**として、変数に代入したり、引数に渡したりできる

次の節では、ラムダ式が代入できる「型」 ―― **関数型インターフェース**の正体を学びます。
