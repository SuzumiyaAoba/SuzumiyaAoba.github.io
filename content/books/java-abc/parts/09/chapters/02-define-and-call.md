## メソッドを定義して呼び出す

メソッドを使うには、2つのステップがあります。

1. **定義する**（define）… メソッドを作る（名前と中身を決める）
2. **呼び出す**（call）… 作ったメソッドを実行する

もっとも単純な「引数も戻り値もないメソッド」から始めましょう。

---

## メソッドを定義する

「こんにちは」と表示するだけのメソッド `greet` を定義してみます。

```java
void greet() {
    IO.println("こんにちは");
}
```

各部分の意味は、次のとおりです。

| 部分 | 名前 | 意味 |
|---|---|---|
| `void` | 戻り値の型 | このメソッドは値を返さない、という印（次の説明） |
| `greet` | メソッド名 | このメソッドの名前。呼び出すときに使う |
| `()` | 引数リスト | 受け取る値（引数）。ここでは空＝受け取らない |
| `{ ... }` | 本体（ブロック） | 実行する処理 |

**`void`**（ボイド）は「空っぽ」という意味の英語で、**「このメソッドは値を返さない」**ことを表します。
`greet` は表示するだけで、結果を返さないので `void` です。（値を返すメソッドは、第4節で学びます。）

---

## メソッドを呼び出す

定義しただけでは、メソッドは実行されません。**呼び出して**はじめて動きます。
呼び出すには、メソッド名のうしろに `()` を付けて書きます。

```java
greet();
```

```text
こんにちは
```

jshell で試すと、定義したときと呼び出したときで、表示が違うことがわかります。

```text
jshell> void greet() {
   ...>     IO.println("こんにちは");
   ...> }
|  次を作成しました: メソッド greet()

jshell> greet()
こんにちは
```

メソッドを定義すると `次を作成しました: メソッド greet()` と表示され（まだ実行はされていません）、`greet()` と呼び出してはじめて `こんにちは` が表示されます。

呼び出すときの `()` を**忘れない**ようにしましょう。`greet`（かっこなし）だけでは、メソッドを呼び出したことになりません。

---

## ファイルに書く場合

jshell ではメソッドを直接定義できますが、ファイルに書くときは、第3章で学んだ `main` の中から呼び出します。

```java
void main() {
    greet();
    greet();
}

void greet() {
    IO.println("こんにちは");
}
```

これを `Greet.java` という名前で保存して実行すると、

```text
$ java Greet.java
こんにちは
こんにちは
```

`main` の中で `greet()` を2回呼んだので、「こんにちは」が2回表示されました。
このように、一度定義したメソッドは、**何度でも呼び出せます**。

> **メソッドを書く順番について**
>
> 上の例では `main` のうしろに `greet` を定義していますが、**順番は自由**です。`greet` を先に書いても、同じように動きます。
> 「`main` を先頭に書いて、そこから呼ぶメソッドを下に並べる」と、プログラムの流れが上から読めて分かりやすくなります。

---

## まとめ

- メソッドは「**定義する → 呼び出す**」の2ステップで使う
- 定義の形は `void メソッド名() { 処理 }`（`void` は「値を返さない」印）
- 呼び出しは `メソッド名();`。**`()` を忘れない**
- 一度定義すれば、**何度でも呼び出せる**
- ファイルでは `main` の中からメソッドを呼ぶ（定義の順番は自由）
