## オーバーロード ― 同じ名前で複数のメソッド

「2つの数を足す」と「3つの数を足す」は、どちらも「足す」処理です。
できれば、どちらも同じ `add` という名前で呼びたいものです。

Java では、**引数の組み合わせが違えば、同じ名前のメソッドをいくつも定義できます**。
これを**オーバーロード**（Overload、多重定義）と呼びます。

---

## オーバーロードの例

`add` という名前で、3つのメソッドを定義してみましょう。

```java
int add(int a, int b) {
    return a + b;
}

int add(int a, int b, int c) {
    return a + b + c;
}

double add(double a, double b) {
    return a + b;
}
```

呼び出すと、Java が**渡した引数に合うメソッドを、自動的に選んで**くれます。

```text
jshell> add(1, 2)
$1 ==> 3

jshell> add(1, 2, 3)
$2 ==> 6

jshell> add(1.5, 2.5)
$3 ==> 4.0
```

- `add(1, 2)` → 引数は `int` 2つ → 1つめのメソッドが呼ばれる
- `add(1, 2, 3)` → 引数は `int` 3つ → 2つめのメソッドが呼ばれる
- `add(1.5, 2.5)` → 引数は `double` 2つ → 3つめのメソッドが呼ばれる

呼ぶ側は、いつも `add` と書くだけでよく、「2つ用」「3つ用」と名前を覚え分けずに済みます。

---

## 何で見分けているのか ― シグネチャ

Java は、メソッドを**名前と引数の組み合わせ**（引数の個数・型・並び順）で区別しています。
この「名前＋引数の組み合わせ」を、メソッドの**シグネチャ**（Signature）と呼びます。

オーバーロードできるのは、シグネチャが違う場合です。

- `add(int, int)` と `add(int, int, int)` … 引数の**個数**が違う → OK
- `add(int, int)` と `add(double, double)` … 引数の**型**が違う → OK

一方、**戻り値の型だけ**が違っても、オーバーロードにはなりません。
たとえば `int add(int, int)` と `double add(int, int)` は、引数が同じ `(int, int)` なので区別できず、エラーになります。
「見分けるのは**引数**であって、戻り値ではない」と覚えておきましょう。

---

## 身近なオーバーロード ― println

実は、私たちが使ってきた `IO.println` や `System.out.println` も、オーバーロードされています。

```java
IO.println("こんにちは");   // String を渡す
IO.println(42);             // int を渡す
IO.println(3.14);           // double を渡す
IO.println(true);           // boolean を渡す
```

文字列でも数値でも `println` の一言で表示できるのは、引数の型ごとに `println` が用意されている（オーバーロードされている）からです。
オーバーロードは、こうして「**同じ目的の処理を、同じ名前で使える**」ようにし、使う人の負担を減らしてくれます。

---

## 補足: 個数が決まらないとき ― 可変長引数

「いくつ渡すか決まっていない」場合は、**可変長引数**（Varargs）という書き方もあります。
型のうしろに `...`（ピリオド3つ）を付けると、その引数を**何個でも**受け取れ、メソッドの中では**配列**として扱えます。

```java
int sum(int... nums) {
    int s = 0;
    for (int n : nums) {
        s += n;
    }
    return s;
}
```

```text
jshell> sum(1, 2, 3, 4)
$1 ==> 10

jshell> sum(10, 20)
$2 ==> 30

jshell> sum()
$3 ==> 0
```

`sum(1, 2, 3, 4)` でも `sum(10, 20)` でも、同じ `sum` で受けられました。
いまは「こういう書き方もある」と知っておけば十分です。

---

## まとめ

- **オーバーロード**は、引数の組み合わせが違えば、同じ名前のメソッドを複数定義できるしくみ
- Java は「名前＋引数の組み合わせ」（**シグネチャ**）でメソッドを見分ける
- 見分けるのは**引数**。**戻り値の型だけ**違ってもオーバーロードにはならない
- `println` も型ごとにオーバーロードされている
- 個数が決まらない引数は、**可変長引数**（`int...`）でも書ける

次の節では、メソッドに値を渡したときに、その値が**どう扱われるか**（値渡し）を学びます。初心者がつまずきやすい、大事なポイントです。
