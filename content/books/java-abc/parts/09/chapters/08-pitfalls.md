## よくあるつまずき

メソッドでつまずきやすいポイントを、まとめて確認します。

---

## 1. 値を返すメソッドで、return を忘れる

戻り値の型が `void` でないメソッドは、**必ず値を返さなければなりません**。
返し忘れると、コンパイルエラーになります。

```java
int add(int a, int b) {
    int sum = a + b;
    // return を書き忘れた！
}
```

```text
エラー: return文が指定されていません
```

`int add(...)` と宣言したからには、「`int` の値を返す」と約束したことになります。
その約束が果たされていないので、エラーになります。最後に `return sum;` を書けば解決します。

`if` で分岐する場合も、**すべての経路で値を返す**必要があります。
たとえば「`if` のときだけ `return` する」と、`if` が成り立たなかったときに返す値がなく、エラーになります。

---

## 2. void なのに値を返そうとする

逆に、`void`（値を返さない）と宣言したメソッドで、`return 値;` を書くこともできません。

```java
void greet() {
    return "こんにちは";   // void なのに値を返そうとしている → エラー
}
```

「値を返さない」と宣言したのに値を返そうとしているので、つじつまが合いません。
表示するだけなら `IO.println("こんにちは");` のように書き、`return` で値は返しません。
（途中で抜けるための、値なしの `return;` は `void` でも書けます。）

---

## 3. 呼び出すときに () を忘れる

メソッドを呼び出すときは、`()` が必要です。

```java
greet();    // ◯ 正しい呼び出し
greet;      // × これは呼び出しにならない（エラー）
```

引数があるメソッドは、`()` の中に値を入れます。`greet("佐藤");` のようにです。

---

## 4. 引数の数や型が合っていない

呼び出すときの実引数は、定義した仮引数と、**数も型も**合っている必要があります。

```java
void greet(String name) {
    IO.println(name + "さん、こんにちは");
}

greet();   // 引数が足りない → エラー
```

```text
エラー: ... メソッド greetは指定された型に適用できません。
  期待値: String
  検出値: 引数がありません
  理由: 実引数リストと仮引数リストの長さが異なります
```

`greet` は `String` を1つ受け取るのに、何も渡していないため、「長さが異なります」と指摘されています。
`greet("佐藤");` のように、定義どおりの数・型で渡しましょう。

---

## 5. 値渡しを勘違いする

第7節で学んだとおり、**基本型はコピー**が渡るので、メソッドの中で書き換えても呼び出し元は変わりません。

```java
void addOne(int x) {
    x = x + 1;        // ここで増やしても…
}

int n = 5;
addOne(n);
// n は 5 のまま（6 にはならない）
```

呼び出し元の値を変えたいなら、**戻り値で返して、受け取り直す**のが基本です。

```java
int addOne(int x) {
    return x + 1;
}

int n = 5;
n = addOne(n);   // 戻り値で受け取る → n は 6
```

一方、**配列は中身が共有される**ので、メソッド内での要素の書き換えは呼び出し元にも反映されます。この違いを混同しないようにしましょう。

---

## まとめ

- 値を返すメソッドは、**すべての経路で `return 値;`** を書く（忘れると「return文が指定されていません」）
- `void` のメソッドでは、値を返せない（値なしの `return;` は可）
- 呼び出すときは **`()` を忘れない**。引数は**数も型も**定義に合わせる
- **基本型はコピー**なので、変更を反映したいなら**戻り値で受け取り直す**

次は、この章で学んだ言葉を、用語集としてまとめます。
