## 引数の渡され方 ― 値渡しと参照

メソッドに値を渡したとき、その値はメソッドの中で**どう扱われる**のでしょうか。
ここには、初心者がとてもつまずきやすい、大事なルールがあります。
**基本型**（`int` など）と、**配列のような型**で、ふるまいが違うのです。

---

## 基本型は「値のコピー」が渡る

まず、`int` を渡す例を見てみましょう。
メソッドの中で引数を書き換えたら、呼び出し元の変数も変わるでしょうか？

```java
void tryChange(int x) {
    x = 100;          // メソッドの中で書き換える
}

int v = 5;
tryChange(v);
```

呼び出したあと、`v` を確認してみます。

```text
jshell> v
v ==> 5
```

`v` は **`5` のまま**でした。
メソッドには、`v` の**値のコピー**（`5`）が渡されます。
`tryChange` の中の `x` は、そのコピーを受け取った別の変数なので、`x` を `100` にしても、元の `v` には影響しません。

このように、**値のコピーを渡す**しくみを、**値渡し**（Pass by Value）と呼びます。
基本型（`int`・`double`・`boolean` など）は、すべてこの値渡しです。

```text
呼び出し元:  v = 5
              │ コピーを渡す
              ▼
メソッド:    x = 5  →  x = 100 に変更（でも v は 5 のまま）
```

---

## 配列は「中身を共有」する

では、配列を渡すとどうなるでしょうか。
配列の全要素を2倍にするメソッドで試します。

```java
void doubleAll(int[] a) {
    for (int i = 0; i < a.length; i++) {
        a[i] = a[i] * 2;
    }
}

int[] nums = {1, 2, 3};
doubleAll(nums);
```

呼び出したあと、`nums` を確認してみます。

```text
jshell> nums
nums ==> int[3] { 2, 4, 6 }
```

今度は、元の `nums` が **`{2, 4, 6}` に変わりました**。
基本型のときと、結果が逆です。なぜでしょうか。

配列のような型（**参照型**）では、メソッドに渡されるのは「**配列そのもの**」ではなく、「配列が**どこにあるか**を指し示す情報」です。
コピーされるのはこの「指し示す情報」だけで、**指している先（配列の中身）は、呼び出し元とメソッドで同じもの**になります。

つまり、`doubleAll` の中の `a` と、呼び出し元の `nums` は、**同じ1つの配列を指して**います。
だから `a[i]` を書き換えると、それは `nums` の中身を書き換えるのと同じことになり、呼び出し元にも反映されるのです。

```text
呼び出し元:  nums ──┐
                    ├──→ [ 1, 2, 3 ]  ← 同じ配列を共有
メソッド:    a ─────┘     （a[i] を書き換えると nums からも見える）
```

> **補足: Java は「常に値渡し」**
>
> 正確にいうと、Java は基本型も参照型も、つねに「値のコピー」を渡します（**常に値渡し**）。
> ただし参照型では、コピーされる「値」が「配列の場所を指す情報」なので、コピーしても**同じ配列**を指します。その結果、中身の書き換えが共有されるのです。
> 初心者のうちは、「**基本型はコピー（元は変わらない）／配列などは中身を共有（書き換えると元も変わる）**」と覚えておけば十分です。

---

## なぜ重要なのか

この違いを知らないと、次のような勘違いをしてしまいます。

- 「メソッドに `int` を渡して中で増やしたのに、呼び出し元が変わらない！」→ 基本型は**コピー**だから（戻り値で受け取る必要がある）
- 「メソッドに配列を渡しただけなのに、いつのまにか中身が変わっていた！」→ 配列は**共有**されているから

逆に、この性質をうまく使えば、`doubleAll` のように「**配列を渡して、メソッドにまとめて加工してもらう**」ことができます。配列を扱うメソッドでは、とてもよく使うパターンです。

---

## まとめ

- Java の引数は**値渡し**（値のコピーを渡す）
- **基本型**（`int` など）はコピーが渡るので、メソッド内で書き換えても**呼び出し元は変わらない**
- **配列などの参照型**は「中身を共有」するので、メソッド内で要素を書き換えると**呼び出し元にも反映される**
- 初心者は「基本型はコピー、配列などは中身を共有」と覚えておけばよい
