## メソッドとは

**メソッド**（Method）とは、**一連の処理に名前をつけて、ひとまとめにした「部品」**のことです。
一度作っておけば、その名前を書くだけで、何度でも同じ処理を呼び出せます。

---

## なぜメソッドが必要なのか

たとえば、「あいさつを表示する」という処理を、プログラムのあちこちで何度も使いたいとします。
メソッドがなければ、そのたびに同じコードを書くことになります。

```java
IO.println("=============");
IO.println(" こんにちは！ ");
IO.println("=============");

// …別の場所でも、また同じ3行を書く…
IO.println("=============");
IO.println(" こんにちは！ ");
IO.println("=============");
```

同じコードがあちこちに散らばると、こんな困りごとが起きます。

- **書くのが大変** … 使うたびに、同じ行を何度も書かなければならない
- **直すのが大変** … 表示を変えたくなったら、散らばった箇所を**すべて**直す必要がある（直し忘れも起きる）
- **読みにくい** … 「何をしているか」が、コードの量に埋もれてしまう

メソッドを使えば、この処理に `printGreeting` という名前をつけて、1か所にまとめられます。
あとは、使いたい場所で `printGreeting();` と書くだけです。

```java
printGreeting();   // 名前を呼ぶだけ
// …別の場所でも…
printGreeting();   // また呼ぶだけ
```

表示を変えたくなったら、メソッドの中身を1か所直すだけで、呼び出しているすべての場所に反映されます。

---

## メソッドは「入力 → 処理 → 出力」の部品

メソッドは、**自動販売機**にたとえるとイメージしやすくなります。

- **お金を入れる**（入力）→ **中で処理する** → **ジュースが出てくる**（出力）

メソッドも同じで、次の3つの要素でできています。

| 要素 | 自動販売機でいうと | メソッドでいうと |
|---|---|---|
| 入力 | 入れたお金・押したボタン | **引数**（ひきすう） |
| 処理 | 内部の動き | メソッドの**中身**（処理） |
| 出力 | 出てくるジュース | **戻り値**（もどりち） |

たとえば、`Arrays.sort(配列)` は「配列（入力）を渡すと、並べ替えてくれる（処理）」メソッドですし、`"abc".length()` は「文字列の長さ `3`（出力）を返してくれる」メソッドです。

ただし、3つの要素は**いつもすべてそろうとは限りません**。

- 入力（引数）がないメソッドもある（例: ただ決まった文字を表示するだけ）
- 出力（戻り値）がないメソッドもある（例: 表示するだけで、何も返さない）

この章では、まず単純な形から始めて、引数・戻り値を1つずつ足していきます。

---

## まとめ

- **メソッド**は、一連の処理に名前をつけてひとまとめにした「部品」
- 同じ処理を何度も書かずに済み、**直す場所が1か所**にまとまる
- メソッドは「**入力（引数）→ 処理 → 出力（戻り値）**」の3要素でできている
- 引数・戻り値は、ない場合もある

次の節では、もっとも単純な「引数も戻り値もないメソッド」を、実際に**定義して呼び出して**みます。
