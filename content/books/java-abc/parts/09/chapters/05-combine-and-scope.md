## メソッドを組み合わせる

メソッドの本当の力は、**いくつものメソッドを組み合わせる**ときに発揮されます。
大きな処理を、小さなメソッドに分けて、それらを呼び合わせることで、プログラムは整理され、読みやすくなります。

---

## メソッドから別のメソッドを呼ぶ

メソッドの中から、別のメソッドを呼び出せます。
例として、「商品の価格に消費税を加えて、表示する」プログラムを、3つのメソッドに分けてみましょう。

```java
void main() {
    int price = 1000;
    int taxIncluded = addTax(price);
    printPrice(taxIncluded);
}

int addTax(int price) {
    return (int) (price * 1.1);
}

void printPrice(int price) {
    IO.println("税込価格: " + price + "円");
}
```

```text
$ java Shop.java
税込価格: 1100円
```

処理の流れを追ってみましょう。

1. `main` が `addTax(1000)` を呼ぶ → `addTax` が `1100` を返す → `taxIncluded` に `1100` が入る
2. `main` が `printPrice(1100)` を呼ぶ → `printPrice` が「税込価格: 1100円」と表示する

`main` は「税の計算」や「表示の細かいやり方」を知らなくても、`addTax`・`printPrice` を**呼ぶだけ**で仕事ができています。
このように、処理を小さなメソッドに分けると、**それぞれが1つの役割に集中**し、全体の流れも読みやすくなります。

> 計算の `(int) (price * 1.1)` は、第4章で学んだ**キャスト**です。`1000 * 1.1` は小数の `1100.0` になるため、`(int)` で整数 `1100` に変換しています。

---

## 変数が使える範囲 ― スコープ

ここで、1つ大事なことに気づきます。
上のプログラムには、`price` という名前の変数が**3か所**に登場しています。

- `main` の中の `int price = 1000;`
- `addTax` の仮引数 `int price`
- `printPrice` の仮引数 `int price`

同じ名前なのに、ぶつからないのでしょうか？　答えは「**ぶつからない**」です。

メソッドの引数や、メソッドの中で宣言した変数は、**そのメソッドの中だけ**で有効だからです。
この「変数が使える範囲」のことを、**スコープ**（Scope）と呼びます。
そして、メソッドの中だけで使える変数を、**ローカル変数**（Local Variable、局所変数）と呼びます。

- `main` の `price` と、`addTax` の `price` は、**名前が同じだけの、まったく別の変数**
- それぞれ、自分のメソッドが実行されている間だけ存在し、メソッドが終わると消える

だから、別のメソッドで同じ名前を使っても、おたがいに影響しません。

---

## メソッドどうしは、引数と戻り値でやりとりする

ローカル変数は、ほかのメソッドからは**直接見えません**。
たとえば、`addTax` の中から `main` の `price` を直接読むことはできません。

では、メソッドどうしは、どうやって値を受け渡すのでしょうか。
その答えが、これまで学んできた **引数**（渡す）と **戻り値**（返す）です。

- `main` → `addTax` へ … **引数**で `1000` を渡す
- `addTax` → `main` へ … **戻り値**で `1100` を返す

「メソッドは、引数と戻り値という、決められた窓口だけで値をやりとりする」――この約束があるおかげで、各メソッドを**部品として独立して**考えられるのです。

---

## まとめ

- メソッドの中から、別のメソッドを呼び出せる。処理を分けると、各メソッドが1つの役割に集中できる
- 変数が使える範囲を**スコープ**、メソッド内だけで使える変数を**ローカル変数**と呼ぶ
- 別々のメソッドのローカル変数は、**名前が同じでも別物**（ぶつからない）
- メソッドどうしの値のやりとりは、**引数（渡す）と戻り値（返す）**で行う
