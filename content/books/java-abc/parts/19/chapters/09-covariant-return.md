## 戻り値の共変性

ここまでは、配列やジェネリクス ―― データの「入れ物」の変性を見てきました。
最後に、少し毛色の違う共変を1つ紹介します。**メソッドの戻り値**にも、共変が現れるのです。
これは、第14章で学んだ**オーバーライド**（メソッドを作り直す）と深く関わっています。

---

## オーバーライドのおさらいと、1つの例外

第14章では、子クラスで親のメソッドをオーバーライドするとき、**シグネチャ（名前・引数・戻り値）をそろえる**のが基本、と学びました。
ですが、戻り値の型については、1つだけゆるやかな例外があります。
**戻り値を、より具体的な型（サブ型）に狭めてよい**のです。
これを、**戻り値の共変性**（Covariant Return Type）と呼びます[^jls-covariant-return]。

例で見てみましょう。動物 `Animal` に「子を産む」メソッド `reproduce` があり、`Cat`（猫）がそれをオーバーライドします。

```java
class Animal {
    Animal reproduce() { return new Animal(); }
    public String toString() { return "Animal"; }
}

class Cat extends Animal {
    @Override
    Cat reproduce() { return new Cat(); }   // 戻り値を Animal → Cat に狭める
    public String toString() { return "Cat"; }
}
```

親の `reproduce` は `Animal` を返しますが、`Cat` ではそれを `Cat` を返すように**狭めて**います。
戻り値の型が親と違う（`Animal` → `Cat`）にもかかわらず、これは正しいオーバーライドとして認められます。

<Column title="Java 5 より前は書けなかった">

戻り値の共変性が使えるようになったのは、実は Java 5（2004年）からです。
それ以前は、オーバーライドで戻り値の型を変えることは一切許されず、`Cat` でも `Animal` を返すしかありませんでした。
そのため、呼び出し側で `(Cat)` とキャストする、という不格好なコードがよく書かれていました。
戻り値の共変性は、ジェネリクスと同じ Java 5 で導入された、地味ですが便利な改善のひとつです。

</Column>

---

## 何がうれしいのか ―― キャストが要らない

戻り値を狭められると、呼び出し側で**キャストが不要**になります。

```java
public class CovRet {
    public static void main(String[] args) {
        Cat c = new Cat();
        Cat kitten = c.reproduce();   // Cat が返るので、Cat で受けられる（キャスト不要）
        System.out.println(kitten);

        Animal a = c;                  // Animal として扱っても…
        System.out.println(a.reproduce());   // 動的束縛で Cat の reproduce が呼ばれる
    }
}
```

```text line-numbers=false
$ java CovRet.java
Cat
Cat
```

もし戻り値を狭められなかったら、`Cat` の `reproduce()` は `Animal` を返すため、`Cat kitten = (Cat) c.reproduce();` とキャストが必要でした。
戻り値の共変性のおかげで、「`Cat` から産まれるのは `Cat`」という自然な型が、キャストなしでそのまま得られます。

## なぜ安全なのか

戻り値を「狭める」方向なら安全なのは、**共変**が読み出し側で安全なのと同じ理屈です。
`Animal` を返すと約束したメソッドを、`Cat` を返すように変えても、呼び出し側は困りません。
なぜなら、`Cat` は `Animal` の一種（第14章の is-a）だからです。
「`Animal` が返ってくるはず」と思って受け取るコードは、実際に `Cat` が返ってきても、`Cat` は `Animal` でもあるので、何も問題が起きません。

これは、第6節で見た「読み出すだけなら共変が安全」の、メソッド版です。
戻り値は「メソッドがデータを**供給する（返す）**」位置なので、サブ型に狭める（共変）のが安全なのです。

> **補足: 引数は「共変に広げる」ことはできない**
>
> 戻り値は共変に狭められますが、**引数の型を変える**と、それはオーバーライドになりません。
> 引数の型を変えたメソッドは、（オーバーライドではなく）名前が同じだけの別メソッド ―― **オーバーロード**（第9章）として扱われます。
> 「戻り値は狭められるが、引数は変えるとオーバーロードになる」と整理しておきましょう。理論上、引数は反変なら安全ですが、Java のオーバーライドはそこまでは許していません。

## 実務での使いどころ

戻り値の共変性は、次のような場面で自然に登場します。

- **自分自身の複製を返すメソッド**（`copy` や `clone` など）。子クラスでオーバーライドし、自分の型を返すようにする。
- **ビルダーやフルーエント API**。メソッドが `this` を返す設計で、サブクラスが自分の型を返して、メソッドチェーンをつなぎやすくする。
- **ファクトリメソッド**（第4部・デザインパターンで学びます）。親が「製品」を返すと宣言し、子が「具体的な製品」を返す。

いずれも、「親が広い型で約束し、子が具体的な型で応える」という、共変の考え方がそのまま活きています。

## まとめ

- **戻り値の共変性**（Covariant Return Type）とは、オーバーライドで**戻り値をサブ型に狭められる**しくみ（Java 5+）
- `Animal reproduce()` を、子で `Cat reproduce()` とオーバーライドできる
- おかげで呼び出し側の**キャストが不要**になる
- 狭める（共変）方向が安全なのは、「返ってくる値は、約束した型の一種である」ことが保証されるから
- 一方、**引数の型を変えると**オーバーライドではなく**オーバーロード**になる

次の節では、この章（ジェネリクスと変性）で、初心者がつまずきやすいポイントを、まとめて確認します。

[^jls-covariant-return]: オーバーライドで戻り値をサブ型に狭められる条件（return-type-substitutable）は、*The Java® Language Specification, Java SE 25 Edition*, §8.4.8.3 "Requirements in Overriding and Hiding," [https://docs.oracle.com/javase/specs/jls/se25/html/jls-8.html#jls-8.4.8.3](<https://docs.oracle.com/javase/specs/jls/se25/html/jls-8.html#jls-8.4.8.3>) に定義されている。共変戻り値（covariant return type）は Java 5 で導入された。
