## コレクションとジェネリクス ― List を読む

ジェネリクスが、もっとも活躍するのは、**コレクション**です。
コレクションとは、たくさんのデータをまとめて扱うための、標準ライブラリのしくみです（くわしくは第3部・第21章で学びます）。
この節では、その代表である **`List`** を例に、ジェネリクスの読み方に慣れておきましょう。
第3部をスムーズに読み始めるための、橋渡しです。

---

## List は「サイズを変えられる配列」

第8章で配列を学びました。配列は便利でしたが、「**作るときに大きさが決まり、後から増やせない**」という制約がありました（第8章のつまずき）。

その制約をなくしたのが、**`List`**（リスト）です。
`List` は、「**後から要素を追加・削除できる、サイズの変わる配列**」のようなものです。
そして、`List` は**ジェネリック**です。「何のリストか」を、型引数で指定します。

```java
List<String> names = new ArrayList<>();   // String のリスト
```

`List<String>` は、「`String` を入れるリスト」と読みます。
`<String>` の部分が、まさにこの章で学んだジェネリクスです。
これで、このリストには `String` しか入らず、取り出すときも `String` として（キャストなしで）扱えます。

> **補足: `List` と `ArrayList`**
>
> 上の例で、左辺は `List<String>`、右辺は `new ArrayList<>()` と、名前が違います。
> `List` は「リストとは、こういうことができる」という**インターフェース**（第16章）で、`ArrayList` はその**実装の1つ**です。
> 「`List` という約束を、`ArrayList` という具体的なしくみで実現する」というわけです。第16章で学んだ「インターフェースの型で扱う」の実例です。
> くわしくは第21章で学びます。ここでは「リストを作るときの、決まった書き方」として見ておけば十分です。

## List を使ってみる

`List` の基本的な使い方を見てみましょう。
`List` と `ArrayList` は `java.util` パッケージにあるので、`import`（第12章）が必要です。

```java
import java.util.List;
import java.util.ArrayList;

public class Main {
    public static void main(String[] args) {
        List<String> names = new ArrayList<>();
        names.add("佐藤");      // 追加
        names.add("鈴木");
        names.add("高橋");

        IO.println("人数: " + names.size());   // 要素数
        IO.println(names.get(0));               // 0 番目を取り出す

        for (String name : names) {              // for-each で全部
            IO.println(name);
        }
    }
}
```

```text
$ java Main.java
人数: 3
佐藤
佐藤
鈴木
高橋
```

- `names.add(...)` … 要素を追加する（配列と違い、後からいくつでも足せる）
- `names.size()` … 要素数を返す（配列の `length` にあたる）
- `names.get(0)` … 指定した位置の要素を取り出す（配列の `names[0]` にあたる）
- `for (String name : names)` … for-each で、全要素を順に処理（配列と同じ）

`<String>` で型を指定しているので、`names.get(0)` は `String` を返し、for-each の変数も `String` です。
キャストは、どこにも必要ありません。これが、ジェネリクスの恩恵です。

## いろいろなコレクション（予告）

`List` 以外にも、目的に応じたコレクションがあります。どれもジェネリックです。

| 型 | 読み方 | 用途 |
|---|---|---|
| `List<String>` | String のリスト | 順番を保って、たくさんのデータを並べる |
| `Set<String>` | String のセット | 重複のない、データの集まり |
| `Map<String, Integer>` | String から Integer への対応 | キーと値のペア（名前→点数 など） |

特に `Map<String, Integer>` は、型引数が**2つ**あります。
`<K, V>`（キーと値）のように、複数の型を指定するジェネリクスです（第2節の補足で触れた、複数の型パラメータの実例です）。
`Map<String, Integer>` は、「`String`（名前）をキーに、`Integer`（点数）を値として対応づける」と読みます。

これらのコレクションは、第3部・第21章で、本格的に学びます。
いまは、「`<>` の中を見れば、何を入れるコレクションか分かる」 ―― この読み方を、つかんでおいてください。

## まとめ

- **`List`** は「サイズを変えられる配列」。`List<String>` で「`String` のリスト」
- `List` はジェネリックなので、`<>` で中身の型を指定する（キャスト不要・型安全）
- `List` は**インターフェース**、`ArrayList` はその**実装**（`List<String> x = new ArrayList<>();`）
- `add`（追加）・`size`（要素数）・`get`（取り出し）・for-each で扱う
- `Set`・`Map` などもジェネリック。`Map<K, V>` は型引数が2つ
- くわしくは第3部・第21章。ここでは「`<>` の読み方」に慣れておく
