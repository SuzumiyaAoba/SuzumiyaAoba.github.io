## はじめに ― この章で学ぶこと

ここまでの第11〜18章では、「自分で型（クラス・レコード・列挙型など）を設計する」さまざまな方法を学びました。

この第19章では、少し視点を変えます。
**ジェネリクス**（Generics、総称型）という、「**いろいろな型に対して使える、共通のしくみ**」を作る方法を学びます。

少し抽象的に聞こえるかもしれませんが、心配いりません。
実は、ジェネリクスは、これまでにも目にしていました。たとえば、第8章のコラムや、これからの第3部で登場する `List` です。

```java
List<String> names = ...;   // String を入れるリスト
List<Integer> scores = ...; // Integer を入れるリスト
```

この `<String>` や `<Integer>` が、ジェネリクスです。
「`List`（リスト）」という共通のしくみに対して、`<String>` で「中身は `String` ですよ」と、**型を後から指定**しているのです。

---

## なぜジェネリクスが必要なのか

「何でも入る入れ物」を作りたい、としましょう。
ジェネリクスがないと、「何でも入る」ようにするには、すべての型の親である `Object`（第14章）を使うことになります。

```java
class Box {
    Object content;   // 何でも入るが…
}
```

ですが、これには問題があります。

- 取り出すとき、`Object` 型なので、**毎回キャスト**が必要
- うっかり違う型を入れても気づけず、取り出すときに **`ClassCastException`**（第15章）になる

つまり、「何でも入る」かわりに、「**型の安全性**」が失われてしまうのです。

ジェネリクスを使えば、「`String` 専用の箱」「`Integer` 専用の箱」を、**1つのクラスから**作れます。
しかも、それぞれの箱は型が決まっているので、キャストが不要で、違う型を入れようとすればコンパイルエラーで防げます。
**「いろいろな型に使える柔軟さ」と「型の安全性」を、両立させる** ―― これがジェネリクスの目的です。

## この章を読み終えると

第19章を読み終えるころには、次のことができるようになっています。

- ジェネリクスが「型を後から決めるしくみ」だと理解できる
- `Box<T>` のようなジェネリッククラスを定義・利用できる
- `<String>` や `<Integer>` といった型指定の意味がわかる
- ジェネリクスが「型の安全性」をどう守るかを説明できる
- `List<String>` のような、第3部で頻出する書き方を読めるようになる

> **補足: この章は「読めるようになる」が目標**
>
> ジェネリクスは、自分でゼロから設計する機会は、最初のうちは多くありません。
> ですが、`List` や `Map` といった**標準ライブラリを使う**ときには、必ず登場します。
> そのため、この章の主な目標は、「ジェネリクスを**自在に設計できる**」ことよりも、「**`List<String>` のような書き方を、自信を持って読める**」ことです。
> 第3部のコレクションを学ぶための、大切な準備運動だと考えてください。

それでは、最初の節「ジェネリクスとは」から始めましょう。
