## 用語集 ― この章で学んだ言葉

第19章では、型を後から決めるしくみ「ジェネリクス」を学びました。
節ごとに、言葉を整理しておきます。

---

## ジェネリクスの基本（第1〜3節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| ジェネリクス | Generics | 扱う型を後から決められるようにするしくみ（総称型） |
| 型パラメータ | Type Parameter | `class Box<T>` の `T`。型の「空欄」 |
| 型引数 | Type Argument | `Box<String>` の `String`。空欄に当てはめる具体的な型 |
| ジェネリッククラス | Generic Class | 型パラメータを持つクラス（`Box<T>`） |
| ジェネリックメソッド | Generic Method | 型パラメータを持つメソッド（`<T> T firstOf(T[])`） |
| ダイヤモンド演算子 | Diamond Operator | `new ArrayList<>()` の `<>`。右辺の型を省略する |

---

## 境界とラッパー（第4節・つまずき）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| 型引数の境界 | Bounded Type Parameter | `<T extends Number>` のように、型パラメータを制限する |
| ラッパークラス | Wrapper Class | 基本型に対応する参照型（`int`→`Integer` など）。型引数に使う |
| オートボクシング | Autoboxing | `int` と `Integer` を自動で相互変換するしくみ |
| 生の型 | Raw Type | 型引数を省いた `List`。型チェックが効かず、避けるべき |

---

## コレクション（第5節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| コレクション | Collection | たくさんのデータをまとめて扱う標準ライブラリのしくみ（第3部で詳説） |
| `List<E>` | リスト | サイズを変えられる、順序つきのデータの並び |
| `Map<K, V>` | マップ | キーと値のペアを対応づける（型引数が2つ） |

---

## おわりに ― 第2部の締めくくりが近づいてきました

第19章で、Java の「型」に関する道具立てが、ほぼ出そろいました。

- クラス・カプセル化・コンストラクタ（第11〜13章）
- 継承・ポリモーフィズム（第14〜15章）
- 抽象クラス・インターフェース（第16章）
- レコード・列挙型（第17〜18章）
- ジェネリクス（第19章）

これらを使えば、安全で、再利用しやすく、変更に強いプログラムを設計できます。

第2部も、いよいよ最後の章です。
これまで、エラーが起きると「例外（Exception）が発生して、プログラムが止まる」場面を、何度も見てきました。
配列の範囲外アクセス、`null` の操作、`ClassCastException`、`valueOf` の失敗 ―― これらは、すべて**例外**でした。

最後の第20章「例外処理」では、この**例外**と正面から向き合います。
エラーが起きても、プログラムを安全に動かし続けるための、`try-catch` をはじめとするしくみを学びましょう。
