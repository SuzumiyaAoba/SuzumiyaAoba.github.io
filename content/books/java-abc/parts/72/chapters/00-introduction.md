## はじめに ― この章で学ぶこと

**Visitor**（ビジター、訪問者）パターンは、**「データ構造と、そこに対する操作を、別々に進化させる**」ための振る舞いパターンです。

たとえば、構文木のような**型が固定**な構造に対して、

- 評価する
- 出力する
- 最適化する
- 静的解析をする

など、**操作はあとから増やしたい**ことがあります。
このとき、構造側のクラスに「**訪問者を受け入れる口**」を 1 つだけ用意しておき、操作は**訪問者の側で書く**ようにします。

現代の Java では、シールドクラスとパターンマッチングが**Visitor の役割を肩代わり**できるので、両方のスタイルを見ていきます。

---

## この章を読み終えると

- Visitor の**意図**（とくに「**操作の追加**」を分離する点）を説明できる
- **ダブルディスパッチ**の意味を理解できる
- シールドクラス + `switch` 式で書く**現代版 Visitor** が選択できる
