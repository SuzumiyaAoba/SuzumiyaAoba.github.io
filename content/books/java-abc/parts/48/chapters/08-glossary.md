## 用語集 ― この章で学んだ言葉

### プロファイリング（profiling）

プログラムを実行しながら、**どこに**時間がかかっているかを計測すること。

### ベンチマーク（benchmark）

**どれだけ**時間がかかっているかを測ること。
プロファイリングと補完的。

### CPU プロファイル

メソッド別の CPU 使用量を集計したプロファイル。
**サンプリング**（軽量）と**インストルメンテーション**（正確だが重い）の 2 方式がある。

### サンプリング

一定間隔で**スレッドのスタック**を記録する計測方式。
本番で使えるほど軽量だが、統計的な精度。

### インストルメンテーション

メソッドの入口・出口にコードを差し込んで計測する方式。
正確だが**重い**ので、本番向きではない。

### メモリプロファイル / アロケーションプロファイル

**どこで**多くのオブジェクトを割り当てているかを計測する。
GC の頻発を解消する手がかりになる。

### ロックプロファイル

`synchronized`・`ReentrantLock`・`LockSupport.park` などの**待ち時間**を集計する。
並行処理のボトルネックを見つける。

### I/O プロファイル

ファイル・ソケットの**読み書き時間**を集計する。
Web アプリではここがボトルネックなことが多い。

### JFR（Java Flight Recorder）

JDK 標準のプロファイラ。
JVM 内部のイベントをローオーバーヘッドでリングバッファに記録する。
Java 11 以降は OpenJDK にも組み込まれ、**無料**で使える。

### `.jfr` ファイル

JFR の記録ファイル。バイナリ形式で、複数のイベントを時系列で含む。

### `jfr` コマンド

`.jfr` ファイルの**CLI 解析ツール**。
- `jfr summary` ―― 全体サマリ
- `jfr print --events ...` ―― 特定イベント抽出
- `jfr metadata` ―― イベントタイプ一覧

### `jdk.ExecutionSample`

JFR のサンプリングイベント。スレッドのスタックトレースを定期記録する。
CPU プロファイルの基礎データ。

### `jdk.GarbageCollection`

JFR の GC イベント。GC の開始・終了・停止時間を記録する。

### `jdk.ObjectAllocationInNewTLAB`

JFR のアロケーションイベント。TLAB 内のオブジェクト割り当てを記録する。

### `jdk.JavaMonitorWait`

JFR のロック待ちイベント。`synchronized` ブロックの待ち時間を記録する。

### JFR 設定プロファイル

`.jfc` という設定ファイル。

- **`default`** ―― オーバーヘッド < 1 %、本番向け
- **`profile`** ―― オーバーヘッド 1〜2 %、詳細分析向け

### 連続記録モード（continuous mode）

`maxage=1h,maxsize=200m,disk=true` のような指定で、**過去 N 時間**をリングバッファに残し続けるモード。
問題発生時に `JFR.dump` でその瞬間を保存する。

### `JFR.start` / `JFR.dump` / `JFR.stop` / `JFR.check`

`jcmd <pid>` で使う JFR のサブコマンド。
実行中のプロセスに対して、記録の開始・ダンプ・停止・確認ができる。

### JFR カスタムイベント

`jdk.jfr.Event` を継承して**自分のアプリのイベント**を発火する仕組み。
業務イベントと JVM イベントをクロス分析できる。

### JMC（Java Mission Control）

JFR の `.jfr` ファイルを解析する**GUI ツール**。
かつては Oracle 商用、現在は独立した OSS プロジェクト。

### 自動分析（Automated Analysis）

JMC が**自動でデータを解析**して、問題点を緑・黄・赤で示す機能。

### Method Profiling ペイン

JMC で CPU プロファイルを階層的に見るビュー。フレームグラフに似た形式。

### TLAB Allocations ペイン

JMC でアロケーションプロファイルを見るビュー。

### Thread Profiling ペイン

JMC でスレッドのタイムラインを見るビュー。
ロック競合・待ち状態の可視化に強い。

### async-profiler

OS のパフォーマンスカウンタ（perf）と `AsyncGetCallTrace` を使う、低オーバーヘッドプロファイラ。
ネイティブまで深く見える、**フレームグラフ**を SVG で出力。

### フレームグラフ（flame graph）

CPU プロファイルの可視化形式。
x 軸が時間、y 軸が呼び出し階層、ブロックの**幅**が CPU 使用時間を表す。

### CPU バウンド / I/O バウンド

- **CPU バウンド** ―― CPU が忙しくて遅い
- **I/O バウンド** ―― CPU は空いていて、何かを待っているから遅い

判別を最初にやらないと、プロファイラの取り方を間違える。

### ヒープダンプ（heap dump）

`jcmd <pid> GC.heap_dump` で取得できる、ヒープ全体のスナップショット。
**Eclipse MAT**（Memory Analyzer Tool）で開き、メモリリークの原因を追う。

### Dominator Tree

Eclipse MAT で、ヒープを**最も占有しているオブジェクト**を階層的に表示するビュー。
リーク調査の主役。

### Stop The World（STW）

GC やセーフポイントで、すべての Java スレッドが**一斉に止まる**期間。
JFR の `Application Halts` で計測できる。
