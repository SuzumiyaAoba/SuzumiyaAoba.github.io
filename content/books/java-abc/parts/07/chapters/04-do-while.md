## do-while 文 ― 先に1回実行する

**do-while 文**は、`while` とよく似ていますが、**条件を調べる順番**が違います。

- `while` … 先に**条件**を調べ、`true` なら処理する（**前判定**）
- `do-while` … 先に**処理**を1回行い、それから条件を調べる（**後判定**）

そのため、do-while は**条件に関係なく、必ず1回は実行される**のが特徴です。

---

## do-while の形

```java
do {
    // 繰り返す処理
} while (条件);
```

`do` のうしろの `{ }` を実行してから、`while (条件)` を調べます。
条件が `true` なら、もう一度 `{ }` に戻り、`false` なら終わります。最後にセミコロン `;` が付く点にも注意しましょう。

例を見てみましょう。`d` が 5 のときに、`d < 3` という条件で繰り返します。

```java
int d = 5;
do {
    IO.println(d);
    d++;
} while (d < 3);
```

```text
5
```

`d` は 5 なので、`d < 3` は最初から `false` です。
それでも do-while は、**先に1回 `{ }` を実行する**ので、`5` が表示されました。
（もしこれが `while` 文だったら、最初の条件チェックで `false` になり、一度も表示されません。）

---

## while と do-while の違い

| | 判定のタイミング | 最低の実行回数 |
|---|---|---|
| `while` | 先に条件（前判定） | 0 回（最初から `false` なら実行しない） |
| `do-while` | あとで条件（後判定） | 1 回（必ず1回は実行する） |

---

## do-while が向く場面

「**まず1回やってみて、それから続けるか決める**」という処理に向いています。
たとえば、「正しい値が入力されるまで、入力をうながし続ける」ような場面です（入力は、少なくとも1回は求める必要があります）。

実際には、3つの繰り返しのなかで do-while を使う場面は多くありません。
まずは `for` と `while` をしっかり使えるようにして、do-while は「先に1回実行する繰り返しもある」と知っておけば十分です。

---

## まとめ

- **do-while 文**は、先に処理を1回行ってから、条件を調べる（**後判定**）
- そのため、条件に関係なく**必ず1回は実行**される
- `while`（前判定）は0回のこともあるが、`do-while` は最低1回
- 「まず1回やってから続けるか決める」場面に向く

次の節では、繰り返しを途中で抜けたり、とばしたりする **break** と **continue** を学びます。
