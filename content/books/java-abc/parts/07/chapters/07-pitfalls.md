## よくあるつまずき

繰り返しでつまずきやすいポイントを、まとめて確認します。

---

## 1. 無限ループ（終わらない繰り返し）

もっとも多い失敗が、**繰り返しが終わらなくなる**ことです。原因は、たいてい次のどちらかです。

- 繰り返しの中で、条件に関わる値を**更新し忘れた**（`while` で `i++` を忘れる など）
- 続ける条件が、**いつまでも `true` のまま**になっている

繰り返しを書いたら、「この条件は、いつ `false` になるか？」を必ず確かめましょう。
万一止まらなくなったら、**Ctrl + C** で中断できます。

---

## 2. 回数が1つずれる（オフバイワン）

繰り返しの回数が、思っていたより1つ多い・少ない、というミスもよくあります。
英語で「1つずれる」を意味する **off-by-one（オフバイワン）エラー**と呼ばれます。

特に、`<` と `<=` の違いに注意しましょう。

```java
for (int i = 1; i <= 10; i++) { ... }   // 1〜10 の 10 回
for (int i = 1; i <  10; i++) { ... }   // 1〜9 の 9 回（10 を含まない）
for (int i = 0; i <  10; i++) { ... }   // 0〜9 の 10 回
```

「`0` から始めて `< 10`」と「`1` から始めて `<= 10`」は、どちらも10回です。
開始の値と条件の組み合わせで、回数が変わることを意識しましょう。

---

## 3. ループ変数を、ループの外で使おうとする

for 文のかっこの中で宣言したループ変数（`int i = 0` の `i`）は、**その for の中だけ**で使えます。
ループの外で使おうとすると、エラーになります。

```java
for (int i = 0; i < 10; i++) {
    // ここでは i を使える
}
// ここで i を使うとエラー（i はもう存在しない）
```

ループの外で結果を使いたいときは、第3節の `sum` のように、**ループの外で宣言した変数**に値をためておきます。

---

## 4. 繰り返しの中での整数のわり算

第5章でも学んだとおり、整数どうしのわり算は切り捨てられます。
繰り返しで平均を求めるようなときは、特に注意しましょう（小数がほしいなら、一方を `double` にします）。

---

## まとめ

- もっとも多い失敗は**無限ループ**。「条件はいつ `false` になるか」を必ず確認する
- `<` と `<=` の違いで回数が1つずれる（**オフバイワン**）に注意
- ループ変数は、その繰り返しの中だけで使える。外で使うなら外で宣言する
- 繰り返しの中でも、整数のわり算の切り捨てに注意

次は、この章で学んだ言葉を、用語集としてまとめます。
