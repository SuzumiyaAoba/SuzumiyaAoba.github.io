## break と continue ― 抜ける・とばす

繰り返しの途中で、流れを変えたいことがあります。
そのための2つの命令が、**`break`** と **`continue`** です。

- **`break`** … 繰り返しを**途中で抜ける**（ループそのものを終える）
- **`continue`** … その回だけ**とばして**、次の回へ進む

---

## break ― 繰り返しを抜ける

`break` は、繰り返しを途中でやめて、ループの外へ出ます。
「探しものが見つかったら、それ以上さがす必要はない」――そんな場面で役立ちます。

1 から 10 まで調べて、4 が見つかったらやめる例です。

```java
int found = 0;
for (int i = 1; i <= 10; i++) {
    if (i == 4) {
        found = i;
        break;
    }
}
```

ループのあと、`found` を確認すると、

```text
jshell> found
found ==> 4
```

`i` が 4 になったところで `break` し、5 以降は調べずにループを終えています。

---

## continue ― その回をとばす

`continue` は、繰り返しを抜けるのではなく、**いまの回の残りをとばして、次の回へ進みます**。
「特定の場合だけ、処理をスキップしたい」ときに使います。

1 から 5 までの合計を求めつつ、3 だけをとばす例です。

```java
int total = 0;
for (int i = 1; i <= 5; i++) {
    if (i == 3) {
        continue;
    }
    total += i;
}
```

```text
jshell> total
total ==> 12
```

`i` が 3 のときだけ `continue` で `total += i` をとばすので、合計は 1 + 2 + 4 + 5 = 12 になりました。

---

## break と continue の違い

| 命令 | はたらき |
|---|---|
| `break` | 繰り返し**そのもの**を終える（ループの外へ出る） |
| `continue` | いまの回だけとばして、**次の回**へ進む |

「ループを抜けるのが `break`、その回だけ飛ばすのが `continue`」と覚えておきましょう。

---

## もう一つの例 ― continue で奇数をとばす

`continue` を使って、1 から 10 のうち**偶数だけ**を表示してみましょう。
「奇数だったら、その回をとばす」と考えます。

```java
for (int i = 1; i <= 10; i++) {
    if (i % 2 != 0) {   // 奇数なら
        continue;       // この回をとばす
    }
    IO.print(i + " ");
}
```

```text
2 4 6 8 10 
```

`i` が奇数（`i % 2 != 0` が `true`）のときは `continue` で表示をとばし、偶数のときだけ `IO.print` にたどり着いています。

> **補足: break・continue は控えめに**
>
> break や continue は便利ですが、多用するとループの流れが追いにくくなります。
> 特に、深いネストの中で何度も使うと、「どこで抜けるのか・どこへ飛ぶのか」が分かりにくくなりがちです。
> まずは `for` や `while` の**条件式で素直に表現できないか**を考え、それが難しいときに break・continue を使う、というくらいの心構えがよいでしょう。

---

## まとめ

- **`break`** は、繰り返しを途中で抜ける（探しものが見つかったとき など）
- **`continue`** は、いまの回だけとばして、次の回へ進む（特定の場合をスキップ）
- どちらも `if` と組み合わせて、「ある条件のときだけ」使うのが基本
