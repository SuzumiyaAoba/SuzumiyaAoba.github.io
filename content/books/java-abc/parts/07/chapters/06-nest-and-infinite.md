## 多重ループと無限ループ

この節では、繰り返しの応用として、**多重ループ**（ループの入れ子）と、**無限ループ**を学びます。

---

## 多重ループ ― 繰り返しの中の繰り返し

繰り返しの中に、さらに繰り返しを書くこともできます。これを**多重ループ**（または、ネストしたループ）と呼びます。

たとえば、九九のような「縦 × 横」の表を作るときに使います。

```java
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        IO.print(i * j + " ");
    }
    IO.println("");
}
```

```text
1 2 3 
2 4 6 
3 6 9 
```

外側の `i` が 1 のあいだに、内側の `j` が 1 → 2 → 3 と最後まで回ります。
内側が終わると、外側の `i` が 2 になり、また内側の `j` が 1 から回り直す ―― これを繰り返します。

つまり、**外側が1回進むごとに、内側はぐるりと1周する**わけです。
（`IO.print` は改行しない表示、`IO.println("")` は空の内容で改行だけを行い、1行ぶんを区切っています。）

多重ループは便利ですが、深く重ねると、繰り返す回数が一気に増えます。
たとえば、外側10回・内側10回なら、内側の処理は 10 × 10 = 100 回実行されます。回数を意識して使いましょう。

---

## 無限ループ ― わざと終わらせない

これまで「無限ループは失敗」として注意してきましたが、実は**わざと**無限ループにして、`break` で抜ける、という書き方もあります。

`while (true)` と書くと、条件がつねに `true` なので、ずっと繰り返します。
そして、ある条件になったら `break` で抜けます。

```java
int count = 0;
while (true) {
    count++;
    if (count >= 3) {
        break;
    }
}
```

ループのあと、`count` を確認すると、

```text
jshell> count
count ==> 3
```

`count` が 3 になったところで `break` し、ループを抜けています。
「条件が複雑で、ループの途中で抜けたい」ようなときに、この書き方が使われます。

---

## わざとの無限ループと、うっかりの無限ループ

同じ無限ループでも、意味はまったく違います。

- **わざとの無限ループ** … `while (true)` と書き、`break` で**必ず抜ける**ように作る
- **うっかりの無限ループ** … 更新を忘れるなどして、**抜けられなくなった**失敗

`while (true)` を使うときは、**どこかに必ず `break`（抜け道）がある**ことを、自分で確認しましょう。抜け道のない無限ループは、ただのバグです。

---

## まとめ

- **多重ループ**は、繰り返しの中に繰り返しを書くこと。外側が1回進むと、内側は1周する
- 多重ループは回数が増えやすいので、実行回数を意識する
- `while (true)` で**わざと**無限ループを作り、`break` で抜ける書き方もある
- `while (true)` を使うときは、**必ず抜け道（`break`）がある**ことを確認する
