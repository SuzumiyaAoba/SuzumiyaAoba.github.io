## 用語集 ― この章で学んだ言葉

第15章では、オブジェクト指向の真骨頂「ポリモーフィズム」を学びました。
節ごとに、言葉を整理しておきます。

---

## ポリモーフィズムの基本（第1〜3節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| ポリモーフィズム | Polymorphism | 多態性。同じ呼び出しに対し、相手に応じて違うふるまいが起こること |
| アップキャスト | Upcast | 子のオブジェクトを、親の型として扱うこと（`Animal a = new Dog();`） |
| 変数の型（見かけの型） | Static Type | 変数の宣言で決まる型。**呼べるメソッド**を決める（コンパイル時） |
| 中身の型（実行時の型） | Runtime Type | 実際に入っているオブジェクトの型。**実際に動くメソッド**を決める |
| 動的束縛（動的ディスパッチ） | Dynamic Binding / Dispatch | 実行時に、中身の型のメソッドが選ばれて動くしくみ |

---

## 活用と型の判定（第4〜5節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| `instanceof` | インスタンス・オブ | オブジェクトの中身の型を調べる演算子（結果は `boolean`） |
| ダウンキャスト | Downcast | 親の型を、子の型として扱い直すこと（`(Dog) a`） |
| `ClassCastException` | ― | 中身と違う型へダウンキャストしたときに起きる例外 |
| パターンマッチング（instanceof） | Pattern Matching | `if (a instanceof Dog dog)` で、型の確認とダウンキャストを同時に行う（Java 16+） |

---

## おわりに

ここまでの第11〜15章で、オブジェクト指向の「3本柱」が出そろいました。

- **カプセル化**（第12章） … データを隠し、安全に守る
- **継承**（第14章） … 共通部分を受け継ぐ
- **ポリモーフィズム**（第15章） … 同じ呼び出しで、相手に応じたふるまい

この3つは、オブジェクト指向を支える、もっとも大切な考え方です。
これらを組み合わせることで、大きく、変更に強く、読みやすいプログラムを組み立てられます。

ここまでは、親クラス `Animal` に、`cry()` の**標準のふるまい**（「（動物の鳴き声）」）を書いていました。
でも、よく考えると、「動物そのもの」が鳴くことはありません。鳴くのは、犬や猫といった**具体的な動物**です。
「`Animal` は、`cry()` が『あるべき』ことだけ決めて、中身は子に任せたい」 ―― そんな場面があります。

次の第16章「抽象クラスとインターフェース」では、「**中身は決めず、『こういうメソッドがある』という約束だけを定める**」しくみを学びます。
ポリモーフィズムを、さらに洗練された形で活かせるようになります。
