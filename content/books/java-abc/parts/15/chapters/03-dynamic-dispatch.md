## 動的束縛 ― なぜ子のメソッドが呼ばれるのか

前の節で、`Animal a = new Dog();` のように、親の型で子を扱えると学びました。
では、その `a` に対して `a.cry()` を呼ぶと、**親（Animal）の `cry()`** と **子（Dog）の `cry()`**、どちらが動くのでしょうか。

この節で、その答えと、しくみを学びます。

---

## 動かして確かめる

まず、実際に動かしてみましょう。

```java
class Animal {
    String name;
    void cry() { IO.println(name + ": （動物の鳴き声）"); }
}

class Dog extends Animal {
    @Override
    void cry() { IO.println(name + ": ワン"); }
}

class Cat extends Animal {
    @Override
    void cry() { IO.println(name + ": ニャー"); }
}

public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog();   // 変数の型は Animal、中身は Dog
        a1.name = "ポチ";
        a1.cry();

        Animal a2 = new Cat();   // 変数の型は Animal、中身は Cat
        a2.name = "タマ";
        a2.cry();
    }
}
```

```text
$ java Main.java
ポチ: ワン
タマ: ニャー
```

注目してください。`a1` も `a2` も、**変数の型は `Animal`** です。
それなのに、`a1.cry()` では `Dog` の「ワン」、`a2.cry()` では `Cat` の「ニャー」が出ました。
親の `cry()`（「（動物の鳴き声）」）は、どちらでも呼ばれていません。

つまり、**呼ばれるのは、変数の型（`Animal`）ではなく、中身の型（`Dog`・`Cat`）の `cry()`** なのです。

## 動的束縛 ― 中身の型で、メソッドが選ばれる

このように、

> メソッドを呼んだとき、**実際に動くのは、その変数の「中身（実行時の型）」のメソッド**

というしくみを、**動的束縛**（Dynamic Binding）、または**動的ディスパッチ**（Dynamic Dispatch）と呼びます。
「動的（実行時）に、どのメソッドを動かすかが決まる（結びつけられる＝束縛される）」という意味です。

第14章で「子クラスでオーバーライドすると、子のメソッドが呼ばれる」と学びました。
その「子のメソッドが呼ばれる」が、変数の型を親（`Animal`）にしても、ちゃんと働く ―― これが動的束縛です。
だからこそ、`Animal` 型で受けても、中身が `Dog` なら `Dog` の鳴き方をしてくれるのです。

---

## 「呼べるか」と「何が動くか」は別

ここで、前の節の「2つの型」が効いてきます。
**「呼べるか」と「何が動くか」は、別々のルールで決まります。**

| | 何で決まるか | いつ決まるか |
|---|---|---|
| **呼べるメソッドか** | 変数の型（見かけの型） | コンパイル時 |
| **実際に動くメソッド** | 中身の型（本当の型） | 実行時（動的束縛） |

`Animal a = new Dog();` の場合、

- `a.cry()` が**呼べるか**は、`Animal` 型に `cry()` があるか、で決まる（→ ある。OK）
- 実際に**動くのは**、中身の `Dog` の `cry()`（→ 「ワン」）

「`Animal` として呼び出すが、中身の `Dog` として動く」 ―― この2段構えが、ポリモーフィズムのしくみです。

---

## これがポリモーフィズムの正体

ここまでをまとめると、ポリモーフィズムは、次の2つの組み合わせで実現されています。

1. **アップキャスト**（第2節） … 子を、親の型でまとめて扱える
2. **動的束縛**（この節） … 親の型で呼んでも、中身の子のメソッドが動く

この2つがあるおかげで、

```java
Animal a = ...;   // 中身は Dog かもしれないし、Cat かもしれない
a.cry();          // でも、中身に応じて、正しい鳴き声が出る
```

と書けます。
呼ぶ側は「`Animal` の `cry()`」と書くだけ。あとは、中身が自分にふさわしいふるまいをしてくれる。
この強力なしくみを、次の節で、実際に活用してみましょう。

---

## まとめ

- 親の型の変数で呼んでも、**実際に動くのは、中身（実行時の型）のメソッド**
- このしくみを**動的束縛**（動的ディスパッチ）と呼ぶ
- 「**呼べるメソッドか**」は変数の型（コンパイル時）、「**実際に動くメソッド**」は中身の型（実行時）で決まる
- ポリモーフィズムは、**アップキャスト**（親の型で扱う）＋**動的束縛**（中身のメソッドが動く）で実現される

次の節では、このしくみを使って、いろいろな種類のオブジェクトを**まとめて扱う**方法を学びます。
