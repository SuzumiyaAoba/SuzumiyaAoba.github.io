## 親の型で子を扱う ― アップキャスト

ポリモーフィズムの鍵は、「**親の型の変数に、子のオブジェクトを入れる**」ことです。
この節では、そのしくみと、なぜそれが可能なのかを学びます。

---

## 親の型の変数に、子を入れる

これまで、`Dog` のオブジェクトは `Dog` 型の変数に入れていました。

```java
Dog dog = new Dog();
```

実は、`Dog` のオブジェクトを、**親である `Animal` 型の変数**に入れることもできます。

```java
Animal animal = new Dog();   // Animal 型の変数に、Dog のオブジェクトを入れる
```

これは、第14章で学んだ **is-a の関係**のおかげです。
「`Dog` は `Animal` の一種（Dog **is a** Animal）」なので、`Dog` のオブジェクトは、`Animal` として扱って問題ありません。
「犬は動物だから、『動物』という箱に入れられる」と考えると、自然ですね。

このように、子の型のオブジェクトを、親の型として扱うことを、**アップキャスト**（Upcast）と呼びます。
クラス階層の図で、子（下）から親（上）へ向かうので「アップ」です。アップキャストは安全なので、特別な書き方は必要なく、ただ代入するだけでできます。

---

## 何が便利なのか

「わざわざ親の型に入れて、何がうれしいの？」と思うかもしれません。
ポイントは、**`Dog` でも `Cat` でも、同じ `Animal` 型の変数で受けられる**ことです。

```java
Animal a1 = new Dog();   // Animal 型に Dog
Animal a2 = new Cat();   // Animal 型に Cat
```

`a1` も `a2` も、同じ `Animal` 型です。
中身が `Dog` か `Cat` かを気にせず、「動物」として、同じように扱えるようになります。
これが、前節で予告した「**まとめて扱う**」の土台です。
種類の違うオブジェクトを、共通の親の型で受けられるからこそ、ひとまとめにできるのです。

---

## 親の型で呼べるのは、親が持つメソッドだけ

ただし、1つ重要な制約があります。
`Animal` 型の変数を通して呼べるのは、**`Animal` が持っているメソッド**だけです。

```java
class Animal {
    String name;
    void cry() { ... }
}

class Dog extends Animal {
    void fetch() { IO.println("ボールを取ってくる"); }   // Dog だけのメソッド
}
```

このとき、`Animal` 型の変数からは…

```java
Animal a = new Dog();
a.cry();      // ◯ cry() は Animal が持っている → OK
a.fetch();    // ✕ fetch() は Animal にはない → コンパイルエラー
```

```text
エラー: シンボルを見つけられません
  シンボル:   メソッド fetch()
  場所: タイプAnimalの変数 a
```

`a` の中身は `Dog` ですが、`a` の**型は `Animal`** です。
Java は、変数の**型**を見て「呼んでよいメソッドか」を判断します。
`Animal` には `fetch()` がないので、「`Animal` 型の `a` から `fetch()` は呼べない」とエラーになります。

これは、こう考えると納得できます。
`Animal` 型の変数には、`Cat` が入っているかもしれません。`Cat` は `fetch()` を持っていません。
だから Java は、「`Animal` 型なら、どの動物にも共通する `Animal` のメソッドだけ呼べる」と、安全側に制限しているのです。

---

## 2つの「型」を区別する

ここで、変数には2つの「型」がある、と整理しておきましょう。

```java
Animal a = new Dog();
```

- **変数の型**（`Animal`） … コンパイル時に「呼べるメソッド」を決める。**見かけの型**
- **中身の型**（`Dog`） … 実際に入っているオブジェクトの型。**本当の型**（実行時の型）

`a.cry()` のように、「呼べるかどうか」は**変数の型（`Animal`）**で決まります。
一方、「実際にどの `cry()` が動くか」は**中身の型（`Dog`）**で決まります。これが、次の節で学ぶ「動的束縛」です。
この2つの型の違いが、ポリモーフィズムを理解する鍵になります。

---

## まとめ

- 子のオブジェクトを、親の型の変数に入れられる（**アップキャスト**）。`Animal a = new Dog();`
- これは **is-a の関係**（犬は動物の一種）のおかげ。安全なので、ただ代入するだけ
- `Dog` でも `Cat` でも、同じ `Animal` 型で受けられる → **まとめて扱える**土台
- ただし、親の型から呼べるのは、**親が持つメソッドだけ**（`Animal` 型から `Dog` 独自の `fetch()` は呼べない）
- 変数には「**変数の型**（呼べるメソッドを決める）」と「**中身の型**（実際のオブジェクト）」がある

次の節では、親の型で呼んでも**子のメソッドが動く**しくみ ―― 動的束縛を学びます。
