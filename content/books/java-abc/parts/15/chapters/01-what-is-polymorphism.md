## ポリモーフィズムとは ― 同じ呼び出し、違うふるまい

**ポリモーフィズム**（Polymorphism）は、ギリシャ語で「多くの（poly）形（morph）」を意味します。
日本語では **多態性**（たたいせい）と訳されます。

その心は、

> **同じ「呼び出し」に対して、相手のオブジェクトに応じて、違う「ふるまい」が起こる**

ことです。
この節では、まずその考え方を、身近な例でつかみます。

---

## 身近なポリモーフィズム

ポリモーフィズムは、日常にもあふれています。

たとえば、いろいろな動物に「鳴いて」とお願いする場面を考えてください。

- 犬に「鳴いて」と言えば → 「ワン」
- 猫に「鳴いて」と言えば → 「ニャー」
- 牛に「鳴いて」と言えば → 「モー」

**「鳴いて」という同じお願い**をしているのに、相手によって**返ってくる鳴き声は違います**。
私たちは、相手が犬か猫かを意識して、お願いの仕方を変えたりはしません。ただ「鳴いて」と言うだけです。あとは、それぞれが自分のやり方で応えてくれます。

これが、ポリモーフィズムの考え方そのものです。
「鳴いて」という共通の呼び出しに対して、犬・猫・牛が、それぞれのふるまいで応える ―― **呼ぶ側は1通り、応える側は多様**、というわけです。

---

## オーバーライドが土台になっている

実は、第14章で学んだ**オーバーライド**が、ポリモーフィズムの土台です。

```java
class Animal {
    String name;
    void cry() { IO.println(name + ": （動物の鳴き声）"); }
}

class Dog extends Animal {
    @Override
    void cry() { IO.println(name + ": ワン"); }
}

class Cat extends Animal {
    @Override
    void cry() { IO.println(name + ": ニャー"); }
}
```

`Dog` と `Cat` は、`Animal` の `cry()` を、それぞれ自分用にオーバーライドしていました。
これにより、「`cry()` という同じ呼び出しに対して、`Dog` なら『ワン』、`Cat` なら『ニャー』」という、**1つの呼び出し・複数のふるまい**の準備ができています。

ここまでは第14章で見たとおりです。
ポリモーフィズムが本当に力を発揮するのは、これを「**親の型でまとめて扱う**」ときです。
そのために必要なのが、次の節で学ぶ「親の型で子を扱う」というしくみです。

---

## ポリモーフィズムの2つの利点（先取り）

これから学ぶポリモーフィズムには、大きく2つの利点があります。
いまは「こういう良いことがあるのか」と、頭の片隅に置いておいてください。

1. **まとめて扱える**
   犬・猫・牛を、すべて「動物」として、同じように扱える。種類ごとに処理を分けなくてよい。
2. **変更に強い**
   新しい動物（鳥など）を追加しても、「すべての動物に鳴いてもらう」部分のコードは、変えなくてよい。

「種類が増えても、使う側のコードは変わらない」 ―― これが、大きなプログラムをすっきり保つ秘訣です。
その具体的なやり方を、この章で順に見ていきましょう。

## まとめ

- **ポリモーフィズム**（多態性）は、「**同じ呼び出しに対して、相手に応じて違うふるまいが起こる**」こと
- 「鳴いて」の一言に、犬は「ワン」、猫は「ニャー」と応えるようなイメージ
- 第14章の**オーバーライド**が、その土台になっている（1つの呼び出し・複数のふるまい）
- ポリモーフィズムにより、(1) いろいろな種類を**まとめて扱え**、(2) 種類が増えても**変更に強い**

次の節では、ポリモーフィズムの鍵となる「**親の型で、子のオブジェクトを扱う**」を学びます。
