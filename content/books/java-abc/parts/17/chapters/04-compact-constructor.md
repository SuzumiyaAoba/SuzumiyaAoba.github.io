## コンパクトコンストラクタ ― 値を検証する

レコードは、コンストラクタを自動で用意してくれました。
ですが、第12章・第13章で学んだように、「**ありえない値を、はじきたい**」こともあります。
たとえば、座標が負の値にならないようにしたい、などです。

レコードでは、**コンパクトコンストラクタ**（Compact Constructor）という特別な書き方で、値の検証を加えられます。

---

## コンパクトコンストラクタの書き方

ふつうのコンストラクタは、`Point(int x, int y)` のように引数リストを書きました。
レコードの**コンパクトコンストラクタ**は、**引数リストを書きません**。レコード名だけを書きます。

```java
record Point(int x, int y) {
    Point {                       // 引数リスト ( ) を書かない！
        if (x < 0) x = 0;         // 負なら 0 に補正する
        if (y < 0) y = 0;
    }
}
```

`Point { ... }` が、コンパクトコンストラクタです。`Point(int x, int y)` の `(int x, int y)` を省いた形です。
中では、コンポーネントと同じ名前（`x`・`y`）の値を、そのまま使えます。

ここがポイントです。
コンパクトコンストラクタの中で `x` や `y` を**書き換えると、その書き換えた値が、最終的にフィールドへ格納されます**。
`this.x = x;` のような代入は、自分で書く必要はありません（その部分は、最後に自動で行われます）。
私たちは、「格納される前に、値をチェック・補正する」ことだけに集中すればよいのです。

---

## 動かして確かめる

負の座標を渡して、補正されるか確かめましょう。

```java
record Point(int x, int y) {
    Point {
        if (x < 0) x = 0;
        if (y < 0) y = 0;
    }
}

public class Main {
    public static void main(String[] args) {
        Point p = new Point(-3, 4);   // x に負の値を渡す
        IO.println(p);
    }
}
```

```text
$ java Main.java
Point[x=0, y=4]
```

`-3` を渡しましたが、コンパクトコンストラクタの補正によって、`x` は `0` になりました。
このように、レコードでも、「生まれた瞬間に、正しい状態を保証する」ことができます。
第13章でふつうのクラスに対して行った検証を、レコードでは、このコンパクトコンストラクタで行うのです。

---

## 不正な値は、はじいて止めることもできる（予告）

補正するのではなく、「**不正な値なら、エラーにして処理を止めたい**」こともあります。
その場合は、コンパクトコンストラクタの中で、**例外を投げて**処理を中断します。

```java
record Point(int x, int y) {
    Point {
        if (x < 0 || y < 0) {
            throw new IllegalArgumentException("座標は負にできません: " + x + ", " + y);
        }
    }
}
```

`throw new IllegalArgumentException(...)` は、「不正な引数だ」というエラー（例外）を発生させ、その場で処理を止めます。
こうすると、`new Point(-3, 4)` のような不正な作成は、その時点でエラーになり、**おかしな `Point` が作られること自体を防げます**。

この「例外を投げる」というしくみは、とても重要なテーマなので、第20章「例外処理」でくわしく学びます。
ここでは、「コンパクトコンストラクタで検証し、ダメな値ははじける」ことだけ、知っておいてください。

> **どちらを使うか: 補正か、エラーか**
>
> - **補正**（`x < 0` なら `0` にする） … 多少おかしな値でも、それなりに動かしたいとき
> - **エラー**（例外を投げる） … おかしな値は、絶対に受け付けたくないとき
>
> 実際の開発では、「ありえない値は、はっきりエラーにする」（例外を投げる）ほうが、バグを早く見つけられるため、好まれることが多いです。
> ただし、どちらが適切かは場面によります。「値を検証する場所を、1か所（コンストラクタ）に集約できる」ことが、ここでの大事な利点です。

---

## まとめ

- レコードでは、**コンパクトコンストラクタ**で、値の検証・補正を加えられる
- 書き方は、**引数リストを省いて** `Point { ... }` とする
- 中でコンポーネント（`x`・`y`）を書き換えると、その値が格納される（`this.x = x;` は不要）
- 補正するほか、**例外を投げて**不正な値をはじくこともできる（くわしくは第20章）
- 検証を1か所（コンストラクタ）に集約できるのが利点
