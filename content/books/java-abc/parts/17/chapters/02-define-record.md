## レコードを定義する

それでは、レコードを実際に定義して、使ってみましょう。
この節では、レコードの基本的な「作り方」と「使い方」を学びます。

---

## レコードを定義して、オブジェクトを作る

座標を表す `Point` レコードを定義し、オブジェクトを作ってみます。

```java
record Point(int x, int y) {}

public class Main {
    public static void main(String[] args) {
        Point p = new Point(3, 4);   // ふつうのクラスと同じく new で作る
        IO.println(p);
    }
}
```

```text
$ java Main.java
Point[x=3, y=4]
```

`record Point(int x, int y) {}` の1行で、`Point` が定義できました。
オブジェクトの作り方は、ふつうのクラスと同じく `new Point(3, 4)` です。
`( )` に渡した `3` と `4` が、それぞれ `x`・`y` に入ります。

そして、`IO.println(p)` で、`Point[x=3, y=4]` という、**中身のわかる表示**が出ました。
これは、レコードが `toString` を**自動で**用意してくれているからです（第14章では、自分で書く必要がありました）。

---

## アクセサ ― 値を取り出す

レコードの中の値を取り出すには、**コンポーネント名と同じ名前のメソッド**を使います。

```java
Point p = new Point(3, 4);
IO.println(p.x());   // x を取り出す → 3
IO.println(p.y());   // y を取り出す → 4
```

```text
3
4
```

ここで注意してほしいのは、メソッド名が `getX()` ではなく、**`x()`** だということです。
これまでのクラスでは、ゲッターを `getX()` という名前で作っていました。
ですが、レコードが自動で用意する取り出しメソッド（**アクセサ**、Accessor と呼びます）は、**コンポーネント名そのまま**（`x()`、`y()`）になります。

`get` が付かないので、最初は少し戸惑うかもしれませんが、レコードの決まりとして覚えておきましょう。

---

## 値は変えられない（不変）

レコードは不変なので、一度作った `Point` の値を変えるメソッド（セッター）は、用意されません。
そして、コンポーネントは `final` なので、直接書き換えることもできません。

```java
Point p = new Point(3, 4);
p.x = 10;    // ✕ x は外から触れず、しかも変更もできない
```

```text
エラー: xはPointでprivateアクセスされます
```

レコードのコンポーネントは、内部では `private` かつ `final` なフィールドとして持たれています。
そのため、外から `p.x` に直接アクセスすること自体ができず（`private`）、仮にできたとしても変更はできません（`final`）。
「`x` を変えたい」ときは、第13章の不変オブジェクトと同じく、**変えた結果を持つ新しいレコードを作ります**。

```java
Point p1 = new Point(3, 4);
Point p2 = new Point(10, p1.y());   // x だけ変えた、新しい Point
```

`p1` はそのまま、`x` だけ変えた `p2` を新しく作る ―― 不変オブジェクトらしい使い方です。

---

## メソッドを追加することもできる

レコードは「データを束ねるだけ」が基本ですが、本体の `{ }` の中に、**メソッドを追加**することもできます。
たとえば、原点からの距離を求めるメソッドを足してみましょう。

```java
record Point(int x, int y) {
    double distanceFromOrigin() {
        return Math.sqrt(x * x + y * y);   // √(x² + y²)
    }
}

public class Main {
    public static void main(String[] args) {
        Point p = new Point(3, 4);
        IO.println(p.distanceFromOrigin());
    }
}
```

```text
$ java Main.java
5.0
```

メソッドの中では、コンポーネント（`x`・`y`）を、そのまま使えます。
`(3, 4)` の原点からの距離は `5.0`（3-4-5 の直角三角形）と、正しく計算できました。
（`Math.sqrt` は、平方根を求める標準ライブラリのメソッドです。）

このように、レコードは「データ＋そのデータに関する計算」を、コンパクトにまとめるのにも使えます。

---

## まとめ

- レコードは、ふつうのクラスと同じく **`new`** でオブジェクトを作る
- `IO.println(レコード)` で、`Point[x=3, y=4]` のように**中身が表示される**（`toString` が自動生成）
- 値を取り出す**アクセサ**は、`getX()` ではなく **`x()`**（コンポーネント名そのまま）
- レコードは**不変**。値を変えるには、新しいレコードを作る
- 本体の `{ }` に、メソッドを追加することもできる
