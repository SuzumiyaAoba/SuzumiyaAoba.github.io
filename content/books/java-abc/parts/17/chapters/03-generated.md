## 自動で用意されるもの

レコードの便利さの正体は、「**たくさんの決まり文句を、自動で用意してくれる**」ことです。
この節では、`record Point(int x, int y) {}` の1行で、具体的に何が用意されるのかを、1つずつ確認します。

レコードが自動で用意してくれるのは、次の4つです。

1. コンストラクタ
2. アクセサ（値を取り出すメソッド）
3. `toString`
4. `equals` と `hashCode`

---

## 1. コンストラクタ（正準コンストラクタ）

`( )` に書いたコンポーネントを、すべて受け取るコンストラクタが、自動で用意されます。
これを **正準コンストラクタ**（Canonical Constructor）と呼びます。

```java
record Point(int x, int y) {}
```

この1行で、次のコンストラクタが自動で作られています。

```java
// 自動で用意される（自分では書かなくてよい）
Point(int x, int y) {
    this.x = x;
    this.y = y;
}
```

だから、`new Point(3, 4)` で、`x` と `y` を渡してオブジェクトを作れたのです。

---

## 2. アクセサ

前の節で見たとおり、各コンポーネントの値を取り出す**アクセサ**（`x()`・`y()`）が自動で用意されます。

```java
Point p = new Point(3, 4);
p.x();   // → 3
p.y();   // → 4
```

名前は、コンポーネント名そのまま（`getX()` ではなく `x()`）でした。

---

## 3. toString

オブジェクトの中身を、読みやすい文字列で返す `toString` が自動で用意されます。

```java
Point p = new Point(3, 4);
IO.println(p);   // → Point[x=3, y=4]
```

第14章では、`Object` の標準の `toString`（`Point@2f92e0f4` のような表示）を、自分でオーバーライドする必要がありました。
レコードなら、`レコード名[コンポーネント名=値, ...]` という形の `toString` が、最初から用意されています。

---

## 4. equals と hashCode ―「中身で等しい」を判定する

ここが、レコードの特にうれしいところです。
レコードは、**中身（コンポーネント）がすべて等しければ「等しい」と判定する** `equals` を、自動で用意してくれます。

第10章・第14章で、「オブジェクトの比較は `==` ではなく `equals`」「中身で比べたいなら `equals` をオーバーライドする」と学びました。
レコードなら、その `equals` が、最初から正しく用意されているのです。

確かめてみましょう。

```java
record Point(int x, int y) {}

public class Main {
    public static void main(String[] args) {
        Point p1 = new Point(3, 4);
        Point p2 = new Point(3, 4);   // 中身は同じだが、別々に作ったオブジェクト

        IO.println(p1 == p2);          // 「同じものか」（場所が同じか）
        IO.println(p1.equals(p2));     // 「中身が同じか」
    }
}
```

```text
$ java Main.java
false
true
```

- `p1 == p2` は **`false`** … `new` で別々に作ったので、「別のもの」（第11章の参照のとおり）
- `p1.equals(p2)` は **`true`** … `x` も `y` も同じなので、「中身が等しい」

ふつうのクラスでこの `equals` を自分で正しく書くのは、意外と難しく、`hashCode` とセットで用意する決まりごともありました（第14章で触れたとおりです）。
レコードは、それを**自動で、正しく**やってくれます。
「中身が同じなら等しい」と判定したいデータには、レコードがうってつけなのです。

> **補足: hashCode とは**
>
> `hashCode()` は、オブジェクトを整数の「番号」で表すメソッドで、`equals` とセットで使われます。
> 第3部で学ぶ `HashMap` や `HashSet` といったしくみが、この `hashCode` を使って、データを高速に探します。
> レコードは、`equals` と矛盾しない `hashCode` も自動で用意してくれるので、こうしたしくみと、安心して組み合わせられます。
> いまは「`equals` と `hashCode` は、セットで自動生成される」と知っておけば十分です。

## まとめると

`record Point(int x, int y) {}` という1行は、次のすべてを自動で用意してくれます。

| 自動で用意されるもの | 内容 |
|---|---|
| 正準コンストラクタ | `Point(int x, int y)`（全コンポーネントを受け取る） |
| アクセサ | `x()`・`y()`（値を取り出す） |
| `toString` | `Point[x=3, y=4]`（中身を表示） |
| `equals`・`hashCode` | 中身が等しいかを判定する |

これらを、ふつうのクラスで手書きすると、何十行にもなります。
レコードは、その手間を1行に凝縮してくれる ―― これが、レコードの大きな魅力です。

---

## まとめ

- レコードは、**コンストラクタ・アクセサ・`toString`・`equals`・`hashCode`** を自動で用意する
- 全コンポーネントを受け取るコンストラクタを、**正準コンストラクタ**と呼ぶ
- `equals` は「**中身がすべて等しいか**」を判定する（`==` は「同じものか」）
- `equals` と矛盾しない `hashCode` も、セットで自動生成される
- 「中身が同じなら等しい」と扱いたいデータに、レコードはうってつけ
