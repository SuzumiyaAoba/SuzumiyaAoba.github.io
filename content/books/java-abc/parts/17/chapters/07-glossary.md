## 用語集 ― この章で学んだ言葉

第17章では、データを束ねる型「レコード」を学びました。
節ごとに、言葉を整理しておきます。

---

## レコードの基本（第1〜3節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| レコード | Record | データを束ねることに特化した、簡潔に書ける不変クラス（Java 16+） |
| コンポーネント | Component | レコードが持つ1つひとつのデータ（`record Point(int x, int y)` の `x`・`y`） |
| アクセサ | Accessor | コンポーネントの値を取り出すメソッド。名前はコンポーネント名そのまま（`x()`） |
| 正準コンストラクタ | Canonical Constructor | 全コンポーネントを受け取る、自動生成のコンストラクタ |

レコードは、コンストラクタ・アクセサ・`toString`・`equals`・`hashCode` を自動生成する。

---

## 検証と使いどころ（第4〜5節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| コンパクトコンストラクタ | Compact Constructor | 引数リストを省いた `Foo { ... }` の形。値の検証・補正に使う |
| 値オブジェクト | Value Object | 「中身が同じなら等しい」と扱う、値そのものを表すデータ。レコードに向く |

レコードはクラスを継承できない（`final`）が、インターフェースは実装できる。

---

## おわりに

第17章では、「データを束ねる」ことに特化したレコードを学びました。

- `record Point(int x, int y) {}` の1行で、不変のデータ型ができる
- コンストラクタ・アクセサ・`toString`・`equals`・`hashCode` は自動生成
- コンパクトコンストラクタで検証を加えられる
- 「値が主役の、変わらないデータ」に最適

レコードは、第13章で学んだ「不変オブジェクト」を、ぐっと手軽にしてくれるしくみでした。

さて、レコードは「データを束ねる」ための型でした。
もう1つ、よく使う特別な型があります。それは、「**決まったいくつかの値の、どれか1つ**」を表す型です。
たとえば「曜日（月〜日のどれか）」「信号の色（赤・黄・青のどれか）」「注文の状態（受付・発送・完了のどれか）」のように、**選択肢が決まっている**データです。

次の第18章「列挙型（enum）」では、こうした「決まった選択肢」を、安全に表すしくみを学びます。
