## レコードとは ― データを束ねる型

**レコード**（Record）は、「**データを束ねて持つ**」ことに特化した、簡潔に書ける型です。
この節では、レコードがどんなもので、なぜ便利なのかを学びます。

---

## ふつうのクラスとの比較

まず、第13章で作った不変の `Point` クラスを思い出しましょう。

```java
class Point {
    private final int x;
    private final int y;

    Point(int x, int y) {
        this.x = x;
        this.y = y;
    }
    public int getX() { return x; }
    public int getY() { return y; }
}
```

`x` と `y` を持つだけのクラスなのに、コンストラクタとゲッターで、これだけの行数が必要でした。
さらに、第14章で学んだ `toString` や `equals` も用意すると、もっと長くなります。

これを、レコードで書くと、こうなります。

```java
record Point(int x, int y) {}
```

**たった1行**です。
しかも、この1行で、

- `x` と `y` を持つ
- コンストラクタ
- `x` と `y` を取り出すメソッド
- `toString`（中身を表示）
- `equals`・`hashCode`（中身での比較）

が、**すべて自動で用意されます**（くわしくは第3節）。
本質的な情報（「`Point` は `x` と `y` を持つ」）だけを書けば、あとは Java がやってくれるのです。

---

## record の読み方

`record Point(int x, int y) {}` を分解してみましょう。

```text
record   Point   (int x, int y)   {}
  │       │            │           │
  │       │            │           └─ 本体（今回は空）
  │       │            └─ このレコードが持つデータ（コンポーネント）
  │       └─ レコードの名前
  └─ 「レコードを定義します」というキーワード
```

`class` のかわりに **`record`** と書き、名前のうしろの **`( )`** の中に、束ねたいデータを並べます。
この `( )` の中に書いた1つひとつ（`int x`、`int y`）を、**コンポーネント**（Component、構成要素）と呼びます。
「`Point` は、`x` と `y` という2つのコンポーネントからなる」というわけです。

本体の `{ }` は、今回は空です。
基本的なものはすべて自動生成されるので、追加したいメソッドがなければ、`{ }` は空のままで構いません。

---

## レコードの最大の特徴 ― 不変である

レコードの、もう1つの大事な特徴は、**不変（イミュータブル）である**ことです。
第13章で学んだ「一度作ったら、中身が変わらない」オブジェクトですね。

レコードのコンポーネント（`x`・`y`）は、自動的に `final` になります。
そのため、一度作った `Point` の `x` や `y` を、あとから書き換えることはできません。

```java
Point p = new Point(3, 4);
// p の x や y を変える手段は、用意されない（不変）
```

第13章では、不変オブジェクトを作るのに「全フィールドを `final`、セッターを作らない…」と、いくつものルールを守る必要がありました。
レコードを使えば、**それらがすべて自動で守られた、安全な不変オブジェクト**が、1行で手に入るのです。

「変える必要のないデータ（座標・金額・日付など）は、不変オブジェクトにするのがよい」 ―― 第13章で学んだこの方針を、レコードは手軽に実現してくれます。

---

## いつレコードを使うのか

レコードは、「**データを束ねて運ぶだけ**」の場面に向いています。

- 座標（x, y）
- 商品（名前, 価格）
- 期間（開始日, 終了日）
- 計算の結果をまとめて返したいとき（複数の値を1つにまとめる）

一方、「複雑なふるまい（メソッド）を中心に持つ」クラスや、「途中で状態が変わる」オブジェクトには、レコードは向きません。
そうしたものは、これまでどおりのクラスで書きます。
使い分けの考え方は、第5節でくわしく整理します。

---

## まとめ

- **レコード**は、「データを束ねて持つ」ことに特化した、簡潔に書ける型
- `record Point(int x, int y) {}` の**1行**で、コンストラクタ・アクセサ・`toString`・`equals` などが自動生成される
- `( )` の中の1つひとつ（`int x` など）を**コンポーネント**と呼ぶ
- レコードは**不変**（コンポーネントは自動的に `final`）。安全な不変オブジェクトが手軽に作れる
- 「データを束ねるだけ」の場面に向く（複雑なふるまいや、変化する状態には不向き）
