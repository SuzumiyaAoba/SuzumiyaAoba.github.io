## はじめに ― この章で学ぶこと

第11章から第16章で、オブジェクト指向の「クラスを設計する」技術を、ひととおり学びました。
カプセル化・継承・ポリモーフィズム・抽象クラス・インターフェース ―― これらを使いこなせば、たいていのクラスは設計できます。

ですが、ここまでの書き方には、1つ「面倒なところ」が残っていました。
それは、**「ただデータを束ねるだけ」の単純なクラスでも、たくさんの決まり文句を書く必要がある**ことです。

たとえば、座標を表す `Point`（第13章の不変オブジェクト）を、きちんと作ると、こうなります。

```java
class Point {
    private final int x;
    private final int y;

    Point(int x, int y) {          // コンストラクタ
        this.x = x;
        this.y = y;
    }
    public int getX() { return x; }   // ゲッター
    public int getY() { return y; }

    @Override
    public String toString() { ... }  // toString
    @Override
    public boolean equals(Object o) { ... }  // equals
    @Override
    public int hashCode() { ... }     // hashCode
}
```

`x` と `y` という、たった2つのデータを束ねたいだけなのに、コンストラクタ・ゲッター・`toString`・`equals`・`hashCode` …と、本質的でない決まり文句が、ずらりと並びます。

この第17章で学ぶ **レコード**（Record）を使うと、この `Point` を、**たった1行**で書けます。

```java
record Point(int x, int y) {}
```

これだけで、上のクラスとほぼ同じもの（しかも安全な不変オブジェクト）が手に入るのです。

## レコードは「データを束ねる」ための型

**レコード**は、Java 16 で正式に導入された、比較的新しいしくみです。
ひとことで言えば、

> **「データを束ねて運ぶ」ことに特化した、簡潔に書ける不変クラス**

です。

世の中のプログラムには、「座標（x, y）」「商品（名前, 価格）」「期間（開始日, 終了日）」のように、**いくつかの値をまとめて持つだけ**のデータが、たくさん登場します。
そうした「データの入れ物」を作るために、毎回たくさんの決まり文句を書くのは、無駄が多いものでした。
レコードは、その決まり文句を **Java が自動で用意してくれる**ことで、私たちの手間を大きく減らしてくれます。

## この章で学ぶこと

第17章は、次の7つの節で構成されています。

| 節 | タイトル | 内容 |
|---|---|---|
| 1 | レコードとは | データを束ねる型。なぜ生まれたか |
| 2 | レコードを定義する | `record` の基本の書き方 |
| 3 | 自動で用意されるもの | コンストラクタ・アクセサ・`toString`・`equals` |
| 4 | コンパクトコンストラクタ | 値の検証を加える |
| 5 | レコードの使いどころ | 何に使い、何に使わないか |
| 6 | よくあるつまずき | 変更できない・継承できない など |

## この章を読み終えると

第17章を読み終えるころには、次のことができるようになっています。

- `record` で、データを束ねる型を1行で定義できる
- レコードが自動で用意してくれるもの（コンストラクタ・アクセサ・`toString`・`equals`・`hashCode`）がわかる
- コンパクトコンストラクタで、値の検証を加えられる
- レコードが「不変」であることを理解し、適切に使い分けられる

それでは、最初の節「レコードとは」から始めましょう。
