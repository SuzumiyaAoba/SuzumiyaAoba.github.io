## 用語集 ― この章で学んだ言葉

### ガベージコレクション（Garbage Collection、GC）

不要になったオブジェクトを、自動で回収する仕組み。
Java では JVM の中で常時動いている。

### 到達可能（reachable） / 到達不可能（unreachable）

ルート（スタックの変数、static フィールドなど）から、参照を辿って届くオブジェクトが「到達可能」。
届かないものが「到達不可能」で、GC の回収対象。

### ルート（root）

GC が参照を辿る出発点。
スタック上の変数、static フィールド、JNI ハンドルなどが含まれる。

### Stop The World（STW）

GC が**全スレッドを止めて**ヒープを回収すること。
止めずに行う**並行**（concurrent）部分との対比で語られる。

### マーク&スイープ（Mark and Sweep）

「生きてるオブジェクトに印 → ヒープを舐めて未印を回収」の 2 段階アルゴリズム。
**断片化**しやすい欠点がある。

### コピー GC（Copying GC）

ヒープを 2 つに分け、生きているものを片方からもう片方にコピーする方式。
**断片化しない**が、ヒープを半分しか使えない。

### マーク&コンパクト（Mark and Compact）

マークしたあと、生きているオブジェクトを**端から詰め直す**方式。
ヒープ全体を使えるが、移動コストが重い。

### 弱世代仮説（Weak Generational Hypothesis）

「オブジェクトの大半はすぐ死に、長寿のものはますます長寿になる」という経験則。
**世代別 GC** の根拠。

### 世代別 GC（Generational GC）

ヒープを「**若い世代（Young）**」と「**古い世代（Old）**」に分けて、それぞれ違うアルゴリズムで回収する方式。

### Young Generation（若い世代）

新しく `new` されたオブジェクトの住処。
**Eden + Survivor 0 + Survivor 1** から成る。

### Eden（エデン）

新しく作られたオブジェクトが、最初に置かれる領域。

### Survivor（サバイバー）

マイナー GC を生き残ったオブジェクトが、一時的に置かれる領域。
**S0 と S1 の 2 つ**を持ち、コピー GC のためにいつも片方が空いている。

### Old Generation（古い世代、Tenured）

何度ものマイナー GC を生き延びた長寿オブジェクトの住処。

### 昇格（Promotion、Tenuring）

オブジェクトの**年齢**が閾値を超えたとき、または Survivor が溢れたときに、Young から Old に移動すること。

### 早期昇格

本当はまだ Young にいたほうがよいオブジェクトが、Survivor 不足で早めに Old へ移動してしまうこと。
後のメジャー GC を重くする。

### マイナー GC（Minor GC、Young GC）

Young Generation だけを対象とする、速くて軽い GC。
頻度は高いが、1 回あたりは ms 単位。

### メジャー GC（Major GC） / フル GC（Full GC）

Young + Old を対象とする、重い GC。
頻度は低いが、1 回あたりが百 ms〜秒単位になることも。

### Humongous（ヒューマンガス）

G1 GC の用語で、**リージョンの半分**を超える大きなオブジェクト。
専用の配置・回収パスを持つ。

### G1 GC（Garbage-First）

Java 9 以降の**既定 GC**。ヒープをリージョンに分けて、ゴミの多いリージョンから優先回収する。
**目標停止時間**（`-XX:MaxGCPauseMillis`）に合わせて、自動でチューニングする。

### ZGC

**STW を 1 ms 以下**に抑えることを狙った、超低レイテンシ GC。
Java 23 以降は、世代別 ZGC が既定。
**読み取りバリア**と**カラー付きポインタ**で、並行回収を実現している。

### Parallel GC

**スループット最優先**の GC。
GC の合計時間を最小化し、アプリの処理量を最大化する。
バッチ・分析用途に向く。

### Serial GC

**シングルスレッド**で動く GC。
小さなコンテナ・低スペック環境で、シンプルさが利点。

### Epsilon GC

「**何もしない**」GC。
超短命ジョブや、性能基準値の計測など、特殊用途のみ。

### 目標停止時間（`-XX:MaxGCPauseMillis`）

G1 に渡す「**ここまでに GC を終わらせて**」というヒント。
絶対の保証ではなく、ベストエフォート。

### `-Xms` / `-Xmx`

ヒープの初期サイズと最大サイズ。
サーバー用途では**同じ値**にすることが多い。

### `-XX:MaxRAMPercentage`

ヒープサイズを**コンテナのメモリ上限の何 %** にするかを指定するオプション。
コンテナで動かすときに便利。

### GC ログ

GC のたびに出力されるログ。Java 9 以降は `-Xlog:gc` で取得。
1 行で「**経過時間・種類・Before → After (Heap)・所要時間**」が分かる。

### `OutOfMemoryError`

ヒープが尽きたときに投げられる Error。
メッセージで領域（`Java heap space` / `Metaspace` / `GC overhead limit exceeded` など）が分かる。

### メモリリーク（memory leak）

不要になったオブジェクトへの参照が**残り続けて**、GC されない状態。
Java では、コレクションへの入れっぱなし、リスナの外し忘れ、`ThreadLocal` の `remove` 忘れが典型。

### `finalize()` / `Cleaner`

オブジェクト回収前に呼ばれる、後始末の仕組み。
**`finalize()` は非推奨**で、使ってはいけない。
リソース管理は **`try-with-resources`** を使う。
