## はじめに ― この章で学ぶこと

第3部も、いよいよ最後の章です。

これまでのプログラムは、データを**プログラムの中だけ**で扱ってきました。
変数やコレクションに入れたデータは、プログラムが終わると、消えてしまいます。

ですが、実用的なプログラムでは、データを**プログラムの外**に保存したり、外から読み込んだりする必要があります。

- 設定を、**ファイルから読み込む**
- 処理の結果を、**ファイルに保存する**
- 動作の記録（ログ）を、**ファイルに書き出す**
- CSV やテキストのデータを、**読み取って処理する**

この、ファイルを**読み書き**する技術を、**ファイル入出力**（File I/O）と呼びます。
「入出力」は英語で Input/Output、略して **I/O**（アイ・オー）です。
この第30章で、その基本を学びます。

---

## モダンな API ― java.nio.file

Java には、ファイルを扱う方法がいくつかありますが、本書では、現代的でかんたんな **`java.nio.file`** パッケージ（`Path` と `Files`）を使います。

```java
Path path = Path.of("memo.txt");                 // ファイルの場所を表す
Files.writeString(path, "こんにちは");             // 書き込む
String content = Files.readString(path);          // 読み込む
```

たった数行で、ファイルへの読み書きができます。
かつての Java では、ファイル入出力は、もっと複雑で長いコードが必要でした。
`Files` クラスのおかげで、いまでは、とても手軽になっています。

## これまでの学びが、活きる章

ファイル入出力は、これまで学んだ知識を**総動員**するテーマです。

- **例外処理**（第20章） … ファイルがない、読めない、などのトラブルに備える（`IOException`）
- **try-with-resources**（第20章） … 開いたファイルを、確実に閉じる
- **コレクション**（第21章） … 複数行を `List` で扱う
- **ストリーム**（第23章） … 大きなファイルを、1行ずつ効率よく処理する

第3部の集大成として、これらが、ここで1つにつながります。

## この章で学ぶこと

第30章は、次の7つの節で構成されています。

| 節 | タイトル | 内容 |
|---|---|---|
| 1 | Path と Files | ファイルの場所と、操作の道具 |
| 2 | テキストの読み書き | `readString`・`writeString` |
| 3 | 行単位の読み書き | 複数行・大きなファイル |
| 4 | ディレクトリの操作 | 存在確認・フォルダ作成 |
| 5 | 入出力と例外 | `IOException`・try-with-resources |
| 6 | よくあるつまずき | パス・文字コード・閉じ忘れ など |

前半（1〜3節）で、ファイルの基本的な読み書きを学びます。
後半（4〜5節）で、ディレクトリ操作と、入出力に欠かせない例外処理を学びます。

## この章を読み終えると

第30章を読み終えるころには、次のことができるようになっています。

- `Path` でファイルの場所を表し、`Files` で操作できる
- ファイルにテキストを書き込み、読み込める
- 複数行を、`List` やストリームで読み書きできる
- ディレクトリの存在確認や作成ができる
- ファイル操作の例外（`IOException`）に、適切に備えられる

> **補足: この章のサンプルは、完全なプログラムで示します**
>
> ファイル入出力は、`IOException`（チェック例外、第20章）を扱う必要があります。
> そのため、この章のサンプルは、`throws Exception` を付けた `main` を含む**完全なプログラム**で示します。
> `Path`・`Files` は `java.nio.file` パッケージにあるので、`.java` ファイルでは `import java.nio.file.*;` が必要です。

それでは、最初の節「Path と Files」から始めましょう。
