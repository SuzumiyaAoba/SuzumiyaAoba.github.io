## 用語集 ― この章で学んだ言葉

第30章では、ファイルを読み書きする「ファイル入出力」を学びました。
節ごとに、言葉を整理しておきます。

---

## Path と Files（第1〜2節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| ファイル入出力 | File I/O | ファイルを読み書きすること |
| `Path` | パス | ファイル・フォルダの場所を表す（`Path.of("memo.txt")`） |
| `Files` | ファイルズ | ファイル操作の道具（`static` メソッド） |
| `Files.writeString` | ― | 文字列をファイルに書き込む（デフォルト上書き） |
| `Files.readString` | ― | ファイル全体を文字列として読み込む |
| `StandardOpenOption.APPEND` | ― | 上書きではなく追記する |

---

## 行単位とディレクトリ（第3〜4節）

| 用語 | 意味 |
|---|---|
| `Files.write(パス, リスト)` | リストの各要素を1行ずつ書き込む |
| `Files.readAllLines(パス)` | 全行を `List<String>` で読み込む |
| `Files.lines(パス)` | 行をストリームで返す（大きいファイル向き。要 try-with-resources） |
| `Files.exists` / `createDirectories` / `list` / `deleteIfExists` | 存在確認 / フォルダ作成 / 一覧 / 削除 |

---

## 入出力と例外（第5節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| `IOException` | ― | 入出力で起きるチェック例外 |
| `NoSuchFileException` | ― | 存在しないファイルを読もうとしたときの例外 |
| try-with-resources | ― | 開いたファイルを確実に閉じる（第20章） |

開いたまま返すもの（`Files.lines`・`Files.list`・`BufferedReader`）は try-with-resources で閉じる。
