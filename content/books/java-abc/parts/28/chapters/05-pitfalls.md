## よくあるつまずき

テキストブロック・文字列フォーマットで、初心者がつまずきやすいポイントを、まとめて確認します。

---

## 1. テキストブロックの開始 `"""` の後に、文字を書く

テキストブロックの開始 `"""` の**直後**には、文字を書けません。必ず改行が必要です。

```java
String s = """こんにちは""";   // ✕ """ の直後に文字
```

```text line-numbers=false
エラー: テキスト・ブロックの開始区切り文字のシーケンスが無効です。行の終了文字がありません
```

開始の `"""` のあとは、いったん改行して、次の行から内容を書きます。

```java
String s = """
    こんにちは""";   // ◯ 改行してから内容
```

「複数行でない、ただの1行文字列」なら、無理にテキストブロックにせず、ふつうの `"..."` を使えば十分です。

---

## 2. インデントが、思ったとおりにならない

テキストブロックは、左側の共通インデントを自動で取り除きます（第2節）。
取り除かれる量は、**終了の `"""` の位置**にも影響されます。
終了の `"""` を、内容より左に置くと、インデントが残ってしまうことがあります。

```java
String s = """
        りんご
        みかん
""";        // 終了の """ がいちばん左 → インデントが残る
```

意図したインデントにならないときは、「内容」と「終了の `"""`」の**位置（字下げ）をそろえる**と、うまくいくことが多いです。
迷ったら、表示して（`"[" + s + "]"` のように囲って）、実際の空白を確認しましょう。

---

## 3. 書式指定子と、引数の型が合わない

`%d`（整数）に文字列を渡すなど、**書式指定子と引数の型が合わない**と、実行時にエラーになります。

```java
String s = "%d".formatted("佐藤");   // %d は整数なのに、文字列を渡した
```

```text line-numbers=false
Exception in thread "main" java.util.IllegalFormatConversionException: ...
```

- `%d` … 整数（`int`・`long`）
- `%s` … 文字列（実は、何でも文字列にして埋め込める）
- `%f` … 小数（`double`）

迷ったときは、**`%s`** を使うと安全です（`%s` は、数値でも何でも文字列として埋め込めます）。
「数値を3桁区切りにしたい」など、特別な整形が必要なときだけ、`%d`・`%f` を使い分けましょう。

---

## 4. 引数の数が、足りない・多すぎる

書式指定子（`%s` など）の数と、渡す引数の数は、**一致**させる必要があります。
足りないと、実行時にエラーになります。

```java
String s = "%sさんは%d歳".formatted("佐藤");   // %d の分の引数がない
```

```text line-numbers=false
Exception in thread "main" java.util.MissingFormatArgumentException: ...
```

`%s` と `%d` で2つの穴があるなら、引数も2つ渡します。
「穴の数と、渡す値の数を合わせる」を、確認しましょう。

---

## 5. ループの中で + 連結して、遅くなる

第4節のとおり、ループの中で文字列を `+` でつなぐのは、回数が多いと効率が悪くなります。

```java
String result = "";
for (int i = 0; i < 100000; i++) {
    result += i;   // △ 大量のループでは遅い
}
```

回数が多いときは、**`StringBuilder`** を使いましょう。

```java
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 100000; i++) {
    sb.append(i);   // ◯ 効率がよい
}
String result = sb.toString();
```

「数個つなぐだけ」なら `+` で問題ありません。「ループで大量に」のときだけ、`StringBuilder` を思い出しましょう。

---

## まとめ

- テキストブロックの開始 `"""` の**直後は改行**（直後に文字はエラー）
- インデントが変なときは、**内容と終了 `"""` の位置をそろえる**
- 書式指定子と**引数の型を合わせる**（迷ったら `%s` が安全）
- **穴の数と引数の数を一致**させる（足りないとエラー）
- ループで大量につなぐなら **`StringBuilder`**（`+` は遅い）

次は、この章で学んだ言葉を、用語集としてまとめます。
