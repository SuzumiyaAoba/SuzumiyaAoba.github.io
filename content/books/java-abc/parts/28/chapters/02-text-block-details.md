## テキストブロックの細かいルール

テキストブロックには、「**コードのインデント**」と「**実際の文字列**」をうまく両立させる、賢いしくみがあります。
この節では、インデントの扱いと、改行を制御する記法を学びます。

---

## コードのインデントは、自動で取り除かれる

テキストブロックを書くと、ふつうはコードの左側に、インデント（字下げ）の空白がたくさん入ります。

```java
String s = """
        りんご
        みかん
        ぶどう
        """;
```

ここで「りんご」などの前には、コード上、たくさんの空白があります。
でも、本当にほしいのは「りんご」という文字だけで、コードの字下げ分の空白は、いりませんよね。

テキストブロックは、これを賢く処理します。
**各行に共通する、左側のインデント（付帯的な空白）を、自動で取り除いてくれる**のです。

```java
String s = """
        りんご
        みかん
        ぶどう
        """;
IO.println("[" + s + "]");
```

```text
[りんご
みかん
ぶどう
]
```

「りんご」「みかん」「ぶどう」の前の空白は、きれいに取り除かれています。
コードを読みやすく字下げしても、その字下げは文字列に含まれない ―― これが、テキストブロックの便利なところです。

---

## どこまで取り除かれるか ― 終了の `"""` が基準

「どれだけのインデントを取り除くか」は、**各行と、終了の `"""` の位置**を見て決まります。
いちばん左にある行（または終了の `"""`）に合わせて、共通の空白が取り除かれます。

そのため、**わざと深く字下げした行は、その差分が残ります**。

```java
String s = """
        りんご
          みかん
        ぶどう
        """;
IO.println("[" + s + "]");
```

```text
[りんご
  みかん
ぶどう
]
```

「みかん」だけ、ほかより2つ深く字下げしていました。
共通分（左側のそろっている空白）は取り除かれますが、「みかん」の**余分な2つ分**は、文字列に残ります。
これを使えば、「箇条書きの中の、さらに下位の項目」のような、**意図的なインデント**を表現できます。

---

## 行末の空白は、取り除かれる

各行の**末尾**にある空白は、自動的に取り除かれます（見えない空白による事故を防ぐためです）。
もし「行末に、意図的に空白を残したい」なら、**`\s`** を使います。

```java
String s = """
    項目:\s
    値""";
IO.println("[" + s + "]");
```

```text
[項目: 
値]
```

`\s` は「空白1つ」を表し、これがあると、その位置の空白は取り除かれません。
「項目: 」の後ろに、意図的に空白を残せました。
（あまり使う場面は多くありませんが、「行末の空白は消える」「残したいなら `\s`」と知っておくと、困ったときに役立ちます。）

---

## 改行を、つなげる ― 行末の `\`

「コードでは改行して書きたいけれど、文字列としては1行にしたい」ときは、行末に **`\`**（バックスラッシュ）を置きます。

```java
String s = """
    これは長い文章を\
    1行につなげる例です。""";
IO.println(s);
```

```text
これは長い文章を1行につなげる例です。
```

行末の `\` は、「ここで改行しない（次の行とつなげる）」という意味です。
コード上は2行ですが、文字列としては「これは長い文章を1行につなげる例です。」という1行になりました。
「コードでは折り返して読みやすく、文字列としては1行に」したいときに便利です。

---

## まとめ

- テキストブロックは、**各行に共通する左側のインデントを、自動で取り除く**（コードは字下げしてOK）
- 取り除く量は、**各行と終了の `"""` の位置**で決まる。深く字下げした分は残る
- **行末の空白は取り除かれる**。残したいなら **`\s`**（空白1つ）
- 行末の **`\`** で、「改行せずに次の行とつなげる」（コードは複数行、文字列は1行）
