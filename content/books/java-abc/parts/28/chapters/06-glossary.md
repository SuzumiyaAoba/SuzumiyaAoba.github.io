## 用語集 ― この章で学んだ言葉

第28章では、テキストブロックと文字列フォーマットを学びました。
節ごとに、言葉を整理しておきます。

---

## テキストブロック（第1〜2節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| テキストブロック | Text Block | `"""` で囲んで複数行の文字列を見たまま書く（Java 15+） |
| 付帯的な空白 | Incidental Whitespace | コードの字下げ分。自動で取り除かれる |
| `\`（行末） | ― | 改行せず次の行とつなげる |
| `\s` | ― | 空白1つ（行末の空白を残す） |

開始 `"""` の直後は改行。取り除かれるインデントは終了 `"""` の位置で決まる。

---

## 文字列フォーマット（第3〜4節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| `formatted` | ― | テンプレートに値を埋め込む（`"%s".formatted(値)`） |
| `String.format` | ― | 同じ整形の別の書き方 |
| 書式指定子 | Format Specifier | `%s`（文字列）・`%d`（整数）・`%f`（小数）・`%.2f`・`%,d` |
| `String.join` | ― | 区切り文字で複数の文字列をつなぐ |
| `StringBuilder` | ― | 効率よく文字列を組み立てる入れ物（`append`） |

---

## おわりに

この章では、文字列を快適に扱う機能を学びました。

- **テキストブロック**（`"""`） … 複数行を見たまま
- **`formatted`・`String.format`** … 値を埋め込んで整形
- **`String.join`・`StringBuilder`** … 文字列の組み立て

第10章の文字列の基礎に、実務でよく使う道具が加わりました。

次の第29章「並行処理入門」では、少し高度なテーマに挑戦します。
「複数の処理を、同時に進める」並行処理です。
これまでのプログラムは、上から下へ、1つずつ順番に処理してきました。
ですが、「重い処理を待つ間に、別の作業を進めたい」「たくさんの依頼を、同時にさばきたい」――
そんなときに使うのが、**スレッド**、そして Java 21 で導入された画期的な **仮想スレッド（Virtual Threads）** です。
現代のアプリケーションを支える、重要な技術の入口を学びましょう。
