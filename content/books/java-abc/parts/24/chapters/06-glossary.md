## 用語集 ― この章で学んだ言葉

第24章では、値の有無を安全に表す「Optional」を学びました。
節ごとに、言葉を整理しておきます。

---

## Optional の基本（第1〜2節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| `Optional<T>` | オプショナル | 値があるかもしれない／ないかもしれない、を表す箱 |
| `isPresent()` / `isEmpty()` | ― | 中身があるか / 空か を調べる |
| `Optional.of(値)` | ― | 中身が必ずある箱を作る（`null` を渡すとエラー） |
| `Optional.ofNullable(値)` | ― | `null` かもしれない値を包む（`null` なら空に） |
| `Optional.empty()` | ― | 空の箱を作る |

---

## 値の取り出し（第3節）

| メソッド | 動き |
|---|---|
| `orElse(値)` | 空ならデフォルト値を返す |
| `orElseGet(() -> 値)` | 空のときだけデフォルト値を計算する |
| `ifPresent(値 -> 処理)` | 中身があるときだけ処理する |
| `orElseThrow()` | 空なら例外を投げる |
| `get()` | そのまま取り出す（**基本使わない**。空だと例外） |

---

## 箱のまま処理（第4節）

| メソッド | 動き |
|---|---|
| `map(変換)` | 中身があれば変換、空なら空のまま |
| `filter(条件)` | 条件を満たせばそのまま、満たさなければ空に |

`map`・`filter`・`orElse` はつなげて書ける（ストリームと同じ発想）。

---

## おわりに

この章では、`null` との戦いに終止符を打つ「Optional」を学びました。

- `Optional` は「**値の有無を、型で示す箱**」
- 作るのは `of`・`ofNullable`・`empty`
- 取り出すのは `orElse`・`ifPresent`・`orElseThrow`（`get` は避ける）
- `map`・`filter` で、箱のまま安全に処理できる

ここまでの第22〜24章（ラムダ式・ストリーム・Optional）は、**Java 8 で導入された「モダン Java」の3本柱**でした。
これらを使いこなせると、コードは格段に簡潔で、安全になります。

第25章からは、また少し趣を変えて、**実務で必ず使う標準ライブラリ**を学んでいきます。
まずは、「日付と時刻」の扱いです。
「3日後の日付」「2つの日付の差」「日付を決まった形式で表示する」――
こうした、一見簡単そうで、実は落とし穴の多い処理を、安全に扱うための **java.time** という API を学びましょう。
