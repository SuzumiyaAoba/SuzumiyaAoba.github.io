## はじめに ― この章で学ぶこと

**Decorator**（デコレーター、装飾者）パターンは、**「既存のオブジェクトに、機能を着せ重ねていく」**ためのパターンです。

たとえば、コーヒーに「ミルク」「砂糖」「シロップ」を順に追加していくイメージ。
継承で「ミルク入りコーヒー」「砂糖入りコーヒー」「ミルクと砂糖入りコーヒー」…とクラスを増やす代わりに、**包む（wrap する）**ことで、組み合わせを自由に作れます。

Java の標準ライブラリで言うと、`BufferedReader` が `Reader` を包み、`InputStreamReader` がさらにその下を包む、あの形が Decorator です。

---

## この章を読み終えると

- Decorator の**意図**を、自分の言葉で説明できる
- **継承**との違いを言える
- `java.io` の入出力ストリームの設計が、すっきり読めるようになる
