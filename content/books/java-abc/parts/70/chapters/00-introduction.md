## はじめに ― この章で学ぶこと

**Mediator**（メディエーター、調停者）パターンは、**「複数のオブジェクトが互いに連絡を取り合う**」状況を、**1 つの調停者にまとめる**振る舞いパターンです。

オブジェクトが N 個あって、それぞれが**互いに直接やりとり**すると、関係は最大 N × (N-1) 本に増えます。これでは保守が大変です。
そこで、**「全員が、調停者だけと話す**」形にすると、関係は N 本に減ります。

UI の部品調整、チャットルーム、コントロールタワー的な役割 ―― それが Mediator です。

## この章を読み終えると

- Mediator の**意図**を、自分の言葉で説明できる
- Observer との違いを区別できる
- イベントバス・モジュール調整の役割を見抜ける
