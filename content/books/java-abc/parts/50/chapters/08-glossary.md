## 用語集 ― この章で学んだ言葉

### パフォーマンスチューニング

プログラムの**速度・メモリ・スループット**を改善する作業。
本書の哲学は「**測ってから判断**」。

### ベンチマーク（benchmark）

「**どれだけの時間がかかるか**」を測ること。
プロファイリング（**どこに**時間がかかるか）と補完的。

### JMH（Java Microbenchmark Harness）

Java 公式の**マイクロベンチマーク**フレームワーク。
ウォームアップ、フォーク、デッドコード除去対策を自動でやる。

### `@Benchmark` / `@Warmup` / `@Measurement` / `@Fork`

JMH のアノテーション。
ベンチマークメソッド・ウォームアップ条件・計測条件・別プロセス回数を指定する。

### `Blackhole`

JMH の API。**戻り値を「使った」ことにする**ためのトリック。
JIT のデッドコード除去対策。

### マクロベンチマーク

アプリ全体の throughput（req/s）やレイテンシを測るタイプの計測。
**wrk・JMeter・k6・Gatling** などのツールを使う。

### `-Xmx` / `-Xms`

ヒープの最大サイズ / 初期サイズ。
サーバー用途では同じ値にすることが多い。

### `-XX:MaxRAMPercentage`

コンテナのメモリ上限の何 % をヒープにするかを指定するフラグ。

### `-XX:MaxGCPauseMillis`

G1 GC への「目標停止時間」のヒント。

### CDS（Class Data Sharing）

クラスロード情報を事前にダンプして、起動を高速化する Java 標準機能。
ネイティブイメージほど劇的でない、中間的な選択肢。

### `LinkedList` の罠

教科書では「**追加削除が O(1) で速い**」とされるが、現代のハードウェアと JIT では、**ほぼ常に `ArrayList` のほうが速い**。
新規コードで使う理由はほぼない。

### `ConcurrentHashMap`

`java.util.concurrent` のスレッドセーフな Map。
内部でセグメント化された CAS を使い、ロックなしに近い読みを提供する。

### プリミティブ専用コレクション

`Map<Integer, V>` のような型のボクシングを避けるためのコレクション。
**Eclipse Collections・HPPC・fastutil** などのライブラリで提供。

### `Stream.toList()`

Java 16 で追加された、ストリームを**不変リスト**に集めるメソッド。
`Collectors.toList()` より推奨される。

### `parallelStream()` の注意

内部で `ForkJoinPool.commonPool` を使うため、**JVM 全体で共有**される。
短時間処理や I/O 待ちでは逆効果。仮想スレッド時代では `StructuredTaskScope` を検討。

### N+1 問題

`findAll()` で 1 回 + 各 entity の関連を取るのに N 回 SQL を発行してしまう問題。
**`@EntityGraph`・`JOIN FETCH`・DTO** で解決。

### `Slice` vs `Page`

Spring Data の戻り値型。
- **`Page`** ―― 件数を `count(*)` で取る。重い
- **`Slice`** ―― 「次のページがあるか」だけ。軽い。無限スクロール向き

### バッチ更新

`saveAll(...)` + `hibernate.jdbc.batch_size` の組み合わせで、**複数の INSERT を 1 つにまとめる**手法。

### HikariCP

Spring Boot 標準のコネクションプール。
プールサイズの**適正値は実測**で決める。

### Caffeine

Java で最も使われる**ローカルキャッシュ**ライブラリ。
W-TinyLFU アルゴリズムで、効率的な eviction を行う。

### Redis

**共有キャッシュ**の定番。複数インスタンスで一貫したキャッシュを共有できる。

### Cache-Aside パターン

アプリが読み書きの両方を直接管理するキャッシュパターン。
最も一般的。

### TTL（Time To Live）

「**書いてから N 分**で消える」キャッシュ寿命戦略。

### LRU（Least Recently Used）

「**最近使われていない**ものから消す」キャッシュ戦略。Caffeine の基本。

### キャッシュスタンピード

キャッシュ期限切れの瞬間、大量リクエストが**同時に DB を叩く**現象。
**再生成のロック**・**`refreshAfterWrite`**・**ジッター**で対策する。
