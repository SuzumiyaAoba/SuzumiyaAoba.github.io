## 用語集 ― この章で学んだ言葉

第40章では、業務システムを支えるロギングの基本を学びました。
節ごとに、言葉を整理しておきます。

---

## なぜログが必要か（第1節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| ロギング | logging | 動作の記録を残すこと |
| 障害調査 | ― | ログを使って原因を究明する活動 |
| 業務イベント記録 | ― | 注文・認証など業務的な出来事の記録 |
| 構造化ログ | structured log | 機械処理しやすい形式（JSON）のログ |

---

## SLF4J と Logback（第2節）

| 用語 | 意味 |
|---|---|
| SLF4J | Simple Logging Facade for Java。ロギングの共通インターフェース |
| Logback | SLF4J の標準実装。Spring Boot のデフォルト |
| Log4j 2 | Logback と並ぶ実装の選択肢 |
| `LoggerFactory.getLogger(クラス.class)` | クラス用ロガーの取得 |
| `{}` プレースホルダ | 遅延評価される引数埋め込み |
| `isDebugEnabled` | メッセージ組み立てが重い場合のガード節 |
| `@Slf4j` | Lombok の自動ロガー生成 |

---

## ログレベル（第3節）

| レベル | 用途 |
|---|---|
| TRACE | きめ細かい追跡。開発時のみ |
| DEBUG | 開発・QA フェーズの内部状態確認 |
| INFO | 正常時の業務イベント |
| WARN | 異常ではないが注意が必要 |
| ERROR | 異常。人間が対応すべき |

| 用語 | 意味 |
|---|---|
| `logging.level.{パッケージ}` | パッケージ単位のレベル設定 |
| Actuator | ログレベルを動的に変えるエンドポイント |

## MDC（第5節）

| 用語 | 意味 |
|---|---|
| MDC | Mapped Diagnostic Context。スレッドごとのログ用キー=値 |
| `%X{key}` | Logback パターンでの MDC 埋め込み |

| Micrometer Tracing | Spring Boot 3 の標準トレーシングライブラリ |

## つまずき（第7節）

| 用語 | 意味 |
|---|---|
| ログの二重出力 | 同じ例外で複数箇所からログを出してしまう |

## 「迷ったとき」の指針

これからプログラミングを続けるあなたが、「**何が正しいか分からない**」と感じたとき。
本書から、3 つのことを思い出してください。

### 1. 「読みやすさ」が、最終の判定基準です

すべての設計原則、パターン、ベストプラクティスは、結局のところ「**コードが読みやすいか、変更しやすいか**」のためです。
原則と読みやすさが衝突したら、**読みやすさを取りましょう**。

### 2. 「困ってから」変えるくらいで、ちょうどよい

「将来困るかも」と思って先回りすると、たいてい外します。
**いま困っていない複雑さは、入れないこと**。
シンプルなコードのほうが、ほぼ常に正解です。

### 3. 「テストできるか」を、設計の物差しにする

テストが書きにくいコードは、たいてい設計に問題があります。
「**このクラス、Mockito で書けるかな?**」と問う癖をつけると、自然と良い設計に近づきます。
