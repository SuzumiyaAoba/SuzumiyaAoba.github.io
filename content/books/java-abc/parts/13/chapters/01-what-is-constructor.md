## コンストラクタとは ― new の正体

`new Car()` と書くと、オブジェクトが作られます。
このとき、最後の `()` の部分で、**コンストラクタ**（Constructor）という特別なしくみが呼ばれています。

この節では、コンストラクタが何者で、なぜ必要なのかを学びます。

---

## コンストラクタは「初期化のための特別なメソッド」

**コンストラクタ**とは、

> オブジェクトが `new` で作られる**瞬間に、一度だけ自動で実行される**、初期化のための特別なメソッド

です。「constructor」は「構築するもの」という意味で、オブジェクトを組み立てる役割を持ちます。

コンストラクタの主な仕事は、**フィールドに、最初の値（初期値）を入れる**ことです。
これまでは `new` でオブジェクトを作ったあと、セッターで1つずつ値を入れていました。
コンストラクタを使うと、その初期化を、**オブジェクトが生まれるのと同時に**まとめて行えます。

---

## ふつうのメソッドとの違い

コンストラクタは「特別なメソッド」と言いましたが、ふつうのメソッドとは、次の点が違います。

| | ふつうのメソッド | コンストラクタ |
|---|---|---|
| 名前 | 自由に決める | **クラス名とまったく同じ**にする |
| 戻り値の型 | `void` や `int` などを書く | **書かない**（`void` も書かない） |
| 呼ばれるタイミング | `オブジェクト.メソッド()` で呼ぶ | `new` のときに**自動で**呼ばれる |
| 呼ばれる回数 | 何度でも | オブジェクトごとに**1回だけ** |

特に重要なのは、最初の2つです。

- **名前は、クラス名と同じ**にする（`Car` クラスなら、コンストラクタの名前も `Car`）
- **戻り値の型を書かない**（`void` すら書かない）

この2つが、「これはコンストラクタですよ」という目印になります。

---

## まず、いまの状態を確認する

コンストラクタを書く前に、第11章のクラスを思い出しましょう。

```java
class Person {
    String name;
    int age;
}
```

このクラスには、コンストラクタを**1つも書いていません**。
それでも `new Person()` と書けました。これはなぜでしょうか。

実は、コンストラクタを1つも書かないと、Java が**引数なしの空っぽのコンストラクタ**を、こっそり用意してくれているのです。
これを**デフォルトコンストラクタ**と呼びます（第5節でくわしく学びます）。

つまり、`new Person()` の `()` は、この「自動で用意された、空のコンストラクタ」を呼んでいたのです。
このコンストラクタは何もしないので、フィールドは初期値（`name` は `null`、`age` は `0`）のままでした。

---

## コンストラクタを書くと、何ができるのか

次の節から、このコンストラクタを**自分で書く**ことで、

- `new` と同時に、フィールドに値を入れる
- 「名前のない人」「年齢が `0` の人」のような、中途半端なオブジェクトができるのを防ぐ
- 第12章のカプセル化と組み合わせて、「**作った瞬間から正しい状態**」のオブジェクトを作る

といったことができるようになります。

イメージとしては、コンストラクタは「**オブジェクトの初期設定係**」です。
製品が工場で組み立てられるとき、最後に初期設定をしてから出荷されるように、オブジェクトも `new` のときにコンストラクタで初期設定されてから、使えるようになるのです。

---

## まとめ

- **コンストラクタ**は、`new` でオブジェクトが作られる瞬間に、**一度だけ自動実行**される特別なメソッド
- 主な仕事は、**フィールドに初期値を入れる**こと
- ふつうのメソッドと違い、(1) 名前は**クラス名と同じ**、(2) **戻り値の型を書かない**
- コンストラクタを1つも書かないと、Java が空の**デフォルトコンストラクタ**を用意する（だから今まで `new Person()` が書けた）
