## オーバーロードと this() ― 複数のコンストラクタ

オブジェクトの作り方は、1通りとは限りません。
「名前と年齢を指定して作る」「名前だけ指定して、年齢は仮の値で作る」のように、**複数の作り方**を用意したいことがあります。

この節では、コンストラクタを複数定義する**オーバーロード**と、コンストラクタどうしを呼び合う **`this()`** を学びます。

---

## コンストラクタもオーバーロードできる

第9章で、引数の組み合わせが違えば、同じ名前のメソッドを複数定義できる**オーバーロード**を学びました。
コンストラクタも同じです。**引数の組み合わせが違えば、複数のコンストラクタを定義できます**。

`Person` に、2つのコンストラクタを用意してみましょう。

```java
class Person {
    String name;
    int age;

    Person(String name, int age) {   // ① 名前と年齢を指定
        this.name = name;
        this.age = age;
    }

    Person(String name) {            // ② 名前だけ指定（年齢は 0 とする）
        this.name = name;
        this.age = 0;
    }
}
```

これで、2通りの作り方ができます。

```java
Person p1 = new Person("佐藤", 25);   // ① が呼ばれる
Person p2 = new Person("鈴木");        // ② が呼ばれる（年齢は 0）
```

`new` に渡した引数の組み合わせ（個数・型）に合わせて、Java が適切なコンストラクタを自動で選んでくれます。
これが、コンストラクタのオーバーロードです。

---

## 重複が気になる

ここで、上の2つのコンストラクタをよく見ると、**似た処理がくり返されている**ことに気づきます。

```java
Person(String name, int age) {
    this.name = name;   // ← ①と②で
    this.age = age;
}
Person(String name) {
    this.name = name;   // ← 同じような処理
    this.age = 0;
}
```

どちらも `this.name = name;` を書いています。
いまは短いので気になりませんが、初期化の処理が増えると、同じコードをあちこちにコピーすることになり、修正もれの原因になります。

そこで使うのが、`this()` です。

---

## this() ― 別のコンストラクタを呼ぶ

**`this(...)`** と書くと、**同じクラスの別のコンストラクタを呼び出せます**。
これを使って、「名前だけのコンストラクタ」から「名前と年齢のコンストラクタ」を呼び、処理をまとめましょう。

```java
class Person {
    String name;
    int age;

    Person(String name, int age) {   // 本体となるコンストラクタ
        this.name = name;
        this.age = age;
    }

    Person(String name) {
        this(name, 0);               // ↑ のコンストラクタを、age = 0 で呼ぶ
    }
}
```

`Person(String name)` の中の `this(name, 0)` は、「`Person(String name, int age)` を、`age` に `0` を渡して呼ぶ」という意味です。
これで、実際の初期化処理は `Person(String name, int age)` の1か所にまとまりました。
「名前だけ」のコンストラクタは、「年齢を `0` にして、本体のコンストラクタにおまかせする」だけになります。

> **注意: `this()` は、ほかの初期化より前に書く**
>
> `this(...)` で別のコンストラクタを呼ぶときは、**ほかの初期化処理よりも前**に書きます。
> 基本は、コンストラクタの**いちばん最初**に書くと覚えておけば、迷いません。
> 「まず別のコンストラクタで初期化を済ませてから、残りの処理をする」という順番です。
> （Java 25 では、インスタンスをまだ使わない準備の文であれば `this(...)` の前に書くこともできますが、初心者のうちは『先頭に書く』を基本にすれば十分です。）

---

## 動かして確かめる

```java
class Person {
    String name;
    int age;

    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    Person(String name) {
        this(name, 0);
    }

    void introduce() {
        IO.println(name + "さん（" + age + "歳）");
    }
}

public class Main {
    public static void main(String[] args) {
        Person p1 = new Person("佐藤", 25);
        Person p2 = new Person("鈴木");

        p1.introduce();
        p2.introduce();
    }
}
```

```text
$ java Main.java
佐藤さん（25歳）
鈴木さん（0歳）
```

`p1` は `Person("佐藤", 25)` で、`p2` は `Person("鈴木")` → `this("鈴木", 0)` 経由で作られました。
そのため `p2` の年齢は `0` になっています。
2つのコンストラクタが、`this()` を通してきちんと連携しているのがわかります。

---

## this と this() の違い

ここで、よく似た2つを整理しておきましょう。

| 書き方 | 意味 |
|---|---|
| `this.name` | このオブジェクトの**フィールド** `name` を指す |
| `this(...)` | 同じクラスの**別のコンストラクタ**を呼ぶ |

`this.` はフィールドへのアクセス、`this(...)` はコンストラクタの呼び出しです。
形は似ていますが、役割はまったく違うので、混同しないようにしましょう。

---

## まとめ

- コンストラクタも、引数の組み合わせが違えば**オーバーロード**（複数定義）できる
- `new` の引数に合わせて、Java が適切なコンストラクタを選ぶ
- **`this(...)`** で、同じクラスの**別のコンストラクタ**を呼べる（初期化処理の重複を防ぐ）
- `this(...)` は、コンストラクタの**いちばん最初**に書く
- `this.フィールド`（フィールドへのアクセス）と `this(...)`（コンストラクタ呼び出し）は別物
