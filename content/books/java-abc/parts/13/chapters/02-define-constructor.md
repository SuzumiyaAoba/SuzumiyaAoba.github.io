## コンストラクタを定義する

それでは、実際にコンストラクタを書いてみましょう。
まずは、いちばん基本となる「引数なしのコンストラクタ」から始めます。

---

## 引数なしのコンストラクタ

コンストラクタは、**クラス名と同じ名前**で、**戻り値の型を書かず**に定義します。

```java
class Person {
    String name;
    int age;

    Person() {              // コンストラクタ（名前は Person、戻り値の型なし）
        name = "名無し";
        age = 0;
        IO.println("Person が作られました");
    }
}
```

`Person()` が、コンストラクタです。
クラス名 `Person` と同じ名前で、`void` などの戻り値の型を書いていない点に注目してください。
この中で、フィールドに初期値（`"名無し"` と `0`）を入れています。

---

## コンストラクタが呼ばれるタイミングを見る

このコンストラクタは、`new` のときに**自動で**呼ばれます。
本当にそうなるか、`IO.println` を入れて確かめてみましょう。

```java
class Person {
    String name;
    int age;

    Person() {
        name = "名無し";
        age = 0;
        IO.println("Person が作られました");
    }
}

public class Main {
    public static void main(String[] args) {
        IO.println("--- new する前 ---");
        Person p = new Person();         // ここでコンストラクタが呼ばれる
        IO.println("--- new した後 ---");
        IO.println("名前: " + p.name + " / 年齢: " + p.age);
    }
}
```

```text
$ java Main.java
--- new する前 ---
Person が作られました
--- new した後 ---
名前: 名無し / 年齢: 0
```

`new Person()` を実行したまさにそのタイミングで、「Person が作られました」が表示されています。
コンストラクタが、`new` のときに自動で呼ばれていることが、はっきり分かりますね。
そして、フィールドにはコンストラクタで入れた初期値（`"名無し"` と `0`）が入っています。

---

## 引数なしコンストラクタの限界

ただ、この引数なしコンストラクタには、限界があります。
**作られる `Person` が、いつも「名無し・0歳」になってしまう**のです。

これでは、「佐藤さん・25歳」のような、具体的な人を作れません。
作ったあとに、結局セッターで値を入れ直すことになり、コンストラクタを書いた意味が薄れてしまいます。

本当に便利なのは、「**作るときに、名前や年齢を渡せる**」コンストラクタです。
それを実現するのが、次の節で学ぶ**引数つきコンストラクタ**です。

> **補足: コンストラクタも「ブロック」が実行される**
>
> コンストラクタの `{ }` の中には、フィールドの初期化だけでなく、ふつうのメソッドと同じように、いろいろな処理を書けます。
> 上の例では `IO.println` で確認用のメッセージを出しました。
> ただし、コンストラクタの本来の役割は「初期化」です。複雑な処理を詰め込みすぎないのが、よい設計のコツです。

---

## まとめ

- コンストラクタは、**クラス名と同じ名前**・**戻り値の型なし**で定義する
- コンストラクタの `{ }` の中で、フィールドに初期値を入れる
- コンストラクタは、`new` のときに**自動で1回**呼ばれる
- 引数なしのコンストラクタは、いつも同じ初期値になってしまうのが限界
