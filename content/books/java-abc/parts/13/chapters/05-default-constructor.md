## デフォルトコンストラクタ ― 暗黙のコンストラクタ

第1節で、「コンストラクタを1つも書かないと、Java が空のコンストラクタを用意してくれる」と触れました。
この、自動で用意されるコンストラクタを、**デフォルトコンストラクタ**（Default Constructor）と呼びます。
この節では、その正体と、知らないとつまずくポイントを学びます。

---

## コンストラクタを書かないと、自動で用意される

次の `Person` には、コンストラクタが1つも書かれていません。

```java
class Person {
    String name;
    int age;
}
```

それでも、`new Person()` と書けました。
これは、Java が、次のような**引数なしで、中身が空のコンストラクタ**を、こっそり用意してくれているからです。

```java
// Java が自動で用意する（見えないが、存在する）
Person() {
}
```

これが**デフォルトコンストラクタ**です。
中身が空なので、フィールドは初期値（`name` は `null`、`age` は `0`）のままになります。
第11章・第12章で、コンストラクタを書かずに `new Person()` が使えていたのは、このおかげでした。

---

## コンストラクタを1つでも書くと、消える

ここが、最も重要で、つまずきやすいポイントです。

> **自分でコンストラクタを1つでも書くと、デフォルトコンストラクタは、もう自動では用意されません。**

たとえば、引数つきコンストラクタだけを書いたとします。

```java
class Person {
    String name;
    int age;

    Person(String name, int age) {   // 引数つきコンストラクタを1つ書いた
        this.name = name;
        this.age = age;
    }
}
```

この状態で、引数なしの `new Person()` を呼ぶと、どうなるでしょうか。

```java
Person p = new Person();   // 引数なしで作ろうとする
```

```text line-numbers=false
エラー: クラス Personのコンストラクタ Personは指定された型に適用できません。
  期待値: String,int
  検出値:    引数がありません
```

エラーになります。
引数つきコンストラクタを書いたことで、デフォルトコンストラクタ（引数なし）は**もう用意されていない**のです。
そのため、引数なしの `new Person()` は「そんなコンストラクタはない」とはじかれます。

これは、第3節で「引数つきコンストラクタだけにすると、中途半端なオブジェクトを防げる」と説明した、まさにそのしくみです。
便利な性質ですが、「コンストラクタを書いたら、引数なしの `new` ができなくなった」と戸惑う人が多いポイントでもあります。

---

## 引数なしでも作りたいなら、自分で書く

「引数つきも、引数なしも、どちらでも作れるようにしたい」場合は、**引数なしのコンストラクタも、自分で書けば**よいだけです。
第4節で学んだ、コンストラクタのオーバーロードです。

```java
class Person {
    String name;
    int age;

    Person() {                       // 引数なしも、自分で用意する
        this("名無し", 0);
    }

    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

これで、両方の作り方ができます。

```java
Person p1 = new Person();             // OK（名無し・0歳）
Person p2 = new Person("佐藤", 25);   // OK
```

「デフォルトコンストラクタが消えたなら、必要なものは自分で書く」 ―― これが対処法です。

---

## まとめ

- コンストラクタを**1つも書かない**と、Java が引数なし・空の**デフォルトコンストラクタ**を自動で用意する
- だから、コンストラクタなしのクラスでも `new Person()` が使えた
- **自分でコンストラクタを1つでも書くと、デフォルトコンストラクタは自動では用意されなくなる**
- そのため、引数つきだけを書くと、引数なしの `new Person()` はエラーになる
- 引数なしでも作りたいなら、**引数なしのコンストラクタを自分で書く**（オーバーロード）
