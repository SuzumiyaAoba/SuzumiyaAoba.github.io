## final と不変オブジェクト

第12章で予告した、**不変オブジェクト**（Immutable Object）を学びます。
これは、コンストラクタを学んだ今だからこそ作れる、より安全なオブジェクトの設計です。

---

## 不変オブジェクトとは

**不変オブジェクト**とは、

> **一度作ったら、中身（フィールドの値）が、二度と変わらない**オブジェクト

のことです。
第10章で学んだ文字列（`String`）が、まさに不変オブジェクトでした。`toUpperCase()` などが元の文字列を変えず、新しい文字列を返したのを思い出してください。

これまで作ってきた `Person` は、セッターで `age` を変えられました。これは「**可変**（ミュータブル、Mutable）」なオブジェクトです。
一方、不変オブジェクトは、作ったあと、誰も中身を変えられません。

---

## final フィールド ― 一度だけ代入できる

不変オブジェクトを作るには、フィールドに **`final`** を付けます。
第4章で、`final` は「書き換えられない変数（定数）」を作る印だと学びました。フィールドに付けると、「**一度値を入れたら、二度と変えられないフィールド**」になります。

`final` フィールドは、**コンストラクタの中で1回だけ**値を入れられます。
（フィールド宣言と同時に初期化してもかまいません。）

```java
class Point {
    final int x;   // 一度だけ代入できる
    final int y;

    Point(int x, int y) {
        this.x = x;   // コンストラクタで1回だけ代入
        this.y = y;
    }
}
```

コンストラクタで `x`・`y` に値を入れたら、もう変更できません。
これにより、`Point` は「作ったときの座標が、ずっと変わらない」オブジェクトになります。

---

## 変更しようとすると、エラーになる

`final` フィールドを、あとから変えようとすると、コンパイルエラーになります。

```java
class Point {
    final int x;
    final int y;
    Point(int x, int y) { this.x = x; this.y = y; }
}

public class Main {
    public static void main(String[] args) {
        Point p = new Point(3, 4);
        p.x = 10;     // final なので変更できない
    }
}
```

```text
エラー: final変数xに値を割り当てることはできません
        p.x = 10;
         ^
```

「`final` な `x` には、値を割り当てられません」と、Java が止めてくれます。
第4章で見た、`final` 変数の再代入エラーと同じです。
これにより、「作ったあとに、うっかり（あるいは勝手に）中身を変えられる」ことを、根本から防げます。

---

## 不変オブジェクトの作り方

不変オブジェクトにするには、次の3つを守ります。

1. **すべてのフィールドを `final` にする**（作ったあと変わらない）
2. **値はコンストラクタで設定する**（`final` なので、ここでしか入れられない）
3. **セッターを作らない**（値を変える手段を提供しない）

`Point` は、この3つを満たしているので、不変オブジェクトです。
値を「読む」ゲッターは付けてもかまいません（読むだけなら、不変性は保たれます）。

```java
class Point {
    private final int x;
    private final int y;

    Point(int x, int y) {
        this.x = x;
        this.y = y;
    }

    public int getX() { return x; }   // 読むだけ（セッターはない）
    public int getY() { return y; }
}
```

---

## 「変える」のではなく「新しく作る」

不変オブジェクトでは、値を変えたいとき、**元を変えるのではなく、変えた結果を持つ新しいオブジェクトを作って返します**。
これも、`String` が `toUpperCase()` で新しい文字列を返したのと、同じ考え方です。

たとえば「x 方向に移動した点」がほしいなら、こう書きます。

```java
class Point {
    private final int x;
    private final int y;

    Point(int x, int y) { this.x = x; this.y = y; }

    // 移動した「新しい Point」を返す（自分自身は変えない）
    Point movedX(int delta) {
        return new Point(x + delta, y);
    }

    public int getX() { return x; }
    public int getY() { return y; }
}

public class Main {
    public static void main(String[] args) {
        Point p1 = new Point(3, 4);
        Point p2 = p1.movedX(5);      // 移動した新しい点

        IO.println("p1: (" + p1.getX() + ", " + p1.getY() + ")");
        IO.println("p2: (" + p2.getX() + ", " + p2.getY() + ")");
    }
}
```

```text
$ java Main.java
p1: (3, 4)
p2: (8, 4)
```

`movedX` は `p1` を変えず、移動後の値を持つ**新しい `Point`**（`p2`）を返しました。
`p1` は `(3, 4)` のまま、`p2` が `(8, 4)` です。元のオブジェクトは、けっして変わりません。

---

## なぜ不変オブジェクトがよいのか

「変えられないと不便では？」と思うかもしれませんが、不変オブジェクトには大きな利点があります。

- **安全に共有できる** … 誰かに渡しても、勝手に書き換えられる心配がない。第11章で見た「同じオブジェクトを複数の変数が指す」場合でも、中身が変わらないので安心
- **状態を追いやすい** … 値が途中で変わらないので、「いつの間にかおかしな値になっていた」というバグが起きにくい
- **予測しやすい** … 一度作ったら同じ、と分かっているので、コードが理解しやすい

そのため、「**変える必要がないデータは、不変オブジェクトにする**」のが、よい設計とされています。
お金（金額）、座標、日付など、「値そのもの」を表すものは、不変にするのが特に向いています。

> **補足: 不変オブジェクトを、もっと簡単に ― レコード**
>
> 「すべて `final`、コンストラクタで設定、ゲッターを用意」という不変オブジェクトの定番の形は、書くのが少し手間です。
> 実は Java には、この定番の形を**ごく短く書ける**「**レコード**（record）」という機能があります。
> レコードについては、第17章でくわしく学びます。ここでは「不変オブジェクトには、便利な書き方がある」と知っておきましょう。

---

## まとめ

- **不変オブジェクト**は、一度作ったら中身が変わらないオブジェクト（`String` が代表例）
- フィールドに **`final`** を付けると、コンストラクタで一度だけ値を入れられ、あとは変更できない
- 不変オブジェクトの作り方: ①全フィールド `final` ②値はコンストラクタで設定 ③セッターを作らない
- 値を変えたいときは、**変えた結果を持つ新しいオブジェクトを作って返す**
- 利点: 安全に共有でき、バグが起きにくく、予測しやすい
- 「変える必要のないデータ」は、不変オブジェクトにするのがよい（簡単に書ける**レコード**は第17章）
