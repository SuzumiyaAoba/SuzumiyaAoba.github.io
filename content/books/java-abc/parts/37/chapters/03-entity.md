## エンティティ

**エンティティ**（Entity、実体）は、**テーブルにマッピングされる Java クラス**です。
JPA を使ううえでの主役のひとつです。

---

## 最小のエンティティ

第2節で動かした `Book` クラスを、もう一度載せます。

```java
package com.example.library;

import jakarta.persistence.*;

@Entity
@Table(name = "books")
public class Book {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String title;

    @Column(nullable = false)
    private String author;

    @Column(name = "published_year")
    private int publishedYear;

    protected Book() {}                              // JPA のためのデフォルトコンストラクタ

    public Book(Long id, String title, String author, int publishedYear) {
        this.id = id;
        this.title = title;
        this.author = author;
        this.publishedYear = publishedYear;
    }

    public Long getId() { return id; }
    public String getTitle() { return title; }
    public String getAuthor() { return author; }
    public int getPublishedYear() { return publishedYear; }

    @Override
    public String toString() {
        return "Book[id=" + id + ", title=" + title
            + ", author=" + author + ", year=" + publishedYear + "]";
    }
}
```

これが、Hibernate に渡される**テーブルの定義書**でもあります。
要素を順番に見ていきましょう。

---

## `@Entity` ― 「テーブルにマッピングするクラス」の宣言

```java
@Entity
public class Book { ... }
```

`@Entity` が付いたクラスは、Hibernate によって**永続化の対象**として扱われます。
このアノテーションがないと、ただの POJO で、テーブルとつなぐ対象になりません。

---

## `@Table` ― テーブル名を指定

```java
@Table(name = "books")
public class Book { ... }
```

`@Table(name = "...")` で、テーブル名を明示します。
省略すると、**クラス名がそのままテーブル名**になります（`Book` → `book`、Hibernate のデフォルトでは小文字化）。

ただし、

- 慣習として、テーブル名は**複数形**（`books`、`users`）で書く流派が多い
- クラス名（単数）とのズレを明示するため、`@Table(name = "books")` を書くのが一般的

---

## `@Id` と `@GeneratedValue` ― 主キー

```java
@Id
@GeneratedValue(strategy = GenerationType.IDENTITY)
private Long id;
```

`@Id` で、「**このフィールドが主キー**」と指定します。
すべてのエンティティに、**1 つの `@Id` フィールドが必須**です。

`@GeneratedValue` は、「主キーの値を**どう生成するか**」を指示します。

| strategy | 動き |
|---|---|
| `IDENTITY` | DB の AUTO_INCREMENT 機能を使う（MySQL・H2 など） |
| `SEQUENCE` | DB のシーケンス機能を使う（PostgreSQL・Oracle 推奨） |
| `AUTO` | DB に合わせて、Hibernate が選ぶ |
| `TABLE` | 専用テーブルで採番する（互換性のための古い方式） |

入門段階では、`IDENTITY` を覚えておけば十分です。
PostgreSQL では `SEQUENCE` の方が高速とも言われていますが、最初はデフォルトで動かしてからチューニングを考えます。

---

## `@Column` ― カラムごとの設定

```java
@Column(nullable = false)
private String title;

@Column(name = "published_year")
private int publishedYear;
```

`@Column` で、カラム単位の細かい設定ができます。

| 属性 | 意味 |
|---|---|
| `name` | カラム名（省略時はフィールド名をスネークケースに） |
| `nullable` | `NOT NULL` 制約（既定: `true`） |
| `length` | `VARCHAR` の長さ（既定: 255） |
| `unique` | `UNIQUE` 制約 |

`@Column` を**書かなくても**、Hibernate がフィールド名から推測してくれます。
ですが、

- **明示的に意図を示す**ため
- **`NOT NULL` などの制約を入れる**ため

書いておくのが推奨です。

`publishedYear` を `published_year` カラムに対応づけているのは、命名規則の慣習を意識した例です。
Hibernate 6 系の標準では、フィールド名 `publishedYear` を自動で `published_year` にスネークケース変換します（**ImplicitNamingStrategyJpaCompliantImpl** などにより）。
それでも `name` を**明示**しておくほうが、安心して読めます。

## デフォルトコンストラクタが必要

```java
protected Book() {}
```

JPA は、データベースから取得した行を Java オブジェクトに変換するときに、

1. **デフォルトコンストラクタ**で空のインスタンスを作る
2. リフレクションで、フィールドに値を**直接代入**する

という流れで動きます。
そのため、**引数なしのコンストラクタが必須**です。
他人に呼ばれたくないので、`protected` か `public` のどちらかにします。
JPA のリフレクションは、`private` でも動くケースもありますが、推奨は `protected` です。

---

## record をエンティティにできないのか?

第17章で学んだ **record** は、`final` で**フィールドが書き換えられない**設計です。
そして、Hibernate はオブジェクトを取得したあとに、フィールドを書き換えながら状態を追跡します（「永続コンテキスト」と呼ばれる仕組み）。

このため、**record はエンティティに使えません**。
record は、エンティティではなく、**DTO**（Data Transfer Object、外部とやり取りするための値オブジェクト）に使うのが定番です。

> **補足: 「エンティティはミュータブルでよいのか?」**
>
> 「最近のオブジェクト指向では、不変オブジェクトを推奨するのでは?」と思うかもしれません。
> JPA のエンティティは、`getter` だけでなく、`setter` も持つ「**ふつうの可変オブジェクト**」です。
> これは、JPA の仕組みが**フィールドの書き換えを追跡**することで成り立っているためです。
> 一方、**外部に渡す値や、リクエスト/レスポンスの DTO** は record で書くのが現代風です。

## いま、何のテーブルができたか

第2節の起動時のログを思い出してください。
`ddl-auto: create-drop` を指定していたので、Hibernate がエンティティを見て**テーブルを自動作成**しています。
H2 の場合、こんな CREATE 文が生成されます（ログから抜粋）。

```sql
create table books (
    id bigint generated by default as identity,
    author varchar(255) not null,
    published_year integer not null,
    title varchar(255) not null,
    primary key (id)
)
```

エンティティの **`@Id`・`@Column(nullable=false)`** などが、そのまま **CREATE 文**になっていることが分かります。
これが「**エンティティはテーブルの定義書でもある**」ということです。

---

## 主キーを `Long` 型にする理由

主キーの `id` フィールドは、ふつう **`Long` 型**にします。
`long`（プリミティブ）ではなく、**`Long`（ラッパー型）**です。
なぜか?

- 新規エンティティはまだ DB に保存されていない → `id` は **`null`**
- 保存されると、DB が採番した `id` がセットされる

`null` を扱う必要があるため、**`Long`** が選ばれます。
`int`/`long` のプリミティブだと、`null` を表せず、`0` が「未保存」を意味することになって、わかりにくくなります。

---

## まとめ

- **`@Entity`** が、Java クラスをテーブルにマッピングする
- **`@Table(name=...)`** で、テーブル名を明示
- **`@Id` + `@GeneratedValue`** で、主キーと採番方式を指定
- **`@Column`** で、`NOT NULL` などの制約を表現
- **デフォルトコンストラクタ（`protected`）**が JPA に必要
- record はエンティティにできない（**DTO に使う**）
- 主キーは **`Long` 型**にして、未保存を `null` で表現する
