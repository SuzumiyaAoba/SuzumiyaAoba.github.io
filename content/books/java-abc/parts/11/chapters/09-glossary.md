## 用語集 ― この章で学んだ言葉

第2部の最初の章である第11章では、オブジェクト指向の中心となる言葉がたくさん登場しました。
ここで、節ごとに整理しておきます。これらは第12章以降でも、ずっと使い続ける基本語です。

---

## クラスとオブジェクト（第1〜2節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| オブジェクト指向 | Object-Oriented | データと操作をまとめた「部品（オブジェクト）」でプログラムを組み立てる考え方 |
| クラス | Class | オブジェクトを作るための**設計図**（たい焼きの型にあたる） |
| オブジェクト | Object | クラスから作られた**実体**（焼いたたい焼きにあたる） |
| インスタンス | Instance | 「あるクラスから作られた実体」を強調する言い方。オブジェクトとほぼ同義 |
| フィールド | Field | クラスの直下に書く変数。オブジェクトが持つ**データ（状態）** |
| メソッド | Method | クラスの中に書く処理。オブジェクトの**ふるまい（操作）** |

---

## オブジェクトの生成と利用（第3〜4節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| `new` | ― | オブジェクトを新しく生成するキーワード（`new Car()`） |
| `.`（ドット） | ― | オブジェクトのフィールドやメソッドにアクセスする記号（`オブジェクト.名前`） |
| インスタンス化 | Instantiation | クラスからオブジェクト（インスタンス）を作ること |
| 参照型 | Reference Type | 値そのものではなく、オブジェクトの「場所」を指す型。クラスから作る型はすべてこれ |

---

## this と static（第5〜7節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| `this` | ジス | いま動いているメソッドが属する**オブジェクト自身**を指す |
| インスタンス変数 | Instance Variable | オブジェクトごとに別々に存在するフィールド |
| インスタンスメソッド | Instance Method | オブジェクトを通して呼ぶメソッド（`static` でないメソッド） |
| `static` | スタティック | クラスに属する印。オブジェクトごとではなく**クラス全体で1つ**だけ存在する |
| クラス変数 | Class Variable | `static` を付けたフィールド。全インスタンスで共有される |
| `static final` | ― | クラスに1つだけある、書き換えられない値（定数）の書き方 |

---

## つまずきの言葉（第8節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| `null` | ヌル | 「何も指していない」状態。参照型の初期値 |
| `NullPointerException` | ― | `null` に対してフィールド・メソッドを使おうとしたときに起きる例外 |

---

## おわりに

この章では、オブジェクト指向の土台となる「クラス」と「オブジェクト」を学びました。

- **クラス**は設計図、**オブジェクト**（インスタンス）はその実体
- クラスには、**フィールド**（データ）と**メソッド**（ふるまい）をまとめて書く
- オブジェクトは `new` で作り、`.` で操作する
- `this` は自分自身、`static` はクラス全体で1つ

ここまでは、フィールドを `c.name = "プリウス";` のように、外から自由に読み書きしていました。
しかし、それでは「ありえない値（マイナスの速度など）」を、誰でも入れられてしまいます。

次の第12章「カプセル化とアクセス修飾子」では、フィールドを**外から勝手にいじれないように守り**、安全なオブジェクトを作る方法を学びます。
オブジェクト指向を、より「らしく」使うための、大切な一歩です。
