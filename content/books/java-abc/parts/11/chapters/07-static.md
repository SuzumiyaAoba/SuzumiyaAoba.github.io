## static ― クラスに属するメンバ

これまでのフィールドやメソッドは、すべて**オブジェクト（インスタンス）に属する**ものでした。
`car1.speed` のように、特定のオブジェクトを通して使いましたね。

しかし、なかには「**オブジェクトごと**ではなく、**クラス全体で1つ**あればよい」ものもあります。
それを表すのが、第3章でも登場した **`static`**（スタティック）です。

---

## インスタンスに属する vs クラスに属する

`static` を理解するには、2つの「属し方」を区別するのが近道です。

| | インスタンスに属する | クラスに属する（`static`） |
|---|---|---|
| 付ける印 | なし（これまでどおり） | **`static`** を付ける |
| 何個ある？ | オブジェクトごとに1つ | クラス全体で**1つだけ**（共有） |
| 使い方 | `オブジェクト.名前` | **`クラス名.名前`** |
| 例 | `car1.speed` | `Car.count` |

- **インスタンスメンバ** … オブジェクトごとに別々（前の節の `speed` など）
- **`static` メンバ**（クラスメンバ） … クラスに1つだけ。すべてのオブジェクトで共有される

---

## static フィールド ― みんなで共有する値

例として、「いままでに作られた車の総数」を数えてみましょう。
これは「車1台ごと」ではなく「`Car` クラス全体で1つ」あればよい値です。だから `static` にします。

```java
class Car {
    static int count = 0;   // クラスに1つだけ（全オブジェクトで共有）
    String name;            // オブジェクトごと

    void register() {
        count++;            // 共有のカウンターを増やす
    }
}

void main() {
    Car a = new Car();
    a.register();
    Car b = new Car();
    b.register();
    Car c = new Car();
    c.register();

    IO.println("作られた車: " + Car.count + " 台");
}
```

```text
$ java CarApp.java
作られた車: 3 台
```

`count` は `static` なので、`a`・`b`・`c` のどれから `count++` しても、**同じ1つの `count`** が増えます。
最後に `Car.count` で取り出すと、3 になっています。

ポイントは、`static` フィールドには **`クラス名.フィールド名`**（ここでは `Car.count`）でアクセスすることです。
特定のオブジェクトのものではなく、クラスのものだからです。

---

## static メソッド ― オブジェクトなしで呼べる処理

メソッドにも `static` を付けられます。
**`static` メソッド**は、オブジェクトを作らなくても、`クラス名.メソッド名(...)` で呼び出せます。

「オブジェクトの状態（インスタンス変数）を使わない、独立した処理」に向いています。

```java
class MathUtil {
    static int square(int n) {   // n を2乗して返す
        return n * n;
    }
}

void main() {
    int r = MathUtil.square(5);   // オブジェクトを作らずに呼べる
    IO.println(r);
}
```

```text
$ java CarApp.java
25
```

`square` は、特定のオブジェクトの状態に依存せず、引数だけで計算が完結します。
こういう処理は、わざわざ `new MathUtil()` でオブジェクトを作る必要がないので、`static` にすると便利です。

---

## 実は、もう何度も使っていた

`static` は、はじめて学ぶ言葉に見えて、実は第1部から何度も使ってきました。

- **`public static void main(...)`** … `main` が `static` なのは、プログラム開始時、まだオブジェクトが1つもない段階で呼ばれるからです（第3章）
- **`Integer.parseInt("5")`** … `Integer` クラスの `static` メソッド。オブジェクトを作らずに呼べました（第4章）
- **`Arrays.sort(data)`** や **`Arrays.toString(data)`** … `Arrays` クラスの `static` メソッド（第8章）
- **`Math.max(a, b)`** や **`Math.random()`** … `Math` クラスの `static` メソッド

これらが、いつも `オブジェクト.` ではなく **`クラス名.`** の形で呼べていたのは、すべて `static` だったからなのです。
「クラス名にドットを付けて呼ぶものは、`static`」と覚えておくと、見分けがつきます。

> **補足: 定数は `static final` で書く**
>
> 第4章で学んだ定数（`final`）は、クラスのフィールドにするとき、ふつう `static final` の組み合わせで書きます。
>
> ```java
> static final int MAX_SPEED = 180;
> ```
>
> 「クラスに1つだけ（`static`）あって、書き換えられない（`final`）値」という意味です。
> `Integer.MAX_VALUE` も、この形で定義された定数です。定数名はすべて大文字にする慣習も、第4章のとおりです。

---

## static の使いすぎに注意

`static` は便利ですが、**何でも `static` にするのは禁物**です。

`static` は「クラス全体で共有」なので、オブジェクトごとに違う状態（`speed` など）を `static` にしてしまうと、すべての車が同じ速度を共有してしまい、めちゃくちゃになります。

- **オブジェクトごとに違う状態** → インスタンスメンバ（`static` を付けない）
- **クラス全体で共通の値・処理** → `static`

迷ったら、「これは1台ごとに必要か、それとも全体で1つでよいか」を考えましょう。
オブジェクト指向の主役は、あくまで**インスタンス**です。`static` は補助的に使う、と心得ておくとよいでしょう。

---

## まとめ

- **`static`** を付けたメンバは、オブジェクトごとではなく、**クラス全体で1つ**だけ存在する（共有される）
- `static` メンバは **`クラス名.名前`** で使う（`Car.count`、`MathUtil.square(5)`）
- `static` メソッドは、オブジェクトを作らずに呼べる（状態に依存しない処理に向く）
- `main`・`Integer.parseInt`・`Arrays.sort`・`Math.max` など、これまで使ってきた「クラス名.〜」は、すべて `static`
- 定数はふつう `static final` で書く
- オブジェクトごとに違う状態は `static` にしない。`static` は補助的に使う
