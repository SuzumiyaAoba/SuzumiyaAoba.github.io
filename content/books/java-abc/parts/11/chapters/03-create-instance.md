## オブジェクトを生成する ― new とインスタンス

クラス（設計図）を定義しただけでは、まだ何も動きません。
たい焼きの型を用意しただけで、まだ1個も焼いていない状態です。

この節では、定義した `Car` クラスから、実際に**オブジェクトを作って**使ってみます。

---

## new でオブジェクトを作る

オブジェクトを作るには、**`new`** というキーワードを使います。

```java
Car myCar = new Car();
```

この1行を分解してみましょう。

```text
Car      myCar    =    new Car();
 │        │       │        │
 │        │       │        └─ Car クラスのオブジェクトを新しく作る
 │        │       └─ 作ったオブジェクトを、左の変数に入れる
 │        └─ 変数の名前
 └─ 変数の型（Car 型）
```

- `new Car()` で、`Car` クラスから**新しいオブジェクトを1つ作り**ます
- 作ったオブジェクトを、`Car` 型の変数 `myCar` に入れます

これで、`myCar` を通じて、作ったオブジェクトを操作できるようになりました。
第8章の配列で `new int[5]` と書いたのを覚えているでしょうか。あの `new` と同じ仲間です。`new` は「新しく実体を作る」合図なのです。

> **補足: うしろの `()` は何？**
>
> `new Car()` の最後の `()` は、**コンストラクタ**（Constructor）という、オブジェクトを作るときに呼ばれる特別なしくみを呼び出しています。
> いまは「`new クラス名()` でオブジェクトを作る、決まった形」と覚えておけば十分です。コンストラクタは、第13章でくわしく学びます。

---

## フィールドにアクセスする ― ドット `.`

作ったオブジェクトのフィールドには、**`.`（ドット）** を使ってアクセスします。
`オブジェクト.フィールド名` と書くと、そのオブジェクトのフィールドを読み書きできます。

```java
myCar.name = "プリウス";   // name フィールドに値を入れる
myCar.speed = 60;          // speed フィールドに値を入れる
```

読み取るときも、同じく `.` を使います。

```java
IO.println(myCar.name);    // → プリウス
IO.println(myCar.speed);   // → 60
```

この `.` は、第3章で `System.out.println` の `.` を「〜の」と読んだのと同じです。
`myCar.name` は「`myCar` の `name`」と読めます。

---

## 完全なプログラムで動かす

ここまでを、1つの完全なプログラムにまとめてみましょう。
`Car` クラスと、それを使う `main` を、同じファイルに書きます。

```java
class Car {
    String name;
    int speed;
}

void main() {
    Car myCar = new Car();   // オブジェクトを作る
    myCar.name = "プリウス";  // フィールドに値を入れる
    myCar.speed = 60;

    IO.println(myCar.name + " / " + myCar.speed + " km/h");
}
```

これを `CarApp.java` という名前で保存し、実行します。

```text
$ java CarApp.java
プリウス / 60 km/h
```

`new` で作ったオブジェクトに名前と速度を入れ、それを取り出して表示できました。

> **jshell でも試せる**
>
> jshell では、クラス定義も `new` も、そのまま打ち込めます。
> `class Car { String name; int speed; }` と定義してから、`Car c = new Car();`、`c.name = "プリウス";` のように1行ずつ試すと、オブジェクトが育っていくようすを対話的に確認できます。

## 初期値を確かめる

前の節で、「`new` で作ると、フィールドには自動的に初期値が入る」と学びました。
本当にそうなるか、値を入れる前に表示して確かめてみましょう。

```java
class Car {
    String name;
    int speed;
}

void main() {
    Car c = new Car();
    IO.println("name : " + c.name);    // まだ入れていない
    IO.println("speed: " + c.speed);   // まだ入れていない
}
```

```text
$ java CarApp.java
name : null
speed: 0
```

`speed` は `0`、`name` は `null` になっています。
`int` のフィールドは `0`、`String` のフィールドは `null` で初期化される、というルールどおりですね。

---

## 変数が持つのは「オブジェクトそのもの」ではない

ここで、少し進んだ大切な話をします。

`Car myCar = new Car();` と書いたとき、変数 `myCar` が持っているのは、実は**オブジェクトそのもの**ではありません。
持っているのは、「**そのオブジェクトが、メモリ上のどこにあるか**」を指し示す情報です。

第9章「引数の渡され方」で、配列が**参照型**で「場所を指し示す情報」を持つ、と学びました。
クラスから作るオブジェクトも、同じ**参照型**です。
変数は、オブジェクトの「住所」を持っていて、その住所をたどってオブジェクトを操作している ―― そうイメージしてください。

この性質は、次の第4節以降や、第15章「ポリモーフィズム」で、何度も効いてきます。
いまは「**オブジェクトを入れた変数は、その場所を指している**」とだけ、頭に入れておきましょう。

---

## まとめ

- オブジェクトは **`new クラス名()`** で作る（生成する）
- 作ったオブジェクトは、`Car` 型などの変数に入れて操作する
- フィールドへのアクセスは **`オブジェクト.フィールド名`**（`.` は「〜の」と読む）
- `new` で作った直後、フィールドには**初期値**が入っている（`int` は `0`、`String` は `null`）
- オブジェクトを入れた変数は、**オブジェクトの場所を指し示す**（参照型）
