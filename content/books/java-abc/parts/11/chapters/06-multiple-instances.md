## 複数のインスタンス ― それぞれが独立した状態を持つ

クラスの大きな利点は、**1つの設計図から、いくつでもオブジェクトを作れる**ことでした。
この節では、`Car` を2台作り、「それぞれが自分だけの状態を持つ」ことを、実際に確かめます。

---

## 2台の車を作る

`new Car()` を2回呼べば、車は2台できます。
それぞれを別の変数に入れて、別々に操作してみましょう。

```java
class Car {
    String name;
    int speed;

    void accelerate(int delta) {
        this.speed += delta;
    }

    void show() {
        IO.println(name + " の速度: " + speed + " km/h");
    }
}

void main() {
    Car car1 = new Car();
    car1.name = "プリウス";

    Car car2 = new Car();
    car2.name = "ロードスター";

    car1.accelerate(50);    // car1 だけ加速
    car2.accelerate(100);   // car2 だけ加速

    car1.show();
    car2.show();
}
```

```text
$ java CarApp.java
プリウス の速度: 50 km/h
ロードスター の速度: 100 km/h
```

`car1` を 50、`car2` を 100 加速したところ、**それぞれの `speed` が別々に増えました**。
`car1` の `speed` と `car2` の `speed` は、まったく別の入れ物なのです。

---

## なぜ、混ざらないのか

たい焼きのたとえを思い出してください。
同じ型から焼いても、1個ずつが別のたい焼きで、あんこの量も別々でしたね。

オブジェクトも同じです。
`new Car()` を呼ぶたびに、**フィールドの入れ物が、オブジェクトごとに新しく用意**されます。
だから `car1.speed` と `car2.speed` は別物で、片方を変えても、もう片方には影響しません。

```text
car1 ──→ ┌────────────────────┐
          │ Car                │
          │ name : プリウス     │
          │ speed: 50          │
          └────────────────────┘

car2 ──→ ┌────────────────────┐
          │ Car                │
          │ name : ロードスター │
          │ speed: 100         │
          └────────────────────┘
```

このように、**オブジェクトごとに用意されるフィールド**を、**インスタンス変数**（Instance Variable）と呼びます。
「インスタンスごとに別々に存在する変数」という意味です。
（次の節で学ぶ `static` なフィールドは、これとは違い、全インスタンスで**共有**されます。）

---

## 同じオブジェクトを指す2つの変数

ここで、第3節で学んだ「変数はオブジェクトの場所を指す」という性質が効いてくる、注意点を1つ紹介します。

`new` で**新しく作る**と別のオブジェクトになりますが、変数を**代入でコピー**しても、新しいオブジェクトはできません。

```java
class Car {
    String name;
    int speed;
}

void main() {
    Car a = new Car();
    a.name = "プリウス";

    Car b = a;          // a を b に代入（新しい車は作っていない！）
    b.name = "別名";    // b 経由で名前を変える

    IO.println(a.name);   // a の名前は？
}
```

```text
$ java CarApp.java
別名
```

`b` の名前を変えただけなのに、`a.name` も「別名」になりました。
これは、`Car b = a;` が**オブジェクトをコピーしたのではなく、同じオブジェクトの場所を `b` にも教えた**だけだからです。
`a` と `b` は、**同じ1台の車**を指しています。だから、どちらから変更しても、同じ車が変わるのです。

```text
a ──┐
    ├──→ [ Car ]   ← 1台の車を、a と b が共有している
b ──┘
```

第9章で学んだ「配列は中身を共有する」のと、まったく同じしくみです。
オブジェクトを別物として2つ用意したいなら、`new` を2回呼ぶ必要があります。

> **これは「バグ」ではなく「しくみ」**
>
> この挙動は、知らないと「いつの間にか値が変わった」と戸惑いますが、参照型の正しいふるまいです。
> 「**`new` は新しい実体を作る／代入は場所を教えるだけ**」 ―― この違いを押さえておけば、混乱しません。

---

## まとめ

- 1つのクラスから、`new` を呼ぶたびに**独立したオブジェクト**が作られる
- オブジェクトごとに用意されるフィールドを、**インスタンス変数**と呼ぶ
- あるオブジェクトのフィールドを変えても、別のオブジェクトには影響しない
- ただし、**変数の代入（`Car b = a;`）はオブジェクトをコピーしない**。同じオブジェクトを2つの変数が指す
- 別の実体がほしいなら、`new` を呼ぶ

次の節では、インスタンスごとではなく、**クラス全体に1つだけ**存在する `static` なメンバを学びます。
