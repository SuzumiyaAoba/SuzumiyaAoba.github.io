## はじめに ― この章で学ぶこと

ここから、本書の**第2部**が始まります。

第1部では、変数・演算子・制御構文・配列・メソッド・文字列という、プログラムを組み立てるための「部品」を学びました。
これらは、どのプログラミング言語にも共通する、基礎の基礎です。

第2部からは、いよいよ Java の本領 ―― **オブジェクト指向**（Object-Oriented）の世界に入っていきます。
その入口となるのが、この第11章「クラスとオブジェクト」です。

---

## クラスは、第3章でも登場した

実は「クラス」という言葉は、第3章ですでに登場していました。

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

このとき、クラスは「**すべての命令を書く、いちばん外側のまとまり**」として説明しました。
そして、「クラスには、この先で学ぶ深い意味がある」「本当の役割は第11章で学ぶ」と、何度か予告してきました。

その「本当の役割」を、ついにこの章で明らかにします。
クラスは単なる「外側の入れ物」ではなく、**オブジェクトを作るための設計図**なのです。

---

## オブジェクト指向とは何か

第1章で、Java は**オブジェクト指向**の言語だと紹介しました。
オブジェクト指向とは、ひとことで言えば、

> プログラムを、**データ**と、**そのデータに対する操作**をひとまとめにした「**部品（オブジェクト）**」の集まりとして組み立てていく考え方

です。

たとえば、買い物アプリを作るなら、「商品」「カート」「注文」といったものを、それぞれ部品として表現し、組み合わせていきます。
現実の世界を「もの（オブジェクト）」の集まりとしてとらえ、それをそのままプログラムに写し取る ―― それがオブジェクト指向の発想です。

最初は少し抽象的に感じるかもしれません。
ですが、この章で「クラス」と「オブジェクト」をていねいに学べば、その考え方が自然と頭に入ってきます。

---

## この章で学ぶこと

第11章は、次の9つの節で構成されています。

| 節 | タイトル | 内容 |
|---|---|---|
| 1 | クラスとオブジェクトとは | 設計図（クラス）と実体（オブジェクト）の関係 |
| 2 | クラスを定義する | フィールド（データ）を持つクラスの書き方 |
| 3 | オブジェクトを生成する | `new` でインスタンスを作る |
| 4 | メソッドを定義する | オブジェクトに「ふるまい」を持たせる |
| 5 | this ― 自分自身への参照 | メソッドの中から自分のフィールドを指す |
| 6 | 複数のインスタンス | オブジェクトは、それぞれが独立した状態を持つ |
| 7 | static ― クラスに属するメンバ | インスタンスに属さない、共通の値や処理 |
| 8 | よくあるつまずき | 初期化忘れ・null・static の誤解 など |

前半（1〜4節）で、クラスを定義し、オブジェクトを作って使う、という基本の流れをつかみます。
後半（5〜7節）で、`this` や `static` といった、一歩進んだしくみを学びます。

---

## この章を読み終えると

第11章を読み終えるころには、次のことができるようになっています。

- クラスとオブジェクトの違いを、自分の言葉で説明できる
- フィールドとメソッドを持つクラスを、自分で定義できる
- `new` でオブジェクトを作り、フィールドを読み書きし、メソッドを呼び出せる
- `this` が何を指すのかがわかる
- 複数のオブジェクトが、それぞれ独立した状態を持つことを理解できる
- `static`（クラスに属するもの）と、インスタンスに属するものの違いがわかる

> **補足: ここからは「完全なプログラム」で示します**
>
> 第4章から第10章までは、jshell にそのまま貼れる「文・式だけ」のサンプルを多く使ってきました。
> 第2部では、クラスという「プログラムの構造」そのものを学ぶため、ふたたび `void main()` を含む**完全なプログラム**を中心に示します。
> もちろん、jshell でもクラスを定義して試せます。気軽に実験したいときは、jshell も活用してください。

それでは、最初の節「クラスとオブジェクトとは」から始めましょう。
