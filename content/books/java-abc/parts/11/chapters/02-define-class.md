## クラスを定義する ― フィールド

それでは、実際にクラスを定義してみましょう。
この章では、**車**を表す `Car` クラスを題材に、少しずつ育てていきます。

まずは、車が持つ**データ（状態）**だけを持った、シンプルなクラスから始めます。

---

## いちばん簡単なクラス

クラスは、`class` というキーワードと、クラス名、そして `{ }` で書きます。

```java
class Car {
}
```

これだけでも、「`Car` という名前のクラス」として成立します。
ただし、中身が空っぽなので、まだ何のデータも持っていません。

> **補足: `public` は、いまは付けなくてよい**
>
> 第3章では `public class Main` と書いていました。`public` は「どこからでも使える」という印で、くわしくは第12章「カプセル化とアクセス修飾子」で学びます。
> この章では、1つのファイルの中でクラスを定義してすぐ使うので、`public` を付けない `class Car` の形で説明します。
> （`public` を付けないクラスは、ファイル名と名前をそろえる必要もありません。）

---

## フィールドを追加する

クラスに、車が持つデータを書き加えてみましょう。
車には「名前」と「速度」がある、と考えます。

```java
class Car {
    String name;   // 車の名前
    int speed;     // 速度（km/h）
}
```

`Car` クラスの `{ }` の中に書いた `String name;` と `int speed;` を、**フィールド**（Field）と呼びます。
フィールドは、第4章で学んだ**変数の宣言**と、まったく同じ形です（`型 名前;`）。

違うのは、**置く場所**です。

- メソッドの**中**で宣言した変数 … その場かぎりの一時的な変数（**ローカル変数**。第9章で学びました）
- クラスの**直下**（メソッドの外）で宣言した変数 … オブジェクトが持ち続けるデータ（**フィールド**）

フィールドは、そのクラスから作られる**1つひとつのオブジェクトが、それぞれ持つデータ**になります。
「名前」「速度」は、車ごとに違いますよね。だから、これらはフィールドとして書くのがふさわしいのです。

---

## クラス図で整理する

クラスが持つフィールド（とメソッド）を、簡単な図で表すことがあります。

```text
┌─────────────┐
│      Car         │  ← クラス名
├─────────────┤
│ String name      │  ← フィールド（データ）
│ int speed        │
├─────────────┤
│ （メソッドは次の節）  │  ← ふるまい（後で追加）
└─────────────┘
```

上の段にクラス名、まん中にフィールド、下にメソッドを書きます。
これは **UML クラス図**と呼ばれる図を、ごく簡単にしたものです。
クラスの全体像を見渡すのに便利なので、頭の片隅に置いておくとよいでしょう。

## フィールドの初期値

第4章で、「値を入れる前の変数を使うとエラーになる」と学びました。
ですが、フィールドは少し事情が違います。

`new` でオブジェクトを作ると（次の節で学びます）、フィールドには**自動的に初期値が入ります**。
初期値は、第8章の配列で学んだものと同じです。

| フィールドの型 | 自動で入る初期値 |
|---|---|
| `int`・`long` など整数 | `0` |
| `double`・`float` など小数 | `0.0` |
| `boolean` | `false` |
| `String` などの参照型 | `null` |

つまり、`int speed;` とだけ書いておけば、オブジェクトを作った時点で `speed` は `0` になっています。
ただし、`String name;` は `null`（何も指していない状態）になる点に注意が必要です。`null` の扱いは、第8節とこのあとの章でくわしく扱います。

---

## まとめ

- クラスは `class クラス名 { ... }` で定義する
- クラスの直下に書いた変数を**フィールド**と呼び、オブジェクトが持つ**データ（状態）**を表す
- フィールドの書き方は変数と同じ（`型 名前;`）。違うのは「クラスの直下に置く」こと
- フィールドは、`new` でオブジェクトを作ると**自動的に初期値**が入る（`int` は `0`、`String` は `null` など）
- この章では、まず `public` を付けない `class Car` の形で進める
