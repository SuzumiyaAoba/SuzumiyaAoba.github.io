## 開発環境の全体像 ― 何を準備するのか

実際にインストール作業を始める前に、まずは**これから何をそろえるのか**、その全体像をつかんでおきましょう。

地図を持たずに歩き始めると、いま自分がどこにいて、あと何が残っているのかがわからなくなります。
反対に、先に全体像を見ておくと、一つひとつの手順が「なぜ必要なのか」を理解しながら進められます。

プログラムを書いて動かすための準備一式のことを、**開発環境**（Development Environment）と呼びます。

この章でそろえる開発環境は、大きく次の2つです。

1. **JDK** ― Java を動かすための本体（必須）
2. **IDE** ― プログラムを書きやすくする道具（あると便利）

全体像を図にすると、次のようになります。

![開発環境の全体像。JDK（必須・エンジン）には javac・java・jshell・標準ライブラリが含まれる。IDE（あると便利・運転席）には入力補完・文法チェックなどがある。IDE だけでは動かず、JDK が必要。](./images/dev-environment-overview.svg)

順番に、それぞれが何者なのかを見ていきましょう。

## JDK ― Java を動かすための本体

第1章で、Java を始めるためにインストールするものは **JDK** である、と学びました。

ここで、混同しやすい **JDK・JRE・JVM** の関係を、もう一度だけ整理しておきます。

| 言葉 | 正式名称 | 役割 |
|---|---|---|
| **JVM** | Java Virtual Machine | バイトコードを実行する「エンジン」 |
| **JRE** | Java Runtime Environment | プログラムを**動かす**ための一式（JVM ＋ 標準ライブラリ） |
| **JDK** | Java Development Kit | プログラムを**作る**ための一式（JRE ＋ 開発ツール） |

この3つは、入れ子（いれこ）の関係になっています。

```text
JDK（作る道具）
 └─ JRE（動かす道具）
     └─ JVM（実行エンジン）
```

私たちはこれから、Java のプログラムを「書いて」「動かす」ことになります。
そのため、いちばん外側の **JDK** を1つ入れておけば、必要なものがすべて手に入ります。

JDK には、たとえば次のような道具が含まれています。

| 含まれるもの | 役割 |
|---|---|
| `javac` | ソースコードをバイトコードに変換する**コンパイラ** |
| `java` | バイトコードを実行する**起動コマンド** |
| `jshell` | Java を1行ずつ試せる対話ツール |
| 標準ライブラリ | あらかじめ用意された、たくさんの便利な機能 |

これらが具体的に何をするものなのかは、後のセクションで実際に使いながら確認していきます。
いまは、「JDK には、Java を書いて動かすための道具がひとまとめに入っている」と理解しておけば十分です。

> **補足: JRE だけを別に入れる必要はある？**
>
> 昔は、「動かすだけなら JRE」「開発するなら JDK」と使い分けることがありました。
> しかし現在では、学習でも開発でも、**JDK を入れておけば十分**です。
> JDK の中に、動かすために必要なもの（JRE 相当）がすべて含まれているからです。
> 「JRE を別に用意しなければ」と心配する必要はありません。

## IDE ― プログラムを書きやすくする道具

JDK さえあれば、実は Java のプログラムを書いて動かすことはできます。

文章を書くための「メモ帳」のような簡単なソフトでソースコードを書き、コマンドで `javac` と `java` を実行すれば、プログラムは動きます。
（この方法は、第7節「コマンドラインで実行する」で実際に体験します。）

しかし、メモ帳だけでプログラムを書くのは、なかなか骨が折れます。

たとえば、次のような場面を想像してみてください。

- 英単語のつづりを間違えても、何も教えてくれない
- かっこ `{` を閉じ忘れても、自分では気づけない
- どこでエラーが出ているのか、すべて自力で探すしかない
- プログラムを動かすたびに、コマンドを手で打ち込む必要がある

これは、定規もコンパスもない机で、フリーハンドで設計図を引くようなものです。
できないことはありませんが、ミスが起きやすく、効率もよくありません。

そこで登場するのが **IDE** です。

**IDE**（Integrated Development Environment、統合開発環境）とは、プログラムを「書く」「動かす」「間違いを見つける」といった作業を、一つのソフトの中でまとめて行えるようにした道具です。

「統合（とうごう）」という言葉のとおり、プログラミングに必要な機能が一つにまとまっています。

IDE には、たとえば次のような便利な機能があります。

| 機能 | どう便利か |
|---|---|
| 入力補完（ほかん） | 途中まで打つと、続きの候補を一覧で出してくれる |
| 文法チェック | 間違いのある場所に、その場で印（赤い波線など）を付けてくれる |
| ワンクリック実行 | ボタン1つでプログラムを動かせる |
| 色分け表示 | 命令・文字・記号などを色で見分けやすくしてくれる |
| 自動整形 | コードのインデント（字下げ）を自動で整えてくれる |

これは、定規・コンパス・消しゴムがそろい、線がずれていればすぐに教えてくれる、賢い製図台のようなものです。
道具が助けてくれるぶん、私たちは「何を作るか」という、本当に考えるべきことに集中できます。

特にプログラミングを始めたばかりのころは、IDE のこうした手助けが、とても心強い味方になります。

## JDK と IDE の関係 ― エンジンと運転席

ここで、JDK と IDE の関係を整理しておきましょう。
この2つは、役割がはっきりと分かれています。

- **JDK** … Java を実際に動かす「エンジン」
- **IDE** … そのエンジンを、人間が使いやすくする「運転席」

車にたとえると、わかりやすいかもしれません。

```text
JDK ＝ エンジン（これがないと、そもそも動かない）
IDE ＝ ハンドルやメーター（運転を楽にしてくれる）
```

エンジン（JDK）がなければ、車は1メートルも進めません。
一方で、ハンドルやメーター（IDE）がなくても、理屈のうえでは車を動かせますが、その運転はとても大変になります。

ここから、大切なポイントが見えてきます。

> **IDE は、JDK の代わりにはならない**

IDE はあくまで「書きやすくする道具」であって、Java を動かす本体ではありません。
そのため、たとえ IDE をインストールしても、**JDK が入っていなければ Java のプログラムは動きません**。

だからこそ、この章では先に **JDK** を準備し、そのあとで **IDE** を準備する、という順番で進めます。

> **補足: IDE が JDK を含んでいることもある**
>
> IDE のなかには、インストール時に JDK を一緒にダウンロードしてくれるものもあります（この章で使う IntelliJ IDEA も、その一つです）。
> とても便利な機能ですが、本書ではまず「JDK と IDE は別物である」という関係をはっきり理解してもらうために、JDK を自分の手で入れる方法から学びます。
> 仕組みがわかっていれば、あとで IDE 任せにしても、裏側で何が起きているのかを見失わずにすみます。

## これから進める順番

この章の後半では、次の順番で作業を進めていきます。

```text
1. JDK を選ぶ         … どの JDK を入れるか決める（第2節）
2. JDK を入れる       … 実際にインストールする（第3節）
3. 入ったか確認する   … java --version で確かめる（第4節）
4. IDE を選ぶ・入れる … IntelliJ IDEA を準備する（第5節）
5. プログラムを動かす … はじめての Hello, World!（第6節）
```

一見たくさんあるように見えますが、一つずつ進めれば、どれも難しくありません。
確実に積み上げていきましょう。

## まとめ

このセクションでは、これから準備する開発環境の全体像を確認しました。

- プログラムを書いて動かすための準備一式を **開発環境** と呼ぶ
- この章でそろえるのは、**JDK**（必須）と **IDE**（あると便利）の2つ
- **JDK** は、Java を作って動かすための本体で、`javac` や `java` などの道具が含まれている
- 学習でも開発でも、いちばん外側の **JDK** を1つ入れておけば、必要なものはそろう
- **IDE** は、プログラムを「書く」「動かす」「間違いを見つける」作業を助けてくれる道具
- IDE は便利だが、**JDK の代わりにはならない**。JDK がなければプログラムは動かない
- そのため、本書では先に **JDK**、次に **IDE** という順番で準備する
