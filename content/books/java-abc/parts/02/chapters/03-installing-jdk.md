## JDK をインストールする

前の節で、本書では **Eclipse Temurin** の **Java 25** を使うと決めました。

この節では、いよいよ実際に、自分のコンピュータへ JDK をインストールしていきます。

手順は OS によって異なるので、自分の環境に合わせて、該当する部分を読んでください。

- [Windows にインストールする](#windows)
- [macOS にインストールする](#macos)
- [Linux にインストールする（補足）](#linux)

どの OS でも、大きな流れは次の3ステップです。

```text
1. インストーラー（またはパッケージ）を入手する
2. 画面の案内に従ってインストールする
3. 正しく入ったかを確認する（次の第4節で行う）
```

この章では、各 OS について「初心者におすすめの方法」と「コマンドで入れる方法」の2つを紹介します。
**迷ったら、おすすめの方法（方法A）を選べば確実です。**

それでは始めましょう。

> **補足: ダウンロードは、必ず公式サイトから**
>
> JDK をダウンロードするときは、必ず**公式サイト**から入手してください。
> Temurin の公式サイトは [https://adoptium.net](https://adoptium.net) です。
> 検索結果の上のほうに、公式によく似た見慣れないサイトが表示されることがあります。
> ソフトのダウンロードでは、こうしたサイトを避け、公式の場所から入手する習慣をつけておきましょう。

<Column title="ダウンロードしたファイルが本物かを確かめる（余裕があれば）">

公式サイトでは、配布しているファイルが途中で壊れたり、すり替えられたりしていないかを確認するための **チェックサム**（SHA256 などと呼ばれる、ファイルごとに決まる短い符号）を一緒に公開しています。
ダウンロードしたファイルから計算した符号と、サイトに載っている符号が一致すれば、正しいファイルだと確かめられます。

いまは無理に行う必要はありません。「ファイルの正しさを確かめる方法がある」と知っておくだけで十分です。
仕事でソフトを扱うようになると役立つ知識なので、頭の片隅に置いておきましょう。

</Column>

## Windows にインストールする {#windows}

Windows では、主に2つの方法があります。

- **方法A: インストーラーを使う**（初心者におすすめ）
- **方法B: winget コマンドを使う**（コマンドに慣れている人向け）

迷ったら、**方法A** で進めてください。

### 方法A: インストーラーを使う（おすすめ）

#### 1. インストーラーをダウンロードする

1. ブラウザで [https://adoptium.net](https://adoptium.net) を開きます。
2. ダウンロードのページで、次のように選びます。
   - **Operating System（OS）**: Windows
   - **Architecture（CPU）**: x64（多くの Windows パソコンはこちら）
   - **Version（バージョン）**: 25 - LTS
3. **`.msi`** という形式のインストーラーをダウンロードします。

`.msi` ファイルは、Windows 用のインストーラーの形式です。
ダウンロードが終わると、`OpenJDK25...-x64_windows...msi` のような名前のファイルが保存されます。

#### 2. インストーラーを実行する

1. ダウンロードした `.msi` ファイルをダブルクリックします。
2. セットアップ画面が開くので、「Next（次へ）」を押して進みます。
3. **「Custom Setup（カスタムセットアップ）」** の画面では、次の2つの設定を**有効にする**ことをおすすめします。
   - **Add to PATH** … `java` コマンドを、どこからでも使えるようにする
   - **Set JAVA_HOME variable** … `JAVA_HOME` という設定を自動で行ってくれる

   これらの項目の左にあるアイコンをクリックし、メニューから「**Will be installed on local hard drive**（ローカルにインストールする）」を選ぶと、有効になります。

   > **なぜ有効にするの？**
   > **PATH** や **JAVA_HOME** は、コンピュータに「Java はここにあるよ」と教えるための設定です。
   > あとから手作業でも設定できますが、インストーラーに任せておくと、この先の手順がスムーズになります。
   > これらの言葉の意味は、次の第4節でくわしく説明します。いまは、案内どおりに有効にしておけば大丈夫です。

4. 「Install（インストール）」を押すと、インストールが始まります。
5. 途中で「このアプリがデバイスに変更を加えることを許可しますか？」と聞かれたら、「はい」を選びます。
6. 「Finish（完了）」が表示されれば、インストール完了です。

これで Windows への JDK のインストールは完了です。
正しく入ったかどうかは、次の第4節で確認します。

### 方法B: winget コマンドを使う

Windows には、**winget**（ウィンゲット）というツールが標準で用意されています。
これは、コマンドを1行打つだけでソフトをインストールできる便利な仕組みです。

1. スタートメニューから **「ターミナル」** または **「PowerShell」** を開きます。
2. 次のコマンドを入力して、Enter キーを押します。

   ```text
   winget install -e --id EclipseAdoptium.Temurin.25.JDK
   ```

   （`-e --id` は「この名前のものを正確に1つだけ入れる」という指定です。）
3. インストールが自動で進みます。完了のメッセージが出れば終わりです。

> **補足: 開いている画面は、いったん閉じる**
>
> インストール後、すでに開いていたターミナルでは、設定がまだ反映されていないことがあります。
> 確認の前に、いったんターミナルを閉じて、新しく開き直しておきましょう。

## macOS にインストールする {#macos}

macOS でも、主に2つの方法があります。

- **方法A: インストーラー（.pkg）を使う**（初心者におすすめ）
- **方法B: Homebrew を使う**（コマンドに慣れている人向け）

迷ったら、**方法A** で進めてください。

### 方法A: インストーラー（.pkg）を使う（おすすめ）

#### 1. インストーラーをダウンロードする

1. ブラウザで [https://adoptium.net](https://adoptium.net) を開きます。
2. ダウンロードのページで、次のように選びます。
   - **Operating System（OS）**: macOS
   - **Architecture（CPU）**:
     - Apple Silicon（M1・M2・M3・M4 など）の Mac → **aarch64**
     - Intel の Mac → **x64**
   - **Version（バージョン）**: 25 - LTS
3. **`.pkg`** という形式のインストーラーをダウンロードします。

> **自分の Mac がどちらかわからないとき**
> 画面左上のリンゴマーク → 「このMacについて」で確認できます。
> 「チップ」に `Apple M…` とあれば **aarch64**、`Intel` とあれば **x64** です。

#### 2. インストーラーを実行する

1. ダウンロードした `.pkg` ファイルをダブルクリックします。
2. インストーラーが開くので、「続ける」を押して進みます。
3. 「インストール」を押します。
4. Mac のパスワード（または Touch ID）を求められたら、入力して許可します。
5. 「インストールが完了しました」と表示されれば完了です。

`.pkg` でインストールすると、JDK は macOS の標準の置き場所（`/Library/Java/JavaVirtualMachines/` の中）に収められ、`java` コマンドがそのまま使えるようになります。

正しく入ったかどうかは、次の第4節で確認します。

### 方法B: Homebrew を使う

**Homebrew**（ホームブルー）は、macOS でソフトを管理するための、とても人気のあるツールです[^homebrew]。
コマンドを1行打つだけで、ソフトのインストールや更新ができます。

すでに Homebrew を使っている方は、次のコマンドで Temurin 25 をインストールできます。

```text
brew install --cask temurin@25
```

完了すると、Java 25 が使えるようになります。
（`--cask` は「アプリとしてまとめて入れる」という指定です。）

> **補足: Homebrew をまだ入れていない場合**
>
> Homebrew のインストールには、もう少し準備が必要です。
> まだ使っていない場合は、無理に Homebrew を使わず、上の **方法A（.pkg インストーラー）** で進めるのがおすすめです。
> Homebrew は、Java の学習に慣れてから導入しても、まったく遅くありません。

## Linux にインストールする（補足） {#linux}

Linux を使っている場合も、もちろん JDK をインストールできます。
本書は Windows・macOS を中心に説明しているため、ここでは要点の紹介にとどめます。

Linux では、主に次のような方法があります。

### tar.gz を使う方法

1. [https://adoptium.net](https://adoptium.net) から、Linux 用・自分の CPU 用の `.tar.gz` をダウンロードします。
2. 任意の場所に展開し、その中の `bin` ディレクトリに `PATH` を通します。

### パッケージ管理ツールを使う方法

ディストリビューションによっては、Adoptium が提供するリポジトリを登録すると、`apt` や `dnf` などのコマンドでインストールできます。
詳しい手順は、Adoptium の公式サイト（[https://adoptium.net/installation/linux/](https://adoptium.net/installation/linux/)）に、ディストリビューションごとにまとめられています。

> **補足: SDKMAN! という選択肢**
>
> Linux や macOS では、**SDKMAN!**（エスディーケイマン）という、複数の Java バージョンを手軽に切り替えられるツールもよく使われます[^sdkman]。
> 学習が進み、複数のバージョンを扱いたくなったときに、選択肢として覚えておくとよいでしょう。

## 補足: 更新とアンインストール

JDK は、一度入れたら終わりではなく、ときどき更新版（`25.0.1` → `25.0.3` など）が公開されます。
セキュリティの修正が含まれることもあるので、余裕があれば更新しておくとよいでしょう。

- **インストーラーで入れた場合**: 新しい版のインストーラーを、同じ手順で実行すれば更新できます。
- **winget で入れた場合**: `winget upgrade EclipseAdoptium.Temurin.25.JDK` で更新できます。
- **Homebrew で入れた場合**: `brew upgrade --cask temurin@25` で更新できます。

不要になった JDK を削除したいときは、OS の標準的な方法でアンインストールできます。

- **Windows**: 「設定」→「アプリ」から、入れた JDK を選んで削除します。
- **macOS**: `.pkg` で入れた場合は `/Library/Java/JavaVirtualMachines/` の中の該当フォルダを削除します。Homebrew で入れた場合は `brew uninstall --cask temurin@25` を実行します。

ただし、学習中は無理に削除する必要はありません。
「更新や削除も、こうしてできる」と知っておけば十分です。

## インストールが終わったら

OS に合わせて JDK のインストールが終わったら、次は「本当に正しくインストールできたのか」を確認します。

インストーラーの画面で「完了」が出ても、それだけで安心するのは禁物です。
コンピュータが、ちゃんと Java を見つけられる状態になっているかどうかを、**自分の目で確かめる**ことが大切です。

その確認方法を、次の第4節で詳しく説明します。

## まとめ

この節では、JDK を実際にインストールしました。

- JDK のダウンロードは、必ず**公式サイト**（[adoptium.net](https://adoptium.net)）から行う
- どの OS でも、流れは「入手 → インストール → 確認」の3ステップ
- 迷ったら、各 OS の「おすすめの方法（方法A）」を選べば確実
- **Windows**: `.msi` インストーラーがおすすめ。「Add to PATH」「Set JAVA_HOME variable」を有効にする。`winget` でも入れられる
- **macOS**: `.pkg` インストーラーがおすすめ。CPU（Apple Silicon か Intel か）に合ったものを選ぶ。`Homebrew` でも入れられる
- **Linux**: `.tar.gz` やパッケージ管理ツール、SDKMAN! などで入れられる
- インストールが終わったら、必ず次の第4節で「本当に入ったか」を確認する

次の節では、`java --version` というコマンドを使って、インストールが成功したかどうかを確認します。

[^homebrew]: Homebrew, "The Missing Package Manager for macOS (or Linux)," [https://brew.sh/](<https://brew.sh/>)。Homebrew は macOS／Linux 向けのオープンソースパッケージマネージャで、`brew install --cask` で GUI アプリやランタイムを Cask 形式で管理できる。

[^sdkman]: SDKMAN!, "The Software Development Kit Manager," [https://sdkman.io/](<https://sdkman.io/>)。SDKMAN! は JDK・Maven・Gradle など JVM 関連 SDK のバージョン管理を行うツールで、`sdk install java 25-tem` のように複数の JDK ディストリビューションを並列導入・切り替えできる。
