## コマンドラインで実行する

前のセクションでは、IntelliJ IDEA の ▶ ボタンで、ボタン1つでプログラムを動かしました。

とても便利でしたが、その裏側では、

1. ソースコードを `javac` で**コンパイル**する
2. できたバイトコードを `java` で**実行**する

という2つの作業が、自動的に行われていました。

このセクションでは、この2つの作業を、**自分の手で**やってみます。
IDE を使わず、第4節で使った**コマンドライン（ターミナル）**から Java を動かすのです。

> **なぜ、わざわざ手でやるのか**
>
> 普段の開発では、IDE のボタンを使うほうが圧倒的に楽です。
> それでも、ここで一度だけ手作業を体験しておくと、第1章で学んだ「コンパイル → 実行」という流れが、頭ではなく**手で**理解できます。
>
> IDE が裏で何をしているのかがわかっていると、トラブルが起きたときにも、「いまはコンパイルの段階だな」「実行の段階で失敗しているな」と、落ち着いて切り分けられるようになります。
> 一度だけでも体験しておく価値は、十分にあります。

---

## 1. ソースコードのファイルを用意する

まず、プログラムを書いたファイルを用意します。

ここでは、デスクトップなど、わかりやすい場所に `hello` という名前のフォルダを作り、その中で作業することにします。

そのフォルダの中に、**`Main.java`** という名前のファイルを作り、次の内容を書いて保存します。

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

ファイルは、IntelliJ で作っても、Windows のメモ帳や macOS のテキストエディットなど、簡単なソフトで作ってもかまいません。
（メモ帳などで作る場合は、拡張子が `.txt` にならないよう、保存時に `Main.java` とファイル名全体を指定してください。）

> **ファイル名は、クラス名と合わせる**
>
> `public class Main` と書いた場合、ファイル名は必ず **`Main.java`**（大文字・小文字も同じ）にする必要があります。
> これは Java のルールです。`main.java` や `Hello.java` ではエラーになります。
> なぜそうなるのかは、第3章でクラスを学ぶときに説明します。
> いまは「クラス名とファイル名をそろえる」とだけ覚えておきましょう。

---

## 2. フォルダへ移動する

ターミナル（コマンドライン）を開き、いま作ったファイルがあるフォルダへ移動します。

フォルダの移動には、**`cd`**（change directory、ディレクトリの変更）というコマンドを使います。

```text
cd （フォルダの場所）
```

たとえば、デスクトップに `hello` フォルダを作った場合は、次のようになります。

```text
（Windows の例）
cd Desktop\hello
```

```text
（macOS の例）
cd Desktop/hello
```

> **コツ: ドラッグ＆ドロップで場所を入力する**
>
> フォルダの場所を手で入力するのが難しい場合は、`cd ` と打って半角スペースを空けたあと、対象のフォルダをターミナルの画面に**ドラッグ＆ドロップ**すると、場所が自動で入力されます。
> 長い場所を打ち間違える心配がなくなる、便利な方法です。

移動できたら、フォルダの中に `Main.java` があるか確認しておきましょう。
ファイルの一覧を表示するコマンドは、次のとおりです。

```text
（Windows）
dir
```

```text
（macOS）
ls
```

一覧の中に `Main.java` が表示されれば、正しい場所にいます。

---

## 3. コンパイルする（javac）

それでは、ソースコードをコンパイルします。

第1章で学んだように、人間が書いたソースコードは、そのままでは実行できません。
**`javac`** を使って、JVM が読める**バイトコード**に変換（コンパイル）します。

次のコマンドを実行します。

```text
javac Main.java
```

エラーが出なければ、コンパイル成功です。
何もメッセージが表示されず、次の入力待ちに戻れば、成功のサインです。

> **「便りがないのはよい便り」**
>
> プログラミングのコマンドでは、成功したときには何も表示されず、失敗したときだけメッセージが出る、ということがよくあります。
> 最初は「本当に動いたのかな？」と不安になるかもしれませんが、何も言われなければ成功、と覚えておきましょう。

成功すると、同じフォルダの中に、新しく **`Main.class`** というファイルができています。
これが、コンパイルでできた**バイトコード**です。

一覧を表示して、確認してみましょう。

```text
（macOS の例）
ls
```

```text
Main.class	Main.java
```

`Main.java`（あなたが書いたソースコード）に加えて、`Main.class`（コンパイル結果）が増えていますね。

---

## 4. 実行する（java）

では、できあがったバイトコードを実行します。

実行には **`java`** コマンドを使います。
このとき指定するのは、ファイル名ではなく、**クラス名**です。

```text
java Main
```

ここで注意してほしいのは、`Main.class` や `Main.java` ではなく、**`Main`** とだけ書くことです。
拡張子（`.class` や `.java`）は付けません。

実行すると、次のように表示されます。

```text
Hello, World!
```

IDE のときと同じ結果が、コマンドラインでも表示されました。

ここまでの流れを、もう一度まとめると次のとおりです。

```text
Main.java（ソースコード）
   │  javac Main.java  でコンパイル
   ▼
Main.class（バイトコード）
   │  java Main  で実行
   ▼
画面に Hello, World! と表示される
```

第1章で学んだ「コンパイル → JVM で実行」という流れを、自分の手で再現できました。

---

## 5. もっと手軽に: 1つのコマンドで実行する

「コンパイルして、実行する」の2ステップを見てきましたが、実は **Java 11 以降**では、ソースコードを**1つのコマンドで直接実行**することもできます。

`javac` を使わず、`java` にソースファイルを直接渡します。

```text
java Main.java
```

これだけで、次のように結果が表示されます。

```text
Hello, World!
```

このとき Java は、裏側でコンパイルを行ってから実行しています（`Main.class` ファイルは作られません）。

ちょっとしたプログラムを試すときには、この方法がとても手軽です。

> **`java Main` と `java Main.java` の違い**
>
> 名前は似ていますが、意味がはっきり違います。混同しないように気をつけましょう。
>
> - `java Main` … すでにコンパイルした **バイトコード（`Main.class`）** を実行する（**拡張子なし**）
> - `java Main.java` … **ソースコード（`.java`）** を、その場でコンパイルして実行する（**拡張子あり**）

> **補足: Java 25 の新しい書き方も、そのまま実行できる**
>
> 第6節で紹介した、Java 25 の短い書き方（`void main()` と `IO.println`）で書いたファイルも、この `java ファイル名.java` の形でそのまま実行できます。
> 簡単なプログラムを手早く試すには、とても相性のよい組み合わせです。

## 6. jshell ― Java を1行ずつ試す

JDK には、**`jshell`**（ジェイシェル）という、便利な道具も含まれています。

`jshell` は、Java を**1行ずつ、その場で試せる**ツールです。
ファイルを作ったりコンパイルしたりしなくても、入力した Java の命令がすぐに実行されます。
（このような対話型の実行環境を **REPL**（レプル）と呼びます。）

ターミナルで、次のように入力して起動します。

```text
jshell
```

起動すると、`jshell>` という入力待ちの表示になります。
そこに、Java の命令を打ち込んでみましょう。

```text
jshell> System.out.println("こんにちは")
こんにちは
```

打った命令が、その場ですぐに実行されました。

計算も試せます。

```text
jshell> 1 + 2
$1 ==> 3
```

`jshell` は、ちょっとした文法を試したいときや、計算結果を確かめたいときに便利です。
学習を進めるなかで、「この書き方で合っているかな？」と思ったら、`jshell` で気軽に試せます。

実は、次の第4章からは、変数や計算、条件分岐といった Java の機能を、一つずつ学んでいきます。
本書では、こうした**機能を試すための短いサンプル**の多くを、`jshell` にそのまま貼り付けて動かせる形で紹介していきます。
ファイルを作ってコンパイル・実行する手間なく、その場で結果を確かめられるので、これからの学習で頼もしい相棒になってくれます。

`jshell` を終わるときは、次のように入力します。

```text
jshell> /exit
```

> **補足: 必須ではないが、本書のサンプルを試すのに便利**
>
> `jshell` は、必ず使わなければならないものではありません。
> ただ、第4章から登場する短いサンプルは、`jshell` に貼ればその場で動かせるものが多くあります。
> 「こういう道具もあるんだな」と知っておき、サンプルを手早く試したくなったら使ってみる、という姿勢で十分です。

> **コラム: 「ターミナル」「シェル」という名前の由来**
>
> `jshell` の「シェル（shell）」とは、いったい何でしょうか。
> shell は、英語で「貝がら」「殻」を意味します。コンピュータの世界では、**OS の中心部分（カーネル＝核）を包み、人間との橋渡しをするソフト**を「シェル」と呼びます。私たちが打ったコマンドを受け取り、中心部分へ伝えてくれる「殻」というわけです。`jshell` は、いわば「Java の殻」なのですね。
> 一方の「**ターミナル**（terminal、端末）」は、もともと、大型コンピュータに文字で命令を送るための装置（キーボードと画面のセット）を指す言葉でした。いまは、その装置の役割をソフトで再現した「文字で操作する画面」を、ターミナルと呼んでいます。
> 黒い画面に文字を打ち込むこの操作には、コンピュータがまだ大きく高価だった時代の名残が、いまも息づいているのです。

## まとめ

このセクションでは、IDE を使わず、コマンドラインから Java を動かしました。

- IDE の ▶ ボタンの裏では、`javac`（コンパイル）と `java`（実行）が自動で行われている
- `cd` コマンドで、ソースコードのあるフォルダへ移動する
- `javac Main.java` でコンパイルすると、バイトコード（`Main.class`）ができる
- `java Main` でバイトコードを実行する（拡張子は付けない／クラス名を指定する）
- ファイル名は、クラス名（`public class Main`）と同じ **`Main.java`** にする
- Java 11 以降は、`java Main.java` のように、ソースを1コマンドで直接実行できる
- `jshell` を使うと、Java を1行ずつ、その場で試せる（終了は `/exit`）

これで、IDE でもコマンドラインでも Java を動かせるようになりました。
