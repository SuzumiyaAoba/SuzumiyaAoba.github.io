## IDE で最初のプログラムを実行する

いよいよ、はじめてのプログラムを動かすときが来ました。

この節では、IntelliJ IDEA で新しいプロジェクトを作り、画面に **「Hello, World!」** という文字を表示します。

「Hello, World!」（ハロー・ワールド、こんにちは世界）は、プログラミングを学ぶときに、世界中で最初に書かれる定番のプログラムです。
まずはこれを動かして、「自分のコンピュータで Java が動いた！」という手応えをつかみましょう。

<Column title="なぜ最初は「Hello, World!」なのか">

「Hello, World!」を最初に書く習慣は、1970 年代に書かれた、ある有名なプログラミングの教科書から広まったと言われています[^kr-hello-world]。
以来、多くの言語の入門が、この小さなプログラムから始まるようになりました。

単純なプログラムにも見えますが、これを動かせたということは、「道具がそろい、書いて、コンパイルし、実行する」という一連の流れが、すべて正しく動いた証拠でもあります。
入門の第一歩として、これ以上ふさわしいものはありません。

</Column>

> **この章での目標は「動かすこと」**
>
> このあとコードが出てきますが、いまはその中身を完全に理解する必要はありません。
> コードの一つひとつの意味は、第3章「はじめてのプログラム」でじっくり説明します。
> この節での目標は、**環境がちゃんと動くことを確認する**ことです。
> 第1章で学んだ「写経」の気持ちで、見本どおりに進めてみましょう。

---

## 1. 新しいプロジェクトを作る

まず、IntelliJ IDEA を起動します。

起動すると、**ウェルカム画面**（最初のメニュー画面）が表示されます。
（すでに別のプロジェクトが開いている場合は、メニューの「File（ファイル）」→「New（新規）」→「Project（プロジェクト）」から進めます。）

1. ウェルカム画面の **「New Project（新規プロジェクト）」** をクリックします。
2. プロジェクトの設定画面が開くので、次のように入力・選択します。

| 項目 | 設定する内容 |
|---|---|
| **Name（名前）** | `HelloWorld`（半角英数字で。好きな名前でもよい） |
| **Location（場所）** | そのままで OK（保存先のフォルダ） |
| **Language（言語）** | **Java** を選ぶ |
| **Build system（ビルドシステム）** | **IntelliJ** を選ぶ |
| **JDK** | 第3節で入れた **25** 系（Temurin）を選ぶ |
| **Add sample code（サンプルコードを追加）** | **チェックを入れる** |

いくつか補足します。

- **Build system（ビルドシステム）**: ここは、後の章で学ぶ Maven や Gradle を選ぶこともできますが、いまは最もシンプルな **IntelliJ** を選びます。
- **JDK**: ここに、第3節で入れた Java 25 が表示されているはずです。`temurin-25` や `25` のような表記を選びます。
  - もし一覧に出てこない場合は、「Add JDK...」でインストール先を指定するか、「Download JDK」から入れ直すこともできます（第5節の補足を参照）。
- **Add sample code（サンプルコードを追加）**: これにチェックを入れると、すぐに動かせる見本のコードを、IntelliJ が自動で用意してくれます。

3. 設定できたら、**「Create（作成）」** を押します。

しばらく待つと、プロジェクトが作られ、編集画面が開きます。

> **補足: 初回は少し時間がかかる**
>
> 初めてプロジェクトを作るときは、IntelliJ が裏側で準備（インデックスの作成など）を行うため、少し時間がかかることがあります。
> 画面の下のほうに進行状況のバーが出ていれば、準備中のサインです。終わるまで待ちましょう。

## 2. 作られたコードを見てみる

はじめて IntelliJ の画面を見ると、情報が多くて戸惑うかもしれません。
ですが、これから使う場所は、大きく次の3つだけです。まずはこれだけ覚えれば十分です。

| 場所 | 名前 | 役割 |
|---|---|---|
| **左側** | プロジェクトツール | ファイルやフォルダの一覧。ここから開きたいファイルを選ぶ |
| **中央〜右側** | エディタ | 実際にコードを書く、いちばん広い場所 |
| **下側** | 実行結果（Run）ウィンドウ | プログラムを動かした結果が表示される（実行すると現れる） |

残りのボタンやメニューは、必要になったときに少しずつ覚えれば大丈夫です。
最初から全部を理解しようとしなくてよい、という第1章の心構えは、IDE の画面にもそのまま当てはまります。

それでは、左側の一覧から、コードを開いてみましょう。

`src`（source、ソースの意味）というフォルダの中に、**`Main`** という名前のファイルがあるはずです。
これをダブルクリックすると、中央のエディタにコードが表示されます。

中身は、だいたい次のようになっています。

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

> **表示が少し違っても大丈夫**
>
> IntelliJ のバージョンによっては、表示される文字（`Hello world!` など）や細かい書き方が、上の例と少し異なることがあります。
> 中身が違っても問題ありません。次の手順で、自分で書き換えていきます。

いまは、このコードの中に、

```java
System.out.println("Hello, World!");
```

という行があることに注目してください。
第1章でも登場した、**画面に文字を表示する命令**です。
`"` で囲まれた **`Hello, World!`** の部分が、実際に画面に表示される文字になります。

## 3. コードを確認・修正する

もし、自動で作られたコードが上の例と違っていたら、次のとおりに書き換えてみましょう。
（同じであれば、そのままで大丈夫です。）

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

第1章の「写経」を思い出してください。
コピー＆ペーストではなく、できるだけ自分の手で打ち込んでみると、コードの形に慣れていきます。

入力していると、IntelliJ が**入力候補**を出してくれることに気づくと思います。
たとえば `System.` まで打つと、続きの候補が一覧で表示されます。
これが、IDE の便利な「入力補完」機能です。
候補が出たら、矢印キーで選んで Enter キーを押すと、続きが自動で入力されます。

> **注意: 記号は、すべて半角で入力する**
>
> プログラムの記号（`(` `)` `{` `}` `"` `;` など）は、すべて**半角**で入力します。
> 日本語入力（全角）のまま打つと、見た目が似ていてもコンピュータには別物として扱われ、エラーになります。
> エラーが出たら、まず「全角の記号が混ざっていないか」を確認してみましょう。

## 4. プログラムを実行する

いよいよ実行です。

IntelliJ では、`main` の行の**左側**に、緑色の **▶（再生）ボタン** が表示されています。
このボタンをクリックし、表示されたメニューから **「Run 'Main'（Main を実行）」** を選びます。

> **別の実行方法**
> 画面右上にある緑色の ▶ ボタンを押す方法や、メニューの「Run（実行）」→「Run...」から実行する方法もあります。どれでも同じように動きます。

実行すると、画面の**下側**に **実行結果のウィンドウ**（Run ツールウィンドウ）が開き、次のように表示されます。

```text
Hello, World!
```

このように表示されれば、**大成功**です！

あなたのコンピュータで、Java のプログラムが初めて動いた瞬間です。
ここまでの環境構築が、すべて正しくできていたという証拠でもあります。

> **裏側で何が起きているのか**
>
> ▶ ボタンを押したとき、IntelliJ は裏側で、次のことを自動的に行っています。
>
> 1. あなたが書いたソースコードを、`javac` で**コンパイル**して、バイトコードに変換する
> 2. そのバイトコードを、`java` で **JVM** に渡して実行する
>
> 第1章で学んだ「コンパイル → JVM で実行」という流れが、ボタン1つの裏側で動いているのです。
> この手順を自分の手で行う方法は、次の第7節で体験します。

## 5. 少し変えてみる

第1章で学んだように、写経のあとは「少しだけ変えてみる」と理解が深まります。

表示する文字を、自分の好きな言葉に変えてみましょう。
`"` で囲まれた部分を書き換えます。

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("はじめての Java プログラム");
    }
}
```

書き換えたら、もう一度 ▶ ボタンで実行します。

```text
はじめての Java プログラム
```

このように、`"` の中の文字を変えると、表示される内容が変わることがわかります。

さらに、表示する行を増やしてみましょう。

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("はじめての Java プログラム");
        System.out.println("環境構築ができました");
    }
}
```

実行すると、2行表示されます。

```text
はじめての Java プログラム
環境構築ができました
```

「書く → 実行する → 結果を見る → 少し変える → もう一度実行する」。
第1章で学んだこの流れを、さっそく自分の手で体験できました。

## 補足: Java 25 の新しい書き方

ここまで、次のような「決まった形」のプログラムを書いてきました。

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

第1章の用語集でも少し触れましたが、実は **Java 25** では、初心者向けに、これをもっと短く書ける書き方が用意されました。
（**コンパクトソースファイル**や**インスタンスメインメソッド**と呼ばれる機能です。）

たとえば、同じ「Hello, World!」を、次のように書くこともできます。

```java
void main() {
    IO.println("Hello, World!");
}
```

`public class` の囲みがなくなり、`System.out.println` が `IO.println` という短い形になりました。
ずいぶんすっきりしましたね。

ただし、この書き方を快適に使うには IDE 側の対応も必要で、まだ見慣れない部分も多いと思います。
本書では、まずは多くの教材や現場で使われている「決まった形」から学び、こうした新しい書き方は、第3章以降であらためて紹介します。

いまは「Java 25 には、もっと簡単に書ける方法もあるらしい」と、頭の片隅に置いておくだけで十分です。
そして、こうした初心者にやさしい新機能が用意されている点も、本書が最新の **Java 25** を選んだ理由の一つです。

[^kr-hello-world]: Brian W. Kernighan and Dennis M. Ritchie, *The C Programming Language* (Prentice Hall, 1978)。"hello, world" の例題プログラムは本書で広く知られるようになった。前身の社内資料 Brian W. Kernighan, "Programming in C: A Tutorial," Bell Labs（1974年頃, [https://www.bell-labs.com/usr/dmr/www/ctut.pdf](<https://www.bell-labs.com/usr/dmr/www/ctut.pdf>)）にも同様の例題が記載されている。

## まとめ

この節では、IntelliJ IDEA で最初のプログラムを動かしました。

- 「New Project」から、Language に **Java**、JDK に **25（Temurin）** を選んでプロジェクトを作った
- 「Add sample code」にチェックを入れると、すぐに動かせる見本コードが用意される
- `System.out.println("...")` は、`"` の中の文字を画面に表示する命令
- `main` の左の緑色の **▶ ボタン**で、プログラムを実行できる
- 実行結果は、画面下の実行結果ウィンドウに表示される
- ▶ ボタンの裏では、第1章で学んだ「コンパイル → JVM で実行」が自動で行われている
- `"` の中を書き換えたり、行を増やしたりして、結果の変化を確かめた
- Java 25 には、`void main()` と `IO.println` を使った、より短く書ける新しい書き方もある（詳しくは第3章以降で）

次の節では、IDE を使わずに、コマンドラインから同じプログラムを動かしてみます。
