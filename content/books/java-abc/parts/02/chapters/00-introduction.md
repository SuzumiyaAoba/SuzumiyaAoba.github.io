## はじめに ― この章で学ぶこと

第1章では、プログラムとは何か、コンピュータはどのような仕組みで動くのか、そして Java とはどんな言語なのかを学びました。

ここまでは、紙とえんぴつで考える「準備運動」のような内容でした。
自分のコンピュータには、まだ何もインストールしていません。

この第2章では、いよいよ**自分のコンピュータで Java を動かすための準備**をします。

料理にたとえてみましょう。
これまでは、「料理とは何か」「キッチンにはどんな道具があるのか」を、本を読んで学んできた段階です。
この章では、実際に**自分のキッチンに道具をそろえ、コンロに火が点くことを確かめる**段階に入ります。
そして章の終わりには、その道具を使って、はじめての小さな一品 ―― 画面に文字を表示するプログラム ―― を実際に作り上げます。

---

## なぜ「環境構築」が必要なのか

第1章で学んだとおり、Java のプログラムを動かすには **JDK**（Java Development Kit）という一式が必要でした。

ところが、私たちのコンピュータには、最初から JDK が入っているとは限りません。
むしろ、ほとんどのパソコンには入っていません。

そのため、Java を学ぶには、まず JDK を自分でインストールする必要があります。

このように、プログラミングを始めるために、必要な道具をそろえてコンピュータを準備することを、**環境構築**（かんきょうこうちく）と呼びます。

環境構築は、よく「プログラミング学習の最初の関門」と言われます。

その理由は、たとえば次のようなものです。

- 普段はあまり触らない「設定」をいじることがある
- 使っている OS（Windows や macOS）によって手順が違う
- うまくいかないと、コードを1行も書かないうちにつまずいてしまう

つまり、環境構築でのつまずきは、プログラミングそのものが難しいからではなく、**準備に特有の理由**で起きるのです。

ですから、ここでつまずいても、「自分にはプログラミングは向いていないのかもしれない」などと考える必要はまったくありません。

> **コラム: つまずくのは、あなただけではありません**
>
> 環境構築は、経験豊富なエンジニアでも、新しいパソコンや新しい言語ではつまずくことがあります。
> 「設定がうまく反映されない」「コマンドが見つからない」といったトラブルは、いわば通過儀礼のようなものです。
>
> この章では、一つひとつの手順を、画面の操作レベルまで分けて説明します。
> さらに、つまずきやすいポイントは第8節「うまくいかないとき」にまとめてあります。
> 焦らず、上から順番に進めれば大丈夫です。

## この章で準備する2つのもの

この章で準備するものは、大きく分けると、たった2つです。

| 準備するもの | 役割 | たとえると |
|---|---|---|
| **JDK** | Java のプログラムを作って動かすための本体 | 料理の道具とコンロ |
| **IDE** | プログラムを書きやすくするための専用エディタ | 使いやすく整ったキッチン |

**JDK** は、第1章でも登場した「Java を作って動かすための一式」です。
これがなければ、そもそも Java のプログラムを動かせません。**必須**の道具です。

**IDE**（アイ・ディー・イー）は、プログラムを書くための高機能なソフトです。
まだ聞き慣れない言葉だと思いますが、第1節と第5節で詳しく説明します。
こちらは必須ではありませんが、入れておくと学習がぐっと楽になります。

いまは、

- **JDK は必須**
- **IDE はあると便利**

この2つだけ、頭に入れておけば十分です。

## この章で学ぶこと

第2章は、次の8つのセクションで構成されています。

| 節 | タイトル | 内容 |
|---|---|---|
| 1 | 開発環境の全体像 | JDK と IDE という、これからそろえる道具の全体像 |
| 2 | JDK を選ぶ | たくさんある JDK のなかから、どれを選べばよいか |
| 3 | JDK をインストールする | Windows・macOS への JDK の入れ方 |
| 4 | インストールを確認する | 本当に入ったかを `java --version` で確かめる |
| 5 | IDE を選ぶ・準備する | IDE とは何か、何を選び、どう入れるか |
| 6 | IDE で最初のプログラムを実行する | はじめての「Hello, World!」を動かす |
| 7 | コマンドラインで実行する | IDE を使わずに、コマンドで動かす方法 |
| 8 | うまくいかないとき | よくあるトラブルと、その対処法 |

前半（1〜4節）では、必須の道具である **JDK** を準備します。
後半（5〜7節）では、**IDE** を準備し、実際に最初のプログラムを動かします。
そして最後の8節では、つまずいたときのために、よくあるトラブルへの対処法をまとめます。

それぞれのセクションは、前のセクションの内容を受けて積み上がるように書かれています。
できるだけ、順番に読み進めることをおすすめします。

## この章を読むうえでの注意

環境構築の手順を読むうえで、最初にお伝えしておきたいことが2つあります。
どちらも、この先の不安を減らすための大切なポイントです。

### 画面や手順は、少しずつ変わっていく

JDK や IDE は、日々改良され、新しい版が公開されています。
そのため、本書に書かれたボタンの位置や文言が、あなたの画面と少し違うことがあります。

それでも、**やることの大きな流れ**は変わりません。

```text
公式サイトからダウンロードする
   ↓
自分の OS に合ったものを選ぶ
   ↓
画面の案内に従ってインストールする
   ↓
正しく入ったか確認する
```

この流れさえ押さえておけば、細かい違いがあっても落ち着いて対応できます。
迷ったときは、本書の手順と合わせて、各**公式サイト**の案内も確認してください。

### バージョンの数字に神経質にならない

本書は **Java 25** を対象としています。

ただし、あなたが読んでいる時点では、Java 26 や 27 など、より新しい版が出ているかもしれません。

この章で学ぶ内容のほとんどは、バージョンが変わっても通用します。
「必ず 25 でなければならない」と気負う必要はありません。
まずは、本書と同じ **Java 25**（または、それ以上の新しい版）を入れておけば十分です。

> **補足: Java 25 は「長く使える」バージョン**
>
> 第1章で学んだように、Java 25 は **LTS**（長期サポート版）です。
> 本書が使う Eclipse Temurin の場合、少なくとも 2031 年ごろまではセキュリティ更新が提供される予定です（提供期間はのちに延長されることもあります）。
> 数年先まで安心して使えるので、いま腰を据えて学ぶ土台として、まさにうってつけのバージョンです。

## この章を読み終えると

第2章を読み終えるころには、次のことができるようになっています。

- 自分のコンピュータに JDK をインストールできる
- `java --version` を使って、正しくインストールできたかを確認できる
- IDE（IntelliJ IDEA）をインストールし、最初のプロジェクトを作れる
- 画面に「Hello, World!」と表示する、はじめてのプログラムを動かせる
- コマンドラインから Java のプログラムを実行できる
- うまくいかないときに、どこを確認すればよいかがわかる

ここまで準備できれば、第3章からは、いよいよ本格的にプログラムを書き始められます。

長い準備に思えるかもしれませんが、一度この環境を整えてしまえば、あとは何度でも使えます。
それでは、最初のセクション「開発環境の全体像」から始めましょう。
