## 用語集 ― この章で学んだ言葉

この第2章でも、たくさんの新しい言葉が登場しました。

ここでは、それらの用語を**節ごとに**まとめておきます。
「あの言葉は何だったかな」と思ったときに、辞書のように使ってください。

すべてを覚える必要はありません。
環境構築は、一度きちんとできてしまえば、毎日使うものではありません。
わからなくなったらここに戻ってくる ―― そんな使い方で十分です。

## コマンドラインで実行する（第7節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| `cd` | change directory | フォルダ（ディレクトリ）を移動するコマンド |
| `ls` / `dir` | ― | フォルダの中のファイル一覧を表示するコマンド（macOS は `ls`、Windows は `dir`） |
| `javac` | ジャバ・シー | ソースコードをバイトコードにコンパイルするコマンド |

| バイトコード（.class） | Bytecode | `javac` でできる、JVM が読める形式の命令。`Main.class` のように作られる |
| 単一ファイル実行 | ― | `java Main.java` のように、ソースを1コマンドで直接実行する方法（Java 11 以降） |
| jshell | ジェイシェル | Java を1行ずつ、その場で試せる対話ツール |
| REPL | レプル | 入力した命令をすぐに実行して結果を返す、対話型の実行環境。jshell もその一つ |

## おわりに

ここに挙げた用語は、いずれも第2章の各節で説明したものです。

環境構築でつまずいたときは、症状から第8節「うまくいかないとき」を、言葉の意味があいまいになったときは、この用語集を見返してください。

さて、これで Java を書いて動かす準備が、すべて整いました。

ここまでお疲れさまでした。
環境構築という最初の関門を越えたあなたは、もう「プログラムを動かせる人」です。

第3章では、いよいよ本格的にプログラムを書き始めます。
この章で動かした「Hello, World!」を題材に、`public class` とは何か、`main` とは何か、`System.out.println` はどういう仕組みなのか ―― プログラムの中身を、一つひとつ読み解いていきましょう。
