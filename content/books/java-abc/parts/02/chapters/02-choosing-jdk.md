## JDK を選ぶ ― ディストリビューションの違い

JDK をインストールしようと思って調べ始めると、多くの人が最初の関門にぶつかります。

それは、「**JDK にはたくさんの種類があって、どれを選べばよいのかわからない**」という問題です。

たとえば、検索すると次のような名前が次々と出てきます。

- Oracle JDK
- Eclipse Temurin
- Amazon Corretto
- Azul Zulu
- Microsoft Build of OpenJDK

「Java の JDK が欲しいだけなのに、どうしてこんなに種類があるの？」と戸惑うのは当然です。

この節では、なぜ JDK に種類があるのか、そして本書ではどれを選ぶのかを説明します。

先に結論からお伝えします。

> **本書では、Eclipse Temurin（テムリン）という JDK を使います。**

なぜこれを選ぶのか、その理由も含めて見ていきましょう。

---

## なぜ JDK に種類があるのか

JDK に種類がある理由を理解するには、第1章で学んだ **OpenJDK** を思い出すと役立ちます。

Java の中身は、**OpenJDK** という名前で、オープンソースとして公開されています。

オープンソースとは、設計図にあたるソースコードが公開されていて、誰でも自由に使ったり、それをもとに製品を作ったりできる、という意味です。

つまり、OpenJDK という「共通の土台（設計図）」があり、いろいろな団体や会社が、それをもとにそれぞれの JDK を組み立てて配布しているのです。

このように、共通の土台から作られて配布される、それぞれの JDK のことを、**ディストリビューション**（Distribution、配布物）と呼びます。

料理にたとえると、わかりやすいかもしれません。

```text line-numbers=false
OpenJDK    ＝ 公開されている共通のレシピ
Temurin    ＝ A団体が、そのレシピで作って提供する料理
Corretto   ＝ B社が、そのレシピで作って提供する料理
Oracle JDK ＝ C社が、そのレシピで作って提供する料理
```

同じレシピ（OpenJDK）をもとにしているので、できあがる料理（JDK）の中身は、基本的にはほとんど同じです。
**どれを選んでも、書いた Java のプログラムが動くという点では変わりません。**

では、何が違うのでしょうか。
主に違うのは、料理そのものではなく、次のような「提供のされ方」の部分です。

- 提供している団体・会社
- 利用するときの料金やライセンス（使ってよい条件）
- サポート（サポートが続く期間や手厚さ）
- ダウンロードのしやすさ

<Column title="なぜ種類が増えたのか">

かつては、Java といえば Oracle 社（とその前の Sun 社）が提供する JDK がほぼ唯一の選択肢でした。
しかし、Java の中身が OpenJDK としてオープンソース化され、さらに 2019 年ごろに Oracle の JDK のライセンスや提供方針が変わったことをきっかけに[^oracle-jdk-license-2019]、「自分たちでも OpenJDK をもとに JDK を作って配ろう」という動きが各社・各団体に広がりました。
いま見られるたくさんのディストリビューションは、その結果です。
選択肢が増えたぶん最初は迷いますが、利用者にとっては「自由に選べる」よい状況だと言えます。

</Column>

## 代表的な JDK ディストリビューション

代表的な JDK を、いくつか紹介します。
すべてを覚える必要はありません。「こういうものがあるんだな」と眺めるだけで十分です。

| 名前 | 提供元 | 特徴 |
|---|---|---|
| **Eclipse Temurin** | Eclipse Foundation（Adoptium） | 特定の会社に偏らない中立的な立場。無料で、用途を問わず使いやすい。広く使われている定番 |
| **Oracle JDK** | Oracle 社 | Java を主導する Oracle 社の公式版。現在のライセンスでは無料で利用できる |
| **Amazon Corretto** | Amazon 社 | Amazon が提供。クラウドサービス（AWS）でよく使われる |
| **Azul Zulu** | Azul 社 | 幅広いバージョンに対応。古い Java も含めて選びやすい |
| **Microsoft Build of OpenJDK** | Microsoft 社 | Microsoft が提供。同社のサービスとの相性がよい |

繰り返しになりますが、これらはどれも **OpenJDK** という同じ土台から作られています。
そのため、本書のサンプルコードは、どの JDK を選んでも動きます。

> **補足: 「OpenJDK」そのものも入手できる**
>
> OpenJDK のプロジェクト自身も、ソースコードや一部のビルドを公開しています。
> ただし、初心者がそのまま使うには少し手間がかかるため、上の表のような「使いやすく整えて配布された JDK」を選ぶのが一般的です。

## 本書が Eclipse Temurin を選ぶ理由

本書では、数ある JDK のなかから **Eclipse Temurin**（エクリプス・テムリン）を使います。

Temurin は、**Adoptium**（アドプティウム）という、Eclipse Foundation のプロジェクトが提供している JDK です。
（以前は「AdoptOpenJDK」という名前で知られていました[^adoptium-rename]。）

Temurin を選ぶ理由は、次のとおりです。

- **無料で使える** ― 学習用途でも、将来の仕事でも、料金を気にせず使える
- **特定の会社に偏らない** ― 中立的な立場のプロジェクトが提供しているので、安心して選べる
- **用途を選ばない** ― 個人の学習から本格的な開発まで、幅広く使える
- **広く使われている** ― 利用者が多く、困ったときに情報を見つけやすい
- **対応 OS が広い** ― Windows・macOS・Linux に対応している

つまり、「迷ったらこれを選んでおけば、まず問題ない」という、安心できる選択肢です。

> **補足: ほかの JDK でも学習はできる**
>
> もし、すでに Oracle JDK など別の JDK を使っていたり、職場や学校で指定された JDK があったりする場合は、無理に Temurin へ入れ替える必要はありません。
> 本書のサンプルコードは、Java 25 に対応した JDK であれば、どれでも動きます。
> ただし、本書では手順の説明を Temurin に統一しているので、特にこだわりがなければ Temurin を使うことをおすすめします。

## どのバージョンを選ぶか

JDK を選ぶときには、ディストリビューション（種類）だけでなく、**バージョン**も選ぶ必要があります。

第1章で学んだように、本書が対象とするのは **Java 25** です。
そして Java 25 は、長期間サポートされる **LTS**（Long-Term Support）のバージョンでした。

ダウンロードページでは、たくさんのバージョンが並んでいることがあります。
そのなかから、本書では次の方針で選びます。

> **Java 25（LTS）を選ぶ。**

LTS のバージョンは、サポートが長く続くため、安心して長く使えます。
学習を始めるなら、まさにうってつけです。

### バージョン番号の細かい数字について

JDK のバージョンには、`25` のあとに、さらに細かい数字が付くことがあります。

```text line-numbers=false
25        … 最初に公開された版
25.0.1    … その後の更新版
25.0.3    … さらに後の更新版
```

`25.0.1` や `25.0.3` のような番号は、Java 25 に対して、不具合の修正やセキュリティの更新を加えた**更新版**です。

これらは、すべて「Java 25 の仲間」です。
ダウンロードするときは、**いちばん新しい 25 系**（その時点で最新の `25.0.x`）を選んでおけば大丈夫です。
細かい数字の違いは、学習において気にする必要はありません。

> **補足: もっと新しい LTS が出ていたら？**
>
> あなたが読んでいる時点で、Java 25 より新しい LTS（たとえば次の LTS）が出ているかもしれません。
> その場合は、より新しい LTS を選んでも、本書の内容はほとんどそのまま通用します。
> ただし、本書と完全に同じ動作を確かめたい場合は、**Java 25** を選ぶのが確実です。

## どの「形」でダウンロードするか

JDK をダウンロードするときは、自分のコンピュータに合ったものを選ぶ必要があります。
選ぶときの観点は、主に次の2つです。

### 1. OS の種類

自分が使っている OS に合ったものを選びます。

- **Windows** を使っているなら、Windows 用
- **macOS**（Mac）を使っているなら、macOS 用
- **Linux** を使っているなら、Linux 用

### 2. CPU の種類（アーキテクチャ）

同じ OS でも、コンピュータの CPU の種類によって、選ぶものが変わることがあります。
特に **Mac** では、この違いが重要です。

| 表記 | 別の言い方 | どんなコンピュータ向けか |
|---|---|---|
| **x64**（x86_64） | Intel | Intel 製 CPU のパソコン（古めの Mac、多くの Windows PC） |
| **AArch64** | ARM64 / Apple Silicon | Apple Silicon（M1・M2・M3・M4 など）の Mac |

たとえば、最近の Mac の多くは **Apple Silicon**（M1 以降）なので、**AArch64（ARM64）** を選びます。
少し前の Mac や、多くの Windows パソコンは **x64（Intel）** を選びます。

Apple Silicon の Mac で、あえて Intel 用（x64）を選ぶ必要はありません。
**AArch64（ARM64）** を選べば、その Mac に最適化されたネイティブ版が手に入り、性能を十分に引き出せます（変換のしくみである Rosetta も必要ありません）。

> **補足: 自分の Mac がどちらかわからないとき**
>
> Mac では、画面左上のリンゴマーク → 「このMacについて」を開くと確認できます。
> 「チップ」または「プロセッサ」の欄に、`Apple M1`／`Apple M2`… などと書かれていれば **Apple Silicon（AArch64）**、`Intel` と書かれていれば **x64** です。
>
> なお、次の節で紹介する **インストーラー** や **Homebrew** を使う方法なら、CPU の種類を自動で判別してくれることが多いので、神経質になりすぎる必要はありません。

## まとめ

この節では、たくさんある JDK のなかから、どれを選べばよいかを学びました。

- Java の中身は **OpenJDK** という名前でオープンソース公開されている
- いろいろな団体・会社が OpenJDK をもとに JDK を組み立てて配布しており、それらを **ディストリビューション** と呼ぶ
- 代表例として、Temurin・Oracle JDK・Corretto・Zulu などがある
- どれも同じ土台（OpenJDK）から作られているため、書いた Java のプログラムはどれでも動く
- 本書では、無料・中立・広く使われているという理由から **Eclipse Temurin** を使う
- バージョンは、長期サポートされる LTS の **Java 25** を選ぶ（`25.0.x` のような更新版でよい）
- ダウンロード時は、自分の **OS** と **CPU の種類**（Intel か Apple Silicon か）に合ったものを選ぶ
