## 算術演算子 ― 四則演算と剰余

**算術演算子**（Arithmetic Operator）は、たし算・ひき算などの計算を行う演算子です。
学校で習う四則演算に加え、Java には「余りを求める」演算子もあります。

| 演算子 | 意味 | 例 | 結果 |
|---|---|---|---|
| `+` | たし算 | `7 + 3` | `10` |
| `-` | ひき算 | `7 - 3` | `4` |
| `*` | かけ算 | `7 * 3` | `21` |
| `/` | わり算 | `7 / 3` | `2` |
| `%` | 余り（剰余） | `7 % 3` | `1` |

かけ算は `×` ではなく **`*`**（アスタリスク）、わり算は `÷` ではなく **`/`**（スラッシュ）を使う点に注意しましょう。

jshell で試してみます。

```text
jshell> int wa = 7 + 3;
wa ==> 10
jshell> int seki = 7 * 3;
seki ==> 21
```

## わり算の注意 ― 整数どうしは整数に

第4章でも学んだ、いちばんの落とし穴を、もう一度確認します。
**整数どうしのわり算は、結果も整数**になり、小数点以下は切り捨てられます。

```text
jshell> int sho = 7 / 3;
sho ==> 2
```

`7 ÷ 3 = 2.33…` ですが、整数どうしなので `2` になります。
小数で計算したいときは、どちらかを小数（`double`）にします。

```text
jshell> double real = 7.0 / 3;
real ==> 2.3333333333333335
```

`7.0` にすると、結果も小数になりました。

> **補足: 小数の最後が中途半端なのはなぜ？**
>
> 結果の末尾が `…35` と中途半端なのは、コンピュータが小数を**2進数**で扱うため、`0.333…` のような数をぴったり表せず、わずかな誤差が出るからです。
> これは Java にかぎらず、多くのプログラミング言語に共通する性質です。お金の計算など、誤差が問題になる場面での対処は、のちの章で触れます。

## 余りを求める ― %（剰余演算子）

`%`（パーセント）は、わり算の**余り**を求める演算子で、**剰余演算子**（じょうよえんざんし）と呼びます。

```text
jshell> int amari = 7 % 3;
amari ==> 1
```

`7 ÷ 3` は「2 余り 1」なので、`7 % 3` は余りの `1` になります。

剰余は、地味に見えて、とても役に立ちます。たとえば、

- **偶数か奇数かを調べる**：`n % 2` が `0` なら偶数、`1` なら奇数
- **一定の間隔で区切る**：`n % 3` が `0` のときだけ処理する、など

こうした使い方は、第6章（条件分岐）や第7章（繰り返し）で活躍します。

## まとめ

このセクションでは、算術演算子を学びました。

- 算術演算子は `+`・`-`・`*`・`/`・`%`。かけ算は `*`、わり算は `/`
- 整数どうしのわり算は、小数点以下が切り捨てられる（`7 / 3` は `2`）
- `%` は**余り**を求める（`7 % 3` は `1`）。偶数・奇数の判定などに役立つ
