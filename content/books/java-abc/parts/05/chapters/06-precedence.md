## 演算子の優先順位

1つの式に複数の演算子が混ざると、「どれから計算するか」が問題になります。
この計算する順番を、**演算子の優先順位**（Operator Precedence）と呼びます。

算数と同じで、かけ算・わり算は、たし算・ひき算より先に計算されます。

```text
jshell> int a = 1 + 2 * 3;
a ==> 7
```

`1 + 2 * 3` は、先に `2 * 3`（＝ 6）が計算され、その後 `1 + 6` で `7` になります。
（`1 + 2` を先に計算して `3 * 3 = 9`、にはなりません。）

---

## かっこで順番を変える

先に計算したい部分は、**かっこ `( )`** で囲みます。かっこの中が、最優先で計算されます。

```text
jshell> int b = (1 + 2) * 3;
b ==> 9
```

`(1 + 2)` が先に計算されて `3`、その後 `3 * 3` で `9` になりました。

迷ったら、かっこを使うのが安全です。優先順位を完全に覚えていなくても、かっこで囲めば、意図したとおりの順番で計算できます。読む人にとっても、計算の順番がはっきりして親切です。

---

## 主な優先順位

よく使う演算子の優先順位を、高い順にまとめます。

| 優先順位 | 演算子 | 種類 |
|---|---|---|
| 高い | `!` | 論理（でない） |
| ↑ | `*` `/` `%` | かけ算・わり算・剰余 |
| │ | `+` `-` | たし算・ひき算 |
| │ | `<` `>` `<=` `>=` | 比較（大小） |
| │ | `==` `!=` | 比較（等価） |
| │ | `&&` | かつ |
| ↓ | `\|\|` | または |
| 低い | `=` `+=` など | 代入 |

たとえば `score >= 0 && score <= 100` は、`>=` や `<=`（比較）が `&&` より先に計算されるので、`(score >= 0) && (score <= 100)` と同じ意味になります。前の節で、かっこなしで書けたのは、このためです。

すべてを暗記する必要はありません。「かけ算・わり算が先」「比較は計算より後」「論理演算はさらに後ろ」というおおまかな感覚をつかみ、**迷ったらかっこ**を使えば十分です。

## 文字列の連結も、左から順に

第4章・第5章で見た文字列の `+` も、順番の影響を受けます。

```text
jshell> String m1 = "答え: " + 1 + 2;
m1 ==> "答え: 12"
jshell> String m2 = "答え: " + (1 + 2);
m2 ==> "答え: 3"
```

- `"答え: " + 1 + 2` … 左から順に処理されます。まず `"答え: " + 1` で `"答え: 1"`（連結）、続けて `+ 2` で `"答え: 12"`（連結）
- `"答え: " + (1 + 2)` … かっこの中の `1 + 2` が先に計算されて `3`、その後 `"答え: " + 3` で `"答え: 3"`

数値を足してから連結したいときは、かっこで囲む ―― これも「迷ったらかっこ」の好例です。

## まとめ

この節では、演算子の優先順位を学びました。

- **演算子の優先順位**は、式の中でどの演算子から計算するかの決まり
- かけ算・わり算（`*` `/` `%`）は、たし算・ひき算（`+` `-`）より先
- 比較や論理演算は、算術より後ろ。代入（`=`）はいちばん後
- **迷ったら、かっこ `( )` で囲む**のがいちばん安全で、読みやすい
