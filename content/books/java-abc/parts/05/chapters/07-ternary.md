## 三項演算子 ― 条件で値を選ぶ

最後に学ぶのは、少し変わった形の演算子、**三項演算子**（さんこうえんざんし、Ternary Operator）です。
**条件によって、2つの値のどちらかを選ぶ**演算子で、**条件演算子**とも呼ばれます。

これまでの演算子は被演算子が2つでしたが、この演算子は3つの部分（条件・真のときの値・偽のときの値）からできているので、「三項」と呼ばれます。

---

## 書き方

三項演算子は、`?` と `:` を使って、次のように書きます。

```text
条件 ? 条件が true のときの値 : 条件が false のときの値
```

実際に使ってみましょう。点数が 60 以上なら「合格」、そうでなければ「不合格」を選ぶ例です。

```text
jshell> int score = 80;
score ==> 80
jshell> String result = score >= 60 ? "合格" : "不合格";
result ==> "合格"
```

`score >= 60` が `true`（80 は 60 以上）なので、`?` の直後の `"合格"` が選ばれ、`result` に入りました。
もし `score` が 50 だったら、`score >= 60` は `false` なので、`:` の後ろの `"不合格"` が選ばれます。

---

## if 文との関係

「条件によって変える」と聞くと、次の章で学ぶ **if 文**を思い浮かべるかもしれません。実際、上の例は if 文でも書けます。

三項演算子は、**2つの値のどちらかを選んで変数に入れる**ような、短い場面で便利です。
一方、条件によって**複数の処理をまとめて切り替えたい**ときは、if 文のほうが向いています。
if 文は、次の第6章でくわしく学びます。

> **補足: 使いすぎに注意**
>
> 三項演算子は短く書けて便利ですが、入れ子にしたり、長い式に使ったりすると、かえって読みにくくなります。
> 「短い値の選択」にとどめ、複雑になりそうなら if 文を使う ―― そう心がけると、読みやすいコードになります。

## まとめ

このセクションでは、三項演算子を学びました。

- 「2つの値のどちらかを選んで変数に入れる」ような、短い場面で便利
- 複雑な条件分岐は、次章で学ぶ **if 文**のほうが向いている
