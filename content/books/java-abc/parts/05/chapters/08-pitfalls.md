## よくあるつまずき

この章で学んだ演算子について、初心者がつまずきやすいポイントをまとめます。

---

## 1. 整数どうしのわり算

何度も登場する、いちばんの落とし穴です。

```text
jshell> int a = 7 / 2;
a ==> 3
```

整数どうしのわり算は、小数点以下が切り捨てられます。小数がほしいときは `7.0 / 2` のように、どちらかを小数にします。

---

## 2. `=` と `==` の取り違え

`=` は代入（入れる）、`==` は比較（等しいか調べる）です。
特に、条件を書くつもりで `=` を1つだけ書いてしまうミスに注意しましょう。

```java
int x = 5;            // 代入
boolean b = x == 5;   // 比較（こちらが「等しいか」の判定）
```

---

## 3. 文字列の比較に `==` を使う

文字列が同じ内容かを比べるときは、`==` ではなく `equals` を使います（第10章でくわしく学びます）。

```java
name.equals("太郎")   // 文字列の比較はこちら
```

---

## 4. かけ算の記号を省略する

数学では `2a` や `2(x + 1)` のように、かけ算の記号を省略できますが、**Java では省略できません**。
必ず `*` を書きます。

```java
int y = 2 * a;     // OK
// int y = 2a;     // エラー（* がない）
```

---

## 5. 優先順位を思い込む

`1 + 2 * 3` は `9` ではなく `7` です（`*` が先に計算されるため）。
複数の演算子が混ざって不安なときは、**かっこ `( )`** で囲んで、意図をはっきりさせましょう。

---

## 6. 小数の計算には、わずかな誤差が出る

コンピュータは小数を2進数で扱うため、`0.1` のような数をぴったり表せず、計算にわずかな誤差が出ることがあります。

```text
jshell> 0.1 + 0.2
$1 ==> 0.30000000000000004
```

`0.1 + 0.2` は、ちょうど `0.3` にはならず、末尾にわずかな誤差が出ました。
そのため、**小数どうしを `==` で比べるのは避けましょう**（`0.1 + 0.2 == 0.3` は、なんと `false` になってしまいます）。
お金など、正確さが必要な計算での対処は、のちの章で扱います。

---

## 7. 整数の上限を超える（オーバーフロー）

`int` で扱える整数の最大値は `2147483647`（約21億）です。これを超えると、最小値（マイナス）に一周してしまいます。

```text
jshell> Integer.MAX_VALUE + 1
$1 ==> -2147483648
```

大きな数を扱うときは、`int` の上限に注意し、必要なら `long` 型を使いましょう。

---

## まとめ

この節では、演算子のつまずきやすい点を確認しました。

- 整数どうしのわり算は切り捨て（小数がほしいなら一方を小数に）
- `=`（代入）と `==`（比較）を取り違えない
- 文字列の比較は `equals`
- かけ算の `*` は省略できない
- 優先順位に迷ったら、かっこで囲む
- 小数の計算には誤差が出る（小数どうしを `==` で比べない）
- `int` には上限がある（超えると**オーバーフロー**。必要なら `long`）

次は、この章で学んだ言葉を、用語集としてまとめます。
