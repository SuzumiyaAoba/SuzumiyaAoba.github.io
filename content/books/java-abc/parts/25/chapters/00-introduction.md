## はじめに ― この章で学ぶこと

プログラムで「日付」や「時刻」を扱う場面は、とてもたくさんあります。

- 「今日から3日後は、何月何日か」
- 「2つの日付の間は、何日あるか」
- 「誕生日から、年齢を計算する」
- 「日時を `2026年06月09日` のような形式で表示する」

この章では、Java で日付・時刻を安全に扱うための **java.time**（ジャバ・タイム）という標準ライブラリを学びます。

日付・時刻の処理は、一見かんたんそうに見えて、実は**落とし穴がたくさん**あります。
「月は0から数える」「うるう年は？」「タイムゾーンは？」「2月30日を作ったらどうなる？」――
こうした厄介な問題を、`java.time` は、きれいに解決してくれます。

---

## なぜ java.time を学ぶのか

実は、Java には古くから `Date` や `Calendar` という日付クラスがありました。
ですが、これらは**とても使いにくく、間違いを起こしやすい**ことで有名でした（くわしくは第1節）。

そこで **Java 8** で、まったく新しく設計し直された日付・時刻 API ―― それが **java.time** です。
古い `Date`・`Calendar` の問題点を、ことごとく解消した、モダンで安全な API です。

現在の Java では、日付・時刻を扱うなら、**`java.time` を使うのが当たり前**です。
この章で、その基本をしっかり押さえましょう。

第25章は、次の8つの節で構成されています。

| 節 | タイトル | 内容 |

後半（4〜5節）で、表示・読み取りの整形と、タイムゾーンを学びます。

## この章を読み終えると

第25章を読み終えるころには、次のことができるようになっています。

- `java.time` が、古い `Date`・`Calendar` より安全な理由を説明できる
- `LocalDate`・`LocalTime`・`LocalDateTime` で、日付・時刻を作って扱える
- 「3日後」「2つの日付の差」などの計算ができる
- `DateTimeFormatter` で、日付を好きな形式で表示・読み取りできる
- タイムゾーンの基本（`ZonedDateTime`・`Instant`）がわかる

> **補足: import について**
>
> `java.time` のクラスは、`java.time` パッケージ（と、整形用の `java.time.format` パッケージ）にあります。
> `.java` ファイルに書くときは、`import java.time.*;`、整形を使うなら `import java.time.format.DateTimeFormatter;` が必要です。
> この章のサンプルは jshell で試せます（jshell では `java.time` を自動で import します）。

それでは、最初の節「なぜ java.time か」から始めましょう。
