## よくあるつまずき

抽象クラスとインターフェースで、初心者がつまずきやすいポイントを、まとめて確認します。

---

## 1. 抽象クラス・インターフェースを new しようとする

抽象クラスもインターフェースも、**それ自体からは実体を作れません**。

```java
Animal a = new Animal();   // ✕ 抽象クラスは new できない
Shape s = new Shape();     // ✕ インターフェースは new できない
```

```text
エラー: Animalはabstractです。インスタンスを生成することはできません
```

これらは「未完成の設計図」なので、`new` できません。
実体を作りたいときは、それを**継承（実装）した、具体的なクラス**を `new` します（`new Dog()`、`new Circle()` など）。

---

## 2. 抽象メソッド・約束のメソッドを実装し忘れる

抽象クラスを継承、またはインターフェースを実装したら、中身のないメソッドを**すべて実装**しなければなりません。
1つでも忘れると、コンパイルエラーになります。

```java
class Circle implements Shape {
    double radius;
    // area() を実装し忘れた！
}
```

```text
エラー: Circleはabstractでなく、Shape内のabstractメソッドarea()をオーバーライドしません
```

「`area()` を実装していない」と教えてくれます。
このエラーは、「**約束を果たしていませんよ**」というサインです。約束されたメソッドを、すべて実装しましょう。
（どうしても全部は実装できない場合、そのクラス自身を `abstract` にする、という選択肢もあります。）

---

## 3. 実装するメソッドに public を付け忘れる

インターフェースのメソッドは `public` の約束です。
実装する側で `public` を付け忘れると、「公開範囲を狭めようとしている」とみなされ、エラーになります。

```java
class Circle implements Shape {
    double radius;
    @Override
    double area() { return radius * radius * 3.14; }   // ✕ public がない
}
```

```text
エラー: Circleのarea()はShapeのarea()を実装できません
  (public)より弱いアクセス権限を割り当てようとしました
```

「`(public)` より弱いアクセス権限を割り当てようとしました」とあります。
インターフェースの約束は `public` なのに、実装側がそれより狭い公開範囲にしようとしている、という意味です。
インターフェースを実装するメソッドには、必ず **`public`** を付けましょう。`@Override` を付けておけば、こうしたミスにも気づきやすくなります。

---

## 4. extends と implements を取り違える

- クラスを継承するのは **`extends`**
- インターフェースを実装するのは **`implements`**

```java
class Dog extends Animal { ... }        // クラスは extends
class Circle implements Shape { ... }   // インターフェースは implements
```

両方を書くときは、**`extends` が先、`implements` が後**の順で書きます。

```java
class Dog extends Animal implements Swimmer { ... }   // この順
```

「クラスは extends（1つ）、インターフェースは implements（複数）」と、セットで覚えておきましょう。

---

## 5. インターフェースのフィールドは「定数」になる

インターフェースにも変数を書けますが、それは自動的に **`public static final`**（公開された定数）として扱われます。
ふつうの「状態を持つフィールド」にはなりません。

```java
interface Config {
    int MAX = 100;   // 実は public static final（定数）。MAX = 200 のような変更は不可
}
```

インターフェースは「状態を持たない」のが基本、という性質の表れです。
「インターフェースに書く変数は、書き換えできない定数になる」と知っておきましょう（オブジェクトごとに変わる状態は、クラスのフィールドで持ちます）。

---

## まとめ

- 抽象クラス・インターフェースは **`new` できない**。実体は、実装した具体クラスで作る
- 中身のないメソッドは、**すべて実装**する（しないとエラー）
- インターフェースを実装するメソッドには、**`public`** を付ける
- クラスは **`extends`**、インターフェースは **`implements`**（`extends` が先）
- インターフェースのフィールドは、自動的に **`public static final`**（定数）になる

次は、この章で学んだ言葉を、用語集としてまとめます。
