## インターフェースとは ― できることの約束

抽象クラスは、「共通の土台」を提供しつつ、一部のメソッドの実装を子に任せるしくみでした。
この節で学ぶ **インターフェース**（Interface）は、もう一歩進んで、「**中身は一切持たず、『こういうことができる』という約束だけ**」を定めるしくみです。

---

## インターフェースは「純粋な約束」

第12章で、「外から使うための窓口」という意味で「インターフェース」という言葉が出てきました。
Java の `interface` は、まさにその「窓口の約束」を、言語のしくみとして表したものです。

インターフェースは、**`interface`** というキーワードで定義します。
例として、「面積を計算できる」という約束を表す `Shape`（図形）インターフェースを作ってみましょう。

```java
interface Shape {
    double area();   // 「面積を返す」という約束（中身はない）
}
```

`Shape` は、「`area()` というメソッドを持つ」という**約束だけ**を定めています。
`area()` には中身（`{ }`）がなく、抽象メソッドと同じように `;` で終わっています。
「図形なら、面積を求められるはずだ。でも、その計算方法は、図形の種類ごとに違う」 ―― それを表しているのです。

---

## クラスとの違い

インターフェースは、クラスと似ていますが、いくつか大きな違いがあります。

| | クラス（抽象クラスを含む） | インターフェース |
|---|---|---|
| キーワード | `class` | `interface` |
| 主な中身 | フィールド＋メソッド（実装あり） | メソッドの**約束**（基本は実装なし） |
| インスタンス（実体） | （抽象でなければ）作れる | 作れない |
| 状態（フィールド）を持つ | 持てる | 基本は持たない |

インターフェースの大きな特徴は、**「データ（状態）を持たず、できることの約束だけを定める」**ことです。
クラスが「何であるか（what it is）」を表すのに対し、インターフェースは「何ができるか（what it can do）」を表す、と考えるとよいでしょう。

> **補足: メソッドは自動的に public abstract**
>
> インターフェースに書いたメソッド（`double area();`）は、何も書かなくても、自動的に `public` で `abstract`（中身なし）として扱われます。
> そのため、`public abstract double area();` と書かなくても、`double area();` だけで同じ意味です。
> 「インターフェースのメソッドは、外に公開される約束である」ことが、言語のレベルで決まっているのです。

---

## インターフェースだけでは、何もできない

抽象クラスと同じく、インターフェースも、**それ自体からは実体を作れません**。

```java
Shape s = new Shape();   // ✕ インターフェースは new できない
```

```text
エラー: Shapeはabstractです。インスタンスを生成することはできません
```

`Shape` は「約束」にすぎないので、それ自体に実体はありません。
この約束を、具体的なクラス（円や長方形）が**実装する**ことで、初めて意味を持ちます。
その「実装する」しくみが、次の節で学ぶ `implements` です。

---

## なぜ約束だけを定めるのか

「中身がないのに、何の役に立つの？」と思うかもしれません。
インターフェースの価値は、**「中身を知らなくても、約束さえあれば、まとめて扱える」**ことにあります。

たとえば、「面積を求める」という約束（`Shape`）さえあれば、

- 円だろうが、長方形だろうが、三角形だろうが、
- その計算方法を知らなくても、
- 「`area()` を呼べば、面積が返ってくる」と**信頼して**、まとめて扱える

のです。
これは、第15章のポリモーフィズムを、「共通の親クラス」がなくても実現できる、ということでもあります。
円と長方形は、継承関係になくても、同じ `Shape` を実装していれば、`Shape` として一緒に扱えるのです。

---

## まとめ

- **インターフェース**は、`interface` で定義し、「**こういうメソッドを持つ**」という約束だけを定める
- メソッドは、基本的に中身を持たない（自動的に `public abstract`）
- クラスが「何であるか」を表すのに対し、インターフェースは「**何ができるか**」を表す
- インターフェース自体は **`new` できない**（実体を持たない）
- 約束だけを定めることで、「中身を知らなくても、まとめて扱える」ようになる

次の節では、`implements` を使って、インターフェースの約束を実際に**実装**します。
