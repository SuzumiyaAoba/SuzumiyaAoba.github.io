## 抽象クラスとは ― 実体を作れないクラス

まず、**抽象クラス**（Abstract Class）から学びます。
抽象クラスは、「**それ自体からは実体を作れない、共通の土台となるクラス**」です。

---

## 「動物」を直接作るのは、おかしい

第15章までの `Animal` クラスは、`new Animal()` で実体を作れました。
ですが、よく考えると、これは少し変です。

```java
Animal a = new Animal();   // 「動物そのもの」を作る？
```

世の中に、「犬でも猫でもない、ただの動物」は存在しません。
存在するのは、犬・猫・鳥…といった**具体的な動物**だけです。
「動物」は、それらをまとめて呼ぶための、**抽象的な概念**にすぎません。

このような「**直接、実体を作るべきではないクラス**」を、`new` できないように指定するのが、**抽象クラス**です。

---

## abstract で抽象クラスにする

クラスの宣言に **`abstract`**（アブストラクト）を付けると、抽象クラスになります。

```java
abstract class Animal {   // 抽象クラス
    String name;
    void eat() {
        IO.println(name + " はごはんを食べる");
    }
}
```

`abstract class Animal` は、「`Animal` は抽象クラスです＝直接は実体を作れません」という宣言です。
抽象クラスにしても、フィールド（`name`）やふつうのメソッド（`eat()`）は、これまでどおり書けます。
違うのは、「`new` で実体を作れない」という1点です。

---

## 抽象クラスは new できない

抽象クラスを `new` しようとすると、コンパイルエラーになります。

```java
abstract class Animal {
    String name;
    void eat() { IO.println(name + " はごはんを食べる"); }
}

public class Main {
    public static void main(String[] args) {
        Animal a = new Animal();   // 抽象クラスを new しようとする
    }
}
```

```text
エラー: Animalはabstractです。インスタンスを生成することはできません
        Animal a = new Animal();
                   ^
```

「`Animal` は `abstract`（抽象）なので、インスタンス（実体）を生成できません」と、Java が止めてくれます。
これにより、「ただの動物」のような、中途半端なオブジェクトが作られるのを防げます。

---

## 抽象クラスは「継承して使う」

「実体を作れないなら、何の役に立つの？」と思うかもしれません。
抽象クラスは、**継承して、具体的な子クラスを作るための土台**として使います。

```java
abstract class Animal {           // 土台（抽象クラス）
    String name;
    void eat() { IO.println(name + " はごはんを食べる"); }
}

class Dog extends Animal {        // 具体的な子クラス
    void cry() { IO.println(name + ": ワン"); }
}

public class Main {
    public static void main(String[] args) {
        Animal a = new Dog();     // ◯ 子クラス Dog なら作れる
        a.name = "ポチ";
        a.eat();
    }
}
```

```text
$ java Main.java
ポチ はごはんを食べる
```

`new Animal()` はできませんが、`new Dog()` はできます。
`Dog` は「具体的な動物」なので、実体を作れるのです。
そして、作った `Dog` を `Animal` 型で扱えるのは、第15章で学んだアップキャストのとおりです。

つまり、抽象クラス `Animal` は、

- 共通のフィールド・メソッド（`name`・`eat()`）を、子クラスに**受け継がせる**
- でも、`Animal` 自体は**実体化させない**（具体的な子だけを実体化させる）

という役割を持ちます。
「共通の土台は提供するが、それ単体では使わせない」 ―― これが抽象クラスです。

---

## まとめ

- **抽象クラス**は、`abstract` を付けたクラス。**それ自体からは `new` で実体を作れない**
- 「動物」のような、直接は実体化すべきでない**抽象的な概念**を表すのに使う
- 抽象クラスでも、フィールドやふつうのメソッドは書ける
- 抽象クラスは、**継承して、具体的な子クラスを作るための土台**として使う
- 子クラス（`Dog` など）は `new` できる
