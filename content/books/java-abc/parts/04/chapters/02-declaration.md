## 変数を宣言して使う ― 宣言・初期化・代入

前の節では、変数とは「名前をつけた箱」だ、というイメージをつかみました。
この節では、その箱を実際に**作って使う**ための書き方を、順を追って学びます。

変数を使うときの流れは、大きく次の3つです。

1. **宣言**：箱を用意する
2. **代入**：箱に値を入れる
3. 名前で取り出して使う

1つずつ見ていきましょう。

---

## 宣言と初期化 ― 箱を用意して、値を入れる

変数を使うには、まず「どんな箱を、どんな名前で用意するか」を Java に伝えます。これを**宣言**（Declaration）といいます。
そして、用意した箱に最初の値を入れることを、**初期化**（Initialization）といいます。

宣言と初期化は、次のように1行でまとめて書くのが基本です。

```text line-numbers=false
jshell> int age = 20;
age ==> 20
```

この1行を、部品に分けて読んでみましょう。

```text line-numbers=false
int     age     =     20     ;
 │       │      │      │      │
 │       │      │      │      └─ 文の終わりの印（セミコロン）
 │       │      │      └─ 箱に入れる値
 │       │      └─ 「右の値を左に入れる」記号
 │       └─ 変数の名前（自分で決める）
 └─ データ型（箱の種類。ここでは整数を表す int）
```

| 部品 | 名前 | 役割 |
|---|---|---|
| `int` | データ型 | 入れる値の種類（ここでは整数） |
| `age` | 変数名 | 箱につける名前。自分で決める |
| `=` | 代入演算子 | 右の値を、左の箱に入れる |
| `20` | 値 | 箱に入れる中身 |
| `;` | セミコロン | 文の終わり |

つまりこの1行は、「`int` 型の `age` という箱を用意して、そこに `20` を入れる」と読めます。

---

## `=` は「等しい」ではなく「入れる」

ここで、前の節でも触れた大切なポイントを、もう一度確認します。

プログラミングの `=` は、数学の「左と右が等しい」という意味ではありません。
**右の値を、左の箱に入れる**という意味の記号です。この `=` を、**代入演算子**（Assignment Operator）と呼びます。

![代入のイメージ。age = 20 では、右の値 20 が age という箱に入る。= は「等しい」ではなく「入れる」という意味。](./images/variable-assignment.svg)

この違いが、はっきりわかる例があります。

```text line-numbers=false
jshell> int age = 20;
age ==> 20
jshell> age = age + 1;
age ==> 21
```

`age = age + 1` を数学の等式として読むと、「age と age + 1 が等しい」となり、おかしな式に見えます。
ですがプログラミングでは、こう読みます。

> 右の `age + 1`（いまの age に 1 を足した値）を計算して、その結果を、左の `age` に入れ直す。

いまの `age` は `20` なので、`age + 1` は `21`。それを `age` に入れ直すので、`age` は `21` になります。
このように、**変数は自分自身を使って、自分を更新できる**のです。

---

## 宣言と代入を、分けて書くこともできる

宣言（箱を用意する）と、代入（値を入れる）は、分けて書くこともできます。

```text line-numbers=false
jshell> int count;
count ==> 0
jshell> count = 5;
count ==> 5
jshell> IO.println(count);
5
```

`int count;` で箱だけを用意し、あとから `count = 5;` で値を入れています。

> **補足: 「宣言だけ」のときの jshell の表示に注意**
>
> 上の例で、`int count;` と宣言しただけの段階で、jshell は `count ==> 0` と表示しました。
> これは、jshell が学習しやすいように、**仮の値（`int` なら 0）を入れておいてくれている**ためです。
>
> しかし、ファイルに書いたふつうのプログラム（メソッドの中）では、**値を入れる前の変数を使うとエラーになります**。
> 「箱を用意したら、使う前に必ず値を入れる」と覚えておきましょう。このことは、この章の第10節でもう一度くわしく扱います。

実際には、最初の例のように**宣言と初期化を1行で書く**ことが多くなります。
そのほうが「入れ忘れ」を防げて安全だからです。

---

## 値は、あとから何度でも書き換えられる

一度値を入れた変数は、あとから何度でも**書き換え**られます。新しい値を代入するだけです。

```text line-numbers=false
jshell> int score = 10;
score ==> 10
jshell> score = 50;
score ==> 50
jshell> score = 100;
score ==> 100
```

`score` の中身が、`10` → `50` → `100` と変わっていきました。
**代入するたびに、前の値は上書きされ、最後に入れた値だけが残ります**。

これが、変数が「変数（変わりうる数）」と呼ばれるゆえんです。
ゲームの得点や、買い物の合計金額のように、**変化していく値**を扱うときに、この書き換えが活躍します。

---

## まとめ

この節では、変数を宣言して使う書き方を学びました。

- 変数を使う流れは、**宣言**（箱を用意）→ **代入**（値を入れる）→ 名前で取り出して使う
- **宣言**は箱を用意すること、**初期化**は最初の値を入れること。`int age = 20;` のように1行で書くのが基本
- `=` は「等しい」ではなく「右の値を左の箱に入れる」という**代入演算子**
- `age = age + 1` のように、変数は自分自身を使って更新できる
- 値はあとから何度でも書き換えられる（最後に入れた値だけが残る）

次の節では、ここまで `int` を例にしてきた「箱の種類」＝**データ型**そのものに目を向け、なぜ型が必要なのかを考えます。
