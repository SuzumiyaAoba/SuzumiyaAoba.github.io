## はじめに ― この章で学ぶこと

第3章では、Hello, World! のプログラムを部品ごとに読み解きました。
クラスや `main`、`System.out.println`、文とセミコロン、コメント、エラーの読み方、そして Java 25 の短い書き方まで学びましたね。

ここまでのプログラムに共通していたのは、**表示する内容を、その場で直接書いていた**ことです。

```java
System.out.println("Hello, World!");
```

`"Hello, World!"` という文字も、`1 + 2` のような計算も、すべて「その場かぎり」でした。
表示したらおしまいで、プログラムはその値を**覚えてはいません**でした。

ですが、実際のプログラムを思い浮かべてみてください。

- 入力された名前を覚えておき、あいさつに使う
- 買い物の合計金額を計算し、その金額を消費税の計算にも使う
- ゲームの得点を覚えておき、敵を倒すたびに書き換える

このように、多くのプログラムは値を**覚えておき**、**何度も使い**、ときには**書き換え**ます。
そのための仕組みが、この章で学ぶ **変数**（Variable）と **データ型**（Data Type）です。

ひとつ、たとえてみましょう。
これまでのプログラムは、決まったセリフをそのまま繰り返す**オウム**のようなものでした。
変数を使えば、聞いた名前を**メモ帳に書き留めておき**、あとで「○○さん、こんにちは」と返せるようになります。
プログラムが、ぐっと「賢く」なるための第一歩です。

## この章で学ぶこと

第4章は、次の10のセクションで構成されています。

| 節 | タイトル | 内容 |
|---|---|---|
| 1 | 変数とは | 値に名前をつけて保存する仕組みと、その必要性 |
| 2 | 変数を宣言して使う | 宣言・初期化・代入。`=` は「等しい」ではなく「入れる」 |
| 3 | データ型とは | なぜ型が必要なのか。型が守ってくれること |
| 4 | 基本データ型 | 整数・小数・真偽・文字を表す型 |
| 5 | 文字列型 String | 文字のまとまりを扱う型 |
| 6 | リテラル | プログラムに値を直接書く書き方 |
| 7 | 型変換 | 型どうしの変換と、つまずきやすい落とし穴 |
| 8 | var | 型を Java に推論してもらう書き方 |
| 9 | 定数 final と命名 | 書き換えない値と、わかりやすい名前のつけ方 |
| 10 | よくあるつまずき | 変数・型でつまずきやすい点と、その対処 |

前半（1〜2節）で変数そのものを、中盤（3〜6節）でデータ型とさまざまな値の書き方を、後半（7〜10節）で型変換・便利な書き方・つまずき対策を学びます。

それぞれのセクションは、前の内容を受けて積み上がるように書かれています。順番に読み進めるのがおすすめです。

## この章から、jshell で試しながら学ぼう

この章からは、第2章で紹介した **jshell** が活躍します。

変数や計算は、実際に値を入れて動かしてみると、いちばんよくわかります。
そこで本書では、この章以降、試せるサンプルの多くを、**jshell にそのまま貼り付けて動かせる形**で紹介していきます。
ファイルを作ってコンパイル・実行する必要はありません。ターミナルで `jshell` を起動し、1行ずつ打ち込むだけです。

たとえば、`jshell` で次のように打つと、

```text
jshell> int price = 100;
price ==> 100
```

`price ==> 100` と表示されます。これは「変数 `price` に `100` が入りました」という、jshell からの返事です。

> **読み方のヒント: `jshell>` と `==>`**
>
> 本書のサンプルで、行のはじめに `jshell>` が付いているものは、`jshell` に打ち込む**入力**です。
> （`jshell>` 自体は打ちません。これは jshell が表示している「入力待ち」の印です。）
> その下に出てくる `==>` は、jshell が「こうなりました」と返してくれる**結果**を表します。
>
> もちろん、`jshell` を使わず、これまでどおりファイルに書いて実行してもかまいません。
> その場合は、確かめたい値を `IO.println(...)` で表示してみましょう。

## この章の読み方

これまでと同じく、最初からすべてを完璧に覚えようとしなくて大丈夫です。

特にこの章には、`int` や `double` のような「型の名前」がいくつも登場します。
ですが、これらを丸暗記する必要はありません。
「値にはいくつか種類があって、種類ごとに入れ物（型）が違うんだな」とつかめれば十分です。
実際に jshell で手を動かしているうちに、自然と身についていきます。

## この章を読み終えると

第4章を読み終えるころには、次のことができるようになっています。

- 変数とは何か、なぜ使うのかがわかる
- 変数を宣言し、値を入れ、あとから書き換えられる
- 整数・小数・真偽・文字・文字列など、値の種類（データ型）を使い分けられる
- 値をプログラムに書く方法（リテラル）がわかる
- 型どうしの変換と、整数の割り算などの落とし穴に気をつけられる
- `var` や `final` といった、より実践的な書き方を知る

これらは、次の第5章「演算子と式」をはじめ、この先のすべての章の土台になります。

それでは、最初のセクション「変数とは ― 値に名前をつけて保存する」から始めましょう。
