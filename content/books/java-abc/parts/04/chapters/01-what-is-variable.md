## 変数とは ― 値に名前をつけて保存する

前の節で触れたように、これまでのプログラムは、値をその場で書くだけで、覚えてはいませんでした。

たとえば、月額 100 円のサービスについて、「月額」と「年額（12 か月分）」を表示することを考えてみましょう。
変数を使わなければ、次のように書くことになります。

```text
jshell> IO.println(100);
100
jshell> IO.println(100 * 12);
1200
```

これでも動きますが、2 つの困りごとがあります。

- この `100` が**何の数字なのか**、コードを見ただけではわかりません。
- 料金が 120 円に変わったら、コード中の `100` を**すべて探して書き直す**必要があります（直し忘れも起こります）。

ここで役に立つのが、変数です。

---

## 変数は「名前をつけた入れ物」

**変数**とは、**値に名前をつけて保存しておく仕組み**です。

イメージは、中身を表すラベルを貼った**箱**です。
箱に値を入れておき、必要なときに、その名前で中身を取り出します。

![変数は、名前をつけた箱。monthlyPrice という名前の箱に、値 100 が入っている様子。](./images/variable-box.svg)

jshell で、実際に箱を 1 つ用意してみましょう。

```text
jshell> int monthlyPrice = 100;
monthlyPrice ==> 100
```

これで、`monthlyPrice`（月額料金）という名前の箱に、`100` という値が入りました。
`monthlyPrice ==> 100` は、「`monthlyPrice` に `100` が入りました」という jshell からの返事です。

あとは、この名前を書くだけで、中の値を何度でも取り出せます。

```text
jshell> IO.println(monthlyPrice);
100
jshell> IO.println(monthlyPrice * 12);
1200
```

`monthlyPrice` と書いた場所が、すべて中身の `100` として扱われていますね。
（`monthlyPrice * 12` は「100 × 12」と計算され、年額の `1200` が表示されます。）

---

## 変数を使うと、何がうれしいのか

先ほどの「変数なし」と比べると、変数を使う利点が見えてきます。

1. **値を覚えて、何度でも使える**
   一度入れておけば、`monthlyPrice` と書くだけで、同じ値を繰り返し使えます。
2. **名前で意味が伝わる**
   `100` という数字だけでは何の値かわかりませんが、`monthlyPrice` なら「月額料金のことだな」と一目でわかります。コードが読みやすくなります。
3. **後から中身を変えられる**
   料金が変わっても、変数に入れる値を 1 か所変えるだけで済みます。

3 つ目を、実際に試してみましょう。変数の中身は、あとから**書き換える**ことができます。

```text
jshell> monthlyPrice = 200;
monthlyPrice ==> 200
jshell> IO.println(monthlyPrice * 12);
2400
```

`monthlyPrice` の中身を `200` に書き換えたので、年額も自動的に `2400` に変わりました。
このように、**箱の中身は入れ替えられます**。だからこそ「変数（変わりうる数）」と呼ばれるのです。

> **補足: `=` は「等しい」ではなく「入れる」**
>
> `monthlyPrice = 200` の `=` は、数学のように「左と右が等しい」という意味ではありません。
> プログラミングの `=` は、**右の値を、左の箱に入れる**という意味の記号です。
> ですから `monthlyPrice = 200` は、「monthlyPrice に 200 を入れる」と読みます。
> この `=` の読み方は、次の節でもう少しくわしく見ていきます。

## 数値だけでなく、文字も入れられる

変数に入れられるのは、数値だけではありません。文字（文字列）も入れられます。

```text
jshell> String name = "太郎";
name ==> "太郎"
jshell> IO.println(name);
太郎
```

`name` という箱に、`"太郎"` という文字を入れました。
（`name ==> "太郎"` のように、文字のときは前後に `"` が付いて表示されます。これは「これは文字（文字列）ですよ」と jshell が示しているためで、実際に表示すると `太郎` とだけ出ます。）

ここで注目してほしいのは、数値を入れるときは `int`、文字を入れるときは `String` と、**最初の言葉が違う**ことです。
これは、**入れる値の種類によって、箱の種類が違う**ためです。
この「箱の種類」のことを、**データ型**（または単に**型**）と呼びます。
データ型については、この章の第 3 節でくわしく学びます。

> **コラム: 数学の「変数」とは少し違う**
>
> 「変数」という言葉は、数学にも出てきます。`x + 3 = 5` の `x` のような、「まだわからない数」のことです。
> プログラミングの変数は、これとは少し違います。プログラミングの変数は「まだわからない数」ではなく、**値を入れておく、名前のついた箱**です。
> そして、その中身は後から自由に入れ替えられます。
> 同じ「変数」という言葉でも、数学とプログラミングではイメージが異なる ―― そう知っておくと、混乱しにくくなります。

## まとめ

このセクションでは、変数とは何かを学びました。

- **変数**とは、値に名前をつけて保存しておく仕組み。イメージは「名前（ラベル）をつけた箱」
- 変数を使うと、(1) 値を覚えて何度も使える、(2) 名前で意味が伝わる、(3) 後から中身を書き換えられる
- 変数の中身は入れ替えられる。だから「変数」と呼ばれる
- `=` は「等しい」ではなく、「右の値を左の箱に入れる」という意味
- 入れる値の種類（数値・文字など）によって、箱の種類＝**データ型**が違う

次の節では、変数を実際に使うための文法 ―― **宣言・初期化・代入**を、順を追って学びます。
