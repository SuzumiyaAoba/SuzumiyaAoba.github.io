## よくあるつまずき

この章では、変数とデータ型について学んできました。
最後に、初心者がとくにつまずきやすいポイントを、まとめて確認しておきましょう。
ここを押さえておくと、エラーが出ても落ち着いて対処できます。

---

## 1. 値を入れる前に、変数を使ってしまう

変数は、「箱を用意する（宣言）」だけでは、まだ中身が空です。
ファイルに書いたふつうのプログラム（メソッドの中）では、**値を入れる前の変数を使うとエラー**になります。

```text line-numbers=false
エラー: 変数xは初期化されていない可能性があります
        System.out.println(x);
                           ^
```

第2節でも触れたように、jshell では仮の値（`int` なら `0`）が入るため、このエラーには気づきにくいので注意が必要です。
**「箱を用意したら、使う前に必ず値を入れる」** ―― これを習慣にしましょう。宣言と初期化を1行で書く（`int x = 0;`）のが、いちばん安全です。

---

## 2. 整数どうしの割り算で、小数が消える

第7節で学んだ、いちばんはまりやすい落とし穴です。

```text line-numbers=false
jshell> int a = 7 / 2;
a ==> 3
```

`7 / 2` は `3.5` ではなく `3` になります。整数どうしの割り算は、小数点以下が切り捨てられるためです。
小数の結果がほしいときは、`7.0 / 2` のように、どちらかを小数にするのを忘れないようにしましょう。

---

## 3. 型に合わない値を入れてしまう

第3節で見たように、型に合わない値を入れるとエラーになります。

```text line-numbers=false
jshell> int bad = "abc";
|  エラー:
|  不適合な型: java.lang.Stringをintに変換できません:
```

`不適合な型`（incompatible types）というエラーが出たら、「箱の型」と「入れようとした値の型」が合っていないということです。
落ち着いて、型をそろえましょう。

---

## 4. 大文字・小文字を打ち間違える

Java は、大文字と小文字をきびしく区別します。`score` と `Score` は、別の変数として扱われます。

```text line-numbers=false
jshell> int score = 5;
score ==> 5
jshell> IO.println(Score);
|  エラー:
|  シンボルを見つけられません
|    シンボル:   変数 Score
```

`シンボルを見つけられません`（cannot find symbol）は、第3章でも出てきたエラーです。
「そんな名前の変数はありません」という意味なので、**つづりや大文字・小文字の打ち間違い**を疑いましょう。

---

## 5. 記号を全角で打ってしまう

第2章・第3章でも注意したとおり、プログラムの記号（`=`・`;`・`"` など）は、すべて**半角**で打ちます。
日本語入力のまま打つと、見た目が似ていても別の文字として扱われ、エラーになります。
エラーが出たら、「全角の記号やスペースが混ざっていないか」も確認してみましょう。

---

## 6. 文字列の比較に、`==` を使ってしまう（予告）

最後に、これから先で出てくる注意点を、ひとつ予告しておきます。

2つの文字列が同じ内容かどうかを比べるとき、数値と同じ感覚で `==` を使うと、**正しく比較できないことがあります**。
文字列の比較には、`==` ではなく `equals` という専用の方法を使います。

```java
name.equals("太郎")   // 文字列どうしの比較は、こちらを使う
```

くわしいしくみと理由は、第10章「文字列の操作」や、第2部であらためて学びます。
いまは、**「文字列が同じか比べるときは、`==` ではなく `equals`」** と、頭の片隅に置いておいてください。

---

## まとめ

この節では、変数とデータ型でつまずきやすい点を確認しました。

- 値を入れる前の変数は使えない（jshell では気づきにくい）。宣言と初期化は1行で書くと安全
- 整数どうしの割り算は、小数点以下が切り捨てられる（`7 / 2` は `3`）
- 型に合わない値を入れると、`不適合な型` のエラーになる
- 大文字・小文字は区別される。打ち間違いは `シンボルを見つけられません`
- 記号は、すべて半角で打つ
- 文字列が同じか比べるときは、`==` ではなく `equals`（くわしくは後の章で）

次は、この章で学んだ言葉を、用語集としてまとめます。
