## リテラル ― プログラムに値を直接書く

ここまで、`20` や `3.14`、`"Hello"`、`'A'`、`true` といった値を、コードの中に書いてきました。
このように、**ソースコードに直接書いた値そのもの**を、**リテラル**（Literal）と呼びます。

たとえば `int age = 20;` の `20` がリテラルです。第3章のエラーの節でも、`"Hello"` を「文字列リテラル」と呼んでいましたね。

リテラルは、値の種類ごとに書き方が決まっています。このセクションで、整理しておきましょう。

---

## 整数リテラル

整数は、そのまま数字で書きます。

```text
jshell> int million = 1_000_000;
million ==> 1000000
```

ここで使った **アンダースコア `_`** に注目してください。
大きな数は桁が読みづらいので、`_` で桁を区切ることができます。
`1_000_000` と `1000000` は**まったく同じ値**で、`_` は見やすさのためのものです（値には影響しません）。

ふだんの「1,000,000」というカンマ区切りのかわり、と考えるとわかりやすいでしょう。

---

## 小数リテラル

小数点を含む数は、そのまま書きます。小数リテラルは `double` 型になります。

```text
jshell> double pi = 3.14;
pi ==> 3.14
```

---

## 文字リテラルと文字列リテラル

文字と文字列は、囲む記号が違います。第4・第5節でも見たとおりです。

```text
jshell> char grade = 'A';
grade ==> 'A'
jshell> String letter = "A";
letter ==> "A"
```

- `'A'` … **文字リテラル**。シングルクォート `'` で囲んだ1文字（`char`）
- `"A"` … **文字列リテラル**。ダブルクォート `"` で囲んだ文字のまとまり（`String`）

---

## 真偽リテラル

真偽を表すリテラルは、`true` と `false` の2つだけです。

```text
jshell> boolean yes = true;
yes ==> true
```

`true` と `false` は、`"true"` のようにクォートで囲みません。これらは Java があらかじめ用意した、特別な値だからです。

---

## リテラルの一覧

リテラルの書き方を、表にまとめます。

| 種類 | 書き方 | 例 | 型 |
|---|---|---|---|
| 整数リテラル | そのまま数字（`_` で区切れる） | `100`, `-5`, `1_000_000` | `int` |
| 小数リテラル | 小数点を含む数 | `3.14`, `0.5` | `double` |
| 文字リテラル | `'` で囲んだ1文字 | `'A'`, `'あ'` | `char` |
| 文字列リテラル | `"` で囲んだ文字 | `"Hello"`, `""` | `String` |
| 真偽リテラル | `true` または `false` | `true`, `false` | `boolean` |

> **補足: 大きな整数や小数には、末尾の印を付けることもある**
>
> 第4節で見たように、`int` に収まらない大きな整数のリテラルには、末尾に `L` を付けて `10000000000L` のように書きます（`long` リテラル）。
> 同じように、`float` の小数リテラルには末尾に `f` を付けて `3.14f` と書きます。
> いまは「ふつうの整数は `int`、ふつうの小数は `double`」と覚えておけば十分です。

---

## まとめ

このセクションでは、リテラルを学びました。

- **リテラル**とは、ソースコードに直接書いた値そのもの（`20`、`3.14`、`"Hello"` など）
- 整数リテラルは `_` で桁を区切れる（`1_000_000`。値には影響しない）
- 小数リテラルは `double`、文字リテラル（`'A'`）は `char`、文字列リテラル（`"A"`）は `String`
- 真偽リテラルは `true` と `false` の2つだけ（クォートで囲まない）

次の節では、型の違う値どうしを扱うときに起こる**型変換**と、初心者がつまずきやすい「整数の割り算」などの落とし穴を学びます。
