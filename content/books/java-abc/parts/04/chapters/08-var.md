## var ― 型を推論してもらう

ここまで、変数を宣言するときは、`int age = 20;` のように、必ず先頭に型（`int` など）を書いてきました。

ですが、よく見ると、右側に `20` という整数が書いてあるので、型が `int` であることは**見ればわかります**。
それなら、わざわざ `int` と書かなくても、Java が判断してくれてもよさそうです。

そこで使えるのが、**`var`**（バー）です。
型のかわりに `var` と書くと、**Java が右側の値から型を判断してくれます**。
このしくみを、**型推論**（Type Inference）と呼びます[^jep286-var]。

---

## var の使い方

実際に、`var` を使って変数を宣言してみましょう。

```text
jshell> var v1 = 20;
v1 ==> 20
jshell> var v2 = "太郎";
v2 ==> "太郎"
jshell> var v3 = 100.0;
v3 ==> 100.0
```

`int` や `String` と書かなくても、ちゃんと値が入りました。

ここで大切なのは、**`var` を使っても、型がなくなるわけではない**ということです。
型を書く手間が省けるだけで、変数にはちゃんと型があります。Java が、右側の値を見て型を決めてくれているのです。

それを確かめてみましょう。jshell の **`/vars`** コマンドを使うと、いま定義されている変数を、型つきで一覧表示できます。

```text
jshell> /vars
|    int v1 = 20
|    String v2 = "太郎"
|    double v3 = 100.0
```

`v1` は `int`、`v2` は `String`、`v3` は `double` になっていますね。
つまり、`var v1 = 20;` は `int v1 = 20;` と**まったく同じ意味**なのです。

> **補足: `/vars` は jshell のコマンド**
>
> `/vars` は、Java の文ではなく、jshell に用意された便利なコマンドです（`/exit` と同じ仲間です）。
> いま箱の中に何が入っているかを確認したいときに使えます。Java のプログラム本体には書きません。

## var が便利な場面

いまの例（`var v1 = 20;`）だけを見ると、「`int` と書くのと手間はあまり変わらない」と感じるかもしれません。

`var` の便利さがはっきりするのは、**型の名前が長くなったとき**です。
本書の後半では、`int` のように短くない、長い名前の型も登場します。
そうした型を何度も書くのは大変ですが、`var` を使えば、その手間を省いて、コードをすっきりさせられます。

いまの段階では、「型を書かずにすませる便利な書き方がある」と知っておけば十分です。
実際に長い型名が出てきたときに、`var` のありがたみを実感できるはずです。

---

## var の注意点

便利な `var` ですが、いくつか気をつける点があります。

### 1. 初期値が必要

`var` は、右側の値を見て型を決めます。
そのため、**値を入れずに `var` だけで宣言することはできません**。

```text
jshell> var y;
|  エラー:
|  ローカル変数yの型を推論できません
|    (初期化子なしで変数に'var'を使用することはできません)
```

`var y;` だけでは、右側に手がかりがなく、型を判断できないため、エラーになります。
`var` を使うときは、必ず `var y = 10;` のように、最初の値もいっしょに書きます。

### 2. 使えるのは、メソッドの中だけ

`var` が使えるのは、メソッドの中で使う変数（ローカル変数）です。
（メソッドやローカル変数については、第9章でくわしく学びます。いまは「使える場所に決まりがある」とだけ知っておけば大丈夫です。なお、jshell に打ち込んだ各行は、メソッドの中のように扱われます。だから、先ほどのように `var` をそのまま試せたのです。）

### 3. 使いすぎると、かえって読みにくいことも

`var` を使うと型が画面に出てこないため、使いすぎると「この変数は何の型だっけ？」とわかりにくくなることがあります。
`var price = 100;` のように、右側を見れば型が一目でわかるときに使うと、読みやすさを保てます。

---

## まとめ

この節では、型推論の `var` を学びました。

- **`var`** と書くと、Java が右側の値から型を判断してくれる（**型推論**）
- `var` を使っても型がなくなるわけではない。`var v1 = 20;` は `int v1 = 20;` と同じ
- jshell の **`/vars`** コマンドで、変数を型つきで確認できる
- `var` は型名が長いときに特に便利
- 注意点: (1) 初期値が必要、(2) メソッドの中でしか使えない、(3) 使いすぎると読みにくくなる

次の節では、書き換えたくない値を守る **`final`** と、変数にわかりやすい名前をつけるためのルールを学びます。

[^jep286-var]: JEP 286: Local-Variable Type Inference, <https://openjdk.org/jeps/286>。Java 10（2018年3月）で導入された機能。ローカル変数宣言にのみ `var` を使え、フィールド・メソッド戻り値・引数には使えない。詳細は *JLS §14.4*（<https://docs.oracle.com/javase/specs/jls/se25/html/jls-14.html#jls-14.4>）を参照。
