## 用語集 ― この章で学んだ言葉

第12章では、オブジェクトを安全に使うための「カプセル化」と、それを支えるしくみを学びました。
節ごとに、言葉を整理しておきます。

---

## カプセル化（第1節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| カプセル化 | Encapsulation | データ（フィールド）を隠し、決められたメソッドを通してのみ操作させる考え方 |
| インターフェース（窓口の意味） | Interface | オブジェクトを外から使うための、公開された操作（メソッド）の集まり |
| 実装 | Implementation | 外からは見えない、内部の作り（フィールドの持ち方や処理の中身） |

---

## アクセス修飾子（第2節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| アクセス修飾子 | Access Modifier | フィールド・メソッド・クラスの「公開範囲」を決める印 |
| `public` | パブリック | どこからでもアクセスできる（全公開） |
| `private` | プライベート | 同じクラスの中だけアクセスできる（最も狭い） |
| `protected` | プロテクテッド | 同じパッケージ＋継承した子クラスからアクセスできる（第14章で詳説） |
| パッケージプライベート | package-private | 修飾子なし。同じパッケージの中だけアクセスできる |

---

## ゲッターとセッター（第3〜4節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| ゲッター | getter | フィールドの値を読み取るメソッド（`getXxx()`） |
| セッター | setter | フィールドに値を設定するメソッド（`setXxx(値)`）。設定前に値を検証できる |
| 検証（バリデーション） | Validation | 値が正しいかを確かめること。セッターやメソッドの中で行う |
| 読み取り専用 | Read-only | ゲッターだけ用意し、セッターを作らないことで実現する |

---

## パッケージ（第5節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| パッケージ | Package | 関連するクラスをまとめて整理するしくみ（フォルダのようなもの） |
| `package` 文 | ― | ファイル先頭で、所属パッケージを宣言する文 |
| `import` | インポート | 別パッケージのクラスを、短い名前で使えるようにする宣言 |
| 完全修飾名 | Fully Qualified Name | パッケージ名を含む正式なクラス名（`java.util.Arrays` など） |
| `java.lang` | ― | `String`・`System`・`IO` などがある特別なパッケージ。自動で import される |

---

## おわりに

この章では、オブジェクトを「安全な部品」にするカプセル化を学びました。

- フィールドは `private` で**隠し**、メソッドを**入口**にする
- セッターで値を**検証**すれば、ありえない状態を防げる
- 公開範囲は、`public`／`private` を基本に、目的に応じて選ぶ
- パッケージでクラスを整理し、`import` で別パッケージのクラスを使う

ここまでで作ってきたオブジェクトには、まだ1つ不便な点があります。
`new` した直後は、フィールドが初期値（`0` や `null`）のままで、毎回セッターで値を入れ直す必要がありました。

```java
Person p = new Person();
p.setName("佐藤");   // 作ってから、別途 設定
p.setAge(25);
```

次の第13章「コンストラクタと初期化」では、**オブジェクトを作るのと同時に、初期値を設定する**しくみを学びます。
`new Person("佐藤", 25)` のように、最初から正しい状態のオブジェクトを作れるようになります。
カプセル化と組み合わせれば、「**作った瞬間から、つねに正しいオブジェクト**」を実現できます。
