## アクセス修飾子 ― 公開範囲を決める

カプセル化を実現する道具が、**アクセス修飾子**（Access Modifier）です。
フィールドやメソッドの前に付けて、「**どこから使ってよいか**（公開範囲）」を決めます。

第3章で `public` を「どこからでも使える印」として紹介しましたね。
その `public` も、このアクセス修飾子の1つです。

---

## 4つの公開範囲

Java のアクセス修飾子には、次の4つの段階があります。
上ほど公開範囲が広く、下ほど狭くなります。

| 修飾子 | 公開範囲 | ひとことで |
|---|---|---|
| `public` | どこからでも | 全公開 |
| `protected` | 同じパッケージ＋継承した子クラス | 身内＋子ども |
| （なし） | 同じパッケージの中だけ | 同じグループ内 |
| `private` | 同じクラスの中だけ | 自分だけ |

- **`public`**（パブリック） … どこからでもアクセスできます。広く使ってほしいメソッドなどに付けます。
- **`private`**（プライベート） … **そのクラスの中からしか**アクセスできません。隠したいフィールドに付けます。
- **`protected`**（プロテクテッド） … 同じパッケージと、継承した子クラスからアクセスできます。継承と関係するので、第14章で改めて触れます。
- **何も付けない**（修飾子なし） … 同じパッケージ（次の節以降で学ぶ、クラスのグループ）の中だけでアクセスできます。**パッケージプライベート**（package-private）と呼ばれます。

初心者のうちは、まず両端の **`public` と `private` の2つ**をしっかり押さえれば十分です。
「**外に公開するものは `public`、隠すものは `private`**」が基本になります。

## private の効果を確かめる

`private` を付けたフィールドに、外からアクセスするとどうなるでしょうか。
実際に試してみましょう。

> **この章からのプログラムの書き方**
>
> 第11章では、1つのクラスを試すだけだったので、`void main()` だけの短い形で書いていました。
> 第12章からは、「**あるクラスを、別のクラスから使う**」場面（カプセル化や、のちの継承など）を扱います。
> そこで、1つのファイルに**部品となるクラス**（`BankAccount` など）と、それを使う **`Main` クラス**を並べ、第3章で学んだ `public class Main { public static void main(String[] args) { ... } }` の形で示します。
> ファイル名は、`public` を付けた `Main` に合わせて **`Main.java`** とし、`java Main.java` で実行します。
> （出力には、引き続き短い `IO.println` を使います。`IO.println` は、`public static void main` の中でもそのまま使えます。）

```java
class BankAccount {
    private int balance;   // private にして隠す
}

public class Main {
    public static void main(String[] args) {
        BankAccount acc = new BankAccount();
        acc.balance = 1000;    // ← クラスの外（Main）から private にアクセス
    }
}
```

これを `Main.java` という名前で保存してコンパイル（実行）すると、エラーになります。

```text line-numbers=false
エラー: balanceはBankAccountでprivateアクセスされます
        acc.balance = 1000;
           ^
```

「`balance` は `private` なので、外からはアクセスできません」と、Java が**コンパイルの段階で**止めてくれました。
これこそが、`private` の力です。
「外から直接さわってはいけない」というルールを、Java が強制的に守らせてくれるのです。

> **補足: エラーは「失敗」ではなく「守り」**
>
> このエラーは、私たちのうっかりミスを防いでくれている、ありがたいものです。
> 「`private` にしたフィールドを、うっかり外からいじろうとしていますよ」と教えてくれているのです。
> 正しい使い方（次の節で学ぶメソッド経由）に直せば、エラーは消えます。

## 同じクラスの中からは、private でもさわれる

`private` は「**そのクラスの中だけ**」アクセスできる、という意味でした。
裏を返せば、**同じクラスの中に書いたメソッドからは、`private` フィールドに自由にアクセスできます**。

```java
class BankAccount {
    private int balance;

    void deposit(int amount) {
        balance += amount;   // 同じクラスの中なので、private でもアクセスできる
    }
}
```

`deposit` は `BankAccount` クラスの中にあるので、`private` な `balance` を読み書きできます。
つまり、「**外からは禁止、中（自分のメソッド）からは自由**」というのが `private` です。
これにより、「フィールドは隠しつつ、それを扱うメソッドは中に用意する」という、カプセル化の形が作れます。

---

## クラスにも付けられる

アクセス修飾子は、フィールドやメソッドだけでなく、**クラス自体**にも付けられます。
第3章で `public class Main` と書いたのが、その例です。

- `public class` … どこからでも使えるクラス
- （修飾子なしの）`class` … 同じパッケージの中だけで使えるクラス

第11章でクラスに `public` を付けず `class Car` と書いていたのは、「1つのファイルの中だけで使うので、パッケージ内で見えれば十分」だったからです。
他のファイルからも広く使うクラスには、`public` を付けます（パッケージについては第6節で学びます）。

---

## まとめ

- **アクセス修飾子**は、フィールド・メソッド・クラスの「公開範囲」を決める印
- 範囲は4段階：`public`（全公開）＞ `protected`（身内＋子）＞ なし（同じパッケージ）＞ `private`（自分だけ）
- まずは **`public`（公開）と `private`（隠す）** の2つを押さえる
- `private` フィールドに**外から**アクセスすると**コンパイルエラー**になる
- ただし、**同じクラスの中のメソッドからは** `private` フィールドにアクセスできる

次の節では、フィールドを `private` にして実際に隠し、安全なクラスを作ります。
