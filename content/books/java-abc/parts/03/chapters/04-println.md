## 画面に文字を表示する ― System.out.println

ここまでで、「クラス（外枠）」と「入口（main）」という枠組みを読み解いてきました。
この節では、いよいよプログラムの**やりたいこと本体**である、次の1行を読み解きます。

```java
System.out.println("Hello, World!");
```

この1行こそが、画面に `Hello, World!` と表示している、命令の正体です[^java-system-out]。
第1章や第2章でも何度か登場した、おなじみの命令ですね。

---

## 左から読み解く

`System.out.println("Hello, World!");` を、ピリオド（`.`）で区切って、左から読んでみましょう。

```java
System . out . println ( "Hello, World!" ) ;
   │     │      │           │              │
   │     │      │           │              └─ 文の終わりの印（セミコロン）
   │     │      │           └─ 表示したい内容（文字列）
   │     │      └─ 「1行表示する」という命令（メソッド）
   │     └─ 出力先（標準出力＝ふつうは画面）
   └─ コンピュータの標準機能をまとめたもの
```

ピリオド（`.`）は、「〜の」と読むとイメージしやすくなります。
すると、この命令は次のように読めます。

> **System（システム）の out（出力先）に、println（1行表示）しなさい。表示する内容は "Hello, World!" です。**

一つずつ見ていきましょう。

| 部分 | 読み | 役割 |
|---|---|---|
| `System` | システム | コンピュータの標準的な機能をまとめたもの |
| `out` | アウト | 出力先（**標準出力**。ふつうは画面） |
| `println` | プリントライン | 受け取った内容を**1行**表示する命令（`ln` = line） |
| `("…")` | ― | 表示したい内容を渡す部分（かっこの中が引数） |
| `;` | セミコロン | 文の終わりの印 |

つまり、`System.out` で「コンピュータの、画面への出力口」を指し示し、その出力口に対して `println(…)` で「これを1行表示して」とお願いしている、というわけです。

> **補足: いまは「決まった言い回し」として覚えてよい**
>
> `System` が何で、`out` がどういう仕組みなのか ―― その正体は、クラスやオブジェクトを学ぶ第2部で、よりはっきりします。
> いまは、`System.out.println(…)` を「**画面に1行表示するための、決まった言い回し**」として、丸ごと覚えてしまって大丈夫です。
> よく使う命令なので、写経しているうちに自然と手が覚えていきます。

## かっこの中が「表示する内容」

`println` のうしろのかっこ `( )` の中に書いたものが、実際に表示される内容になります。
このように、メソッドに渡す値のことを、**引数**（ひきすう、Argument）と呼びます。

```java
System.out.println("Hello, World!");
//                  └──── ここに書いた内容が表示される ────┘
```

ためしに、かっこの中を書き換えてみましょう。

```java
System.out.println("はじめまして");
```

実行すると、次のように表示されます。

```text
はじめまして
```

第2章でも体験したとおり、**かっこの中を変えれば、表示される内容が変わる**のです。

---

## 文字は `" "`（ダブルクォート）で囲む

ここで一つ、大切なルールがあります。
**文字（文字列）を表示したいときは、その文字を `"`（ダブルクォート）で囲む**、という決まりです。

```java
System.out.println("Hello, World!");   // "..." で囲まれた文字がそのまま表示される
```

`"` で囲まれた文字のまとまりを、**文字列**（String、ストリング）と呼びます。
`"` から `"` までが「ひとかたまりの文字」だと、Java に伝えているのです。

もし `"` で囲まずに書くと、Java はその文字を「命令」や「名前」だと勘違いして、エラーになります。

```java
System.out.println(Hello, World!);   // ← " がないのでエラーになる
```

> **注意: `"` は半角で、開いたら必ず閉じる**
>
> `"` は、第2章でも注意したとおり**半角**で入力します（全角の「”」ではありません）。
> また、`"` は必ず**開いたら閉じる**こと。閉じ忘れると、「文字列が終わっていない」というエラーになります（第7節で紹介します）。

---

## `println` と `print` ―― 改行の違い

`println` の末尾の `ln` は、**line（行）** を表しています。
`println` は、内容を表示したあと、**自動的に改行**します。

よく似た命令に `print`（プリント）があります。
こちらは、表示したあとに**改行しません**。

この違いを、実際に比べてみましょう。

まず `println` を2回続けた場合です。

```java
System.out.println("おはよう");
System.out.println("こんにちは");
```

それぞれの表示のあとに改行が入るので、2行に分かれて表示されます。

```text
おはよう
こんにちは
```

次に `print` を2回続けた場合です。

```java
System.out.print("おはよう");
System.out.print("こんにちは");
```

改行が入らないので、2つの文字がそのままつながって表示されます。

```text
おはようこんにちは
```

| 命令 | 表示後の改行 | 主な使いどころ |
|---|---|---|
| `println` | する | 1行ずつ表示したいとき（基本はこちら） |
| `print` | しない | 続けて表示したいとき |

ふだんは、1行ずつ表示できる `println` を使うことが多くなります。
「改行までしてくれるのが `println`、しないのが `print`」と覚えておきましょう。

## 文字だけでなく、数値も表示できる

`println` で表示できるのは、文字だけではありません。
かっこの中に**数値**を書くと、その数値が表示されます。

数値を表示するときは、`"` で囲みません。

```java
System.out.println(100);       // 数値の 100
System.out.println(1 + 2);     // 計算した結果の 3
```

実行すると、次のように表示されます。

```text
100
3
```

`1 + 2` のように計算式を書くと、**計算した結果**が表示される点に注目してください。
ここでは詳しく立ち入りませんが、計算（演算）については第5章「演算子と式」で学びます。

ここで、`"` で囲む場合と囲まない場合の違いを、はっきりさせておきましょう。

```java
System.out.println("1 + 2");   // ← "..." で囲むと、文字としてそのまま表示
System.out.println(1 + 2);     // ← 囲まないと、計算した結果が表示
```

```text
1 + 2
3
```

- `"1 + 2"` … `"` で囲んだので、**文字（文字列）**としてそのまま表示される
- `1 + 2` … 囲まないので、**計算式**とみなされ、結果の `3` が表示される

`"` で囲むかどうかで、扱いがまったく変わります。
この違いは、第4章「変数とデータ型」で、文字列と数値の違いを学ぶと、よりはっきり理解できます。

> **補足: `'`（シングルクォート）との違い**
>
> 文字を囲む記号には、`"`（ダブルクォート）のほかに `'`（シングルクォート）もあります。
> 似ていますが、役割が違います[^jls-literals]。
>
> - `"…"` … **文字列**（0文字でも、何文字でもよい文字のまとまり）
> - `'…'` … **1文字だけ**を表す（`char` という型。第4章で学ぶ）
>
> 画面に文章を表示したいときは、`"`（ダブルクォート）を使う、と覚えておきましょう。

<Column title="覚えられなくても大丈夫 ― プロも「調べながら」書いている">

`System.out.println` のような書き方を見て、「これを全部暗記しないといけないの？」と不安に思うかもしれません。
でも、安心してください。**経験を積んだプログラマーでも、すべてを頭の中に覚えているわけではありません。**
よく使うものは自然と手が覚えますが、それ以外は、必要になるたびに公式ドキュメントで調べたり、インターネットで検索したりしながら書いています。

さらに、第2章で入れた IDE には、`System.` まで打つと続きの候補を出してくれる**入力補完**があります。これは、プロが毎日使っている道具でもあります。

大切なのは、すべてを暗記することではなく、「**こういうことができる**」と知っていて、「**必要なときに正しい形を見つけられる**」ことです。覚えていないことを調べるのは、まったく恥ずかしいことではありません。

</Column>

## まとめ

この節では、画面に文字を表示する命令 `System.out.println` を読み解きました。

- `System.out.println("…");` は、**画面に文字を1行表示する**ための決まった言い回し
- `.`（ピリオド）を「〜の」と読むと、「**System の out に println しなさい**」と読み解ける
- かっこ `( )` の中に書いた**引数**が、表示される内容になる
- 文字（**文字列**）を表示するときは、`"`（ダブルクォート、半角）で囲む。開いたら必ず閉じる
- `println` は表示後に**改行する**、`print` は**改行しない**（`ln` = line）
- 数値や計算式は `"` で囲まずに書くと、その値や計算結果が表示される

これで、Hello, World! の3つの部品 ―― クラス・main・println ―― を、すべて読み解けました。
