## はじめに ― この章で学ぶこと

第2章では、自分のコンピュータに JDK と IDE をそろえ、はじめてのプログラム「Hello, World!」を**動かす**ところまで進みました。

画面に文字が表示されたとき、「とりあえず動いた」という手応えはつかめたと思います。
その一方で、こんな気持ちも残っているのではないでしょうか。

- あの呪文のような英語の並びは、いったい何だったのか
- `public class` や `main` は、何のために書くのか
- どうして、たった1行表示するだけで、あれだけの文字が必要なのか

この第3章では、その「動いたけれど、よくわからなかった」コードを、**一つひとつ読み解いていきます**。

料理にたとえてみましょう。
第2章は、レシピを見ながら、初めての一品を作り上げた段階でした。
この章は、そのレシピを横に置いて、「なぜこの手順なのか」「この材料は何の役割なのか」を、ゆっくり確かめていく段階です。

意味がわかると、レシピを少し変えてみたり、自分なりにアレンジしたりできるようになります。
プログラムも同じです。中身を読み解けるようになると、**自分で書ける範囲**が広がり、**エラーが出ても落ち着いて直せる**ようになります。

---

## この章で読み解くプログラム

この章を通して題材にするのは、第2章で動かした、次のプログラムです。

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

たった3行の中身（と、それを囲むかっこ）に、Java の基本となる考え方が、ぎゅっと詰まっています。

この章では、このコードを次のように「部品」へ分け、一つずつ正体を明らかにしていきます。

- `public class Main { … }` ―― プログラムを入れる**箱**
- `public static void main(String[] args) { … }` ―― プログラムが**始まる場所**
- `System.out.println("Hello, World!");` ―― 画面に文字を**表示する命令**

## この章で学ぶこと

第3章は、次の8つのセクションで構成されています。

| 節 | タイトル | 内容 |
|---|---|---|
| 1 | プログラムの全体像をつかむ | Hello, World! を「部品」に分け、入れ子の構造を俯瞰する |
| 2 | クラス | `public class Main` ―― プログラムを入れる箱とファイル名のルール |
| 3 | main メソッド | `public static void main(String[] args)` ―― プログラムの入口 |
| 4 | 画面に文字を表示する | `System.out.println` を左から読み解く |
| 5 | 文・ブロック・セミコロン | 文と `;`、`{ }` のブロック、インデントという文法の骨組み |
| 6 | コメント | `//` と `/* */` ―― コードに残す、人間のためのメモ |
| 7 | エラーと向き合う | はじめてのコンパイルエラーと、その読み方・直し方 |
| 8 | Java 25 のシンプルな書き方 | コンパクトソースファイルとインスタンス main |

前半（1〜4節）では、Hello, World! を部品ごとに読み解きます。
中盤（5〜6節）では、文・ブロック・コメントといった、すべてのプログラムに共通する文法の骨組みを学びます。
後半（7〜8節）では、つまずいたときのエラーの読み方と、Java 25 ならではの新しい書き方を紹介します。

それぞれのセクションは、前のセクションの内容を受けて積み上がるように書かれています。
できるだけ、順番に読み進めることをおすすめします。

## この章の読み方

第2章までと同じく、この章でも一つだけお伝えしておきたいことがあります。

それは、**最初からすべての意味を完璧に理解しようとしなくてよい**、ということです。

この章には、`public`・`static`・`void` のように、いまの段階ではすぐに説明しきれない言葉も登場します。
これらは、本書の第2部（クラスやメソッドの章）で、あらためてじっくり解説します。

ですからこの章では、

- それぞれの部品が、**どんな役割を持っているか**
- どこを変えてよくて、どこは決まった形なのか

という「役割」と「形」をつかむことを目標にします。
細かい仕組みは、先の章で必ず回収しますので、いまは安心して読み進めてください。

> **補足: 「おまじない」を、おまじないで終わらせない**
>
> 入門書のなかには、`public static void main(String[] args)` のような部分を「おまじない」と呼び、「いまは気にせず、そのまま書きましょう」と説明するものがあります。
> 動かすことを優先するなら、それも一つのやり方です。
>
> ですが本書は、できるだけ「なぜそう書くのか」を大切にします。
> すべてを今すぐ理解する必要はありませんが、「ここには、ちゃんと意味があるんだな」と知っておくだけで、この先の学習がぐっと楽になります。

## この章を読み終えると

第3章を読み終えるころには、次のことがわかるようになっています。

- Hello, World! のコードが、どんな部品からできているか
- クラス（`class`）が「プログラムを入れる箱」であること、ファイル名との関係
- `main` が「プログラムの始まる場所」であること
- `System.out.println` が、どんな仕組みで画面に文字を出しているか
- 文の終わりの `;` や、`{ }` のブロック、インデントといった、文法の基本ルール
- コードにメモを残す「コメント」の書き方
- コンパイルエラーが出たときの、メッセージの読み方と直し方
- Java 25 で使える、もっと短いプログラムの書き方

ここまで読み解ければ、第4章からの「変数とデータ型」「演算子」といった、プログラムの中身づくりに、迷わず進めるはずです。

それでは、最初のセクション「プログラムの全体像をつかむ」から始めましょう。
