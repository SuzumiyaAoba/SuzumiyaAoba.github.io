## main メソッド ― プログラムが始まる場所

クラスの `{ }` の中を見ていくと、次の部品が見えてきます。

```java
public static void main(String[] args) {
    …
}
```

これを、**main メソッド**（メインメソッド）と呼びます。
Hello, World! のプログラムでいえば、前の節で見た「クラス」の中にある「入口」にあたる部分です。

このセクションでは、なぜ `main` が必要なのか、そして `public static void main(String[] args)` という長い並びが、それぞれ何を表しているのかを読み解きます。

---

## プログラムは `main` から始まる

Java のプログラムを実行すると、コンピュータは、まず **`main` という名前のメソッドを探します**。
そして、見つけた `main` の中身を、上から順に実行していきます。

つまり、`main` は**プログラムの始まる場所**です。
このような「プログラムが最初に実行を始める入口」のことを、**エントリーポイント**（Entry Point、入口）と呼びます。

第2章で Hello, World! を動かしたとき、画面に文字が表示されたのは、

```text
プログラムを実行する
   ↓
main を探して、その中を上から実行する
   ↓
System.out.println("Hello, World!"); が実行され、画面に表示される
```

という流れが起きていたからです。

もし `main` が見つからないと、Java は「どこから始めればいいのかわからない」ため、プログラムを実行できません。
（このときのエラーは、第7節で紹介します。）

> **補足: 「メソッド」って何？**
>
> **メソッド**（Method）とは、ひとまとまりの処理に名前をつけたものです。
> いまは「**処理のかたまり**」くらいの理解で十分です。
>
> `main` も、そのメソッドの一つで、「プログラムを実行したら、まず動かす」という特別な役割を与えられた、決まった名前のメソッドです。
> メソッドを自分で作る方法は、第9章「メソッド」でくわしく学びます。

## `public static void main(String[] args)` を分解する

それでは、この長い並びを、左から1語ずつ分解してみましょう。
一見すると暗号のようですが、それぞれにちゃんと意味があります。

```java
public static void main(String[] args)
  │      │     │    │         │
  │      │     │    │         └─ 起動時に受け取る追加の文字（引数）
  │      │     │    └─ メソッドの名前（「main」は決まった名前）
  │      │     └─ 結果を返さない、という印
  │      └─ オブジェクトを作らずに実行できる、という印
  └─ どこからでも呼び出せる（公開）、という印
```

表にまとめると、次のようになります。

| 部分 | 読み | いまの理解 | くわしく学ぶ章 |
|---|---|---|---|
| `public` | パブリック | どこからでも呼び出せる（公開） | 第12章 |
| `static` | スタティック | オブジェクトを作らずに実行できる | 第11章 |
| `void` | ボイド | 結果（戻り値）を返さない | 第9章 |
| `main` | メイン | メソッドの名前。Java が探す決まった名前 | この章 |
| `(String[] args)` | ― | 起動時に受け取る追加の文字（コマンドライン引数） | 第8章・第10章 |

`public static void main(String[] args)` は、Java が「プログラムの入口」として認識するための、**決まった形**です。
この形でないと、Java は `main` を入口だと認識できません。

ですから、いまの段階では、**この並びは丸ごとそのまま書く**もの、と考えてかまいません。
ただし、「おまじない」と丸暗記するのではなく、「`public` は公開、`void` は戻り値なし……と、一つひとつに意味があるんだな」と知っておくことが大切です。
それぞれの本当の意味は、上の表に挙げた章で、必ず回収していきます。

## いまは深入りしなくてよい部分

`public`・`static`・`void`・`String[] args` は、どれもこの先の章で本格的に学ぶ、奥の深い言葉です。
ここでは、いまの段階で「なんとなくのイメージ」だけ持っておきましょう。

- **`public`（公開）** … この `main` を、外（Java の実行のしくみ）から呼び出せるようにしておく印です。`main` は実行のしくみから呼び出されるので、公開しておく必要があります。
- **`static`（静的）** … ふつう、メソッドを使うには、前の節で触れたように、クラス（設計図）をもとに「**実体（オブジェクト）**」を作る必要があります（くわしくは第11章で学びます）。`static` を付けると、その実体を作らなくても、クラスから直接呼び出せるようになります。プログラムの入口である `main` は、いちばん最初に呼ばれ、まだ実体を作る前の段階で動く必要があるため、`static` を付けます。
- **`void`（戻り値なし）** … メソッドには、処理の結果を呼び出し元へ「返す」ものと、返さないものがあります。`main` は結果を返す必要がないので、「返しません」という印の `void` を付けます。
- **`String[] args`（引数）** … プログラムを起動するときに、外から文字を受け取るための受け皿です。たとえばコマンドラインで `java Main 東京` のように渡した「東京」を、`args` で受け取れます。Hello, World! では使っていないので、いまは「そういう受け皿がある」とだけ知っておけば十分です。

くり返しになりますが、これらは**いま完全に理解する必要はありません**。
「役割の見当はついた」という状態で、先へ進んで大丈夫です。

> **コラム: なぜ、これほど長い形なのか**
>
> 「文字を表示するだけなのに、入口の宣言が長すぎる」と感じた人もいるでしょう。
> その感覚は、まったく正しいものです。
>
> この `public static void main(String[] args)` という形は、Java が誕生した当初からの、長い歴史を持つ書き方です。
> 初心者にとってハードルが高い、という声は以前からありました。
>
> そこで **Java 25** では、この入口を、もっと短く書ける新しい方法が用意されました。
> それがどんなものかは、この章の第8節「Java 25 のシンプルな書き方」で紹介します。
> まずは、長い歴史を持つ「決まった形」をきちんと読めるようになっておきましょう。

## main の中身が、実行される

`main` の `{ }` の中に書いた命令が、上から順番に実行されます。

```java
public static void main(String[] args) {
    System.out.println("1行目");
    System.out.println("2行目");
    System.out.println("3行目");
}
```

このプログラムを実行すると、次のように表示されます。

```text
1行目
2行目
3行目
```

書いた順番どおりに、上から実行されていることがわかります。
第1章で学んだ「プログラムは、基本的に書かれた順番に処理を実行する」という考え方が、ここでも生きています。

`main` の中に何を書くか ―― それが、プログラムの「やりたいこと」そのものです。
次のセクションでは、その中身として書かれている `System.out.println(…)` を、いよいよ読み解いていきます。

## まとめ

このセクションでは、プログラムの入口である「main メソッド」について学びました。

- Java のプログラムは、**`main` という名前のメソッドから実行が始まる**（エントリーポイント）
- **メソッド**とは「処理のかたまり」のこと。`main` は実行の入口という特別な役割を持つ
- `public static void main(String[] args)` は、Java が入口として認識するための**決まった形**
  - `public`（公開）・`static`（実体を作らず呼べる）・`void`（戻り値なし）・`main`（名前）・`String[] args`（受け皿）
- これらの語の本当の意味は、第9章・第11章・第12章で学ぶ。いまは丸ごと書いてよい
- `main` の `{ }` の中身が、**上から順番に**実行される

入口の正体がつかめたら、次のセクションでは、その中で実際に動いている命令 `System.out.println` を、左から読み解いていきます。
