## プログラムの全体像をつかむ

まずは、これから読み解くプログラムの全体を、もう一度ながめてみましょう。

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

「文字を1行表示するだけなのに、ずいぶん長いな」と感じるかもしれません。
たしかに、いまの段階では記号や英単語が多く、どこからどう読めばよいのか戸惑うところです。

そこでこの節では、いきなり細かい意味に立ち入る前に、**コードを大きな「部品」に分けて、全体の構造をつかむ**ことから始めます。
建物を見るとき、まず外観や間取りを把握してから、部屋の中を見て回るのと同じ考え方です。

---

## 3つの部品に分けてみる

先ほどのコードは、大きく分けると**3つの部品**でできています。

```java
public class Main {                                  // ① 箱

    public static void main(String[] args) {         // ② 入口

        System.out.println("Hello, World!");          // ③ 命令

    }
}
```

| 部品 | コード | 役割 |
|---|---|---|
| ① 箱 | `public class Main { … }` | プログラムを入れる、いちばん外側のまとまり |
| ② 入口 | `public static void main(String[] args) { … }` | プログラムが始まる場所 |
| ③ 命令 | `System.out.println("Hello, World!");` | 実際に行いたいこと（画面への表示） |

私たちが「やりたいこと」は、③の「画面に Hello, World! と表示する」だけです。
①の箱と②の入口は、その命令を正しく実行するために、Java が必要としている**枠組み**だと考えてください。

それぞれの部品は、このあとの節で1つずつ詳しく見ていきます。

- 第2節 …… ① クラス（箱）
- 第3節 …… ② main メソッド（入口）
- 第4節 …… ③ System.out.println（命令）

この節では、これらが**どう組み合わさっているのか**という、全体の形を押さえておきましょう。

---

## 「入れ子」になっている

3つの部品は、横に並んでいるのではありません。
**箱の中に入口があり、入口の中に命令がある**、という具合に、だんだん内側へ入っていく形になっています。

このように、あるものの中に別のものが入っている構造を、**入れ子**（ネスト、Nest）と呼びます。

文章で表すと、次のような関係です。

```text
箱（クラス Main）
 └─ 入口（main メソッド）
     └─ 命令（System.out.println …）
```

身近な例でいえば、

- **建物**（クラス）の中に、**玄関**（main）があり、玄関を入ると**作業**（命令）が始まる
- **おべんとう箱**（クラス）の中に、**仕切り**（main）があり、その中に**おかず**（命令）が入っている

といったイメージです。
外側の大きなまとまりから、内側の細かい中身へと、順にたどっていく構造になっているのです。

---

## 入れ子を支える `{ }`（波かっこ）

この入れ子の構造を作っているのが、**`{ }`（波かっこ、なみかっこ）** です。

`{` から `}` までが「ひとまとまり」を表します。
先ほどのコードには、`{ }` が2組あります。

```java
public class Main {          // ← クラスの { が始まる

    public static void main(String[] args) {   // ← main の { が始まる

        System.out.println("Hello, World!");

    }                        // ← main の } で閉じる
}                            // ← クラスの } で閉じる
```

- 外側の `{ }` …… クラス Main の中身を囲む
- 内側の `{ }` …… main の中身を囲む

`{` で開いたら、必ずどこかで `}` を書いて閉じる必要があります。
この対応がくずれると、プログラムは動きません（よくある間違いとして、第7節で取り上げます）。

`{ }` のくわしい話は第5節「文・ブロック・セミコロン」でも扱いますが、いまは「**`{ }` がまとまりを作り、入れ子の構造を支えている**」とだけ覚えておけば十分です。

---

## インデント（字下げ）は、なぜついているのか

もう一つ、コードの見た目で気になるのが、行の先頭に入っている**空白**です。

```java
public class Main {
    public static void main(String[] args) {        // ← 4文字ぶん下がっている
        System.out.println("Hello, World!");          // ← さらに4文字ぶん下がっている
    }
}
```

このように、行の先頭に空白を入れて文字を右へずらすことを、**インデント**（字下げ）と呼びます。

インデントは、**入れ子の深さ**を目で見てわかるようにするためのものです。
内側に入るほど、右へ下げて書きます。

- クラスの中身は、1段下げる
- main の中身は、さらにもう1段下げる

実は、Java はこの空白を読み飛ばすので、インデントがなくてもプログラム自体は動きます。
ですが、インデントのないコードはとても読みにくく、間違いのもとになります。
インデントは「人間が読みやすくするための工夫」であり、プログラミングでは欠かせない習慣です。

> **補足: インデントは IDE が手伝ってくれる**
>
> 第2章で入れた IntelliJ IDEA のような IDE は、コードを書くと自動でインデントを整えてくれます。
> 形がくずれてしまっても、整形（フォーマット）の機能を使えば、きれいに直すことができます。
> インデントについては、第5節でもう少しくわしく説明します。

---

## まとめ

このセクションでは、Hello, World! のプログラム全体を、大きな部品に分けてながめました。

- プログラムは、大きく **① 箱（クラス）・② 入口（main）・③ 命令** の3つの部品からできている
- 私たちが本当にやりたいのは ③ の命令で、① と ② はそれを支える枠組みである
- 3つの部品は、**箱の中に入口、入口の中に命令**という「**入れ子**」の構造になっている
- 入れ子を作っているのが **`{ }`（波かっこ）** で、`{` を開いたら必ず `}` で閉じる
- 行頭の空白（**インデント**）は、入れ子の深さを見やすくするための工夫である

全体の地図が見えたところで、次のセクションからは、いちばん外側の部品である「クラス（箱）」を、くわしく見ていきます。
