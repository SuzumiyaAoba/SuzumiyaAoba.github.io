## コメント ― コードにメモを残す

プログラムを書いていると、「この処理は何のためのものか」「なぜこう書いたのか」を、書き残しておきたくなることがあります。

そんなときに使うのが、**コメント**（Comment）です。
コメントとは、**コードの中に書いておける、人間のためのメモ**です。

コメントは、プログラムの一部のように見えますが、**実行はされません**。
Java は、コメントの部分を読み飛ばします。
そのため、コメントに何を書いても、プログラムの動きには影響しません。

この節では、コメントの書き方と、上手な使い方を学びます。

---

## 2種類のコメント

Java のコメントには、おもに2つの書き方があります[^jls-comments]。

### 1. 行コメント `//`

`//`（スラッシュ2つ）を書くと、**その行の `//` 以降**がコメントになります。
1行ぶん、または行の途中から行末までをメモにしたいときに使います。

```java
// これは行コメントです（この行はすべてメモ）
System.out.println("Hello, World!");   // 行の途中からでも書ける
```

上のコードを実行すると、コメントは読み飛ばされるので、表示されるのは次の1行だけです。

```text
Hello, World!
```

`//` から行末まではメモなので、画面には何の影響もありません。

### 2. ブロックコメント `/* */`

`/*` で始めて `*/` で終わると、その間にはさまれた部分が、**複数行にわたって**コメントになります。
長い説明を書きたいときに便利です。

```java
/*
  ここはブロックコメントです。
  複数行にわたって、
  まとめてメモを書けます。
*/
System.out.println("Hello, World!");
```

`/*` と `*/` で囲まれた部分は、何行あってもすべて読み飛ばされます。

2種類のコメントを、表にまとめておきましょう。

| 書き方 | 範囲 | 使いどころ |
|---|---|---|
| `//` | その行の `//` 以降（行末まで） | 短いメモ、1行の説明 |
| `/* … */` | `/*` から `*/` まで（複数行 OK） | 長い説明、まとまった注釈 |

> **補足: もう一つの `/** */`（Javadoc）**
>
> `/**` で始まるコメントは、**Javadoc**（ジャバドック）と呼ばれ、クラスやメソッドの説明を書くために使われます。
> これをもとに、説明書（ドキュメント）を自動で生成できる、という特別な役割を持っています[^javadoc-tool]。
> いまは「そういうコメントもある」と知っておくだけで十分です。くわしくは、メソッドを学ぶ第9章などで触れます。

## コメントは、何のために書くのか

「実行されないなら、なぜわざわざ書くの？」と思うかもしれません。
コメントは、**未来の自分や、コードを読むほかの人**のために書きます。

プログラムは、書いた直後は内容を覚えていても、数週間・数か月たつと、自分でも「これは何をしているコードだっけ？」とわからなくなることがよくあります。
そんなとき、コメントに「なぜこう書いたのか」が残っていると、すぐに思い出せます。

コメントには、たとえば次のようなことを書きます。

- そのコードが**何のためのもの**か（目的）
- **なぜ**そう書いたのか（理由・背景）
- 注意すべき点や、あとで直したいことのメモ

## よいコメント・避けたいコメント

コメントは便利ですが、**書きすぎると、かえって読みにくくなる**こともあります。
ここで、コメントの上手な使い方のコツを、少しだけ紹介します。

避けたいのは、**コードを見ればすぐわかること**を、そのままなぞるコメントです。

```java
// 「Hello, World!」と表示する
System.out.println("Hello, World!");
```

このコメントは、コードを読めばわかることをくり返しているだけで、あまり役に立ちません。

一方、役に立つのは、**コードだけでは伝わらない「なぜ」**を書いたコメントです。

```java
// 動作確認用に、まずあいさつを表示している（あとで本来の処理に置きかえる）
System.out.println("Hello, World!");
```

このように、「何のために」「どういう意図で」書いたのかを残すと、あとから読む人の助けになります。

> **補足: いまは、気負わなくてよい**
>
> 「よいコメントとは何か」は、経験を積むなかで少しずつわかってくるものです。
> 学習を始めたばかりのいまは、「自分が理解するためのメモ」として、気軽にコメントを使ってかまいません。
> 写経したコードに「ここはクラスの宣言」「ここが表示する命令」と、自分用のメモを書き込むのも、よい練習になります。

## コメントアウト ―― 一時的に実行から外す

コメントには、メモを残すほかに、もう一つ便利な使い方があります。
それは、**コードを一時的に実行されない状態にする**ことです。

動いているコードの行頭に `//` を付けると、その行はコメントとして読み飛ばされ、実行されなくなります。
このように、コードをコメントにして実行から外すことを、**コメントアウト**と呼びます。

```java
System.out.println("1行目");
// System.out.println("2行目");   // ← この行はコメントアウトされ、実行されない
System.out.println("3行目");
```

実行すると、2行目は飛ばされて、次のように表示されます。

```text
1行目
3行目
```

コメントアウトは、「この行を消さずに、ちょっとだけ動きを止めてみたい」というときに役立ちます。
たとえば、「どの行が原因でうまく動かないのか」を調べるときに、あやしい行を一時的にコメントアウトして確かめる、といった使い方ができます。

不要になったら `//` を外せば、また実行されるようになります。
コードを消してしまう前に、まずコメントアウトして試す ―― これも、覚えておくと便利なテクニックです。

## まとめ

この節では、コードにメモを残す「コメント」について学びました。

- **コメント**は、コードの中に書ける、人間のためのメモ。**実行はされない**（Java が読み飛ばす）
- `//` …… その行の `//` 以降がコメントになる（**行コメント**）
- `/* … */` …… `/*` から `*/` までがコメントになる（**ブロックコメント**、複数行 OK）
- コメントは、**未来の自分やほかの人**のために、目的や「なぜ」を書き残すもの
- コードを見ればわかることのなぞりは避け、**コードだけでは伝わらない意図**を書くとよい
- 行頭に `//` を付けてコードを一時的に実行から外すことを、**コメントアウト**と呼ぶ

ここまでで、Java のプログラムを「読み解く」ための知識が、ひととおりそろいました。
次の節では、書いたコードがうまく動かないとき ―― **エラー**と向き合う方法を学びます。
