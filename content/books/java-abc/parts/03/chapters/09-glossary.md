## 用語集 ― この章で学んだ言葉

この第3章でも、たくさんの新しい言葉が登場しました。

ここでは、それらの用語を**セクションごとに**まとめておきます。
「あの言葉は何だったかな」と思ったときに、辞書のように使ってください。

第3章で扱った言葉の多くは、`public` や `static` のように、本書の第2部であらためてくわしく学ぶものです。
いまの段階では、「役割」をつかめていれば十分です。すべてを暗記する必要はありません。

---

## プログラムの全体像（第1節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| 入れ子 | Nest（ネスト） | あるまとまりの中に、別のまとまりが入っている構造。クラスの中に main、main の中に命令 |
| 波かっこ | `{ }` | まとまり（ブロック）を作る記号。開いたら必ず閉じる |
| インデント | ― | 行頭に空白を入れて、入れ子の深さを見やすくする字下げ |

---

## クラス（第2節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| クラス | Class | すべての命令を書く、いちばん外側のまとまり。命令はすべてクラスの `{ }` の中に書く。本当の役割（オブジェクトを作る設計図）は第11章で学ぶ |
| オブジェクト | Object | クラス（設計図）をもとに作られる「もの」。1つのクラスから複数作れる。第11章で学ぶ |
| `class` | ― | クラスを定義するときに書くキーワード |
| `public` | パブリック | 「どこからでも使える（公開）」という印。くわしくは第12章 |
| オブジェクト指向 | Object-Oriented | プログラムをクラスという単位でまとめて組み立てる考え方。第2部で学ぶ |
| パスカルケース | Pascal Case | 単語の先頭をすべて大文字にしてつなげる書き方（例: `HelloWorld`）。クラス名の慣習 |
| クラス名とファイル名の一致 | ― | `public` なクラスは、クラス名とファイル名を同じにする（`public class Main` → `Main.java`） |

## main メソッド（第3節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| メソッド | Method | ひとまとまりの処理に名前をつけたもの。「処理のかたまり」。第9章で学ぶ |
| main メソッド | ― | プログラムが最初に実行する、入口となるメソッド |
| エントリーポイント | Entry Point | プログラムが実行を始める入口。Java では `main` がこれにあたる |
| `static` | スタティック | 実体（オブジェクト）を作らずに呼び出せる、という印。第11章で学ぶ |
| `void` | ボイド | 結果（戻り値）を返さない、という印。第9章で学ぶ |
| `String[] args` | ― | 起動時に外から受け取る文字（コマンドライン引数）の受け皿。第8章・第10章で学ぶ |

## System.out.println（第4節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| `System.out.println(…)` | ― | 画面に内容を1行表示する、決まった言い回し |
| 標準出力 | Standard Output | プログラムの出力先。ふつうは画面。`System.out` が指す |
| `println` / `print` | ― | `println` は表示後に改行する、`print` は改行しない（`ln` = line） |
| 引数 | ひきすう、Argument | メソッドのかっこ `( )` に渡す値。表示したい内容など |
| 文字列 | String、ストリング | `"…"`（ダブルクォート）で囲まれた文字のまとまり |
| ダブルクォート / シングルクォート | `"` / `'` | `"…"` は文字列、`'…'` は1文字（`char`）。第4章で学ぶ |

## 文・ブロック・セミコロン（第5節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| 文 | Statement、ステートメント | 命令の1単位。終わりに `;` を付ける |
| セミコロン | `;` | 文の終わりを表す区切り。日本語の「。」にあたる |
| ブロック | Block | `{ }` で囲んだ、文のまとまり |
| 大文字・小文字の区別 | ケースセンシティブ | `System` と `system` を別の文字として扱うこと |
| 全角・半角 | ― | 記号はすべて半角で書く。全角の記号はエラーになる |

## コメント（第6節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| コメント | Comment | コードに書ける、人間のためのメモ。実行されない |
| 行コメント | `//` | その行の `//` 以降をコメントにする |
| ブロックコメント | `/* … */` | `/*` から `*/` までをコメントにする（複数行 OK） |
| Javadoc | ジャバドック | `/** … */` で書く、説明書の生成に使える特別なコメント |
| コメントアウト | ― | コードの行頭に `//` を付けて、一時的に実行から外すこと |

## エラーと向き合う（第7節）

エラーメッセージは、コンピュータの言語設定によって日本語か英語で表示されます。下表では「日本語環境での表示（英語環境での表示）」の形で示します。

| 用語 | 英語・読み | 意味 |
|---|---|---|
| コンパイルエラー | Compile Error | コンパイルの段階で、文法の間違いが見つかったときに出るエラー |
| 実行時エラー | Runtime Error | プログラムを実行する段階で起きるエラー（例: `main` が見つからない） |
| シンボル | Symbol | コードに出てくる名前（変数名・メソッド名など）。`シンボルを見つけられません`＝その名前が見つからない |
| リテラル | Literal | `"Hello"` や `100` のように、コードに直接書いた値。`文字列リテラル`＝直接書いた文字列 |
| `';'がありません` | `';' expected` | セミコロンの付け忘れを知らせるメッセージ |
| `シンボルを見つけられません` | `cannot find symbol` | 名前のつづり・大文字小文字の間違いを知らせるメッセージ |
| `…は不正な文字です` | `illegal character` | 全角の記号など、使えない文字が混ざっていることを知らせるメッセージ |
| `文字列リテラルが閉じられていません` | `unclosed string literal` | `"` の閉じ忘れを知らせるメッセージ |

## Java 25 のシンプルな書き方（第8節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| コンパクトソースファイル | Compact Source File | クラスの宣言を省略して書けるソースファイル（Java 25 で正式導入） |
| インスタンス main メソッド | Instance Main Method | `public static` を付けない `void main()` の形 |
| `IO.println` / `IO.print` / `IO.readln` | ― | `IO` を使った、短い入出力の命令。`System.out.println` などの短い書き方 |
| プレビュー機能 | Preview Feature | 正式導入の前に試験的に提供される機能。Java 21〜24 ではこの扱いだった |

## おわりに

ここに挙げた用語は、いずれも第3章の各セクションで説明したものです。

この章では、第2章で動かした「Hello, World!」を題材に、プログラムの中身を一つひとつ読み解いてきました。

- クラスは、すべての命令を書く、いちばん外側のまとまりであること（本当の役割は第11章で学ぶ）
- `main` は、プログラムが始まる入口であること
- `System.out.println` は、画面に文字を表示する命令であること
- 文の終わりには `;`、まとまりは `{ }`、読みやすさのためのインデント
- コメントの書き方と、エラーとの向き合い方
- そして、Java 25 で使える、もっとシンプルな書き方

`public` や `static` のように、いまはまだ「役割」しかわからない言葉も出てきましたが、心配はいりません。
それらは、本書の第2部で、一つひとつ正体を明らかにしていきます。

ここまでで、Java のプログラムが「どんな形をしていて、どう動くのか」を、自分の目で読み解けるようになりました。

第4章からは、いよいよプログラムの**中身**を作っていきます。
最初のテーマは「**変数とデータ型**」 ―― 数値や文字といったデータを、プログラムの中でどう扱うのか、を学びましょう。
