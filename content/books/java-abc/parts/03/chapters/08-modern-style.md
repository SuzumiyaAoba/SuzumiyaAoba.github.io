## Java 25 のシンプルな書き方 ― コンパクトソースファイルとインスタンス main

この章では、Hello, World! のプログラムを部品ごとに読み解いてきました。

その過程で、こんなふうに感じた人も多いと思います。

> たった1行「Hello, World!」を表示するだけなのに、`public class` や `public static void main(String[] args)` という、長い枠組みを書かなければならない。

その感覚は、とても自然なものです。
実は、Java を作っている人たちも、同じように「初心者には、この枠組みのハードルが高い」と考えていました。

そこで **Java 25** では、この枠組みを**大幅に省略できる、新しい書き方**が正式に導入されました。
第2章でも2度ほど予告した、その書き方を、いよいよこのセクションで紹介します。

---

## まず、見比べてみる

ここまで書いてきた「従来の形」と、Java 25 の「新しい形」を、並べてみましょう。
どちらも、画面に `Hello, World!` と表示する、同じプログラムです。

**従来の形**（この章で読み解いてきた形）

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

**Java 25 の新しい形**

```java
void main() {
    IO.println("Hello, World!");
}
```

ずいぶんすっきりしましたね。
変わったのは、大きく次の2点です。

| | 従来の形 | Java 25 の新しい形 |
|---|---|---|
| クラスの宣言 | `public class Main { … }` で囲む | **囲まなくてよい** |
| 入口の宣言 | `public static void main(String[] args)` | `void main()` と短く書ける |
| 表示の命令 | `System.out.println(…)` | `IO.println(…)` と短く書ける |

この章で「枠組み」として読み解いてきた部分が、ごっそり省略できるようになっているのがわかります。

---

## なぜ、こんなに短く書けるのか

短くなったとはいえ、魔法ではありません。
省略された部分は、**Java が裏側で自動的に補ってくれている**のです。

この新しい書き方を支えているのが、Java 25 で正式に導入された、次の2つのしくみです。

### コンパクトソースファイル

クラスの宣言（`public class Main { … }`）を書かなくても、Java が自動的に「見えないクラス」を用意してくれます。
このように、**クラスの宣言を省略して書けるソースファイル**を、**コンパクトソースファイル**（Compact Source File）と呼びます。

第2節で学んだ「Java の命令はクラスの中に書く」というルールは変わっていません。
ただ、そのクラスの宣言を**書く手間を Java が肩代わりしてくれる**、というわけです。

### インスタンス main メソッド

入口の宣言も、`public` や `static` を省いて、`void main()` とだけ書けるようになりました。
このような、`public static` を付けない `main` を、**インスタンス main メソッド**（Instance Main Method）と呼びます。

第3節で見たとおり、`public` や `static` には、それぞれちゃんとした意味があります。
ですが、初心者がいちばん最初に書くプログラムでは、その意味を知らなくても始められるように、省略が認められたのです。

### IO による、短い入出力

`System.out.println` も、`IO.println` という短い形で書けるようになりました。
`IO`（アイ・オー）は、画面への表示や、キーボードからの入力をまとめた、初心者にやさしい道具です。

| 新しい書き方 | はたらき |
|---|---|
| `IO.println("…")` | 内容を1行表示する（`System.out.println` にあたる） |
| `IO.print("…")` | 改行せずに表示する（`System.out.print` にあたる） |
| `IO.readln("…")` | キーボードからの入力を1行読み取る |

> **補足: しくみの名前は、覚えなくてよい**
>
> 「コンパクトソースファイル」「インスタンス main メソッド」という名前は、いま覚える必要はありません。
> 「クラスや `public static` を省略して、短く書けるしくみがある」とだけ知っておけば十分です。
> これらの正体は、本書の第2部でクラスやメソッドを学ぶと、自然とわかるようになります。

## 実行してみる

新しい形のプログラムも、実行のしかたは、第2章で学んだ方法とほとんど同じです。

たとえば、上のコードを `hello.java` という名前で保存し、コマンドラインから次のように実行します。

```text
java hello.java
```

すると、これまでと同じように表示されます。

```text
Hello, World!
```

> **ファイル名の自由度**
>
> 新しい形では、クラスの宣言（`public class Main`）を書きません。
> そのため、第2節で学んだ「クラス名とファイル名をそろえる」というルールに、しばられません。
> `hello.java` でも `test.java` でも、好きな名前で保存できます。
>
> IDE（IntelliJ IDEA）でも、これまでと同じように ▶ ボタンで実行できます（IDE のバージョンによっては、設定や見え方が少し異なることがあります）。

## いつ使う？ ―― 本書の方針

ここまで読んで、「最初から、この短い書き方を教えてくれればよかったのに」と思うかもしれません。
それでも本書が、**従来の形を中心に**学んでいくのには、理由があります。

- **世の中の大半のコードは、従来の形で書かれている** … これまでに書かれた膨大な Java のプログラムや、多くの入門書・参考書・サンプルコードは、`public class` と `public static void main` の形です。これを**読めること**は、これからも欠かせません。
- **現場では、従来の形が基本** … 実際の開発では、1つのファイルに複数のクラスを書いたり、クラスどうしを連携させたりします。新しい短い書き方は、あくまで「小さなプログラムを手軽に書く」ためのものです。
- **省略された部分にも、意味がある** … `public`・`static`・`class` は、第2部で学ぶ大切な考え方につながっています。短い書き方は、これらを「省略している」だけで、なくなったわけではありません。

そこで本書では、

- **基本は、従来の形**（`public class` と `public static void main`）で学ぶ
- 新しい短い書き方は、「**こう書くこともできる**」と知っておく

という方針で進めます。
ちょっとしたプログラムを手早く試したいときには、この章で学んだ新しい形が役立ちます。
ぜひ、両方の書き方を知っておいてください。

> **補足: `IO.println` は jshell でもそのまま試せる**
>
> 第2章で紹介した `jshell` でも、`IO.println("…")` はそのまま動きます。
> しかも `jshell` では `void main()` の枠すら要らず、`IO.println("こんにちは")` の**1行だけ**を打てば、すぐに結果が表示されます。
> 次の第4章からは、こうした「1行だけで試せる」コードを `jshell` に貼りながら、気軽に確かめて学んでいきます。

> **注意: 新しい書き方が使えるのは Java 25 以降**
>
> この短い書き方が、追加の設定なしにそのまま使えるのは、**Java 25 以降**です。
> Java 21 から 24 では「プレビュー機能」という試験的な扱いで、実行するには特別な指定が必要でした。
> さらに古いバージョンの Java では、そもそもこの書き方は使えません。
>
> もし、新しい形のプログラムがうまく動かないときは、第2章で確認したとおり、`java --version` で **Java 25 以上**が入っているかを確かめてみましょう。
> こうした初心者にやさしい新機能がそろっている点も、本書が最新の **Java 25** を選んだ理由の一つです。

## まとめ

このセクションでは、Java 25 で正式に導入された、シンプルな書き方を学びました。

- Java 25 では、`public class` の宣言や `public static` を省略して、プログラムを短く書ける
- クラスの宣言を省ける書き方を**コンパクトソースファイル**、`public static` を付けない `main` を**インスタンス main メソッド**と呼ぶ
- `System.out.println` は、`IO.println` という短い形でも書ける（`IO.print`・`IO.readln` もある）
- 省略された部分は、Java が裏側で自動的に補っている。クラスの考え方がなくなったわけではない
- 本書は、世の中の大半のコードに合わせて**従来の形を中心**に学び、新しい形は「こう書くこともできる」と知っておく方針で進める
- この書き方がそのまま使えるのは **Java 25 以降**

これで、第3章「はじめてのプログラム」の本編は終わりです。
