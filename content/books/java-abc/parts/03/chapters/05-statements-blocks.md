## 文・ブロック・セミコロン ― 文法の骨組み

ここまで、Hello, World! を部品ごとに読み解いてきました。
その過程で、`;`（セミコロン）や `{ }`（波かっこ）、インデント（字下げ）といった記号が、くり返し登場しました。

これらは、Hello, World! に限らず、**すべての Java プログラムに共通する「文法の骨組み」**です。
日本語に「。」や「、」、段落といったルールがあるように、Java にも「ここで文が終わる」「ここからここまでがひとまとまり」といったルールがあります。

この節では、その骨組みを、あらためて整理します。

---

## 文（ステートメント）と、セミコロン

Java のプログラムは、**文**（ステートメント、Statement）の積み重ねでできています[^jls-statements]。
文とは、**命令の1単位**のことです。日本語の「一文」に近いものだと考えてください。

そして、**文の終わりには、必ず `;`（セミコロン）を付ける**という決まりがあります。
これは、日本語の文の終わりに付ける「。」のようなものです。

```java
System.out.println("おはよう");      // ← 文の終わりに ;
System.out.println("こんにちは");    // ← 文の終わりに ;
```

`;` は、Java に「ここで1つの文が終わりましたよ」と伝える区切りです。
この区切りがあるおかげで、Java は文と文の切れ目を判断できます。

> **注意: セミコロンの付け忘れは、最も多い間違い**
>
> プログラミングを始めたばかりのころ、いちばん多い間違いが、この **`;` の付け忘れ**です。
> 文を書いたら、終わりに `;` を付ける ―― これを習慣にしましょう。
> もし忘れても、コンパイラがちゃんと教えてくれます（第7節で、そのエラーを紹介します）。

### 改行と文は、別もの

ここで一つ、おさえておきたいことがあります。
Java では、**文の区切りは「改行」ではなく「`;`」で決まる**、という点です。

たとえば、次の2つは、Java にとってまったく同じ意味です。

```java
System.out.println("A"); System.out.println("B");
```

```java
System.out.println("A");
System.out.println("B");
```

上は1行に2つの文を、下は2行に分けて書いていますが、どちらも `;` で2つの文に区切られているので、同じように動きます。

とはいえ、読みやすさのために、**ふつうは1行に1文**を書きます。
「`;` で区切る」というルールを知ったうえで、見やすい書き方を選びましょう。

## ブロック ―― `{ }` でまとめる

第1節でも触れたとおり、`{ }`（波かっこ）で囲んだ部分を、**ブロック**（Block）と呼びます[^jls-blocks]。
ブロックは、複数の文を「ひとまとまり」にする入れ物です。

Hello, World! には、2つのブロックがありました。

```java
public class Main {                                 // ← クラスのブロック開始
    public static void main(String[] args) {        // ← main のブロック開始
        System.out.println("Hello, World!");
    }                                                // ← main のブロック終了
}                                                    // ← クラスのブロック終了
```

- 外側のブロック … クラス Main の中身をまとめる
- 内側のブロック … main の中身をまとめる

そして、この2つのブロックは**入れ子**になっています。
クラスのブロックの中に、main のブロックがすっぽり入っている形です。

### `{` と `}` は、必ずペアにする

ブロックで最も大切なルールは、**`{` で開いたら、必ずどこかで `}` で閉じる**ことです。
`{` と `}` は、いつもペアになっていなければなりません。

```text
{ で開いた数  ＝  } で閉じた数
```

この数が合わないと、Java は「まとまりがどこで終わるのか」がわからず、エラーになります。
かっこの対応がくずれるのも、初心者によくある間違いの一つです（第7節で扱います）。

> **コツ: `{` を書いたら、先に `}` も書く**
>
> ブロックを書くときは、`{` を打った時点で、対応する `}` も先に打っておくと、閉じ忘れを防げます。
> IDE（IntelliJ IDEA）を使っていると、`{` を打った瞬間に `}` を自動で補ってくれることもあります。
> 開きかっこと閉じかっこを、いつもセットで意識する習慣をつけましょう。

## インデント（字下げ）

第1節で少し触れた**インデント**（字下げ）を、ここでもう少しくわしく見ておきましょう。

インデントとは、**行の先頭に空白を入れて、文字を右へずらすこと**です。
ブロックの**入れ子の深さ**に合わせて、内側に入るほど右へ下げて書きます。

```java
public class Main {
    public static void main(String[] args) {     // 1段下げる
        System.out.println("Hello, World!");        // 2段下げる
    }
}
```

インデントの幅は、**半角スペース4つぶん**にするのが、Java では一般的です[^java-style-indent]。

### Java は空白を読み飛ばす

実は、Java はインデント（行頭の空白）を**読み飛ばします**。
そのため、インデントがまったくなくても、プログラムは動いてしまいます。

たとえば、次のコードも、文法上は正しく動きます。

```java
public class Main {
public static void main(String[] args) {
System.out.println("Hello, World!");
}
}
```

ですが、どこからどこまでが、どのまとまりなのか、とても読み取りにくいですね。
さらに極端な例として、次のように1行に詰め込んでも、Java は動かせてしまいます。

```java
public class Main{public static void main(String[] args){System.out.println("Hello, World!");}}
```

これでは、人間にはほとんど読めません。

### 「人間のための」ルール

このように、インデントや改行は、Java にとっては「あってもなくてもよい」ものです。
しかし、**コードを読む人間にとっては、とても重要**です。

- 入れ子の深さがひと目でわかる
- ブロックの始まりと終わりが見つけやすい
- どの命令が、どのまとまりに属しているかがわかる

読みやすいコードは、間違いが減り、あとから直すのも楽になります。
これは第1章で学んだ「よいプログラム」の条件 ―― **読みやすい** ―― にもつながる、大切な習慣です。

> **補足: IDE の自動整形にたよってよい**
>
> インデントを自分で1つずつ数える必要はありません。
> IntelliJ IDEA には、コードのインデントや空白を、決められたルールに沿って自動で整える**フォーマット**（整形）機能があります。
> 形がくずれてしまったら、この機能で一気に整えられます。
> まずは「内側ほど下げる」という考え方をつかんでおき、細かい調整は IDE に手伝ってもらいましょう。

## 大文字・小文字、全角・半角に気をつける

文法の骨組みとあわせて、Java のコードを書くうえで欠かせない、2つの注意点を確認しておきます。

### 大文字と小文字は区別される

Java は、**大文字と小文字を、別の文字として区別します**（これを「**大文字・小文字を区別する**」「ケースセンシティブ」と言います）。

```text
○  System.out.println("...");
×  system.out.println("...");      // System の S が小文字
×  System.Out.Println("...");      // out / println が大文字始まり
```

`System` を `system` と書いたり、`println` を `Println` と書いたりすると、Java は「そんな名前は知らない」とみなし、エラーになります。
見た目はそっくりでも、コンピュータにとっては別物なのです。

### 記号は、すべて半角で

第2章でも注意したとおり、プログラムの記号 ―― `(` `)` `{` `}` `"` `;` `.` など ―― は、すべて**半角**で入力します。

日本語入力（全角）のまま打つと、見た目が似ていても、コンピュータには別の文字として扱われ、エラーになります。

```text
○  半角:  ( ) { } " ;
×  全角:  （ ） ｛ ｝ ” ；
```

うまく動かないときは、まず「**全角の記号が混ざっていないか**」を確認しましょう。
これは、初心者がとてもつまずきやすいポイントです。

## 小さく書いて、こまめに動かす

文の書き方がわかったところで、プログラムを書くときの、とても大切なコツをひとつお伝えします。
それは、**一気にたくさん書かず、少し書いては動かす**、ということです。

たとえば、`System.out.println` を1つ書いたら、すぐ実行して結果を確かめます。
うまくいったら、次の1行を足して、また実行します。

なぜ、こまめに動かすのがよいのでしょうか。
もしエラーが出ても、**直前に書き足したところに原因がある**と、すぐにわかるからです。

反対に、30行ほど一気に書いてから初めて実行すると、いくつものエラーがまとめて出て、どこが原因なのかを探すだけで一苦労です。
1行ずつ確かめながら進めていれば、こうした「原因さがし」で立ち止まることが、ぐっと減ります。

```text
少し書く → 動かす → 結果を確かめる → また少し書く
```

これは、第1章で学んだ「書く → 実行する → 結果を見る」の流れそのものです。
遠回りに見えて、初心者にとっては、いちばん速く確実に進める方法です。

> **補足: 完成してから動かす、ではない**
>
> 「プログラムは、全部書き上げてから実行するもの」と思いがちですが、実際は逆です。
> プロほど、ほんの少し書いては動かし、また少し書いては動かす、を細かく繰り返しています。
> 「まだ途中だから動かさない」ではなく、「途中でも、動くところまでこまめに動かす」と覚えておきましょう。

## まとめ

この節では、すべての Java プログラムに共通する「文法の骨組み」を整理しました。

- プログラムは**文**（命令の1単位）の積み重ねでできており、文の終わりには **`;`（セミコロン）** を付ける
- 文の区切りは「改行」ではなく「`;`」で決まる。ただし、読みやすさのため、ふつうは1行に1文を書く
- `{ }` で囲んだまとまりを**ブロック**と呼び、`{` で開いたら必ず `}` で閉じる（数を合わせる）
- **インデント**（字下げ）は、入れ子の深さを見やすくするための工夫。半角スペース4つが一般的
- Java はインデントや改行を読み飛ばすが、**人間のために**整えて書くことが大切
- Java は**大文字・小文字を区別する**。記号は**すべて半角**で書く
- **小さく書いて、こまめに動かす**と、エラーの原因をすぐに見つけられる

文法の骨組みがわかったところで、次の節では、コードに「人間のためのメモ」を残す**コメント**を学びます。

[^jls-blocks]: *The Java® Language Specification, Java SE 25 Edition*, §14.2 "Blocks," [https://docs.oracle.com/javase/specs/jls/se25/html/jls-14.html#jls-14.2](<https://docs.oracle.com/javase/specs/jls/se25/html/jls-14.html#jls-14.2>)。"A block is a sequence of statements, local variable declaration statements, and local class and interface declarations within braces." と定義されている。

[^java-style-indent]: Oracle, "Code Conventions for the Java Programming Language: §4 Indentation," [https://www.oracle.com/java/technologies/javase/codeconventions-indentation.html](<https://www.oracle.com/java/technologies/javase/codeconventions-indentation.html>)。"Four spaces should be used as the unit of indentation." と推奨されている。Google Java Style Guide（[https://google.github.io/styleguide/javaguide.html](<https://google.github.io/styleguide/javaguide.html>)）は2スペースを採用するなど、組織により流儀が分かれる。
