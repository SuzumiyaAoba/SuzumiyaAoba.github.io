## はじめに ― この章で学ぶこと

オブジェクトの作り方は、いつも **`new`** からとは限りません。

**Prototype**（プロトタイプ、原型）パターンは、**「既存のインスタンスをコピーして、新しいインスタンスを作る」**ためのパターンです。
原型となるオブジェクトを 1 つ持っておき、必要に応じて**複製**する ―― そんな発想です。

設定の重い・組み立てが大変なオブジェクトを、毎回ゼロから作らずに済ませる場面で使われます。

---

## この章を読み終えると

- Prototype の**意図**を、自分の言葉で説明できる
- **浅いコピーと深いコピー**の違いを説明できる
- Java の `Cloneable` の落とし穴を理解し、**現代的な代替**を選べる
