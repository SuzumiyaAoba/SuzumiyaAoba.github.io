## 継承とは ― is-a の関係

継承を学ぶ前に、まず「**どういうときに継承を使うのか**」を理解しておきましょう。
継承は強力なしくみですが、使いどころを間違えると、かえって分かりにくいコードになります。

---

## 共通部分を、別々に書く無駄

継承を使わずに、`Dog`（犬）と `Cat`（猫）を作ると、こうなります。

```java
class Dog {
    String name;
    void eat() { IO.println(name + " はごはんを食べる"); }
    void cry() { IO.println(name + ": ワン"); }
}

class Cat {
    String name;
    void eat() { IO.println(name + " はごはんを食べる"); }   // ← Dog と同じ
    void cry() { IO.println(name + ": ニャー"); }
}
```

`name` フィールドと `eat()` メソッドが、`Dog` と `Cat` でまったく同じです。
動物の種類が増えるほど（鳥、魚…）、この同じコードが何度もコピーされます。
そして、もし `eat()` の処理を変えたくなったら、**すべてのクラスを直す**ことになります。これは大変ですし、直し忘れも起きます。

---

## 共通部分を「親」にまとめる

そこで、共通部分（`name` と `eat()`）を持つ、**`Animal`**（動物）クラスを作ります。
そして、`Dog` と `Cat` は、この `Animal` を**継承**して、共通部分を受け継ぎます。

```java
class Animal {                 // 親クラス（共通部分）
    String name;
    void eat() { IO.println(name + " はごはんを食べる"); }
}

class Dog extends Animal {     // Animal を継承する子クラス
    void cry() { IO.println(name + ": ワン"); }
}

class Cat extends Animal {     // Animal を継承する子クラス
    void cry() { IO.println(name + ": ニャー"); }
}
```

`extends Animal` と書くことで、`Dog` と `Cat` は、`Animal` の `name` と `eat()` を**受け継ぎます**。
そのため、`Dog` と `Cat` には、それぞれ違う部分（`cry()`）だけを書けば済みます。
共通の `eat()` は、`Animal` の1か所にしかありません。変更があっても、そこを直すだけで、`Dog` も `Cat` も変わります。

（`extends` の書き方は、次の節でくわしく学びます。ここでは「共通部分を親にまとめられる」イメージをつかんでください。）

---

## 「is-a」の関係 ― 継承を使ってよいか

継承を使ってよいかどうかは、**「子 is a 親」（子は、親の一種である）**という関係が成り立つかで判断します。
これを **is-a の関係**と呼びます。

- 「犬は、動物の一種である」（Dog **is a** Animal） … ◯ 成り立つ → 継承してよい
- 「猫は、動物の一種である」（Cat **is a** Animal） … ◯ 成り立つ → 継承してよい

このように、`Dog` も `Cat` も「`Animal` の一種」と自然に言えるので、`Animal` を継承するのは適切です。

逆に、is-a が成り立たないのに継承するのは、間違いのもとです。

- 「車は、エンジンの一種である」（Car **is a** Engine） … ✕ 成り立たない

車はエンジンを**持って**いますが、エンジンの一種**ではありません**。
このような「持っている」関係（has-a の関係）は、継承ではなく、**フィールドとして持つ**のが正しい設計です（車のフィールドにエンジンを持たせる）。

> **覚えておくこと: is-a なら継承、has-a なら「持つ」**
>
> - **is-a**（〜は、…の一種である）→ 継承（`extends`）
> - **has-a**（〜は、…を持っている）→ フィールドとして持つ
>
> 「犬は動物の一種」（is-a）なので継承。「車はエンジンを持つ」（has-a）なのでフィールド。
> 迷ったら、「子 is a 親」と声に出してみて、自然かどうかで判断しましょう。

## まとめ

- **継承**は、共通部分を**親クラス**にまとめ、**子クラス**がそれを受け継ぐしくみ
- 共通コードを1か所にまとめられるので、無駄が減り、修正も1か所で済む
- 継承を使ってよいかは、**「子 is a 親」（子は親の一種）**が成り立つかで判断する
- 「持っている」関係（has-a）は継承ではなく、**フィールドとして持つ**
- クラスのつながりを **クラス階層**と呼ぶ（親が上、子が下）
