## super ― 親を呼ぶ

オーバーライドすると、親のメソッドは子のメソッドに置きかわりました。
でも、「**親の処理も使いたい。そのうえで、子の処理を足したい**」ということがよくあります。

そんなときに使うのが、**`super`**（スーパー）です。
`super` は、子クラスから**親クラス**を指すキーワードです。

---

## super.メソッド() ― 親のメソッドを呼ぶ

オーバーライドした子のメソッドの中で、**`super.メソッド名(...)`** と書くと、**親のほうのメソッド**を呼べます。

例で見てみましょう。`Animal` に、自己紹介する `describe()` があり、`Dog` がそれをオーバーライドします。
ただし、`Dog` は「親の自己紹介」をしたうえで、「犬である」ことを付け足したい、とします。

```java
class Animal {
    String name;
    void describe() {
        IO.println("名前: " + name);
    }
}

class Dog extends Animal {
    @Override
    void describe() {
        super.describe();           // まず、親の describe() を呼ぶ
        IO.println("種類: 犬");      // そのうえで、子の処理を足す
    }
}

public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.name = "ポチ";
        dog.describe();
    }
}
```

```text
$ java Main.java
名前: ポチ
種類: 犬
```

`Dog` の `describe()` の中で、`super.describe()` によって**親の `describe()`**（「名前: ポチ」を表示）が呼ばれ、そのあとに子の処理（「種類: 犬」）が実行されました。

もし `super.describe()` と書かず、ただ `describe()` と書いてしまうと、それは**自分自身の `describe()`** を呼ぶことになり、同じメソッドが何度も呼ばれ続けてしまいます（無限に呼び出してしまう）。
「**親のほうを呼びたいときは `super.`**」を、はっきり区別しましょう。

---

## なぜ super が便利なのか

`super` を使うと、「**親の処理を土台にして、子で機能を足す**」ことができます。
親の処理をコピーして書き直す必要がないので、

- 親と同じ処理を、二重に書かずにすむ
- 親の処理が変わっても、子は `super` 経由で自動的に新しい処理を使える

という利点があります。
オーバーライドは「親を完全に置きかえる」だけでなく、`super` と組み合わせて「**親に少し足す**」という使い方ができるのです。

---

## this と super の対応

第11章・第13章で学んだ `this` と、この `super` は、きれいに対応しています。

| | 指すもの | 例 |
|---|---|---|
| `this` | **自分自身**（このオブジェクト） | `this.name`、`this.メソッド()` |
| `super` | **親クラス**の部分 | `super.メソッド()`、`super(...)`（次の節） |

- `this.メソッド()` … 自分（子）のメソッドを呼ぶ
- `super.メソッド()` … 親のメソッドを呼ぶ

「自分を指すのが `this`、親を指すのが `super`」と、セットで覚えておきましょう。
次の節では、`super(...)` を使って、親の**コンストラクタ**を呼ぶ方法を学びます。

---

## まとめ

- **`super`** は、子クラスから**親クラス**を指すキーワード
- **`super.メソッド名(...)`** で、（オーバーライドする前の）**親のメソッド**を呼べる
- これにより、「親の処理を土台にして、子で機能を足す」ことができる
- `this`（自分自身）と `super`（親）は、対になっている

次の節では、継承とコンストラクタの関係 ―― `super(...)` で親を初期化するしくみを学びます。
