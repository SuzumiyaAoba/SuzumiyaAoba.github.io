## オーバーライド ― メソッドを作り直す

継承では、親から受け継いだメソッドを、子クラスで**作り直す**ことができます。
これを**オーバーライド**（Override、上書き）と呼びます。
継承を「らしく」使うための、とても重要なしくみです。

---

## 受け継いだメソッドを、子で作り直す

前の節では、`cry()` を `Dog` と `Cat` それぞれに**新しく追加**していました。
ですが、「すべての動物は鳴く（ただし鳴き方が違う）」と考えると、`cry()` は**動物に共通のふるまい**です。

そこで、`cry()` を親の `Animal` に持たせ、子クラスで**それぞれの鳴き方に作り直す**ほうが自然です。

```java
class Animal {
    String name;
    void cry() {
        IO.println(name + ": （動物の鳴き声）");   // 親の標準のふるまい
    }
}

class Dog extends Animal {
    @Override
    void cry() {
        IO.println(name + ": ワン");                // Dog 用に作り直す
    }
}

class Cat extends Animal {
    @Override
    void cry() {
        IO.println(name + ": ニャー");              // Cat 用に作り直す
    }
}
```

`Animal` の `cry()` は、標準の鳴き声を表示します。
`Dog` と `Cat` は、同じ `cry()` を**自分用に作り直して**います。これがオーバーライドです。

---

## @Override という目印

メソッドをオーバーライドするときは、その上に **`@Override`**（アットマーク・オーバーライド）と書きます。
これは **アノテーション**（Annotation、注釈）と呼ばれるもので、「**このメソッドは、親のメソッドをオーバーライドしていますよ**」と Java に伝える目印です。

`@Override` を付けると、よい効果があります。
もし、オーバーライドに失敗していたら（メソッド名のつづり間違いなど）、**コンパイルエラーで教えてくれる**のです。

```java
class Dog extends Animal {
    @Override
    void cri() {   // cry のつもりが cri（つづり間違い）
        IO.println("ワン");
    }
}
```

```text
エラー: メソッドはスーパータイプのメソッドをオーバーライドまたは実装しません
```

`@Override` がなければ、`cri()` は「Dog 独自の新しいメソッド」とみなされ、エラーになりません。
そして、`cry()` を呼んでも親の標準の鳴き声が出てしまい、「あれ、ワンと鳴かない？」と原因探しに苦労します。
`@Override` を付けておけば、こうしたうっかりミスを、コンパイルの段階で防げます。
**オーバーライドするときは、必ず `@Override` を付ける** ―― これを習慣にしましょう。

## オーバーライドした結果を確かめる

オーバーライドすると、子クラスのオブジェクトでは、**子のメソッドが呼ばれます**。

```java
class Animal {
    String name;
    void cry() { IO.println(name + ": （動物の鳴き声）"); }
}

class Dog extends Animal {
    @Override
    void cry() { IO.println(name + ": ワン"); }
}

class Cat extends Animal {
    @Override
    void cry() { IO.println(name + ": ニャー"); }
}

public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.name = "ポチ";
        dog.cry();              // Dog の cry() が呼ばれる

        Cat cat = new Cat();
        cat.name = "タマ";
        cat.cry();              // Cat の cry() が呼ばれる
    }
}
```

```text
$ java Main.java
ポチ: ワン
タマ: ニャー
```

`dog.cry()` では `Dog` の `cry()`、`cat.cry()` では `Cat` の `cry()` が呼ばれました。
親の標準の鳴き声（`（動物の鳴き声）`）は、どちらでも表示されていません。
**子クラスでオーバーライドすると、親のメソッドは、子のメソッドに置きかわる**のです。

## オーバーロードとの違い

名前が似ている**オーバーロード**（第9章・第13章）と混同しやすいので、整理しておきましょう。

| | オーバーロード（Overload） | オーバーライド（Override） |
|---|---|---|
| 意味 | 同じ名前で、**引数違い**の複数のメソッドを定義 | 親のメソッドを、子で**作り直す** |
| 場所 | 同じクラスの中 | 親クラスと子クラスの間 |
| 引数 | **違える**（個数・型） | 親と**同じ**にする |
| 例 | `add(int, int)` と `add(double, double)` | 親の `cry()` を子で `cry()` として作り直す |

- **オーバーロード**は「同じ名前で、いろいろな引数を受けられるようにする」（横に増やす）
- **オーバーライド**は「親から受け継いだメソッドを、子で作り直す」（縦に上書きする）

名前は似ていますが、別のしくみです。「ロード（積む）」と「ライド（乗りかえる）」で区別すると、覚えやすいかもしれません。

---

## まとめ

- **オーバーライド**は、親から受け継いだメソッドを、子クラスで**作り直す**こと
- 親と**同じ名前・同じ引数**でメソッドを定義すると、オーバーライドになる
- オーバーライドには、必ず **`@Override`** を付ける（つづり間違いなどをコンパイル時に防げる）
- 子クラスのオブジェクトでは、**オーバーライドした子のメソッド**が呼ばれる
- **オーバーロード**（引数違いを増やす）とは別物
