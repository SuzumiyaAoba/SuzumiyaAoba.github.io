## よくあるつまずき

継承で、初心者がつまずきやすいポイントを、まとめて確認します。

---

## 1. 何でも継承で表そうとする

継承は便利なので、つい「共通点があれば継承」としたくなります。
しかし、第1節で学んだとおり、継承を使ってよいのは **「子 is a 親」（子は親の一種）**が成り立つときだけです。

```java
class Engine { ... }
class Car extends Engine { ... }   // ✕ 車はエンジンの一種ではない
```

「車はエンジンを**持って**いる」（has-a）のであって、「車はエンジンの一種」（is-a）ではありません。
このような has-a の関係は、継承ではなく、**フィールドとして持つ**のが正しい設計です。

```java
class Car {
    Engine engine;   // ◯ 車はエンジンを「持つ」
}
```

「is-a なら継承、has-a なら持つ」を、いつも確認しましょう。
継承は強力なぶん、無理に使うと、かえって関係が分かりにくくなります。

---

## 2. オーバーライドのつもりが、別メソッドになっている

オーバーライドは、親と**同じ名前・同じ引数**でなければなりません。
名前や引数がずれていると、オーバーライドにならず、「子の新しいメソッド」になってしまいます。

```java
class Animal { void cry() { ... } }

class Dog extends Animal {
    void cry(int times) { ... }   // ✕ 引数が違う → オーバーライドではなく、別物（オーバーロード）
}
```

`cry(int times)` は引数が違うので、親の `cry()` のオーバーライドにはなりません。
こうしたうっかりを防ぐため、**オーバーライドには必ず `@Override` を付けます**。
`@Override` を付けておけば、オーバーライドになっていない場合に、コンパイルエラーで教えてくれます（第3節）。

---

## 3. super を付け忘れて、自分を呼んでしまう

オーバーライドしたメソッドの中で、親のメソッドを呼ぶつもりが、`super.` を忘れると、**自分自身**を呼んでしまいます。

```java
class Dog extends Animal {
    @Override
    void describe() {
        describe();        // ✕ super. を忘れた → 自分の describe() を無限に呼ぶ
    }
}
```

これは、自分の `describe()` が自分の `describe()` を呼び続ける、無限の呼び出しになります（やがてエラーで止まります）。
親のメソッドを呼びたいときは、必ず **`super.メソッド()`** と書きましょう。

---

## 4. 親に引数なしコンストラクタがなく、super(...) を忘れる

第5節で学んだとおり、親に引数なしコンストラクタがないと、子は `super(...)` で親を明示的に初期化する必要があります。

```java
class Animal {
    Animal(String name) { ... }   // 引数つきだけ
}

class Dog extends Animal {
    Dog() { }                      // ✕ super(...) がない → エラー
}
```

「親のコンストラクタに、何を渡せばよいか」を考え、子のコンストラクタで `super(...)` を呼びましょう。
このエラーは、「親をどう初期化するか決めていない」というサインです。

---

## 5. 継承を深くしすぎる

継承は、何段にも重ねられます（`Object` → `Animal` → `Dog` → `Puppy` …）。
しかし、継承が深くなりすぎると、「あるメソッドが、どのクラスで定義されているのか」を追うのが大変になり、かえって分かりにくくなります。

- 継承は、**浅く**保つのが基本（深くても2〜3段まで、と意識する）
- 「共通機能を使いまわしたい」だけなら、継承よりも、**他のオブジェクトを持って使う**（has-a・委譲）ほうが、すっきりすることも多い

実際の開発では、「**継承よりも、まず持つこと（コンポジション）を検討する**」という考え方が広く知られています。
継承は便利ですが、「is-a が本当に成り立つか」「深くなりすぎないか」を意識して、慎重に使いましょう。

---

## まとめ

- 継承は **is-a が成り立つときだけ**。has-a は継承ではなく「持つ」
- オーバーライドは**同じ名前・同じ引数**。`@Override` を付けてミスを防ぐ
- 親のメソッドは `super.メソッド()`。`super.` を忘れると自分を呼んでしまう
- 親に引数なしコンストラクタがないときは、子で `super(...)` を呼ぶ
- 継承は**浅く**保つ。使いまわしだけなら「持つこと（コンポジション）」も検討する

次は、この章で学んだ言葉を、用語集としてまとめます。
