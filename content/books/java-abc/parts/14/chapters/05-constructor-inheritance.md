## コンストラクタと継承 ― super(...) で親を初期化する

継承では、子クラスのオブジェクトを作るとき、**親クラスの部分も初期化**する必要があります。
この節では、コンストラクタと継承の関係、そして親のコンストラクタを呼ぶ **`super(...)`** を学びます。

---

## 子を作ると、親のコンストラクタも呼ばれる

まず、大事な事実から。
子クラスのオブジェクトを `new` すると、**親クラスのコンストラクタが、先に呼ばれます**。

確かめてみましょう。

```java
class Animal {
    Animal() {
        IO.println("Animal のコンストラクタ");
    }
}

class Dog extends Animal {
    Dog() {
        IO.println("Dog のコンストラクタ");
    }
}

public class Main {
    public static void main(String[] args) {
        new Dog();
    }
}
```

```text line-numbers=false
$ java Main.java
Animal のコンストラクタ
Dog のコンストラクタ
```

`new Dog()` をしただけなのに、先に `Animal` のコンストラクタが呼ばれ、そのあとに `Dog` のコンストラクタが呼ばれています。
これは、「**まず親の部分を組み立ててから、子の部分を組み立てる**」という、自然な順番です。
家を建てるとき、土台（親）を作ってから、その上に建物（子）を作るのと同じです。

実は、子のコンストラクタの先頭には、`super();`（親の引数なしコンストラクタを呼ぶ）が**自動で補われています**。
だから、何も書かなくても、親のコンストラクタが先に呼ばれるのです。

---

## super(...) ― 親のコンストラクタに値を渡す

親のコンストラクタが**引数を必要とする**場合は、`super(...)` を使って、子から**明示的に**値を渡します。

`Animal` が、名前を受け取るコンストラクタを持つ場合で見てみましょう。

```java
class Animal {
    String name;

    Animal(String name) {        // 名前を受け取るコンストラクタ
        this.name = name;
    }
}

class Dog extends Animal {
    Dog(String name) {
        super(name);             // 親のコンストラクタに name を渡す
    }

    void cry() {
        IO.println(name + ": ワン");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog("ポチ");
        dog.cry();
    }
}
```

```text line-numbers=false
$ java Main.java
ポチ: ワン
```

`Dog("ポチ")` の中で `super(name)` を呼び、親 `Animal` のコンストラクタに `"ポチ"` を渡しています。
これで、`Animal` の `name` フィールドが、きちんと初期化されました。
`super(...)` は、第13章で学んだ `this(...)`（同じクラスの別コンストラクタを呼ぶ）の、**親バージョン**です。

---

## 親に引数なしコンストラクタがないと、super(...) が必須

注意が必要なのは、**親クラスに「引数なしのコンストラクタ」がない**場合です。

子のコンストラクタで `super(...)` を書かないと、自動で `super();`（引数なし）が補われます。
ところが、親に引数なしコンストラクタがないと、その自動の `super();` が呼べず、エラーになります。

```java
class Animal {
    String name;
    Animal(String name) { this.name = name; }   // 引数つきだけ（引数なしがない）
}

class Dog extends Animal {
    Dog() {
        // super(...) を書いていない → 自動の super() が呼べずエラー
    }
}
```

```text line-numbers=false
エラー: クラス Animalのコンストラクタ Animalは指定された型に適用できません。
  期待値: String
  検出値:    引数がありません
```

第13章で学んだ「コンストラクタを書くと、デフォルトコンストラクタ（引数なし）は用意されない」を思い出してください。
`Animal` は引数つきコンストラクタだけなので、引数なしの `Animal()` は存在しません。
そのため、子は `super(name)` のように、**親のコンストラクタを明示的に呼ぶ**必要があります。

「親に引数なしコンストラクタがないなら、子は `super(...)` で親を初期化する」と覚えておきましょう。

> **補足: `super(...)` を書く位置**
>
> `super(...)` は、`this(...)` と同じく、子コンストラクタの**初期化処理より前**に書きます（基本は先頭）。
> 1つのコンストラクタで、`super(...)` と `this(...)` を両方書くことはできません（どちらか一方です）。
> （Java 25 では、インスタンスを使わない準備の文なら `super(...)` の前に書けますが、初心者のうちは『先頭に書く』を基本にすれば十分です。）

---

## コンストラクタは継承されない

最後に、間違えやすい点を1つ。
フィールドやメソッドは継承されますが、**コンストラクタは継承されません**。

`Animal(String name)` というコンストラクタがあっても、`Dog` がそれを自動で受け継ぐわけではありません。
`Dog` で `new Dog("ポチ")` のように作りたいなら、`Dog` に `Dog(String name)` コンストラクタを自分で書き、その中で `super(name)` を呼ぶ必要があります。
「コンストラクタは、各クラスで用意し、`super(...)` で親へつなぐ」と考えましょう。

---

## まとめ

- 子のオブジェクトを作ると、**親のコンストラクタが先に呼ばれる**（土台から組み立てる）
- 子コンストラクタの先頭には、自動で `super();`（親の引数なし）が補われる
- 親のコンストラクタに値を渡すには、**`super(...)`** を使う（`this(...)` の親バージョン）
- 親に引数なしコンストラクタがないときは、子で **`super(...)` が必須**
- **コンストラクタは継承されない**。各クラスで用意し、`super(...)` で親につなぐ
