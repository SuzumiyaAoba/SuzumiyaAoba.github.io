## 用語集 ― この章で学んだ言葉

### JVM（Java Virtual Machine、Java仮想マシン）

バイトコードを実行する「仮想的なコンピュータ」。
OS や CPU の違いを吸収して、`.class` を**どこでも同じように**動かす。

### バイトコード（bytecode）

`.class` に書かれる、JVM 用の**仮想機械語**。
1 命令が原則 1 バイトで、スタックマシン用に設計されている。
`javap -c` で読める。

### マジックナンバ（magic number）

`.class` の先頭 4 バイトに必ず置かれる印 `0xCAFEBABE`。
ファイル形式の判別に使う。

### major version / minor version

`.class` のバージョン番号。Java 25 の major version は **69**。
古い JVM が新しいクラスを読もうとすると `UnsupportedClassVersionError`。

### 定数プール（constant pool）

クラスファイル内で使う**文字列・クラス名・メソッド参照**などを 1 つに集めた表。
バイトコードは「**動詞（命令）＋ 番号（定数プールへの参照）**」で動く。

### `<init>` / `<clinit>`

- `<init>` ― **インスタンスのコンストラクタ**の正式名（バイトコードレベル）
- `<clinit>` ― **クラスの初期化**メソッド（static イニシャライザと static フィールドの代入）

### クラスローダ（Class Loader）

`.class` のバイトを読み込み、`Class<?>` オブジェクトを作る仕組み。
3 階層（**Bootstrap・Platform・Application**）で構成される。

### 親委譲モデル（Parent Delegation Model）

クラスローダが「クラスを読む」前に、**まず親に聞く**ルール。
同じクラス名なら**常に同じ型**になり、`String` のすり替えなどを防ぐ。

### ローディング・リンキング・初期化

クラスが「使える状態」になるまでの 3 段階。

1. **ローディング** ― バイトを読み、`Class<?>` を作る
2. **リンキング** ― 検証・準備・解決
3. **初期化** ― `<clinit>` を実行する

「**初めて使われたとき**」に最後まで走る。

### ランタイムデータ領域（Runtime Data Areas）

実行中の JVM が使うメモリ領域の総称。

- **ヒープ** ― オブジェクト置き場（全スレッド共有）
- **メソッド領域**（**Metaspace**）― クラス情報・static フィールド（全スレッド共有）
- **JVM スタック** ― メソッド呼び出しのフレーム（スレッドごと）
- **PC レジスタ** ― 次に実行するバイトコード位置（スレッドごと）
- **ネイティブメソッドスタック** ― ネイティブ呼び出し用

### フレーム（Frame）

JVM スタックに 1 つずつ積まれる、**メソッド呼び出し 1 回分**の作業領域。
ローカル変数領域・オペランドスタック・戻り先を持つ。

### Metaspace（メタスペース）

Java 8 以降の**メソッド領域**の実装名。
ヒープではなく**ネイティブメモリ**に置かれる。`-XX:MaxMetaspaceSize` で上限制御。

### 実行エンジン（Execution Engine）

バイトコードを実行する JVM 内の部品。
**インタプリタ**と **JIT コンパイラ**から成る。

### JIT（Just-In-Time）コンパイラ

実行時に、**ホットなバイトコードをネイティブコードに翻訳**する仕組み。
HotSpot JVM では **C1**（速いが弱い最適化）と **C2**（遅いが強い最適化）の 2 段構え。

### 階層的コンパイル（Tiered Compilation）

インタプリタ（Tier 0）→ C1（Tier 1〜3）→ C2（Tier 4）と段階的にコンパイルする方式。
立ち上がりの速さと、最終的な性能の両立を狙う。

### ウォームアップ（warmup）

JIT が翻訳と最適化を**走らせている過渡期**。
ベンチマークでは**最初の数回を捨てる**のが鉄則。

### コードキャッシュ（CodeCache）

JIT が翻訳したネイティブコードを置く、ネイティブメモリ上の領域。
溢れると JIT が止まり、性能が落ちる。

### インライン化（Inlining）

メソッド呼び出しを呼び出し先の本体で**置き換える**最適化。
JIT の効果の中でも、特に効きが大きい。

### エスケープ解析（Escape Analysis）

そのオブジェクトがメソッドの外に**漏れない**ことが分かれば、ヒープではなく**スタックに置く**最適化。

### `ClassNotFoundException` / `NoClassDefFoundError` / `UnsupportedClassVersionError`

クラス読み込み関連のエラー。

- **`ClassNotFoundException`** ― `Class.forName(...)` 等で**明示的に読もうとした**ときに見つからない
- **`NoClassDefFoundError`** ― 読み込み or リンク or 初期化が**失敗**した
- **`UnsupportedClassVersionError`** ― クラスファイルのバージョンが、実行 JVM より**新しい**

### フィールドディスクリプタ／メソッドディスクリプタ

JVM 内部での**型表記**。
プリミティブは `I`/`J`/`D` など 1 文字、参照型は `LClassName;`、配列は `[` を頭につける。
メソッドは `(引数...)戻り値`、例: `int sum(int)` は `(I)I`。
