## throw と throws ― 例外を投げる・宣言する

ここまでは、（標準ライブラリなどが起こす）例外を**捕まえる**側でした。
この節では、**自分から例外を起こす（投げる）**側を学びます。
キーワードは、よく似た2つ ―― **`throw`**（投げる）と **`throws`**（宣言する）です。

---

## throw ― 例外を投げる

**`throw`** を使うと、自分で例外を発生させられます。
書き方は、`throw new 例外の型(...)` です。

第12章・第13章で、不正な値を「チェックして、はじく」方法を学びました。
そこでは、おかしな値を `0` に補正したり、メッセージを表示したりしていました。
ですが、より厳密には「**不正な値なら、例外を投げて、処理を止める**」のが、Java らしいやり方です。

年齢を設定するメソッドで、負の値が来たら例外を投げてみましょう。

```java
class Person {
    private int age;

    public void setAge(int age) {
        if (age < 0) {
            throw new IllegalArgumentException("年齢に負の値は設定できません: " + age);
        }
        this.age = age;
    }
}
```

`throw new IllegalArgumentException(...)` で、「不正な引数だ」という例外を発生させています。
`IllegalArgumentException`（不正な引数の例外）は、「引数の値がおかしい」ことを表す、標準の例外です。
`( )` に渡した文字列が、例外のメッセージ（`getMessage()` で取り出せるもの）になります。

`throw` が実行されると、その時点で**メソッドの処理は中断**され、例外が呼び出し元へ伝わっていきます（捕まえられなければ、プログラムが止まります）。

---

## 投げた例外を、捕まえる

投げた例外は、呼び出し側で try-catch で捕まえられます。

```java
class Person {
    private int age;
    public void setAge(int age) {
        if (age < 0) {
            throw new IllegalArgumentException("年齢に負の値は設定できません: " + age);
        }
        this.age = age;
    }
    public int getAge() { return age; }
}

public class Main {
    public static void main(String[] args) {
        Person p = new Person();
        try {
            p.setAge(-5);                      // 不正な値 → 例外が投げられる
            IO.println("設定できました");       // ここは実行されない
        } catch (IllegalArgumentException e) {
            IO.println("エラー: " + e.getMessage());
        }
    }
}
```

```text
$ java Main.java
エラー: 年齢に負の値は設定できません: -5
```

`setAge(-5)` が例外を投げ、それを呼び出し側の `catch` が捕まえました。
「ありえない値を、はっきりエラーにする」ことで、おかしな状態のまま処理が進むのを防げます。
そして、その対応（エラーメッセージの表示など）を、呼び出し側で決められるのです。

> **補足: なぜ「補正」より「例外」がよいことが多いのか**
>
> 第13章では、負の年齢を `0` に補正していました。これも1つの方法です。
> ですが、「こっそり `0` に直す」と、呼び出し側は「おかしな値を渡した」ことに気づけません。
> 例外を投げれば、「その値はダメだ」とはっきり伝わり、バグを早く見つけられます。
> 「**ありえない値は、だまって直さず、はっきりエラーにする**」 ―― これが、堅牢なプログラムの考え方です。

## throws ― 例外を投げることを宣言する

`throw` が「例外を投げる」のに対し、**`throws`**（s が付く）は、「このメソッドは、例外を投げるかもしれません」と**宣言**するものです。
メソッドの宣言の、`( )` のうしろに書きます。

これが必要になるのは、前の節で学んだ**チェック例外**を扱うときです。
チェック例外は「処理（捕捉）するか、宣言するか」のどちらかが必須でした。
メソッドの中で**捕まえずに、呼び出し元に任せたい**ときは、`throws` で「投げるかもしれない」と宣言します。

```java
// このメソッドは InterruptedException（チェック例外）を投げるかもしれない、と宣言
static void waitOneSecond() throws InterruptedException {
    Thread.sleep(1000);   // catch しない代わりに、throws で宣言
}
```

`throws InterruptedException` と宣言することで、`Thread.sleep` のチェック例外を、このメソッドの中で `catch` せずにすみます。
そのかわり、`waitOneSecond()` を**呼ぶ側**が、その例外を処理する責任を負います（catch するか、さらに `throws` で上へ伝えるか）。

「例外を、**ここで処理する**（try-catch）」か、「**呼び出し元に任せる**（throws）」か ―― チェック例外には、この2つの選択肢があるのです。

---

## throw と throws の違い

よく似た2つを、整理しておきましょう。

| | 役割 | 書く場所 |
|---|---|---|
| **`throw`** | 例外を、実際に**投げる** | メソッドの中（`throw new ...`） |
| **`throws`** | 例外を投げるかもしれない、と**宣言する** | メソッドの宣言（`( )` のうしろ） |

- `throw`（s なし） … 動詞。「いま、投げる」
- `throws`（s あり） … 宣言。「投げることがあります」

1文字違いですが、役割はまったく違います。混同しないようにしましょう。

---

## まとめ

- **`throw`** で、自分から例外を投げられる（`throw new IllegalArgumentException("...")`）
- 投げた例外は、呼び出し側の try-catch で捕まえられる
- 「ありえない値は、だまって直さず、**はっきり例外にする**」のが、堅牢なやり方
- **`throws`** は、「このメソッドは例外を投げるかも」という**宣言**（チェック例外で必要）
- チェック例外は、「ここで処理（try-catch）」か「呼び出し元に任せる（throws）」かを選ぶ
- `throw`（投げる）と `throws`（宣言する）は、1文字違いだが別物
