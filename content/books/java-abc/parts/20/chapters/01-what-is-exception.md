## 例外とは ― 実行中に起きる問題

**例外**（Exception）とは、**プログラムの実行中に起きる、予期しない問題**のことです。
この節では、例外が発生すると何が起こるのか、そしてスタックトレースの読み方を学びます。

---

## コンパイルエラーとは違う

第3章で、**コンパイルエラー**を学びました。あれは、「実行する前の段階で見つかる、文法の間違い」でした。
**例外は、それとは違います。** 文法は正しく、コンパイルも成功するのに、**実行している最中に起きる**問題です。

たとえば、`10 ÷ 0`（0 での割り算）を考えます。
これは文法的には正しいので、コンパイルは通ります。ですが、実行すると問題が起きます。

```java
public class Main {
    public static void main(String[] args) {
        int a = 10;
        int b = 0;
        int result = a / b;          // 0 で割る
        IO.println("計算結果: " + result);
    }
}
```

```text
$ java Main.java
Exception in thread "main" java.lang.ArithmeticException: / by zero
	at Main.main(Main.java:5)
```

`ArithmeticException`（算術例外）という例外が発生しました。
`/ by zero`（0 での割り算）と、原因も書かれています。
これが「実行中に起きる問題」＝例外です。
（第5章で「0 での割り算はエラーになる。くわしくは第20章で」と予告したのが、これです。）

---

## 例外が起きると、処理が中断される

注目してほしいのは、出力に **「計算結果: ...」が表示されていない**ことです。

例外が発生すると、**その時点で処理が中断**され、それ以降の行は実行されません。
`int result = a / b;` で例外が起きたので、その次の `IO.println("計算結果: ...")` には、たどり着けなかったのです。

そして、例外が**最後まで対応されない**と、プログラムは異常終了します。
このとき表示されるのが、`Exception in thread "main" ...` から始まる、例外の情報です。
何も対策しなければ、ささいな問題1つで、プログラム全体が止まってしまうのです。

---

## スタックトレースを読む

例外が発生したときに表示される情報を、**スタックトレース**（Stack Trace）と呼びます。
これは、「**どこで、どんな例外が起きたか**」をたどるための、大切な手がかりです。

```text
Exception in thread "main" java.lang.ArithmeticException: / by zero
	at Main.main(Main.java:5)
```

1行ずつ読み解きましょう。

| 部分 | 意味 |
|---|---|
| `Exception in thread "main"` | `main` の実行中に、対応されない例外が起きた |
| `java.lang.ArithmeticException` | 起きた例外の**種類** |
| `/ by zero` | 例外の**メッセージ**（原因の説明） |
| `at Main.main(Main.java:5)` | 例外が起きた**場所**（`Main.java` の5行目） |

特に大事なのは、最後の `at ...(ファイル名:行番号)` です。
ここを見れば、「どのファイルの、何行目で例外が起きたか」が分かります。
複数の `at ...` が並ぶこともありますが（メソッドが別のメソッドを呼んでいる場合）、ふつうは**いちばん上の行**が、問題が起きた場所です。

> **スタックトレースは、こわくない**
>
> 赤い文字でずらりと表示されると、身構えてしまうかもしれません。
> ですが、スタックトレースは「**どこで何が起きたか、ちゃんと教えてくれる、親切なメッセージ**」です（第3章のコンパイルエラーと同じですね）。
> まずは「いちばん上の例外の種類とメッセージ」「`at` の行番号」の2つを読むことから始めましょう。
> それだけで、原因の見当が、かなりつくようになります。

## いろいろな例外

これまでの章で出会った例外を、振り返ってみましょう。どれも `〜Exception` という名前でした。

| 例外 | いつ起きるか | 登場した章 |
|---|---|---|
| `ArithmeticException` | 0 で割ったとき | この章 |
| `ArrayIndexOutOfBoundsException` | 配列の範囲外にアクセス | 第8章 |
| `NullPointerException` | `null` を操作 | 第11章 |
| `ClassCastException` | 不正なダウンキャスト | 第15章 |
| `NumberFormatException` | 数値に変換できない文字列 | この章（次節） |

これらは、すべて例外の仲間です。
共通しているのは、「**実行中に発生し、対応しないとプログラムが止まる**」という点です。
次の節からは、これらの例外を**捕まえて対応する**方法を学んでいきます。

---

## まとめ

- **例外**は、**実行中に起きる、予期しない問題**（コンパイルエラーとは別）
- 例外が起きると、**その時点で処理が中断**され、対応されないとプログラムが止まる
- 例外が起きると、**スタックトレース**が表示される
- スタックトレースでは、**例外の種類・メッセージ**と、**`at` の行番号**を読む
- `ArithmeticException`・`NullPointerException` など、これまで見てきた `〜Exception` は、すべて例外

次の節では、例外を捕まえて対応する **try-catch** を学びます。
