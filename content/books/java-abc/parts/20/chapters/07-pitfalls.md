## よくあるつまずき

例外処理で、初心者がつまずきやすいポイントを、まとめて確認します。
例外処理は「書けること」以上に、「**正しく使うこと**」が大切です。

---

## 1. 例外を握りつぶす（catch で何もしない）

もっとも避けたいのが、例外を `catch` したのに、**何もしない**ことです。

```java
try {
    int n = Integer.parseInt(input);
} catch (NumberFormatException e) {
    // 何も書かない（握りつぶし）
}
```

これは**例外の握りつぶし**と呼ばれ、とても危険です。
例外が起きても、エラーがなかったかのように、こっそり処理が続いてしまいます。
すると、「なぜか結果がおかしいのに、エラーも出ない」という、もっとも原因を追いにくいバグになります。

`catch` の中では、**最低でも何が起きたかを記録・表示**しましょう。

```java
} catch (NumberFormatException e) {
    IO.println("数値に変換できませんでした: " + e.getMessage());
}
```

「捕まえたなら、必ず何か対応する（少なくとも記録する）」を、徹底しましょう。

---

## 2. catch の範囲を、広げすぎる

「とりあえず全部捕まえればいい」と、`catch (Exception e)` で、すべての例外をまとめて捕まえるのは、考えものです。

```java
try {
    // いろいろな処理
} catch (Exception e) {   // △ あらゆる例外をまとめて捕まえる
    IO.println("何かエラーが起きました");
}
```

これだと、本来は別々に対応すべき例外（数値変換の失敗、配列の範囲外…）が、ひとまとめにされてしまいます。
「何かエラー」としか分からず、適切な対応ができません。
基本は、**対応したい例外を、具体的に指定して捕まえる**ようにしましょう（`catch (NumberFormatException e)` など）。

---

## 3. 非チェック例外を、try-catch で隠す

`NullPointerException` のような非チェック例外（プログラムの誤り）を、try-catch で**隠す**のは、よくない使い方です。

```java
try {
    IO.println(name.length());   // name が null かもしれない
} catch (NullPointerException e) {
    // 握りつぶす
}
```

これは、「`null` チェックをサボって、起きた例外を隠している」状態です。
非チェック例外は、第4節で学んだとおり、**コードを直して、そもそも起きないようにする**のが正しい対応です。

```java
if (name != null) {       // まず null かどうか確認する
    IO.println(name.length());
}
```

「try-catch は、エラーをもみ消す道具ではない」と心に留めましょう。

---

## 4. throw と throws を取り違える

第5節で見たとおり、`throw`（投げる）と `throws`（宣言する）は別物です。

```java
throw new IllegalArgumentException("...");   // 例外を投げる（メソッドの中）
void foo() throws IOException { ... }          // 例外を宣言する（メソッドの宣言）
```

- `throw`（s なし） … その場で例外を**投げる**
- `throws`（s あり） … 投げるかも、と**宣言する**

「投げるなら `throw`、宣言なら `throws`」と、区別しましょう。

---

## 5. 例外を、ふつうの分岐がわりに使う

例外は「例外的な（めったに起きない）事態」のためのものです。
ふつうに起こりうる分岐を、例外で表すのは、適切ではありません。

```java
// △ ありがちな「ない場合」を、例外で表してしまう
try {
    String value = map.get(key);
    IO.println(value.length());
} catch (NullPointerException e) {
    IO.println("見つかりませんでした");
}
```

「キーが見つからない」は、ふつうに起こりうることです。
こういう場合は、例外に頼らず、`if` で**事前に確認**するのが基本です。

```java
if (value != null) {
    IO.println(value.length());
} else {
    IO.println("見つかりませんでした");
}
```

「例外は、あくまで**例外的な事態**のために使う。ふつうの分岐は `if` で」と覚えておきましょう。

---

## まとめ

- 例外を **握りつぶさない**（`catch` で何もしないのは厳禁。最低でも記録・表示する）
- `catch (Exception e)` で**広く捕まえすぎない**。対応したい例外を具体的に指定する
- 非チェック例外（プログラムの誤り）は、try-catch で隠さず、**コードを直す**
- `throw`（投げる）と `throws`（宣言する）を取り違えない
- 例外は**例外的な事態**のために使う。ふつうの分岐は `if` で

次は、この章で学んだ言葉を、用語集としてまとめます。
