## 要素とインデックス

配列に入れた値は、**インデックス（添字）**という番号を使って、1つずつ取り出したり、書き換えたりします。

---

## インデックスは 0 から始まる

配列でいちばん大切なルールは、**インデックスは `0` から始まる**ということです。

```text line-numbers=false
scores:  [ 80 ][ 95 ][ 70 ][ 60 ][ 100 ]
   番号:    0     1     2     3      4
```

要素が5個ある配列なら、インデックスは `0`・`1`・`2`・`3`・`4` です。
**最初の要素が `0` 番、最後の要素が `4` 番**になります。「1番から」ではない点に、特に注意しましょう。

要素が5個なら、最後のインデックスは `5` ではなく **`4`**（＝個数 − 1）です。
この「1つずれる」感覚は、はじめは戸惑いますが、配列の基本なので、少しずつ慣れていきましょう。

---

## 要素を取り出す

要素を取り出すには、`配列名[インデックス]` と書きます。

```java
int[] scores = {80, 95, 70, 60, 100};
```

それぞれの要素を取り出してみましょう。

```text line-numbers=false
jshell> scores[0]
$2 ==> 80

jshell> scores[1]
$3 ==> 95

jshell> scores[4]
$4 ==> 100
```

`scores[0]` が最初の要素 `80`、`scores[4]` が最後の要素 `100` です。
`scores[インデックス]` は、ふつうの `int` の値として、計算や表示にそのまま使えます。

```java
int first = scores[0];
IO.println("1人目の点数は " + scores[0] + " 点です");
```

```text line-numbers=false
1人目の点数は 80 点です
```

---

## 要素を書き換える

`配列名[インデックス]` は、`=` の左に置くと、その要素を**書き換える**こともできます。

```java
int[] scores = {80, 95, 70, 60, 100};
scores[0] = 88;
```

書き換えたあとの配列を確認してみましょう。

```text line-numbers=false
jshell> scores
scores ==> int[5] { 88, 95, 70, 60, 100 }
```

0番の要素が、`80` から `88` に書き換わりました。

第2節で `new int[5]` を使って空の配列を作りましたが、その配列に値を入れるのも、この書き換えと同じやり方です。

```java
int[] scores = new int[3];
scores[0] = 80;
scores[1] = 95;
scores[2] = 70;
```

```text line-numbers=false
jshell> scores
scores ==> int[3] { 80, 95, 70 }
```

`0` で埋まっていた配列に、1つずつ値を入れていけました。

---

## まとめ

- 要素は `配列名[インデックス]` で取り出す・書き換える
- インデックスは **`0` から始まる**（最初が `0` 番、最後は「個数 − 1」番）
- `scores[0]` は、ふつうの値として計算や表示に使える
- `scores[0] = 88;` のように、`=` の左に置くと要素を書き換えられる

ここまでは、要素を1つずつ手で指定してきました。
