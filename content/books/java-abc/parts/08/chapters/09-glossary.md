## 用語集 ― この章で学んだ言葉

この第8章で登場した言葉を、まとめておきます。

---

## 配列の基本（第1〜4節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| 配列 | Array | 同じ型の値を並べて、1つの名前でまとめて扱うしくみ |
| 要素 | Element | 配列の中に入っている1つ1つの値 |
| インデックス（添字） | Index | 要素の位置を表す番号。**0 から始まる** |
| 初期化子 | ― | `{80, 95, 70}` のように、値を並べて配列を作る書き方 |
| length | レングス | 配列の要素数を表す（`配列名.length`、**かっこなし**） |

---

## 走査と応用（第5〜7節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| 拡張 for 文（for-each） | Enhanced for | `for (型 変数 : 配列)` で全要素を順に取り出す書き方 |
| 多次元配列 | Multidimensional Array | 配列の中に配列を入れたもの。2次元配列（`int[][]`）が代表 |
| Arrays | アレイズ | 配列を便利に扱う道具（`Arrays.toString`・`Arrays.sort` など） |
| コマンドライン引数 | Command-line Arguments | `main(String[] args)` の `args` で受け取る、起動時に渡す文字列 |

---

## つまずき関連（第8節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| ArrayIndexOutOfBoundsException | ― | 配列の範囲を超えた番号にアクセスしたときに出るエラー（例外） |
| オフバイワン | Off-by-one | 回数や番号が1つずれる間違い（`<` と `<=` の取り違えなど） |
| null | ヌル | 「何もない」状態。`new` で作った参照型の配列の初期値 |

---

## おわりに

この第8章では、たくさんのデータをまとめて扱う**配列**を学びました。

- 配列は、同じ型の値を並べ、**インデックス（0 始まり）**で1つずつ出し入れする
- `length` と **繰り返し**を組み合わせれば、全要素を一気に処理できる（合計・最大値など）
- 読むだけなら **for-each**、番号や書き換えが必要なら通常の for
- **多次元配列**で表やマス目を、**`Arrays`** で表示や並べ替えを扱える

配列は、第7章の繰り返しと並んで、プログラムの土台となる道具です。
「たくさんのデータを、1つずつ順に処理する」というパターンは、これから何度も登場します。

次の第9章「**メソッド**」では、一連の処理に名前をつけて、何度も呼び出せる「部品」にする方法を学びます。
これまで `IO.println` や `Arrays.sort` のように「呼び出して使ってきた」しくみを、いよいよ自分で作れるようになります。
