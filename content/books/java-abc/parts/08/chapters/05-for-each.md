## 拡張 for 文（for-each）

前節では、`for (int i = 0; i < scores.length; i++)` のように、インデックス `i` を使って配列をたどりました。
しかし「全要素を、順に取り出すだけ」なら、もっと簡潔に書ける専用の構文があります。
それが**拡張 for 文**（Enhanced for statement）、通称 **for-each**（フォーイーチ）です。

---

## for-each の形

```java
for (型 変数 : 配列) {
    // 取り出した要素を使う処理
}
```

実際に使ってみましょう。

```java
int[] scores = {80, 95, 70};
for (int s : scores) {
    IO.println(s);
}
```

```text
80
95
70
```

`for (int s : scores)` は、「**`scores` の中の各要素を、1つずつ `s` に取り出して繰り返す**」という意味です。
`:`（コロン）を「〜の中の」と読み、「scores の中の各要素 s について」と読むとわかりやすいでしょう。

前節の書き方と比べてみましょう。

```java
// 通常の for（インデックスを使う）
for (int i = 0; i < scores.length; i++) {
    IO.println(scores[i]);
}

// for-each（要素を直接受け取る）
for (int s : scores) {
    IO.println(s);
}
```

for-each には、こんな利点があります。

- インデックス（`i`）や `length` を書かなくてよい
- `scores[i]` のような添字アクセスがないので、**範囲外アクセスの心配がない**
- 「全部を順に処理する」という意図が、ひと目で伝わる

合計を求める処理も、for-each ですっきり書けます。

```java
int[] scores = {80, 95, 70};
int total = 0;
for (int s : scores) {
    total += s;
}
```

```text
jshell> total
total ==> 245
```

## for-each が使えない・向かない場面

便利な for-each ですが、万能ではありません。次のような場面では、通常の for を使います。

**1. 「何番目か」を知りたいとき**

for-each は要素そのものを受け取るだけで、インデックスを教えてくれません。
「3番目だけ特別扱いしたい」「番号を一緒に表示したい」ようなときは、通常の for を使います。

**2. 要素を書き換えたいとき**

for-each のループ変数は、要素の**コピー**です。そのため、ループ変数に代入しても、**元の配列は変わりません**。

```java
int[] nums = {1, 2, 3};
for (int x : nums) {
    x = x * 2;   // x は変わるが…
}
```

```text
jshell> nums
nums ==> int[3] { 1, 2, 3 }   // 元の配列は変わらない！
```

配列の中身を書き換えたいときは、インデックスを使う通常の for で `nums[i]` に代入します。

```java
for (int i = 0; i < nums.length; i++) {
    nums[i] = nums[i] * 2;
}
```

```text
jshell> nums
nums ==> int[3] { 2, 4, 6 }   // こちらは書き換わる
```

## 使い分けのまとめ

| やりたいこと | 向いている書き方 |
|---|---|
| 全要素を、順に読むだけ | **for-each**（`for (int s : scores)`） |
| 何番目かを使いたい | 通常の for（インデックス `i`） |
| 要素を書き換えたい | 通常の for（`nums[i] = ...`） |

迷ったら、「**読むだけなら for-each、番号や書き換えが必要なら通常の for**」と覚えておきましょう。

---

## まとめ

- **for-each（拡張 for 文）**は、`for (型 変数 : 配列)` で全要素を順に取り出す
- インデックスや `length` が不要で、範囲外アクセスの心配がない
- ただし「何番目か」は使えず、ループ変数への代入は**元の配列に反映されない**
- 番号を使う・書き換えるなら、通常の for を使う
