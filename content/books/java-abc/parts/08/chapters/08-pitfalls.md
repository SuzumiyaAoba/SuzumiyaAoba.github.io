## よくあるつまずき

配列でつまずきやすいポイントを、まとめて確認します。

---

## 1. 範囲外のインデックスにアクセスする

配列でもっとも多い失敗が、**存在しない番号にアクセスする**ことです。

要素が3個の配列なら、使えるインデックスは `0`・`1`・`2` だけです。
`3` 以上を指定すると、実行時にエラーが起きます。

```java
int[] nums = {10, 20, 30};
IO.println(nums[5]);   // 5 番は存在しない！
```

`jshell` で実行すると、次のようなエラー（**例外**）が表示されます（行頭の `|` は、jshell がエラーなどの説明に付ける印です）。

```text
|  例外java.lang.ArrayIndexOutOfBoundsException: Index 5 out of bounds for length 3
```

`ArrayIndexOutOfBoundsException`（アレイ・インデックス・アウトオブバウンズ・エクセプション）は、「配列の範囲を超えた」という意味です。
メッセージの `Index 5 out of bounds for length 3` は、「長さ3の配列に、5番でアクセスした」ことを表しています。

特に for 文では、条件を `i <= nums.length`（`<=`）にすると、最後に `nums[3]`（存在しない）へアクセスしてしまいます。
**`i < nums.length`（`<`）**が正しい形でした（第4節）。これは第7章で学んだ**オフバイワン**の代表例です。

> エラーの一種である**例外**（Exception）については、第20章でくわしく学びます。ここでは「配列の範囲外にアクセスすると、このエラーで止まる」と知っておきましょう。

## 2. 配列の大きさは、あとから変えられない

配列は、作るときに大きさが決まり、**あとから増やしたり減らしたりできません**。

```java
int[] nums = new int[3];   // 3個の配列
// あとから「4個に増やす」ことはできない
```

「データが何個になるか、最初はわからない」「あとから追加したい」という場合は、配列ではなく、第21章で学ぶ**コレクション**（`List` など）が向いています。
配列は「**個数が決まっているデータ**」に使うもの、と覚えておきましょう。

---

## 3. `length` と `length()` を混同する

配列の長さは **`length`**、文字列の長さは **`length()`** です。
**かっこ `( )` が付くかどうか**が違うので、混同しやすい点です。

```java
int[] nums = {10, 20, 30};
String s = "abc";
```

```text
jshell> nums.length     // 配列はかっこなし
$3 ==> 3

jshell> s.length()      // 文字列はかっこあり
$4 ==> 3
```

| 対象 | 長さの取り方 | かっこ |
|---|---|---|
| 配列 | `nums.length` | **なし** |
| 文字列（String） | `s.length()` | **あり** |

理由は先取りになりますが、配列の `length` は「フィールド（値そのもの）」、文字列の `length()` は「メソッド（処理を呼び出す）」だからです（メソッドは第9章で学びます）。
今は「**配列はかっこなし、文字列はかっこあり**」と形で覚えてしまって構いません。

## 4. 配列をそのまま表示しても、中身が出ない

第7節でも触れたとおり、配列を `IO.println` にそのまま渡しても、中身は表示されません。

```java
int[] nums = {10, 20, 30};
IO.println(nums);        // → [I@1db9742 のような表示になる
```

中身を確認したいときは、**`Arrays.toString(nums)`** を使います。

```java
IO.println(Arrays.toString(nums));   // → [10, 20, 30]
```

「配列の中身が変だな？」と思ったら、`Arrays.toString` を思い出しましょう。

---

## 5. `new` で作った参照型の配列は、中身が `null`

`new` で作った配列は、初期値で埋まります。
`int` なら `0` ですが、`String` などの**参照型**の場合は **`null`**（何もない状態）になります。

```java
String[] names = new String[3];   // 中身は { null, null, null }
```

中身を入れる前に使うと、エラー（`NullPointerException`）の原因になります。
`new` で参照型の配列を作ったら、**使う前に値を入れる**ことを忘れないようにしましょう（`null` は第11章以降でくわしく扱います）。

---

## まとめ

- 範囲外アクセスは **`ArrayIndexOutOfBoundsException`**。for は `i < length`（`<`）が正しい
- 配列の大きさは**固定**。増減させたいなら `List`（第21章）
- 長さは、**配列は `length`（かっこなし）／文字列は `length()`（かっこあり）**
- 配列をそのまま表示すると中身が出ない → **`Arrays.toString`** を使う
- `new` で作った参照型の配列は中身が **`null`**。使う前に値を入れる

次は、この章で学んだ言葉を、用語集としてまとめます。
