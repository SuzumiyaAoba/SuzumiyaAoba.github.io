## 配列とは

**配列**（はいれつ、Array）とは、**同じ型の値を、いくつも並べて、1つの名前でまとめて扱う**しくみです。

---

## 変数を1つずつ用意する大変さ

たとえば、5人ぶんのテストの点数を保存したいとします。
これまでの知識だと、変数を5つ用意することになります。

```java
int score1 = 80;
int score2 = 95;
int score3 = 70;
int score4 = 60;
int score5 = 100;
```

5人ならまだ書けますが、これが30人、100人だったらどうでしょう。
変数を100個も書くのは現実的ではありませんし、「全員の合計を求める」といった処理も、100個の変数を1つずつ足していくことになり、とても繰り返しでは扱えません。

---

## 配列なら1つの名前でまとめられる

配列を使うと、5人ぶんの点数を、**1つの名前**でまとめて持てます。

```java
int[] scores = {80, 95, 70, 60, 100};
```

`scores` という1つの名前の中に、5つの値が並んで入っています。
それぞれの値は、**番号**を使って取り出します（番号の使い方は、第3節でくわしく学びます）。

身近なイメージでいうと、配列は**番号のついたロッカーの列**のようなものです。

```text
scores:  [ 80 ][ 95 ][ 70 ][ 60 ][ 100 ]
   番号:    0     1     2     3      4
```

「`scores` という名前のロッカーの列があり、0番には80、1番には95……が入っている」というイメージです。
番号を指定すれば、そのロッカーの中身を出し入れできます。

---

## 配列に関する言葉

配列を学ぶうえで、次の2つの言葉をおさえておきましょう。

| 用語 | 英語・読み | 意味 |
|---|---|---|
| 要素 | Element（エレメント） | 配列の中に入っている1つ1つの値（上の例の `80`、`95` など） |
| インデックス（添字） | Index（インデックス） | 要素の位置を表す番号（上の例の `0`〜`4`） |

「`scores` という配列の、インデックス `1` の要素は `95`」のように使います。

---

## 配列の大事な性質

配列には、最初に知っておくべき性質が2つあります。

1. **同じ型の値しか入れられない** … `int` の配列には整数だけ、`String` の配列には文字列だけが入ります。整数と文字列を混ぜることはできません。
2. **大きさ（要素の数）は、作るときに決まり、あとから変えられない** … 5個の配列を作ったら、6個に増やしたり、4個に減らしたりはできません。

特に2つめは、初心者がよくつまずく点です。
「あとから個数を増やしたい」という場合は、配列ではなく、第21章で学ぶ**コレクション**（`List` など）を使います。
配列は、まず「**決まった数のデータを、まとめて扱う道具**」だと覚えておきましょう。

---

## まとめ

- **配列**は、同じ型の値をいくつも並べて、1つの名前でまとめて扱うしくみ
- 1つ1つの値を**要素**、その位置を表す番号を**インデックス（添字）**と呼ぶ
- 配列には**同じ型**の値しか入れられない
- 配列の**大きさは作るときに決まり、あとから変えられない**
