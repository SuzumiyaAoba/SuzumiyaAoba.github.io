## 配列を作る

配列の作り方には、おもに2つの方法があります。
「**最初から入れる値が決まっている**」場合と、「**入れ物だけ先に用意する**」場合です。順に見ていきましょう。

---

## 方法1: 値を並べて作る（初期化子）

入れたい値が最初から決まっているときは、値を `{ }` の中に、カンマ `,` で区切って並べます。

```java
int[] scores = {80, 95, 70, 60, 100};
```

これを jshell で打つと、次のように表示されます。

```text
jshell> int[] scores = {80, 95, 70, 60, 100};
scores ==> int[5] { 80, 95, 70, 60, 100 }
```

`int[5]` は「`int` が5個の配列」という意味です。
`{ ... }` の中身が、そのまま配列の要素になっているのがわかります。

型の書き方にも注目しましょう。

- `int[]` … 「`int` の配列」という型。型のうしろに `[ ]`（角かっこ）を付ける
- `int[] scores` … 「`int` の配列型の、`scores` という名前の変数」

文字列の配列なら `String[]`、小数の配列なら `double[]` のように、**「型 + `[]`」** で「その型の配列」を表します。

```java
String[] names = {"佐藤", "鈴木", "高橋"};
```

```text
jshell> String[] names = {"佐藤", "鈴木", "高橋"};
names ==> String[3] { "佐藤", "鈴木", "高橋" }
```

---

## 方法2: 入れ物だけ先に用意する（new）

「個数は決まっているが、値はあとから入れる」という場合は、`new` を使って、**空の配列**（決まった大きさの入れ物）を作ります。

```java
int[] scores = new int[5];
```

`new int[5]` は、「`int` が5個ぶんの入れ物を、新しく用意する」という意味です。
このとき、各要素には自動的に**初期値**が入ります。

```text
jshell> int[] scores = new int[5];
scores ==> int[5] { 0, 0, 0, 0, 0 }
```

`int` の配列なので、すべての要素が `0` で埋まっています。
要素にどんな初期値が入るかは、型によって決まっています。

| 型 | 初期値 |
|---|---|
| `int`・`long` など整数 | `0` |
| `double`・`float` など小数 | `0.0` |
| `boolean` | `false` |
| `String` などの参照型 | `null` |

> **補足: `null` に注意**
>
> `String[]` を `new` で作ると、各要素は `null`（ヌル、「何もない」状態）になります。
> 中身を入れる前にうっかり使うとエラーのもとになります。`null` については第11章以降でくわしく扱いますが、「`new` で作った参照型の配列は、まだ空っぽ」と覚えておきましょう。

`new` で作った配列に、あとから値を入れる方法は、次の第3節で学びます。

---

## どちらを使う？

| 方法 | 使う場面 |
|---|---|
| `{80, 95, ...}`（初期化子） | 入れる値が、最初から決まっているとき |
| `new int[5]`（new） | 個数だけ決まっていて、値はあとから入れるとき |

最初のうちは、値が決まっているなら方法1、繰り返しで順に値を入れていくなら方法2、と考えれば十分です。

---

## まとめ

- 値が決まっているときは、`int[] scores = {80, 95, 70};` のように `{ }` で並べて作る
- 入れ物だけ先に用意するときは、`int[] scores = new int[5];` のように `new` で作る
- `new` で作ると、各要素は型ごとの**初期値**（`int` は `0`、`String` は `null` など）で埋まる
- 型は「**型 + `[]`**」で表す（`int[]`、`String[]`、`double[]` など）

次の節では、配列から値を1つずつ**取り出したり、書き換えたり**する方法を学びます。
