## 配列と繰り返し

配列の本当の便利さは、第7章で学んだ**繰り返し**と組み合わせたときに発揮されます。
「たくさんの要素を、1つずつ順に処理する」――これが、配列と繰り返しの黄金パターンです。

---

## 配列の長さを知る ― length

繰り返しで配列を扱うには、まず「配列にいくつ要素があるか」を知る必要があります。
それを教えてくれるのが、**`length`**（レングス）です。

```java
int[] scores = {80, 95, 70, 60, 100};
```

```text
jshell> scores.length
$2 ==> 5
```

`配列名.length` と書くと、要素の数（ここでは `5`）が得られます。
`length` のうしろに**かっこ `( )` は付けません**。（第10章で学ぶ文字列の `length()` はかっこが付きます。混同しやすいので、第8節であらためて整理します。）

---

## for 文ですべての要素を処理する

`length` がわかれば、for 文ですべての要素をたどれます。
インデックスは `0` から始まり、最後は「個数 − 1」なので、条件は `i < scores.length` とします。

```java
int[] scores = {80, 95, 70, 60, 100};
for (int i = 0; i < scores.length; i++) {
    IO.println(i + "番目: " + scores[i]);
}
```

```text
0番目: 80
1番目: 95
2番目: 70
3番目: 60
4番目: 100
```

ループ変数 `i` が `0` → `4` と変化し、`scores[i]` でそのつど対応する要素を取り出しています。
ここで条件を `i <= scores.length`（`<=`）にしてしまうと、存在しない `scores[5]` にアクセスしてエラーになります。**`i < scores.length`（`<`）**が定番の形だと覚えておきましょう。

---

## 定番パターン1: 合計を求める

第7章で「合計を求める」を学びましたが、配列と組み合わせると、より実用的になります。
集計用の変数 `sum` を用意し、全要素を足していきます。

```java
int[] scores = {80, 95, 70, 60, 100};
int sum = 0;
for (int i = 0; i < scores.length; i++) {
    sum += scores[i];
}
```

```text
jshell> sum
sum ==> 405
```

5人ぶんの合計 405 が求められました。
配列が何要素になっても、`length` を使っているので、このコードはそのまま使えます。

> **補足: 平均を求めるなら**
>
> 平均は「合計 ÷ 個数」です。ただし第5章で学んだとおり、整数どうしのわり算は切り捨てられます。
> `sum / scores.length` は `405 / 5` で `81` になりますが、割り切れない場合に小数まで求めたいなら、`(double) sum / scores.length` のように一方を `double` にします。

---

## 定番パターン2: 最大値を求める

「いちばん大きい要素」を探すのも、よくある処理です。
まず最初の要素を「仮の最大値」とし、残りの要素と比べて、より大きければ更新します。

```java
int[] scores = {80, 95, 70, 60, 100};
int max = scores[0];
for (int i = 1; i < scores.length; i++) {
    if (scores[i] > max) {
        max = scores[i];
    }
}
```

```text
jshell> max
max ==> 100
```

ポイントは2つです。

- `max` の初期値を `0` ではなく **`scores[0]`**（最初の要素）にする。`0` から始めると、すべての要素がマイナスの配列で正しく求められません。
- ループを **`i = 1`** から始める。0番はすでに `max` に入れたので、1番から比べれば十分です。

同じ考え方で、`>` を `<` に変えれば**最小値**も求められます。

---

## まとめ

- `配列名.length` で要素の数がわかる（**かっこは付けない**）
- `for (int i = 0; i < 配列名.length; i++)` が、全要素をたどる定番の形
- **合計**は集計用の変数に足していく、**最大値**は最初の要素を仮の最大値として比べていく
- 配列の要素数が変わっても、`length` を使えば同じコードが通用する

ここまでは、インデックス `i` を使う for 文を見てきました。
次の節では、「全要素を順に取り出す」だけならもっと簡潔に書ける、**拡張 for 文（for-each）**を学びます。
