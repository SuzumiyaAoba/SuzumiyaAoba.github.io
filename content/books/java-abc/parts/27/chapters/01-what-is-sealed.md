## シールドクラスとは ― 継承を限定する

**シールドクラス**（Sealed Classes）は、「**継承（実装）できるクラスを、限定する**」しくみです[^jep409-sealed]。
この節では、なぜ継承を限定したいのか、その動機を学びます。

---

## 自由な継承の、困りごと

第14章・第16章では、継承や実装は「**誰でも自由にできる**」のが当たり前でした。

```java
interface Shape {
    double area();
}
// Shape は、世界中の誰でも、いくらでも実装できる
class Circle implements Shape { ... }
class Rectangle implements Shape { ... }
// 知らないところで、誰かが UFO implements Shape を作るかもしれない…
```

たいていの場合、この「自由さ」は便利です。
ですが、ときには困ることがあります。

たとえば、「図形を受け取って、種類ごとに処理する」コードを書くとします。

```java
if (shape instanceof Circle c) {
    // 円の処理
} else if (shape instanceof Rectangle r) {
    // 長方形の処理
} else {
    // それ以外…？ でも、「それ以外」って何があるの？
}
```

`Shape` を誰でも実装できると、「**いったい何種類あるのか**」が分かりません。
そのため、「それ以外」（`else`）に、何を書けばいいか決められず、もし新しい図形が増えても、それに気づけません。

---

## 「種類を限定する」という発想

そこで、発想を変えます。

> `Shape` を実装してよいのは、**円・長方形・三角形の3つだけ**、と決めてしまう。

こうすれば、「`Shape` は、この3種類しかない」と、はっきりします。
すると、

- 「それ以外」を考える必要がなくなる（3つを処理すれば、もれがない）
- 新しい種類を勝手に増やされる心配がない
- コンパイラも「3種類しかない」と知っているので、**もれをチェック**できる

といった利点が生まれます。

この「**継承できる相手を、決まったものだけに限定する**」のが、シールドクラスです。
`sealed`（封をする）という言葉のとおり、継承の「入口」を、許可した相手だけに絞るのです。

---

## enum との違い ― 「種類」と「データ」

「種類を限定する」と聞くと、第18章の **enum**（列挙型）を思い出すかもしれません。
たしかに、どちらも「決まった選択肢」を表します。ですが、得意なことが違います。

| | enum | シールドクラス |
|---|---|---|
| 限定するもの | 決まった**値**（RED・GREEN…） | 決まった**型**（Circle・Rectangle…） |
| 各要素がデータを持つ | 全員、同じ形のデータ | **種類ごとに違う形**のデータを持てる |

- **enum** … 「信号の色」のように、選択肢が**単純な値**で、みな同じ形のとき
- **シールドクラス** … 「図形」のように、**種類ごとに違うデータ**を持つとき（円は半径、長方形は幅と高さ…）

円は「半径」、長方形は「幅と高さ」と、種類によって持つデータが違います。
こういう「**種類は限られているが、それぞれ中身が違う**」ものを表すのが、シールドクラスの得意分野です。
（そして、その「中身」をレコード（第17章）で表し、パターンマッチング（第26章）で分解する、という組み合わせが定番です。）

---

## まとめ

- **シールドクラス**は、「**継承（実装）できるクラスを、決まったものだけに限定する**」しくみ
- 自由な継承だと「何種類あるか分からず」、もれや、勝手な追加の心配がある
- 種類を限定すると、「もれなく処理できる」「勝手に増やされない」「コンパイラがチェックできる」
- **enum** は「決まった値」、**シールドクラス**は「決まった型（種類ごとに違うデータを持てる）」
- レコード・パターンマッチングと組み合わせるのが定番

次の節では、`sealed` と `permits` を使った、シールドクラスの**書き方**を学びます。

[^jep409-sealed]: JEP 409: Sealed Classes, <https://openjdk.org/jeps/409>。Java 17（2021年9月）で正式機能（permanent feature）となった。`sealed`／`non-sealed`／`permits` キーワードで継承可能なサブクラスを明示する。詳細は *JLS §8.1.1.2*（<https://docs.oracle.com/javase/specs/jls/se25/html/jls-8.html#jls-8.1.1.2>）参照。
