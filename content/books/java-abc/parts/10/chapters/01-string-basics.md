## 文字列のおさらいと不変性

メソッドを使う前に、文字列（String）について、いくつか大事なことをおさらい・整理しておきましょう。

---

## String と char ― 文字列と1文字

Java には、「文字」を表す型が2つあります。混同しやすいので、ここで整理します。

| 型 | 表すもの | 書き方 | 例 |
|---|---|---|---|
| `String` | 文字の**並び**（0文字以上） | **ダブル**クォート `"..."` | `"Hello"`、`"A"`、`""` |
| `char` | **1文字**だけ | **シングル**クォート `'...'` | `'A'`、`'あ'`、`'5'` |

ポイントは**クォートの違い**です。

- `"A"` … ダブルクォート → **String**（長さ1の文字列）
- `'A'` … シングルクォート → **char**（1文字）

`'A'` と `"A"` は見た目が似ていますが、別の型です。
`char` は「文字列を構成する、1つ1つの文字」だと考えてください。`String` は、その `char` が並んだものです。

```text
"Hello" という String は、'H' 'e' 'l' 'l' 'o' という char の並び
```

---

## 文字列にもインデックスがある

文字列は文字の並びなので、配列と同じように、各文字に**インデックス（番号）**が付いています。
そして、配列と同じく **`0` から始まります**。

```text
"Hello"
 H  e  l  l  o
 0  1  2  3  4
```

「先頭の文字が `0` 番」というルールは、配列（第8章）と共通です。
次の節で、このインデックスを使って文字を取り出します。

---

## 文字列は「変えられない」 ― 不変性

文字列について、最も大切な性質があります。
それは、**一度作った文字列は、あとから中身を変えられない**ということです。
この性質を**不変**（ふへん、Immutable／イミュータブル）と呼びます。

「えっ、でも文字列を加工するメソッドがあるのでは？」と思うかもしれません。
たしかにあります。しかし、それらは**元の文字列を変えるのではなく、変えた結果を「新しい文字列」として返す**のです。

例として、全部を大文字にする `toUpperCase()` を見てみましょう。

```java
String s = "hello";
String up = s.toUpperCase();
```

結果を確認します。

```text
jshell> up
up ==> "HELLO"

jshell> s
s ==> "hello"
```

`up` は大文字の `"HELLO"` になりましたが、**元の `s` は `"hello"` のまま**です。
`toUpperCase()` は `s` を書き換えたのではなく、大文字にした**新しい文字列**を作って返しただけなのです。

このことから、とても大事な使い方のルールが導けます。

> **加工の結果は、必ず受け取る**
>
> `s.toUpperCase();` とだけ書いても、`s` は変わりません（結果を捨てているだけ）。
> 加工した結果を使いたいなら、`s = s.toUpperCase();` のように、**戻り値を変数で受け取る**必要があります。
> これは第9章で学んだ「メソッドの戻り値を受け取る」と同じ考え方です。

この「不変」という性質は、これからこの章で何度も出てきます。「文字列のメソッドは、元を変えず、新しい文字列を返す」と覚えておきましょう。

---

## まとめ

- `String` は文字の並び（ダブルクォート `"..."`）、`char` は1文字（シングルクォート `'...'`）
- 文字列の各文字にはインデックスがあり、配列と同じく **`0` から始まる**
- 文字列は**不変（Immutable）** ―― 一度作ると中身を変えられない
- 加工メソッドは元を変えず、**新しい文字列を返す**。結果は `s = s.xxx()` のように受け取る
