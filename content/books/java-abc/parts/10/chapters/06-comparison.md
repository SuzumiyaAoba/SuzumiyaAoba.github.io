## 文字列の比較（==と equals）

この章で**最も重要**なテーマです。
2つの文字列が「同じ内容か」を比べる方法には、初心者がほぼ必ずつまずく落とし穴があります。
結論を先に言うと、**文字列の比較には `==` ではなく `equals` を使う**――これを必ず守ってください。理由を順に見ていきます。

---

## == は「同じものか」を比べる

第5章で、`==` は値が等しいかを調べる比較演算子として学びました。
`int` などの基本型では、`5 == 5` のように、**値そのもの**を比べてくれます。

ところが、文字列（`String`）のような**参照型**で `==` を使うと、比べるのは中身ではなく、**「2つが、まったく同じ1つのものを指しているか」**になります。
第9章で学んだ「参照型は、場所を指し示す情報を持つ」を思い出してください。`==` は、その「場所」が同じかどうかを見るのです。

これだけだと分かりにくいので、実際の例で確かめましょう。

```java
String x = new String("hello");
String y = new String("hello");
```

`x` と `y` は、どちらも内容は `"hello"` です。これを2通りの方法で比べてみます。

```text
jshell> x == y
$3 ==> false

jshell> x.equals(y)
$4 ==> true
```

- `x == y` は **`false`** … `new` で別々に作ったので、「別のもの」と判定された
- `x.equals(y)` は **`true`** … 中身が同じ `"hello"` なので、「等しい」と判定された

内容は同じなのに、`==` は `false` になってしまいました。
これが「文字列に `==` を使ってはいけない」理由です。`==` は中身ではなく「同じものか」を見るため、内容が同じでも `false` になることがあるのです。

---

## equals は「中身が同じか」を比べる

文字列の**中身（内容）が同じか**を調べるのが、**`equals`** メソッドです。

```java
String a = "Hello";
String b = "Hello";
```

```text
jshell> a.equals(b)
$3 ==> true
```

`equals` は、文字を1つずつ見て、内容が一致するかを判定します。
私たちがふつう「文字列が同じ」と言うときの意味は、こちらの「中身が同じ」です。だから、**文字列の比較は `equals` を使う**のが正解です。

---

## なぜ == で動くように見えてしまうのか

やっかいなことに、次のように**リテラルで書いた文字列**どうしだと、`==` でも `true` になってしまいます。

```java
String a = "hello";
String b = "hello";
```

```text
jshell> a == b
$3 ==> true
```

これは、Java が「同じ内容の文字列リテラルは、1つを使い回す」という工夫（**文字列プール**と呼びます）をしているため、`a` と `b` がたまたま同じものを指すからです。

ここに罠があります。
リテラルで試すと `==` でも動いてしまうので、「文字列は `==` で比べられる」と勘違いしがちです。
しかし、ユーザーが入力した文字列や、メソッドで作った文字列（先ほどの `new String` のような実行時に作られる文字列）どうしになると、`==` は `false` になり、**ある日突然うまく動かなくなります**。

> **覚えておくこと**
>
> 文字列の比較は、**いつでも `equals` を使う**。`==` は使わない。
> 「リテラルだと `==` でも通る」のは、たまたまであって、頼ってはいけません。

---

## 便利な比較メソッド

`equals` の仲間に、知っておくと便利なメソッドがあります。

**大文字・小文字を無視して比べる ― `equalsIgnoreCase`**

```text
jshell> "Hello".equalsIgnoreCase("hello")
$1 ==> true
```

`"Hello"` と `"hello"` は、`equals` では `false` ですが、`equalsIgnoreCase` なら**大文字・小文字の違いを無視して** `true` になります。

**順番（辞書順）を比べる ― `compareTo`**

```text
jshell> "apple".compareTo("banana")
$1 ==> -1

jshell> "apple".compareTo("apple")
$2 ==> 0
```

`compareTo` は、2つの文字列の**辞書（アルファベット）順**を比べ、`int` を返します。

- 結果が**負**（マイナス）… 自分のほうが辞書順で**前**（`"apple"` は `"banana"` より前）
- 結果が **`0`** … 同じ
- 結果が**正**（プラス）… 自分のほうが辞書順で**後**

文字列を五十音順・アルファベット順に並べ替えたいときなどに使います（並べ替えのしくみは、コレクションを学ぶ第21章以降でくわしく扱います）。

---

## まとめ

- 文字列の比較は、**`==` ではなく `equals` を使う**（この章で最も大切なルール）
- `==` は「**同じものか**（場所が同じか）」、`equals` は「**中身が同じか**」を比べる
- リテラルどうしは `==` でも `true` になることがあるが（文字列プール）、**頼ってはいけない**
- 大文字小文字を無視するなら `equalsIgnoreCase`、辞書順を比べるなら `compareTo`
