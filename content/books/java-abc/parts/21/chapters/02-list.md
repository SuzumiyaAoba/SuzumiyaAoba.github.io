## List ― 順序のある並び

**List**（リスト）は、「**データを順番に並べて持つ**」ための、もっともよく使うコレクションです。
配列にいちばん近い感覚で使えて、しかも大きさが自由に変わります。
この節では、List の基本 ―― 作り方と、要素の並び方を学びます。

---

## List の特徴

List には、次の特徴があります。

- **順番（順序）を保つ** … 入れた順に並び、`0` 番目・`1` 番目…と位置で取り出せる（配列と同じ）
- **重複を許す** … 同じ値を、何個でも入れられる
- **大きさが可変** … `add` でいくらでも増やせる

「待ち行列の順番」「買い物リスト」「投稿の一覧」のように、**順番に意味があり、同じものが複数あってもよい**データに向いています。

---

## List を作る ― ArrayList

List を作るときの、もっとも基本的な書き方は、次のとおりです。

```java
List<String> names = new ArrayList<>();
```

ここで、左辺と右辺で名前が違うことに注目してください。

- 左辺 `List<String>` … 「`String` のリスト」という**型**（インターフェース）
- 右辺 `new ArrayList<>()` … 実際に作る、**具体的な実装**

第16章・第19章で学んだとおり、`List` は「リストとはこういうもの」という**インターフェース**で、`ArrayList`（アレイリスト）は、その**実装の1つ**です。
「`List` という約束を、`ArrayList` という具体的なしくみで実現する」というわけです。

> **補足: なぜ `List<String> x = new ArrayList<>()` と書くのか**
>
> 「`ArrayList<String> x = new ArrayList<>()` でもいいのでは？」と思うかもしれません。それでも動きます。
> ですが、**左辺はインターフェース（`List`）にする**のが、よい習慣とされています。
> こうしておくと、あとで「`ArrayList` ではなく別の実装（`LinkedList` など）に変えたい」と思ったとき、右辺だけ直せばよく、使う側のコードに影響しません。第16章で学んだ「インターフェースの型で扱う」の実践です。
> 右辺の `<>` は、第19章で学んだ**ダイヤモンド演算子**（左辺から型が分かるので省略）です。

## 要素を追加して、表示する

`add(...)` で、要素を末尾に追加していきます。

```java
List<String> names = new ArrayList<>();
names.add("佐藤");
names.add("鈴木");
names.add("高橋");
IO.println(names);
```

```text line-numbers=false
[佐藤, 鈴木, 高橋]
```

`add` を呼んだ順に、`佐藤`・`鈴木`・`高橋` と並んでいます。
List をそのまま表示すると、`[要素, 要素, ...]` という形で、中身が見えます（第14章で学んだ `toString` が、List 用に用意されているのです）。

大きさを最初に決めていないのに、3つの要素が、ちゃんと順番に入りました。
配列の `new String[3]` のような「大きさの指定」が、まったく不要なことに注目してください。

---

## 同じ値も、複数入れられる

List は重複を許すので、同じ値を何度でも入れられます。

```java
List<String> votes = new ArrayList<>();
votes.add("りんご");
votes.add("みかん");
votes.add("りんご");      // 同じ「りんご」をもう一度
IO.println(votes);
```

```text line-numbers=false
[りんご, みかん, りんご]
```

「りんご」が2つ入っています。
たとえば「投票の集計」のように、同じ選択肢が何度も出てくるデータには、この性質が役立ちます。
（逆に、「重複させたくない」場合は、第4節で学ぶ **Set** を使います。）

---

## 最初から要素を入れて作る ― List.of

「最初から中身が決まっている」リストは、**`List.of(...)`** で、ひといきに作れます[^jep269-collections]。

```java
List<String> colors = List.of("赤", "緑", "青");
IO.println(colors);
```

```text line-numbers=false
[赤, 緑, 青]
```

`List.of(...)` は、`( )` に並べた値を持つリストを、その場で作ってくれます。
`add` を何度も呼ぶより、すっきり書けます。

ただし、1つ大切な注意があります。
**`List.of(...)` で作ったリストは、あとから変更できません**（要素の追加・削除・変更ができない「変更不可リスト」になります）。

```java
List<String> colors = List.of("赤", "緑", "青");
colors.add("黄");   // ✕ 変更不可のリストに追加しようとする
```

```text line-numbers=false
Exception in thread "main" java.lang.UnsupportedOperationException
```

`UnsupportedOperationException`（サポートされない操作の例外）が起きます。
「`List.of` で作ったリストは、読み取り専用」と覚えておきましょう。
**あとから変更したいなら `new ArrayList<>()`、変更しない決まった一覧なら `List.of(...)`** と、使い分けます。
（このつまずきは、第7節でも改めて取り上げます。）

## まとめ

- **List** は、データを**順番に**並べて持つコレクション。**重複OK**で、大きさは**可変**
- 基本の作り方は **`List<型> 名前 = new ArrayList<>();`**（左辺はインターフェース `List`）
- **`add(...)`** で末尾に追加する。表示すると `[要素, 要素, ...]` の形で見える
- 同じ値も複数入れられる（投票の集計など）
- **`List.of(...)`** で、中身の決まったリストを一度に作れる（ただし**変更不可**）

次の節では、List の要素を**取得・変更・削除・検索**する、さまざまな操作を学びます。

[^jep269-collections]: JEP 269: Convenience Factory Methods for Collections, [https://openjdk.org/jeps/269](<https://openjdk.org/jeps/269>)。Java 9（2017年）で導入された。`List.of(...)`／`Set.of(...)`／`Map.of(...)` は不変（immutable）のコレクションを返し、要素の追加・削除・変更を試みると `UnsupportedOperationException` が投げられる。
