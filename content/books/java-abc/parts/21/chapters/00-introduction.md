## はじめに ― この章で学ぶこと

ここから、本書の**第3部**が始まります。

第2部では、クラス・継承・インターフェース・レコード・ジェネリクスなど、**自分で型を設計する**ための道具を学びました。
第3部「モダン Java と標準ライブラリ」では、その土台のうえに、**実務で日々使う、強力な道具**を積み上げていきます。

その最初の一歩が、この第21章「**コレクション**」です。

第8章で**配列**を学びました。配列は「同じ型の値を、たくさん並べて持つ」ための、便利なしくみでした。
ですが、配列には大きな弱点がありました ―― **作るときに大きさが決まり、あとから増やせない**ことです。

```java
int[] scores = new int[3];   // 3個と決めたら、4個目は入れられない
```

現実のプログラムでは、「データが何個になるか、最初は分からない」ことがほとんどです。
利用者が入力するたびにデータが増えたり、検索結果の件数が毎回違ったり ―― そんなとき、固定サイズの配列では、うまく扱えません。

そこで登場するのが、**コレクション**（Collection）です。
コレクションは、「**たくさんのデータを、柔軟にまとめて扱う**」ための、標準ライブラリのしくみです。
あとから要素を増やしたり減らしたり、重複を取り除いたり、キーで引いたり ―― 配列では面倒だったことが、簡単にできるようになります。

## 3つの代表的なコレクション

コレクションには、目的に応じて、いくつかの種類があります。
この章では、もっともよく使う3つを学びます。

| 種類 | ひとことで | 身近なたとえ |
|---|---|---|
| **List**（リスト） | 順番に並べる。重複OK | 買い物リスト、行列の待ち順 |
| **Set**（セット） | 重複を許さない集まり | 出席者の名簿（同じ人は1回） |
| **Map**（マップ） | キーと値の対応 | 辞書（単語→意味）、電話帳（名前→番号） |

- **List** … データを**順番に**並べたいとき。同じ値が何度出てきてもよい
- **Set** … **重複をなくしたい**とき。「いる／いない」だけを管理する
- **Map** … 「**あるもの**から**別のもの**を引きたい」とき（名前から点数、商品コードから価格など）

「何をしたいか」によって、この3つを使い分けます。
その使い分けの感覚をつかむのが、この章のいちばんの目標です。

## この章で学ぶこと

第21章は、次の8つの節で構成されています。

| 節 | タイトル | 内容 |
|---|---|---|
| 1 | コレクションとは | 配列との違い。なぜコレクションか |
| 2 | List ― 順序のある並び | リストの基本と作り方 |
| 3 | List の操作 | 追加・取得・変更・削除・検索 |
| 4 | Set ― 重複のない集まり | 重複を自動で取り除く |
| 5 | Map ― キーと値の対応 | キーから値を引く |
| 6 | コレクションの繰り返し処理 | for-each で全要素を扱う |
| 7 | よくあるつまずき | 基本型・null・変更不可リスト など |

前半（1〜3節）で、もっともよく使う **List** を、じっくり学びます。
後半（4〜6節）で、**Set**・**Map** と、コレクション共通の繰り返し処理を学びます。

> **補足: この章からのサンプルの示し方**
>
> 第3部では、コレクションやストリームのような「**ライブラリを使う**」例が中心になります。
> そこで、第1部の第4〜10章と同じく、サンプルは原則 **jshell にそのまま貼れる「文・式だけ」**の形で示します（`public class Main { ... }` で包みません）。
> jshell は、`List`・`Map` などが属する `java.util` パッケージを**自動で import** してくれるので、`import` を書かずにそのまま試せます。
> 一方、`.java` ファイルとして書くときは、第12章で学んだとおり `import java.util.*;` のような import が必要です（その点は、必要な箇所で改めて案内します）。

## この章を読み終えると

第21章を読み終えるころには、次のことができるようになっています。

- コレクションが配列より優れている点を説明できる
- `List` を作り、要素を追加・取得・変更・削除・検索できる
- `Set` で、重複のないデータの集まりを扱える
- `Map` で、キーから値を引く対応関係を扱える
- for-each で、コレクションの全要素を処理できる
- List・Set・Map を、目的に応じて使い分けられる

それでは、最初の節「コレクションとは」から始めましょう。
