## 用語集 ― この章で学んだ言葉

第21章では、たくさんのデータをまとめて扱う「コレクション」を学びました。
節ごとに、言葉を整理しておきます。

---

## コレクションの基本（第1節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| コレクション | Collection | たくさんのデータを柔軟にまとめて扱う標準ライブラリのしくみ |
| `ArrayList` | アレイリスト | `List` の代表的な実装。可変長の配列のように使える |
| 要素 | Element | コレクションに入っている、1つひとつのデータ |

コレクションはすべてジェネリック。型引数には参照型（`int` ではなく `Integer`）を使う。

---

## List（第2〜3節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| `List` | リスト | 順序を保ち、重複を許すコレクション |
| `add` / `get` / `set` | ― | 追加 / 取得 / 変更（位置は 0 始まり） |
| `remove` | ― | 値または位置で削除（後ろが自動で詰められる） |
| `contains` / `indexOf` | ― | 含むか判定 / 位置を取得（なければ `-1`） |
| `size` | ― | 実際の要素数 |
| `List.of(...)` | ― | 中身の決まったリストを作る（変更不可） |

---

## Set（第4節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| `Set` | セット | 重複を許さないコレクション |
| `HashSet` | ハッシュセット | `Set` の基本実装。順序は保証されない |
| `LinkedHashSet` | ― | 入れた順を保つ Set |
| `TreeSet` | ― | 自動的に並べ替える Set |

重複は自動で取り除かれる。`new HashSet<>(リスト)` でリストの重複を消せる。

---

## Map（第5〜6節）

| 用語 | 英語・読み | 意味 |
|---|---|---|
| `Map` | マップ | キーと値を対応づけるコレクション（辞書のようなもの） |
| キー / 値 | Key / Value | 引くための目印 / 対応する中身 |
| `HashMap` | ハッシュマップ | `Map` の基本実装。順序は保証されない |
| `put` / `get` | ― | 登録（上書き） / 取得 |
| `getOrDefault` | ― | キーがなければデフォルト値を返す |
| `containsKey` | ― | キーの有無を判定 |
| `entrySet` / `keySet` / `values` | ― | ペア / キー / 値 を取り出す（for-each 用） |

---

## おわりに

この章では、データをまとめて扱う3つのコレクションを学びました。

- **List** … 順序つき・重複OK（`ArrayList`）
- **Set** … 重複なし（`HashSet`）
- **Map** … キーと値の対応（`HashMap`）

「順番に並べたいなら List、重複をなくしたいなら Set、キーで引きたいなら Map」 ―― この使い分けが、これからのプログラミングの土台になります。
第8章の配列とあわせて、「たくさんのデータをどう持つか」の選択肢が、ぐっと広がりました。

ところで、第7節の最後に、`names.removeIf(name -> name.equals("佐藤"))` という、見慣れない `->` が出てきました。
これは、**ラムダ式**と呼ばれる、新しい書き方です。
「条件に合うものを削除」「1つずつ処理する」といった操作を、とても簡潔に書けるしくみです。

次の第22章「ラムダ式と関数型インターフェース」では、この `->` の正体を明らかにします。
そして第23章のストリーム API へと進むと、コレクションの操作が、見違えるほどスマートに書けるようになります。
ここからの3つの章（ラムダ・ストリーム）は、モダン Java の中心となる、とても大切なテーマです。
