## コレクションとは ― 配列の限界を超える

**コレクション**（Collection）は、「たくさんのデータを、柔軟にまとめて扱う」ためのしくみです。
この節では、配列の限界をふり返り、コレクションがそれをどう解決するのかを学びます。

---

## 配列の限界

第8章で学んだ配列を思い出しましょう。

```java
int[] scores = new int[3];   // 大きさ3で作る
scores[0] = 80;
scores[1] = 95;
scores[2] = 70;
```

配列は便利でしたが、次のような不便さがありました。

1. **大きさが、最初に固定される**
   `new int[3]` と書いた時点で、入るのは3個まで。4個目を入れたくなっても、増やせません。
2. **要素の追加・削除が面倒**
   「途中に1つ挿入する」「1つ削除して詰める」といった操作を、自分で書くのは大変でした。
3. **「何個入っているか」を、自分で管理する必要がある**
   配列の `length` は「入れ物の大きさ」であって、「実際に使っている数」ではありません。

現実には、「データが何個になるか、あらかじめ分からない」場面がほとんどです。
そのたびに、大きさを決め打ちしたり、入れ替えのコードを書いたりするのは、現実的ではありません。

---

## コレクションなら、柔軟に扱える

コレクションを使うと、これらの不便さが解消されます。
代表的な `List`（リスト）で、配列と同じこと（3つの点数を持つ）を書いてみましょう。

```java
List<Integer> scores = new ArrayList<>();
scores.add(80);
scores.add(95);
scores.add(70);
IO.println(scores);
```

```text line-numbers=false
[80, 95, 70]
```

ここで注目してほしいのは、**最初に大きさを決めていない**ことです。
`add(...)` を呼ぶたびに、リストは**自動的に大きく**なります。4個目、5個目…と、いくつでも追加できます。
「実際に何個入っているか」も、`size()` を呼べば、いつでも分かります。

配列とコレクションの違いを、表で整理しましょう。

| | 配列 | コレクション（List など） |
|---|---|---|
| 大きさ | 固定（あとから変えられない） | **可変**（自動で増減する） |
| 要素の追加・削除 | 自分で書く（面倒） | `add`・`remove` で簡単 |
| 要素数 | `length`（入れ物の大きさ） | `size()`（実際の個数） |
| 便利なメソッド | 少ない（`Arrays` 経由） | 豊富（検索・並べ替えなど） |

## コレクションはジェネリック ―「何の」コレクションか

第19章で学んだ**ジェネリクス**を思い出してください。
コレクションは、すべて**ジェネリック**です。`< >` の中に、「**何を入れるか**」の型を指定します。

```java
List<String> names = new ArrayList<>();    // String のリスト
List<Integer> scores = new ArrayList<>();  // Integer のリスト
```

- `List<String>` … `String`（文字列）を入れるリスト
- `List<Integer>` … `Integer`（整数）を入れるリスト

型を指定しておくことで、**それ以外の型を入れようとすると、コンパイルエラー**になります。
取り出すときも、指定した型のまま（キャストなしで）扱えます。第19章で学んだ「型の安全性」が、ここで効いてくるのです。

> **補足: 型引数には参照型 ―「`int`」ではなく「`Integer`」**
>
> 第19章で学んだとおり、`< >` の中には**参照型**しか書けません。
> そのため、整数のリストは `List<int>` ではなく、ラッパークラスの `List<Integer>` とします。
> `scores.add(80)` のように `int` の値をそのまま入れられるのは、Java が `int` と `Integer` を自動変換（オートボクシング）してくれるからでした。

## コレクションの全体像

Java のコレクションには、たくさんの種類がありますが、大きくは次のように整理できます。

```text line-numbers=false
コレクション（たくさんのデータをまとめる）
├─ List … 順番に並べる。重複OK            （ArrayList など）
├─ Set  … 重複のない集まり                （HashSet など）
└─ Map  … キーと値の対応 ※              （HashMap など）
```

（※ 厳密には、`Map` は「コレクション」の仲間とは少し別の系統ですが、本書では「データをまとめるしくみ」として、まとめて扱います。）

`List`・`Set`・`Map` には、それぞれ**インターフェース**（第16章）と、その**実装クラス**があります。
たとえば `List` はインターフェースで、`ArrayList` がその実装です。
次の節から、この3つを順に見ていきましょう。

## まとめ

- **コレクション**は、たくさんのデータを**柔軟に**まとめて扱うしくみ
- 配列と違い、**大きさが可変**（`add` で自動的に増える）で、便利なメソッドが豊富
- コレクションは**ジェネリック**。`List<String>` のように「何を入れるか」を指定する
- 型引数には**参照型**を使う（`int` ではなく `Integer`）
- 代表は **List**（順序つき）・**Set**（重複なし）・**Map**（キーと値）の3つ

次の節では、もっともよく使うコレクション ―― **List** を学びます。
