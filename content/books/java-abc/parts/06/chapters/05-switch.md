## switch 文と switch 式

`else if` をいくつも続けると、コードが長くなりがちです。
特に「1つの値が、どの値に当てはまるか」で分けたいときは、**switch**（スイッチ）を使うと、すっきり書けます。

たとえば「曜日の番号（1〜7）から、曜日の名前を求める」ような場面です。

---

## switch 式（Java の新しい書き方）

Java 14 以降では、**switch 式**という、わかりやすい書き方が使えます[^jep361-switch]。本書では、こちらをおすすめします。

```java
int day = 3;
String dow = switch (day) {
    case 1 -> "月";
    case 2 -> "火";
    case 3 -> "水";
    default -> "その他";
};
```

`switch (day)` で、`day` の値を調べます。
`case 3 ->` のように、値と、その値だったときに選ぶものを、矢印 `->` で結びます。
`default` は、どの `case` にも当てはまらないときに選ばれます。

jshell で確かめてみましょう。

```text
jshell> int day = 3;
day ==> 3
jshell> String dow = switch (day) {
   ...>     case 1 -> "月";
   ...>     case 2 -> "火";
   ...>     case 3 -> "水";
   ...>     default -> "その他";
   ...> };
dow ==> "水"
```

`day` が 3 なので、`case 3` が選ばれ、`dow` に `"水"` が入りました。
（jshell では、途中の行に `...>` という継続の印が出ます。`{ }` が閉じるまで、入力を待ってくれているしるしです。）

このように、switch 式は**全体で1つの値**になり、その結果を変数に入れられます。

---

## 1つの case に複数の値

カンマで区切れば、1つの `case` に複数の値をまとめられます。

```java
int month = 4;
String season = switch (month) {
    case 12, 1, 2 -> "冬";
    case 3, 4, 5  -> "春";
    case 6, 7, 8  -> "夏";
    default       -> "秋";
};
```

`month` が 4 なら `case 3, 4, 5` に当てはまり、`season` は `"春"` になります。

---

## case で複数の処理をする ― ブロックと yield

`case ... ->` の右側には、ふつう1つの値を書きます。
しかし、「途中で何か表示してから値を返したい」など、複数の処理が必要なこともあります。
そのときは、処理を `{ }`（ブロック）で囲み、返す値を **`yield`**（イールド）で指定します。

```java
int day = 6;
String type = switch (day) {
    case 1, 2, 3, 4, 5 -> "平日";
    case 6, 7 -> {
        IO.println("週末です！");
        yield "週末";
    }
    default -> "不明";
};
```

```text
週末です！
```

```text
jshell> type
type ==> "週末"
```

`day` が 6 なので `case 6, 7` のブロックが実行され、「週末です！」を表示してから、`yield "週末"` で `"週末"` を返しています。
**1行で済むなら `->` のあとに値を直接書き、複数の処理が必要なときだけ `{ }` と `yield` を使う**、と覚えておきましょう。

---

## switch 式は「すべての場合」を網羅する

switch 式には、大事なルールが1つあります。
switch 式は**必ず1つの値を返す**ので、「どの `case` にも当てはまらなかった」という抜け道があってはいけません。
そのため、起こりうる**すべての場合をカバー**する必要があり、たいていは `default` を書きます。

カバーしきれない場合があると、コンパイルエラーになります。

```java
int day = 3;
String dow = switch (day) {
    case 1 -> "月";
    case 2 -> "火";
    // 3 以上のときに返す値がない！
};
```

```text
エラー: switch式がすべての可能な入力値をカバーしていません
```

`day` には 3 や 100 など、`case 1`・`case 2` 以外の値も入りえます。
そのときに返す値がないため、Java が「網羅できていない」と教えてくれるのです。`default` を加えれば解決します。
書き忘れを**コンパイルの時点で**防いでくれる、switch 式の安全な性質です。

## 従来の switch 文と break

switch には、古くからある**switch 文**という書き方もあります。こちらは矢印ではなく、コロン `:` と `break` を使います。

```java
switch (day) {
    case 1:
        IO.println("月");
        break;
    case 2:
        IO.println("火");
        break;
    default:
        IO.println("その他");
}
```

この書き方には、**`break` を書き忘れると、次の `case` の処理にそのまま流れ込んでしまう**という、有名な落とし穴があります。

たとえば、次は `break` を書き忘れた switch 文です。

```java
int day = 1;
switch (day) {
    case 1:
        IO.println("月");
    case 2:
        IO.println("火");
    default:
        IO.println("その他");
}
```

```text
月
火
その他
```

`day` は 1 なので「月」だけが表示されてほしいのに、`break` がないため、`case 2` や `default` の処理にも**そのまま流れ込んで**しまいました。
この「次の case へ流れ込む」現象を、**フォールスルー**（fall-through）と呼びます。意図しないフォールスルーは、見つけにくいバグのもとです。

新しい **switch 式（`->`）では、この `break` が不要**で、当てはまった case だけが実行されます（フォールスルーも起きません）。
そのぶん安全で読みやすいので、本書では switch 式を中心に使います。

## if-else と switch、どちらを使う？

if-else と switch は、どちらも「条件で処理を分ける」しくみですが、得意分野が違います。

| 書き方 | 向いている場面 |
|---|---|
| `if` / `else if` | **範囲**や、複数条件を組み合わせた判定（`score >= 60`、`a && b` など） |
| `switch` | **1つの値**が、どの値に当てはまるか（曜日・月・区分番号など、とびとびの値） |

「`x >= 10` のような大小・範囲」なら if、「`x` が 1 か 2 か 3 か…」のように1つの値をたくさんの候補と照らし合わせるなら switch、と考えるとよいでしょう。

> **発展: switch は「型」でも分けられる（パターンマッチング）**
>
> 新しい Java の switch は、値だけでなく「**型**」によっても分岐できます（パターンマッチング for switch）[^jep441-switch-pattern]。
> たとえば「受け取ったものが文字列なら…、整数なら…」のような分け方です。
> とても強力な機能ですが、クラスや型の知識が必要になるため、本書では第26章「パターンマッチング」でくわしく学びます。ここでは「switch はもっと進化している」とだけ知っておいてください。

## まとめ

- **switch** は、1つの値が「どの値に当てはまるか」で分けるのに向いている
- **switch 式（`->`）** は全体で1つの値になり、変数に入れられる。本書のおすすめ
- カンマ区切りで複数の値をまとめられ（`case 3, 4, 5`）、複数の処理は `{ }` と **`yield`** で書く
- switch 式は**すべての場合を網羅**する必要がある（たいてい `default` を書く）
- 従来の **switch 文（`:` と `break`）** は `break` 忘れの**フォールスルー**に注意。switch 式なら不要
- **範囲・複合条件は if、1つの値の多分岐は switch** と使い分ける

次の節では、条件分岐でつまずきやすいポイントを、まとめて確認します。

[^jep361-switch]: JEP 361: Switch Expressions, [https://openjdk.org/jeps/361](<https://openjdk.org/jeps/361>)。Java 14（2020年3月）で正式機能となった。`->` ラベル、`yield`、網羅性チェック、フォールスルーの除去が導入された。*JLS §15.28* も参照。

[^jep441-switch-pattern]: JEP 441: Pattern Matching for switch, [https://openjdk.org/jeps/441](<https://openjdk.org/jeps/441>)。Java 21（2023年9月）で正式機能となった。型パターン、レコードパターン、`when` ガード節などにより、switch で型に応じた分岐が書ける。
