## else if で多分岐

`if` と `else` では、道は2つにしか分かれません。
ですが、「優・良・可」のように、**3つ以上に分けたい**こともよくあります。

そんなときは、`else` のうしろにさらに `if` を続けた、**`else if`** を使います。

---

## else if の書き方

点数を「優（80以上）・良（70以上）・可（それ以外）」の3つに分ける例を見てみましょう。

```java
int n = 75;
if (n >= 80) {
    IO.println("優");
} else if (n >= 70) {
    IO.println("良");
} else {
    IO.println("可");
}
```

`n` は 75 なので、結果はこうなります。

```text
良
```

`else if` は、いくつでも続けられます。
最後の `else` は、「どの条件にも当てはまらなかったとき」の処理で、省略することもできます。

---

## 上から順に調べ、最初の1つだけを実行する

`else if` のしくみで大切なのは、**上から順に条件を調べ、最初に成り立った（`true` になった）ブロックだけを実行する**という点です。
一度どれかが実行されると、残りの条件は調べられません。

上の例で、`n` が 75 のときの流れを追ってみましょう。

1. `n >= 80` → 75 は 80 以上ではないので `false`。次へ
2. `n >= 70` → 75 は 70 以上なので `true`。**「良」を表示して、ここで終了**
3. （`else` は調べられない）

---

## 順番に注意

`else if` は上から順に調べるので、**条件を書く順番**が大切です。

たとえば、もし `n >= 70` を先に書いてしまうと、80 以上の点数でも先に `n >= 70` が `true` になり、「優」には決してたどり着けません。
「きびしい条件（範囲のせまい条件）から先に書く」と覚えておくと、こうした取りこぼしを防げます。

---

## まとめ

- **`else if`** を使うと、3つ以上の場合に枝分かれできる
- 上から順に条件を調べ、**最初に成り立ったブロックだけ**を実行する
- 最後の `else` は「どれにも当てはまらないとき」。省略も可能
- 条件を書く**順番**に注意する（せまい条件から先に）
