## ブロックと複合条件・ネスト

このセクションでは、if 文をより安全に、わかりやすく書くためのポイントを3つ学びます。
**`{ }` の使い方**、**複数条件のまとめ方**、そして**入れ子（ネスト）**です。

---

## `{ }` は省略しないほうがよい

if 文の処理が1文だけのとき、Java では `{ }` を省略できます。

```java
if (score >= 60)
    IO.println("合格");
```

しかし、本書では**つねに `{ }` を付ける**ことをおすすめします。
`{ }` を省略すると、あとから処理を1行追加したときに、思わぬバグが生まれやすいからです。

```java
if (score >= 60)
    IO.println("合格");
    IO.println("おめでとう");   // これは if と関係なく、いつも実行されてしまう
```

見た目（インデント）では `if` の中のように見えますが、`{ }` がないため、2行目の `おめでとう` は**条件に関係なく、いつも実行**されてしまいます。
最初から `{ }` で囲んでおけば、こうした勘違いは起きません。

---

## 複数の条件をまとめる ― 複合条件

第5章で学んだ論理演算子（`&&`・`||`）を使うと、複数の条件を1つの if にまとめられます。

たとえば「点数が 0 以上 100 以下（正しい範囲）」を確かめるには、`&&` を使います。

```java
int n = 75;
if (0 <= n && n <= 100) {
    IO.println("正しい点数です");
}
```

```text
正しい点数です
```

`0 <= n`（`true`）と `n <= 100`（`true`）の両方が成り立つので、`&&` 全体が `true` になり、処理が実行されます。

---

## 入れ子（ネスト）の if

if 文の中に、さらに if 文を書くこともできます。これを**ネスト**（入れ子）と呼びます。

```java
if (ログインしている) {
    if (管理者である) {
        // 管理者向けの処理
    }
}
```

ネストは便利ですが、深くなりすぎると、とたんに読みにくくなります。
多くの場合、**複合条件（`&&`）を使って、平らに書き直せます**。

```java
if (ログインしている && 管理者である) {
    // 管理者向けの処理
}
```

「ネストが深いな」と感じたら、`&&` でまとめられないか考えてみましょう。

---

## まとめ

- if の処理が1文でも、**`{ }` は省略せず付ける**（あとからのバグを防ぐ）
- `&&` や `||` で、**複数の条件を1つの if にまとめられる**（範囲チェックなど）
- if の中に if を書くことを**ネスト**と呼ぶ。深くなりすぎたら、複合条件で平らにできないか考える
