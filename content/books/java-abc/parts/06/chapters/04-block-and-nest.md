## ブロックと複合条件・ネスト

この節では、if 文をより安全に、わかりやすく書くためのポイントを3つ学びます。
**`{ }` の使い方**、**複数条件のまとめ方**、そして**入れ子（ネスト）**です。

---

## `{ }` は省略しないほうがよい

if 文の処理が1文だけのとき、Java では `{ }` を省略できます。

```java
if (score >= 60)
    IO.println("合格");
```

しかし、本書では**つねに `{ }` を付ける**ことをおすすめします。
`{ }` を省略すると、あとから処理を1行追加したときに、思わぬバグが生まれやすいからです。

```java
if (score >= 60)
    IO.println("合格");
    IO.println("おめでとう");   // これは if と関係なく、いつも実行されてしまう
```

見た目（インデント）では `if` の中のように見えますが、`{ }` がないため、2行目の `おめでとう` は**条件に関係なく、いつも実行**されてしまいます。
最初から `{ }` で囲んでおけば、こうした勘違いは起きません。

---

## 複数の条件をまとめる ― 複合条件

第5章で学んだ論理演算子（`&&`・`||`）を使うと、複数の条件を1つの if にまとめられます。

たとえば「点数が 0 以上 100 以下（正しい範囲）」を確かめるには、`&&` を使います。

```java
int n = 75;
if (0 <= n && n <= 100) {
    IO.println("正しい点数です");
}
```

```text line-numbers=false
正しい点数です
```

`0 <= n`（`true`）と `n <= 100`（`true`）の両方が成り立つので、`&&` 全体が `true` になり、処理が実行されます。

逆に、「どちらか一方でも成り立てば」というときは、`||`（または）を使います。
たとえば「0 未満、または 100 より大きい（＝正しい範囲の外）」を確かめるには、次のように書きます。

```java
int n = 120;
if (n < 0 || n > 100) {
    IO.println("点数の範囲が正しくありません");
}
```

```text line-numbers=false
点数の範囲が正しくありません
```

`n < 0`（`false`）か `n > 100`（`true`）の**どちらか**が成り立てば、`||` 全体が `true` になります。`n` は 120 なので `n > 100` が成り立ち、処理が実行されました。

> **補足: マジックナンバーは、定数で名前をつける**
>
> 上の `100` のような数値が、コードのあちこちに直接書かれていると、「この `100` は何の数字だろう？」と後から読んだときに分かりにくくなります。こうした、意味の説明がないむき出しの数値を**マジックナンバー**（Magic Number）と呼びます。
> 第4章で学んだ `final`（定数）を使い、`final int MAX_SCORE = 100;` のように名前をつけておくと、`if (n < 0 || n > MAX_SCORE)` と書けて、意味が伝わりやすくなります。同じ値を何度も使うときは、特に定数にしておくと、変更も1か所で済みます。

## 入れ子（ネスト）の if

if 文の中に、さらに if 文を書くこともできます。これを**ネスト**（入れ子）と呼びます。
たとえば「ログインしていて、かつ管理者のときだけ、管理者メニューを表示する」を、ネストで書くと次のようになります。

```java
boolean loggedIn = true;
boolean isAdmin = true;
if (loggedIn) {
    if (isAdmin) {
        IO.println("管理者メニューを表示");
    }
}
```

```text line-numbers=false
管理者メニューを表示
```

ネストは便利ですが、深くなりすぎると、`{ }` が何重にもなり、とたんに読みにくくなります。
上の例のような「両方が成り立つとき」のネストは、**複合条件（`&&`）を使って、平らに書き直せます**。

```java
if (loggedIn && isAdmin) {
    IO.println("管理者メニューを表示");
}
```

結果は同じですが、こちらのほうがネストが浅く、条件もひと目で読み取れます。
「ネストが深いな」と感じたら、`&&` でまとめられないか考えてみましょう。

---

## まとめ

- if の処理が1文でも、**`{ }` は省略せず付ける**（あとからのバグを防ぐ）
- `&&` や `||` で、**複数の条件を1つの if にまとめられる**（範囲チェックなど）
- if の中に if を書くことを**ネスト**と呼ぶ。深くなりすぎたら、複合条件で平らにできないか考える
