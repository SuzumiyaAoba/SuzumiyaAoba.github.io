## よくあるつまずき

条件分岐でつまずきやすいポイントを、まとめて確認します。

---

## 1. `=` と `==` の取り違え

条件で「等しいか」を調べるつもりで、`=`（代入）を1つだけ書いてしまうミスです。

数値の場合は、Java が**型のエラー**で気づかせてくれます。if の条件は `boolean` でなければならず、`score = 60`（代入）の結果は数値になるためです。

```java
int score = 50;
if (score = 60) {     // = が1つ → 代入になっている
    IO.println("60点です");
}
```

```text
エラー: 不適合な型: intをbooleanに変換できません
```

`score = 60` は代入で、その結果は `int`（60）です。if の条件は `boolean` でなければならないので、「`int` を `boolean` に変換できない」とエラーになります。`==` に直せば解決します。

ただし、`boolean` の変数では注意が必要です。`if (done = true)` のように書くと、これは「`done` に `true` を代入し、その結果（`true`）を条件にする」と解釈され、**エラーにならずに通ってしまいます**。比較のつもりなら、必ず `==` を使いましょう。

---

## 2. `{ }` の省略によるバグ

第4節でも見たとおり、`{ }` を省略すると、インデントで揃えても、2行目以降は if と無関係になってしまいます。

```java
if (score >= 60)
    IO.println("合格");
    IO.println("おめでとう");   // 条件に関係なく、いつも実行される
```

処理が1文でも、**つねに `{ }` を付ける**ことで防げます。

---

## 3. 文字列の比較に `==` を使う

条件の中でも、文字列が同じ内容かを比べるときは、`==` ではなく `equals` を使います。

```java
if (name.equals("太郎")) {
    IO.println("こんにちは、太郎さん");
}
```

（理由は第10章「文字列の操作」でくわしく学びます。）

---

## 4. switch の break 忘れ

従来の switch 文（`:` を使う形）では、`break` を書き忘れると、次の `case` の処理にそのまま流れ込んでしまいます。
新しい **switch 式（`->`）** を使えば、`break` は不要で、この問題は起こりません。

---

## 5. switch 式で、すべての場合をカバーし忘れる

switch 式（`->`）は、必ず1つの値を返すため、**すべての場合をカバー**しなければなりません。
`default` を書かず、当てはまらない値があると、コンパイルエラーになります。

```java
int day = 3;
String dow = switch (day) {
    case 1 -> "月";
    case 2 -> "火";
};   // 3 以上のときに返す値がない
```

```text
エラー: switch式がすべての可能な入力値をカバーしていません
```

`default` を加えれば解決します。これは裏を返せば、「対応漏れをコンパイルの時点で教えてくれる」という、switch 式の便利な点でもあります。

---

## 6. else if の順番ミス

`else if` は上から順に調べるため、**広い条件を先に書くと、せまい条件に決してたどり着けません**。

```java
// 順番が逆の悪い例
if (n >= 70) {
    IO.println("良");         // 80 以上でも、ここで止まってしまう
} else if (n >= 80) {
    IO.println("優");         // 決して実行されない
}
```

「せまい条件（きびしい条件）から先に書く」と覚えておきましょう。

---

## まとめ

- 条件の「等しい」は `==`。`=`（代入）と取り違えない（特に `boolean` 変数で注意）
- 処理が1文でも `{ }` を付ける
- 文字列の比較は `equals`
- 従来の switch 文は `break` 忘れ（フォールスルー）に注意。switch 式は `default` で網羅する
- `else if` は、せまい条件から先に書く
