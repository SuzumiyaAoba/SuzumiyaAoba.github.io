## はじめに ― この章で学ぶこと

第21章でコレクション、第22章でラムダ式を学びました。
この2つが、いよいよ1つに結びつくのが、この第23章「**ストリーム API**」です。

ストリーム API（Stream API）は、ひとことで言えば、

> **コレクションのデータを、「流れ作業」のように処理する**しくみ

です。
「リストから条件に合うものを絞り込む」「各要素を変換する」「合計する」「並べ替える」――
こうした処理を、**流れるようにつないで**書けます。

たとえば、「点数のリストから、合格点（60点以上）だけを取り出し、高い順に並べる」処理は、こう書けます。

```java
List<Integer> scores = List.of(85, 40, 70, 55, 90);

List<Integer> passed = scores.stream()
    .filter(s -> s >= 60)        // 60点以上だけ
    .sorted(Comparator.reverseOrder())  // 高い順に
    .toList();                   // リストにまとめる

IO.println(passed);
```

```text
[90, 85, 70]
```

`filter`（絞り込む）→ `sorted`（並べ替える）→ `toList`（集める）と、処理が**上から下へ流れて**いきます。
for ループや if を何重にも書かなくても、「**何をしたいか**」を、そのまま順番に並べるだけ。
これが、ストリームの読みやすさです。

---

## なぜストリームが便利なのか

同じ処理を、これまで学んだ for ループで書くと、こうなります。

```java
List<Integer> scores = List.of(85, 40, 70, 55, 90);

List<Integer> passed = new ArrayList<>();
for (int s : scores) {
    if (s >= 60) {
        passed.add(s);          // 絞り込み
    }
}
passed.sort(Comparator.reverseOrder());   // 並べ替え
IO.println(passed);
```

動きは同じですが、「一時的なリストを用意して」「ループを回して」「if で判定して」「add して」…と、**手順（どうやるか）**が前面に出ています。

ストリームは、この「どうやるか」を裏に隠し、「**何をしたいか**（絞る・並べる・集める）」だけを書けるようにしてくれます。
処理が増えても、`.メソッド()` をつなげていくだけなので、見通しがよく、間違いも減ります。

## この章で学ぶこと

第23章は、次の8つの節で構成されています。

| 節 | タイトル | 内容 |
|---|---|---|
| 1 | ストリームとは | 流れ作業の考え方。3つの工程 |
| 2 | ストリームを作る | コレクションや値からストリームを得る |
| 3 | 中間操作 | `filter`・`map`・`sorted` などで加工する |
| 4 | 終端操作 | `forEach`・`count`・`collect`・`sum` で結果を得る |
| 5 | Collectors とグループ化 | `toList`・`joining`・`groupingBy` |
| 6 | ストリームの実践 | 複数の操作を組み合わせる |
| 7 | よくあるつまずき | 使い回し・null・無限ループ など |

前半（1〜4節）で、ストリームの基本の流れ ―― 作る → 加工する（中間操作）→ 結果を得る（終端操作）―― を学びます。
後半（5〜6節）で、データを集約する `Collectors` と、実践的な組み合わせを学びます。

## この章を読み終えると

第23章を読み終えるころには、次のことができるようになっています。

- ストリームの「作る → 中間操作 → 終端操作」の流れを理解できる
- `filter`・`map`・`sorted` で、データを絞り込み・変換・並べ替えできる
- `toList`・`count`・`sum`・`forEach` で、結果を取り出せる
- `Collectors.groupingBy` などで、データをグループ化・集約できる
- for ループで書いていた処理を、ストリームで簡潔に書きかえられる

> **補足: ストリームは Java 8 から**
>
> ストリーム API も、第22章のラムダ式と同じく **Java 8** で導入されました。
> ストリームの各工程には、ラムダ式やメソッド参照を渡します。第22章で学んだことが、ここで存分に活きてきます。
> なお、ここでの「ストリーム」は、第30章で扱うファイルの「入出力ストリーム」とは**別物**です（名前が似ているだけです）。

それでは、最初の節「ストリームとは」から始めましょう。
